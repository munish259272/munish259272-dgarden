# Understanding Property Decorators in Python

## Properties Recap

We’ve explored **properties**, which let us control attribute access with getter and setter methods while keeping simple syntax like `obj.x`. They’re instances of the `property` class, typically set up with `fget` and `fset`:

```python
class MyClass:
    def __init__(self, x):
        self._x = x
    def get_x(self):
        return self._x
    def set_x(self, value):
        self._x = value
    x = property(get_x, set_x)
```

- **`x`**: A class-level `property` object tied to `get_x` and `set_x`.

**Key Point**: **Properties blend method logic with attribute-like access**.

---

## Property Methods: A Flexible Approach

The `property` class offers methods—`getter`, `setter`, and `deleter`—to define behavior incrementally:

```python
class MyClass:
    def __init__(self, x):
        self._x = x
    def get_x(self):
        return self._x
    def set_x(self, value):
        self._x = value
    
    x = property()  # Start empty
    x = x.getter(get_x)  # Add getter
    x = x.setter(set_x)  # Add setter
```

- **`x.getter(get_x)`**: Returns a new `property` with `fget=get_x`.
- **Chaining**: Each method call builds on the previous `property` object.

Alternatively:
```python
x = property(get_x)  # Start with getter
x = x.setter(set_x)  # Add setter
```

**Takeaway**: **Property methods allow step-by-step construction**.

---

## Decorators: A Cleaner Syntax

This process mirrors **decorators**, where a function is passed to another function, and the result replaces the original name. The `property` class can act as a decorator:

```python
class MyClass:
    def __init__(self, language):
        self._language = language
    
    @property
    def language(self):
        return self._language
```

- **`@property`**: Creates a `property` object with `language` as the getter, assigned to `language`.
- **Equivalent**: `language = property(language)`.

**Key Point**: **Decorators streamline property setup**.

---

## Adding Setters with Decorators

To include a setter, use the property’s `setter` method as a decorator:

```python
class MyClass:
    def __init__(self, language):
        self._language = language
    
    @property
    def language(self):
        return self._language
    
    @language.setter
    def language(self, value):
        self._language = value
```

- **`@language.setter`**: Updates the `language` property, adding the setter while keeping the getter.
- **Flow**:
  1. `@property` makes `language` a property with a getter.
  2. `@language.setter` enhances it with a setter.

Test it:
```python
m = MyClass("Python")
print(m.language)  # "Python"
m.language = "Java"  # Sets _language
print(m.__dict__)  # {'_language': 'Java'}
```

**Takeaway**: **Decorators chain to define full properties**.

---

## Why Names Must Match

The function name sets the property’s name:
- `@property def language`: Creates a property named `language`.
- `@language.setter def language`: Updates that same `language` property.

Mismatch example:
```python
class MyClass:
    @property
    def language(self):
        return self._language
    
    @language.setter
    def lang(self, value):  # Wrong name
        self._language = value
```

- **Result**: `m.language` has a getter but no setter; `m.lang` is a method, not a property setter.
- **Fix**: Both must be `language` to target the same property.

**Key Point**: **Matching names ensure proper property linkage**.

## Breaking It Down

Imagine building it manually:
```python
class MyClass:
    def __init__(self, language):
        self._language = language
    
    def lang_getter(self):
        return self._language
    language = property(lang_getter)  # Getter only
    
    def lang_setter(self, value):
        self._language = value
    language = language.setter(lang_setter)  # Add setter
```

- **Decorators Automate**:
  - `@property` handles the getter step.
  - `@language.setter` handles the setter step, reusing the `language` name.

**Takeaway**: **Decorators hide the reassignment dance**.


> [!question]- "<font color="#000000">Decorators hide the reassignment dance</font>" - What do you mean by this ?
> 
> The phrase "decorators hide the reassignment dance" refers to how decorators simplify the process of setting up properties in Python, eliminating the need for explicit reassignment of property objects.
> 
> In traditional property setup without decorators, you often have to manually create a property object and then incrementally add getters, setters, or deleters by reassigning the property object to itself with each new method. This can be seen as a bit of a "dance" because you are repeatedly updating the same property by chaining methods.
> 
> Here's what that process might look like without decorators:
> 
> ```python
> class MyClass:
>     def __init__(self, language):
>         self._language = language
>     
>     def get_language(self):
>         return self._language
>     language = property(get_language)  # Create initial property with getter
> 
>     def set_language(self, value):
>         self._language = value
>     language = language.setter(set_language)  # Reassign to add setter
> ```
> 
> As you can see, there's a lot of manual re-assignment involved.
> 
> However, using decorators streamlines this process:
> 
> ```python
> class MyClass:
>     def __init__(self, language):
>         self._language = language
> 
>     @property
>     def language(self):
>         return self._language
> 
>     @language.setter
>     def language(self, value):
>         self._language = value
> ```
> 
> With decorators like `@property` and `@<property_name>.setter`, you don't need to manually reassign the property object after adding each method. The decorator syntax automatically handles this behind-the-scenes reassignment for you. This makes your code cleaner and easier to read while still providing all the functionality of properties with getters and setters.

## Adding Docs and More

Include a docstring with the getter:
```python
class MyClass:
    def __init__(self, language):
        self._language = language
    
    @property
    def language(self):
        """The language value."""
        return self._language
    
    @language.setter
    def language(self, value):
        self._language = value
```

- **`help(MyClass)`**: Shows "language: The language value."

**Key Point**: **Decorators support docstrings** for clarity.
