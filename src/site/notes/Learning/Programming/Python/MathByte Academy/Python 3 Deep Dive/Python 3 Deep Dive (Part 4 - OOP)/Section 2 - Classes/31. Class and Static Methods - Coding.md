# Class and Static Methods with a Timer Example

## How Functions Behave in Classes

By default, a function in a class binds to the instance when called from it:

```python
class Person:
    def hello(arg="default"):
        print(f"hello with arg={arg}")

Person.hello()      # hello with arg=default (plain function)
p = Person()
p.hello()           # hello with arg=<Person object> (bound method)
```

- **Class Call**: `Person.hello()` treats it as a function.
- **Instance Call**: `p.hello()` binds `p` as `self`.

**Key Point**: **Instance access binds functions** by default.

---

## Class Methods: Binding to the Class

Use `@classmethod` to bind a method to the class, not the instance:

```python
class MyClass:
    def hello(self):
        print("Regular hello")
    
    def instance_hello(self):
        print(f"Instance hello from {self}")
    
    @classmethod
    def class_hello(cls):
        print(f"Class hello from {cls.__name__}")

m = MyClass()
MyClass.hello()         # Regular hello (function)
# m.hello()             # TypeError: missing 'self'
m.instance_hello()      # Instance hello from <MyClass object>
MyClass.class_hello()   # Class hello from MyClass
m.class_hello()         # Class hello from MyClass
```

- **`cls`**: Receives the class (`MyClass`), not the instance.
- **Consistency**: Always class-bound, whether called via `MyClass` or `m`.

**Takeaway**: **Class methods focus on class-level logic**.

---

## Static Methods: No Binding

Use `@staticmethod` for functions that stay unbound:

```python
class MyClass:
    @staticmethod
    def static_hello():
        print("Static method called")

m = MyClass()
MyClass.static_hello()  # Static method called
m.static_hello()        # Static method called
```

- **No Args**: No `self` or `cls` injected—pure function.
- **Debate**: Some say these belong at the module level instead.

**Key Point**: **Static methods ignore binding**, acting standalone.

## Mixing Them in a Timer Class

Let’s build a `Timer` class with all three method types:

```python
from datetime import datetime, timezone, timedelta
from time import sleep

class TimerError(Exception):
    """Custom exception for Timer class."""

class Timer:
    tz = timezone.utc  # Class-level timezone
    
    @classmethod
    def set_tz(cls, offset, name):
        cls.tz = timezone(timedelta(hours=offset), name)
    
    @staticmethod
    def current_dt_utc():
        return datetime.now(timezone.utc)
    
    @classmethod
    def current_dt(cls):
        return datetime.now(cls.tz)
    
    def __init__(self):
        self._time_start = None
        self._time_end = None
    
    def start(self):
        self._time_start = self.current_dt_utc()
        self._time_end = None  # Reset end time
    
    def stop(self):
        if self._time_start is None:
            raise TimerError("Timer must be started before stopping")
        self._time_end = self.current_dt_utc()
    
    @property
    def start_time(self):
        if self._time_start is None:
            raise TimerError("Timer has not been started")
        return self._time_start.astimezone(self.tz)
    
    @property
    def end_time(self):
        if self._time_end is None:
            raise TimerError("Timer has not been stopped")
        return self._time_end.astimezone(self.tz)
    
    @property
    def elapsed(self):
        if self._time_start is None:
            raise TimerError("Timer must be started for elapsed time")
        end = self._time_end if self._time_end else self.current_dt_utc()
        return (end - self._time_start).total_seconds()

# Usage
t1 = Timer()
t1.start()
sleep(2)
t1.stop()
print(f"Start: {t1.start_time}  End: {t1.end_time}  Elapsed: {t1.elapsed} sec")

t2 = Timer()
t2.start()
sleep(3)
t2.stop()
Timer.set_tz(-7, "MST")  # Affects all instances
print(f"T2 Start: {t2.start_time}  End: {t2.end_time}  Elapsed: {t2.elapsed} sec")
```

### Breakdown
- **Instance Methods (`start`, `stop`)**: Manage instance-specific timing.
- **Class Method (`set_tz`)**: Sets `tz` for all instances via `cls`.
- **Static Method (`current_dt_utc`)**: Generic UTC time, no binding needed.
- **Properties**: Convert UTC times to the class `tz` for display.

**Output Example**:
```
Start: 2025-03-24 12:00:00+00:00  End: 2025-03-24 12:00:02+00:00  Elapsed: 2.0 sec
T2 Start: 2025-03-24 05:00:03-07:00  End: 2025-03-24 05:00:06-07:00  Elapsed: 3.0 sec
```

**Takeaway**: **Mix methods and properties** for flexible class design.

> [!question]- What is `timezone.utc`, `timezone(timedelta(hours=offset), name)`, `datetime.now(timezone.utc)` and `datetime.now(cls.tz)` ?
> Let me explain these terms from your `Timer` class example, all related to Python's `datetime` module:
> 
> ### 1. `timezone.utc`
> - **What it is**: A predefined `timezone` object representing Coordinated Universal Time (UTC), provided by the `datetime` module.
> - **Details**: It’s a constant with a zero offset from UTC (no hours/minutes shift).
> - **Use**: Sets a time zone to UTC for consistent, universal timekeeping.
> - **Example**: 
>   ```python
>   from datetime import timezone
>   print(timezone.utc)  # UTC
>   ```
> 
> ### 2. `timezone(timedelta(hours=offset), name)`
> - **What it is**: Creates a custom `timezone` object with a specified UTC offset and name.
> - **Arguments**:
>   - `timedelta(hours=offset)`: A `timedelta` object defining the offset from UTC (e.g., `-7` for MST, Mountain Standard Time).
>   - `name`: A string label for the time zone (e.g., `"MST"`).
> - **Details**: Allows you to define time zones manually, like `-7` hours for MST or `+1` for CET.
> - **Use**: Used in `set_tz` to configure the class-level `tz` attribute.
> - **Example**:
>   ```python
>   from datetime import timezone, timedelta
>   tz = timezone(timedelta(hours=-7), "MST") # "MST" is just a name or label for my custom timezone , it could be anything like "MKS" or "ABC"
>   print(tz)  # MST
>   ```
> 
> ### 3. `datetime.now(timezone.utc)`
> - **What it is**: Returns the current UTC time as a `datetime` object, stamped with the `timezone.utc` time zone.
> - **Details**: Captures the present moment in UTC, ensuring no local system time zone interference.
> - **Use**: In `current_dt_utc` and `start/stop` methods to record times in a universal format.
> - **Example**:
>   ```python
>   from datetime import datetime, timezone
>   utc_now = datetime.now(timezone.utc)
>   print(utc_now)  # e.g., 2025-03-24 12:00:00+00:00
>   ```
> 
> ### 4. `datetime.now(cls.tz)`
> - **What it is**: Returns the current time as a `datetime` object, stamped with the class’s `tz` time zone (e.g., `MST` or `UTC`).
> - **Details**: Uses the `tz` class attribute set by `set_tz` or defaulting to `timezone.utc`.
> - **Use**: In `current_dt` to show the current time adjusted to the class’s time zone.
> - **Example**:
>   ```python
>   from datetime import datetime, timezone, timedelta
>   class Timer:
>       tz = timezone(timedelta(hours=0), "IST")
>   
>   print(datetime.now(Timer.tz))  # e.g., 2025-03-24 05:00:00-07:00
>   print(Timer.tz) # IST
>   ```
> 
> ### Key Differences
> - **`timezone.utc` vs. `timezone(timedelta, name)`**: `utc` is a fixed UTC zone; the latter is customizable.
> - **`datetime.now(timezone.utc)` vs. `datetime.now(cls.tz)`**: Both get "now," but the former is always UTC, while the latter uses the class’s time zone.
> 
> These tools help the `Timer` class manage time consistently—storing in UTC internally (`start`, `stop`) and displaying in the user’s time zone (`start_time`, `end_time`, `elapsed`). Let me know if you need more clarification!
