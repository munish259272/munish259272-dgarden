# Slots and Inheritance: A Comparison

## No Slots Baseline

Start with a simple hierarchy without slots:

```python
class Person:
    def __init__(self, name):
        self.name = name

class Student(Person):
    pass

s = Student("Alex")
print(s.__dict__)  # {'name': 'Alex'}
```

- **Storage**: `name` goes into the instance dictionary.
- **Key Point**: **No slots means dictionaries by default**.

---

## Slots in Base Class

Add slots to the parent:

```python
class Person:
    __slots__ = ('name',)
    def __init__(self, name):
        self.name = name

class Student(Person):
    pass

s = Student("Alex")
s.age = 18
print(s.__dict__)  # {'age': 18}
print(s.name)      # Alex
# print(s.__dict__['name'])  # KeyError: 'name' not in dict
```

- **Behavior**: `name` uses slots from `Person`; `age` uses `__dict__`.
- **Takeaway**: **Subclasses get slots + a dictionary unless specified**.

---

## Slots in Subclass Too

Add slots to the subclass:

```python
class Person:
    __slots__ = ('name',)
    def __init__(self, name):
        self.name = name

class Student(Person):
    __slots__ = ('school', 'student_number')
    def __init__(self, name, school, student_number):
        super().__init__(name)
        self.school = school
        self.student_number = student_number

s = Student("James Bond", "MI6 Prep", "007")
print(s.name, s.school, s.student_number)  # James Bond MI6 Prep 007
# s.__dict__  # AttributeError: no dictionary
# s.age = 18  # AttributeError: 'age' not in slots
```

- **Rule**: Add only new slots (`school`, `student_number`), not `name`.
- ⚠️ **Warning**: Redefining `name` in `Student.__slots__` hides `Person`’s slot—avoid it!
- **Key Point**: **All slots = no dictionary**.

---

## Slots in Subclass Only

Reverse it—slots in subclass, not parent:

```python
class Person:
    def __init__(self, name):
        self.name = name

class Student(Person):
    __slots__ = ('school', 'student_number')
    def __init__(self, name, school, student_number):
        super().__init__(name)
        self.school = school
        self.student_number = student_number

s = Student("James", "MI6 Prep", "007")
print(s.__dict__)  # {'name': 'James'}
print(s.school)    # MI6 Prep
```

- **Result**: `school` and `student_number` use slots; `name` uses `__dict__`.
- **Takeaway**: **Parent’s dictionary persists**.

---

## Slots vs. Properties as Descriptors

Both use **data descriptors**:

```python
class Person:
    __slots__ = ('_name', 'age')
    def __init__(self, name, age):
        self.name = name
        self.age = age
    @property
    def name(self):
        return self._name
    @name.setter
    def name(self, value):
        self._name = value

p = Person("Eric", 78)
print(p.name, p.age)  # Eric 78
# p.__dict__  # AttributeError: no dictionary
print(type(Person.name), type(Person.age)) # <class 'property'> <class 'member_descriptor'>
print(hasattr(Person.name, '__get__'), hasattr(Person.name, '__set__'))  # True True (property descriptor)
print(hasattr(Person.age, '__get__'), hasattr(Person.name, '__get__'))   # True True (slot descriptor)
```

- **Descriptors**: Implement `__get__`, `__set__`, etc.
- **Slots**: Auto-generate descriptors for `age`.
- **Properties**: Manual descriptors for `name`.
- **Key Point**: **Slots are built-in descriptors**.

---

## Mixing Slots and Dictionaries

Enable both in one class:

```python
class Person:
    __slots__ = ('name', '__dict__')
    def __init__(self, name, age):
        self.name = name
        self.age = age

p = Person("Alex", 18)
print(p.name, p.age)  # Alex 18
print(p.__dict__)     # {'age': 18}
p.grade = "A"
print(p.__dict__)     # {'age': 18, 'grade': 'A'}
```

- **Trick**: Add `__dict__` to `__slots__`.
- **Trade-Off**: `name` uses slots; `age` uses `__dict__`—less memory saving.
- **Takeaway**: **Hybrid approach balances flexibility and efficiency**.
