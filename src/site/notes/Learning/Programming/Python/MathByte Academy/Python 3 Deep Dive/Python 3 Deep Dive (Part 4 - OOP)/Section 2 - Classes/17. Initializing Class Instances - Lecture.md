# Initializing Class Instances in Python

## Default Instance Creation

When we instantiate a class, Python handles two steps:
1. **Creates** a new instance.
2. **Initializes** its namespace.

Example:
```python
class MyClass:
    language = "Python"
```

Instantiate it:
```python
obj = MyClass()
```

- **Callable**: `MyClass()` triggers Python’s built-in process.
- **Namespace**: `obj.__dict__` starts as `{}` (empty).

**Key Point**: Python **automatically sets up an instance** with an empty namespace.

---

## Custom Initialization with `__init__`

We can override the default initialization using **`__init__`**:
```python
class MyClass:
    language = "Python"
    def __init__(self, version):
        self.version = version
```

language is a class attribute -> in class namespace  and **init** is a class attribute -> in class namespace (as a function) but when we call `obj = MyClass("3.7")` python creates a new object ( using `__new__` auomatically)

Create an instance:
```python
obj = MyClass("3.7")
print(obj.__dict__)  # {'version': '3.7'}
```

- **`__init__`**: A **bound method** called after the instance is created.
- **`self`**: The newly created instance, passed automatically.
- **`version`**: An argument we provide.

**Takeaway**: **`__init__` customizes the instance** post-creation.

> ### Question Breakdown
> 
> You’re asking:
> 1. If a class has **no `__init__`** (e.g., `class MyClass: pass`), does `obj = MyClass()`:
>    - Initialize `obj` with an empty dictionary (`obj.__dict__`)?
>    - Create a `__init__` bound method inherited from the `object` class?
> 2. If a class has a **custom `__init__`**, does `obj = MyClass()` create a bound `__init__` method, and why don’t we call `obj.__init__()` explicitly?
> 
> ---
> 
> ### Answer
> 
> #### 1. Class Without `__init__` (e.g., `class MyClass: pass`)
> 
> When you define a class without an `__init__` method, Python still provides default behavior for instance creation and initialization, inherited from the **`object` class**, which is the base class for all classes in Python (implicitly, unless specified otherwise).
> 
> **What Happens When You Instantiate `obj = MyClass()`?** 📌:
> - **Instance Creation**:
>   - Python calls the `__new__` method (inherited from `object`) to create a new instance of `MyClass`. This allocates memory for the instance and sets up its basic structure.
>   - `__new__` is a static method that returns the new instance.
> - **Instance Initialization**:
>   - After `__new__` creates the instance, Python automatically calls the `__init__` method (if it exists) to initialize the instance.
>   - Since `MyClass` doesn’t define `__init__`, Python uses the default `__init__` from the `object` class.
>   - The `object.__init__` method does **nothing significant**—it accepts the instance (`self`) and performs no additional initialization, leaving the instance’s namespace (`__dict__`) empty unless class attributes or other methods add to it.
> 
> **Does `obj` Get an Empty Dictionary (`obj.__dict__`)?** ✅:
> - Yes, by default, a new instance of a class has an empty `__dict__` for its **instance namespace**, which stores instance-specific attributes.
> - When `obj = MyClass()` is executed, the instance is created with `obj.__dict__ = {}` because no attributes are set during initialization (since `object.__init__` adds nothing).
> - Example:
>   ```python
>   class MyClass:
>       pass
> 
>   obj = MyClass()
>   print(obj.__dict__)  # Output: {}
>   ```
> 
> **Is a `__init__` Bound Method Created?** 🎯:
> - **No, a bound `__init__` method is not created for the instance.** Here’s why:
>   - The `__init__` method exists as a **class attribute** on `MyClass`, inherited from `object` (i.e., `MyClass.__init__` points to `object.__init__`).
>   - When `obj = MyClass()` is called, Python invokes `MyClass.__init__(obj)` internally to initialize the instance, but this is a **one-time call** during instantiation.
>   - After instantiation, the `__init__` method remains a **method of the class** (`MyClass.__init__`), not a bound method attached to the instance `obj`.
>   - You can verify this:
>     ```python
>     class MyClass:
>         pass
> 
>     obj = MyClass()
>     print(hasattr(obj, '__init__'))  # Output: False
>     print(hasattr(MyClass, '__init__'))  # Output: True
>     print(MyClass.__init__ is object.__init__)  # Output: True
>     ```
>   - The `__init__` method is not “bound” to the instance (i.e., no `obj.__init__` exists). Instead, it’s a class-level method that Python calls automatically during instantiation.
> 
> **Role of `object` Class**:
> - All Python classes implicitly inherit from `object` unless another base class is specified.
> - The `object` class provides a default `__init__` method that does nothing (essentially a no-op):
>   ```python
>   class object:
>       def __init__(self):
>           pass  # No initialization logic
>   ```
> - When `MyClass` lacks `__init__`, `MyClass.__init__` is `object.__init__`, and it ensures the instance is created with an empty `__dict__` (unless modified by other means, like class attributes or descriptors).
> 
> ---
> 
> #### 2. Class With a Custom `__init__`
> 
> When you define a custom `__init__` method, you override the default initialization behavior, allowing you to set instance attributes during creation.
> 
> **What Happens When You Instantiate `obj = MyClass("3.7")`?** 📌:
> - **Instance Creation**:
>   - Python calls `__new__` (typically inherited from `object`) to create the instance.
> - **Instance Initialization**:
>   - Python calls your custom `__init__` method, passing the newly created instance (`self`) and any arguments (e.g., `version`).
>   - The custom `__init__` sets instance attributes, populating `obj.__dict__`.
> - Example:
>   ```python
>   class MyClass:
>       def __init__(self, version):
>           self.version = version
> 
>   obj = MyClass("3.7")
>   print(obj.__dict__)  # Output: {'version': '3.7'}
>   ```
> 
> **Does `obj = MyClass("3.7")` Create a Bound `__init__` Method?** 🎯:
> - **No, a bound `__init__` method is not created for the instance.** Here’s why:
>   - The `__init__` method is a **class attribute** (a function defined in the class), stored in `MyClass.__init__`.
>   - When you call `MyClass("3.7")`, Python internally calls `MyClass.__init__(obj, "3.7")` to initialize the instance, but this is a **one-time operation** during instantiation.
>   - After instantiation, `__init__` remains a **method of the class**, not a bound method attached to `obj`. The instance does not have its own `__init__` method:
>     ```python
>     class MyClass:
>         def __init__(self, version):
>             self.version = version
> 
>     obj = MyClass("3.7")
>     print(hasattr(obj, '__init__'))  # Output: False
>     print(hasattr(MyClass, '__init__'))  # Output: True
>     ```
>   - The `__init__` method is **bound to the class**, and Python’s instantiation process handles calling it automatically. This is why you don’t need to call `obj.__init__()` explicitly.
> 
> **Why Don’t We Call `obj.__init__()` Explicitly?** ⚠️:
> - The `__init__` method is designed to be called **automatically** by Python during instance creation (after `__new__`).
> - Calling `obj.__init__(arg)` manually is possible but rare and generally discouraged:
>   - It reinitializes the instance, potentially overwriting existing attributes:
>     ```python
>     obj = MyClass("3.7")
>     print(obj.__dict__)  # Output: {'version': '3.7'}
>     obj.__init__("3.8")  # Manually reinitialize
>     print(obj.__dict__)  # Output: {'version': '3.8'}
>     ```
>   - Manual calls can lead to unexpected behavior, as `__init__` is meant for one-time setup.
> - Python’s class instantiation (`MyClass()`) handles `__init__` internally, making explicit calls unnecessary.
> 
> **Custom `__init__` vs. `object.__init__`**:
> - With a custom `__init__`, you define the initialization logic (e.g., setting `self.version`).
> - Without `__init__`, `object.__init__` is used, which does nothing, leaving `__dict__` empty unless other attributes are set (e.g., via class attributes or later assignments).
> 
> ---
> 
> ### Key Takeaways
> 
> **Without `__init__` (e.g., `class MyClass: pass`)** 🎯:
> - **Empty `__dict__`**: `obj = MyClass()` creates an instance with `obj.__dict__ = {}`, as `object.__init__` adds no attributes.
> - **No Bound `__init__`**: The `__init__` method is inherited from `object` and exists as a class attribute (`MyClass.__init__`), not a bound method on `obj`. It’s called automatically during instantiation.
> - **Role of `object`**: Provides a default `__init__` that does nothing, ensuring minimal initialization.
> 
> **With Custom `__init__`** 🎯:
> - **Populated `__dict__`**: `obj = MyClass("3.7")` initializes `obj.__dict__` with attributes set in `__init__` (e.g., `{'version': '3.7'}`).
> - **No Bound `__init__`**: The custom `__init__` is a class attribute (`MyClass.__init__`), not a bound method on `obj`. It’s called automatically during instantiation.
> - **No Explicit Call**: You don’t call `obj.__init__()` because Python handles it during `MyClass()`.
> 
> **Why No `obj.__init__()`?** 📌:
> - `__init__` is a **class-level method** invoked by Python’s instantiation process (`__new__` → `__init__`).
> - It’s not bound to the instance as a method; it’s a one-time initialization step.
> - Manual calls are possible but unconventional and risky (e.g., overwriting state).
> 
> **General Insight** ✅:
> - In both cases, `__init__` (whether from `object` or custom) is a **class method** called automatically. The instance’s `__dict__` reflects attributes set during or after initialization. The `object` class ensures a default, minimal `__init__` when none is provided.
>

## How It Works

### Steps:
1. **Instance Creation**: Python makes `obj` with an empty `__dict__`.
2. **Initialization**: Calls `obj.__init__("3.7")`, binding `self` to `obj`.

Equivalent manual call:
```python
MyClass.__init__(obj, "3.7")
```

- **Result**: Adds `"version": "3.7"` to `obj.__dict__`.
- **Namespace**: `language` stays in `MyClass.__dict__`; `version` is in `obj.__dict__`.

**Key Point**: **`__init__` modifies the instance namespace**, not the class.

---

## Why `self`?

- **Convention**: `self` is the standard name for the instance parameter—universally recognized in Python.
- **Clarity**: `self.version = version` shows we’re working on the instance, unlike `obj` or `instance`.

Alternative (works but unconventional):
```python
def __init__(instance, version):
    instance.version = version
```

**Takeaway**: Use **`self`** for readability and consistency.

---

## Timing and Purpose

- **Order**: 
  1. Instance is created with `__dict__ = {}`.
  2. `__init__` runs as a bound method. #doubt
- **Dynamic**: Like adding class attributes later, we add instance attributes in `__init__` later after the object has already been initialized.

**Key Point**: **`__init__` runs after creation**, initializing the instance.

---

## `__new__` vs. `__init__`

- **`__new__`**: Creates the instance (rarely overridden).
- **`__init__`**: Initializes it (commonly used).

Most of the time, we just need `__init__` to set up the instance state.

**Takeaway**: Focus on **`__init__`** for initialization.
