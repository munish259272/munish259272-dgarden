# Implementing Arithmetic Operators with Special Methods

## Basic Arithmetic Methods

Python lets us customize arithmetic operators using special (dunder) methods:

```python
class Number:
    def __init__(self, value):
        self.value = value
    
    def __add__(self, other):  # +
        return Number(self.value + other.value)
    
    def __sub__(self, other):  # -
        return Number(self.value - other.value)
    
    def __mul__(self, other):  # *
        return Number(self.value * other.value)
    
    def __truediv__(self, other):  # /
        return Number(self.value / other.value)
    
    def __floordiv__(self, other):  # //
        return Number(self.value // other.value)
    
    def __mod__(self, other):  # %
        return Number(self.value % other.value)
    
    def __pow__(self, other):  # **
        return Number(self.value ** other.value)

a = Number(10)
b = Number(3)
print((a + b).value)    # 13
print((a - b).value)    # 7
print((a * b).value)    # 30
print((a / b).value)    # 3.333...
print((a // b).value)   # 3
print((a % b).value)    # 1
print((a ** b).value)   # 1000
```

- **Methods**:
  - `__add__`: Addition (`+`).
  - `__sub__`: Subtraction (`-`).
  - `__mul__`: Multiplication (`*`).
  - `__truediv__`: True division (`/`).
  - `__floordiv__`: Floor division (`//`).
  - `__mod__`: Modulus (`%`).
  - `__pow__`: Exponentiation (`**`).
- **Key Point**: **Dunder methods enable operator overloading**.

## Matrix Multiplication (Python 3.5+)

The `@` operator (matrix multiplication) uses `__matmul__`:

```python
class Matrix:
    def __init__(self, value):
        self.value = value
    
    def __matmul__(self, other):  # @
        return Matrix(self.value * other.value)  # Simplified example

m1 = Matrix(2)
m2 = Matrix(3)
print((m1 @ m2).value)  # 6
```

- **Note**: Python doesn’t use `@` natively (e.g., for lists), but NumPy leverages it.
- **Takeaway**: ** `__matmul__` is for custom matrix ops**.

## Handling Unsupported Operations

Use `NotImplemented` to signal an operation isn’t supported:

```python
class Number:
    def __init__(self, value):
        self.value = value
    
    def __add__(self, other):
        if not isinstance(other, Number):
            return NotImplemented
        return Number(self.value + other.value)
```

- **Why?**: Avoid raising exceptions—lets Python try alternatives.
- **Key Point**: **Return `NotImplemented` for unsupported types**.

## Reflected Operators

For `a + b`, Python tries `__add__` on `a`. If it returns `NotImplemented` and types differ, it tries `__radd__` on `b`:

```python
class Text:
    def __init__(self, text):
        self.text = text
    
    def __radd__(self, other):
        if isinstance(other, int):
            return Text(str(other) + self.text)
        return NotImplemented

t = Text("world")
print((5 + t).text)  # 5world
```

- **Flow**: `int.__add__(Text)` → `NotImplemented` → `Text.__radd__(int)`.
- **Applies To**: `__rsub__`, `__rmul__`, `__rtruediv__`, etc.
- **Takeaway**: **Reflected methods handle swapped operands**.

## In-Place Operators

In-place ops (e.g., `+=`) use methods like `__iadd__`:

```python
class Number:
    def __init__(self, value):
        self.value = value
    
    def __iadd__(self, other):  # +=
        self.value += other.value
        return self

n = Number(10)
n += Number(5)
print(n.value)  # 15
```

- **Methods**: `__isub__` (`-=`), `__imul__` (`*=`), `__itruediv__` (`/=`), etc.
- **Key Point**: **In-place methods modify and return self**.

## Unary Operators

Unary operators work on one operand:

```python
class Number:
    def __init__(self, value):
        self.value = value
    
    def __neg__(self):  # -a
        return Number(-self.value)
    
    def __pos__(self):  # +a
        return Number(+self.value)
    
    def __abs__(self):  # abs(a)
        return Number(abs(self.value))

n = Number(-3)
print((-n).value)  # 3
print((+n).value)  # -3
print(abs(n).value)  # 3
```

- **Methods**:
  - `__neg__`: Negation (`-`).
  - `__pos__`: Positive (`+`).
  - `__abs__`: Absolute value (`abs()`).
- **Takeaway**: **Unary methods customize single-operand ops**.
