# Exploring Class Attributes in Python

## Creating a Class with Attributes

Let’s define a class:
```python
class Program:
    language = "Python"
    version = 3.6
```

- **Focus**: These are **class attributes**, part of the class object itself, not instances.
- **`Program`**: A **type** (its `type` is `<class 'type'>`) and an **object**.

**Key Point**: **Class attributes** are set directly in the class definition.

---

## Built-In Attributes

Python adds attributes automatically:
```python
print(Program.__name__)  # "Program"
print(type(Program))     # <class 'type'>
```

- **`__name__`**: A built-in property showing the class name.

---

## Accessing Attributes

### Dot Notation
Simple and common:
```python
print(Program.language)  # "Python"
print(Program.version)   # 3.6
```

### `getattr()`
A built-in function for flexibility:
```python
value = getattr(Program, "version")  # 3.6
```

- **Syntax**: `getattr(object, "attribute_name", [default])`.
- **Default**: Avoids exceptions:
```python
value = getattr(Program, "x", "N/A")  # "N/A" (no "x" exists)
```
- Without default: `getattr(Program, "x")` raises `AttributeError`.

**Takeaway**: Use **dot notation** for ease, **`getattr()`** for safety.

---

## Setting Attributes

### Dot Notation
Update or add attributes:
```python
Program.version = 3.7  # Updates version
Program.x = 100        # Adds new attribute
print(Program.version)  # 3.7
print(Program.x)       # 100
```

### `setattr()`
Equivalent function:
```python
setattr(Program, "version", 3.6)  # Sets version back
setattr(Program, "y", "hello")    # Adds new attribute
```

- **Dynamic**: Python allows **runtime additions** to custom objects.

**Key Point**: **Classes are mutable**—modify state anytime.

---

## Limits of Dynamic Changes

Built-ins resist modification:
```python
a = "hello"
a.x = 100         # Raises AttributeError
str.x = 100       # Raises AttributeError
```

- **Custom objects**: Flexible.
- **Built-ins**: Locked down.

**Takeaway**: **Custom classes support dynamic attributes**, built-ins don’t.

---

## Where State Lives

Class state is stored in **`__dict__`**:
```python
print(Program.__dict__)
# Output: mappingproxy({'__module__': '__main__', 'language': 'Python', 'version': 3.6, 'y': 'hello', ...})
```

- **Mapping Proxy**: A **read-only dictionary-like object**.
- **Contents**: Key-value pairs (e.g., `"language": "Python"`).

**Key Point**: **State is in a namespace**, tracked by `__dict__`.

---

## Modifying State

Update or add via:
```python
setattr(Program, "x", -100)  # Updates x
Program.y = "hi"             # Updates y
print(Program.__dict__)      # Reflects changes
```

- **String Keys**: Attribute names are always strings, often interned for speed.

---

## Deleting Attributes

Remove attributes with:
### `delattr()`
```python
delattr(Program, "x")  # Removes x
```

### `del` Keyword
```python
del Program.y          # Removes y
```

- **Result**: Gone from `__dict__`:
```python
print(Program.__dict__)  # No "x" or "y"
```

**Takeaway**: **Attributes are removable** dynamically.

---

## Using `__dict__`

Inspect state:
```python
print(Program.__dict__["language"])  # "Python"
print(list(Program.__dict__.items()))  # Key-value pairs
```

- ⚠️ **Warning**: Don’t rely on `__dict__` for access—use `getattr()` or dot notation. Not all attributes are here (e.g., `__name__` isn’t).
- **Use Case**: Debugging.

**Key Point**: **`__dict__` is read-only**, but reflects changes indirectly.

---

## Read-Only Nature

Direct mutation fails:
```python
Program.__dict__["language"] = "Java"  # Raises TypeError: mappingproxy doesn’t support assignment
```

- Use **`setattr()`** or **dot notation** instead.
