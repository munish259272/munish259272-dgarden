# Exploring Python’s Exception System

## Exceptions as Objects

Exceptions are **class instances**:

```python
e = Exception()
print(type(e))           # <class 'Exception'>
print(isinstance(e, Exception))  # True
print(isinstance(e, BaseException))  # True
```

- **Fact**: `Exception` inherits from `BaseException`.
- **Key Point**: **Exceptions are objects, not just events**.

---

## Triggering vs. Creating

Creating an instance **doesn’t raise it**:

```python
x = IndexError("Out of range")
print(x)  # "Out of range"
```

- **Raising**: Triggers workflow:
```python
raise IndexError("Out of range")  # Starts propagation
```
- **Key Point**: **`raise` activates the exception**.

---

## Exception Hierarchy

Python’s exceptions form a **tree**:

- **Root**: `BaseException`.
- **Key Branch**: `Exception` (most inherit here).
- **Example**:
  ```python
  print(issubclass(IndexError, LookupError))  # True
  print(issubclass(LookupError, Exception))   # True
  ```
- **Chain**: `IndexError` < `LookupError` < `Exception` < `BaseException`.
- **Key Point**: **Hierarchy defines relationships**.

---

## Propagation Workflow

Exceptions **bubble up** if unhandled:

```python
l = [1, 2]
l[5]  # IndexError propagates to caller (e.g., Jupyter)
```

- **Result**: Stack trace shows origin and path.
- **Key Point**: **Unhandled exceptions escalate**.

---

## Basic Handling

Catch with `try`/`except`:

```python
try:
    l = [1, 2]
    l[5]
except IndexError as e:
    print(f"Error: {e.__class__.__name__}: {e}")  # Error: IndexError: list index out of range
```

- **Guarded**: Code in `try` is watched.
- **Handled**: `except` stops propagation.
- **Key Point**: **Specific catches enable recovery**.

### Broader Catch
```python
try:
    l[5]
except LookupError as e:
    print(f"Caught: {e}")  # Caught: list index out of range
```

- **Works**: `IndexError` is a `LookupError`.
- **Key Point**: **Hierarchy allows flexible catching**.

---

## Avoiding Broad Catches

Broad catches like `Exception` can **hide issues**:

```python
try:
    l[5]
except Exception as e:
    print("Something went wrong")
```

- **Problem**: Catches *everything*—hard to recover meaningfully.
- ⚠️ **Caution**: Avoid unless intentional.

### Bare Catch
```python
try:
    l[5]
except:
    print("Exception occurred")
```

- **Worse**: No type, no object—total mystery.
- **Key Point**: **Specificity aids debugging**.

---

## Stack Trace Insight

Trace shows **call path**:

```python
def func3():
    raise ValueError("Some custom message")
def func2():
    func3()
def func1():
    func2()

func1()  # Trace: func3 → func2 → func1
```

- **Output**: Bottom (origin) to top (caller).
- **Key Point**: **Trace pinpoints the source**.

### Handling in Flight
```python
def func2():
    try:
        func3()
    except ValueError:
        print("Error silenced")
```

- **Result**: Stops at `func2`, no trace.
- **Key Point**: **Handling silences propagation**.

---

## Broad Catch Pitfalls

Example with a generator:

```python
def square(seq, idx):
    return seq[idx] ** 2

def squares(seq, max_n):
    for i in range(max_n):
        try:
            yield square(seq, i)
        except Exception:  # Too broad!
            return  # Stops generator

l = [1, 2, "3"]
print(list(squares(l, 10)))  # [1, 4] (stops early)
```

- **Issue**: `TypeError` from `"3" ** 2` caught, hides intent.
- **Fix**:
```python
        except IndexError:  # Specific
            return
```
- **Result**: `[1, 4, 9]`, `TypeError` propagates.
- **Key Point**: **Broad catches mask bugs**.
