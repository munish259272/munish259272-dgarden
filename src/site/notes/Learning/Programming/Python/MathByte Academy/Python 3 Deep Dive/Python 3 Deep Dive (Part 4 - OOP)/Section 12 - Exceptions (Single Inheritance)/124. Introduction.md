# Understanding Exceptions in Python

## What Are Exceptions?

Exceptions signal **special conditions**, not just errors:

- **Example**: `StopIteration`—raised when an iterator is done.
- **Not an Error**: It’s expected, not a failure.
- **Key Point**: **Exceptions flag events to handle**.

---

## Exception Propagation

Unhandled exceptions **travel up the call stack**:

- **Flow**: If not caught, they bubble up to the top.
- **End**: Uncaught, they crash the program.
- **Key Point**: **Exceptions demand handling or they escalate**.

---

## Built-in Exception Hierarchy

Python organizes exceptions in a **class hierarchy**:

- **Base**: `BaseException` (root of all exceptions).
- **Common**: `Exception` (most user-facing exceptions inherit here).
- **Examples**: `ValueError`, `TypeError`, `StopIteration`.
- **Key Point**: **Built-ins form a structured tree**.

---

## Handling Exceptions

Catch exceptions with `try`/`except`:

```python
try:
    next(iter([]))  # Empty iterator
except StopIteration:
    print("Done iterating")
```

- **Purpose**: Manage specific conditions gracefully.
- **Key Point**: **Handle to control flow**.

---

## Raising Exceptions

Trigger exceptions with `raise`:

```python
raise ValueError("Invalid input")
```

- **Use**: Signal issues explicitly.
- **Key Point**: **Raise to enforce rules**.

---

## Custom Exception Classes

Create your own for **app-specific needs**:

```python
class MyAppError(Exception):
    def __init__(self, message, code):
        super().__init__(message)
        self.code = code

raise MyAppError("Something went wrong", 42)
```

- **Why**: Tailor exceptions to your logic.
- **Key Point**: **Custom exceptions enhance clarity**.

### Custom Hierarchy
```python
class AppBaseError(Exception):
    pass

class InputError(AppBaseError):
    pass

class TimeoutError(AppBaseError):
    pass
```

- **Structure**: Build a tree for related errors.
- **Key Point**: **Hierarchies organize exceptions**.

### Extending Functionality
```python
class DetailedError(Exception):
    def __init__(self, message, details):
        super().__init__(message)
        self.details = details

raise DetailedError("Failed", {"retry": False})
```

- **Add**: Extra attributes or methods.
- **Key Point**: **Extend for richer context**.
