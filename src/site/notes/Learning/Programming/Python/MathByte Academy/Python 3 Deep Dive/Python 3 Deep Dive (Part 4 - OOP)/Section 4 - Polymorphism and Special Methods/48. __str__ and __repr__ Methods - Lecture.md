# Understanding `__str__` and `__repr__` in Python

## String Representation Basics

Python offers two special methods—`__str__` and `__repr__`—to define string representations of objects. Both return strings, but they serve different purposes.

---

## `__repr__`: Developer’s Tool

`__repr__` is for developers, often showing how to recreate an object:

```python
class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y
    
    def __repr__(self):
        return f"Point({self.x}, {self.y})"

p = Point(3, 4)
print(repr(p))  # Point(3, 4)
```

- **Goal**: Provide a detailed, ideally **reconstructible** representation.
- **Use**: Called by `repr()`—great for debugging or logging internals.
- **Flexibility**: If exact recreation isn’t feasible (e.g., too many args), make it descriptive.

**Key Point**: ** `__repr__` aids development and debugging**.

---

## `__str__`: User-Friendly Display

`__str__` is for display, often simpler and aimed at end users:

```python
class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y
    
    def __str__(self):
        return f"({self.x}, {self.y})"
    
    def __repr__(self):
        return f"Point({self.x}, {self.y})"

p = Point(3, 4)
print(str(p))  # (3, 4)
print(p)       # (3, 4)
```

- **Goal**: A readable, concise string for `print()` or `str()`.
- **Use Case**: User output, logging—less technical than `__repr__`.

**Key Point**: ** `__str__` prioritizes readability** for users.

---

## How They Interact

- **Priority**: `print()` and `str()` call `__str__` first. If missing, they fall back to `__repr__`.
- **Default**: If neither is defined, Python uses the inherited `__repr__` from `object`:

```python
class Point:
    pass

p = Point()
print(repr(p))  # < __main__.Point object at 0x... >
print(p)        # Same as above (no __str__, uses __repr__)
```

- **Fallback**: `object`’s `__repr__` gives the class name and memory address.

**Takeaway**: ** `__str__` overrides `__repr__` for display**, but `__repr__` is the backup.

---

## Why Two Methods?

- **`__repr__`**: Internal, technical—think “developer view.”
- **`__str__`**: External, friendly—think “user view.”
- **Example**:
  ```python
  class Point:
      def __init__(self, x, y):
          self.x = x
          self.y = y
      
      def __str__(self):
          return f"Point at {self.x}, {self.y}"
      def __repr__(self):
          return f"Point({self.x}, {self.y})"

  p = Point(1, 2)
  print(p)        # Point at 1, 2
  print(repr(p))  # Point(1, 2)
  ```

**Key Point**: **Two flavors for two audiences**.
