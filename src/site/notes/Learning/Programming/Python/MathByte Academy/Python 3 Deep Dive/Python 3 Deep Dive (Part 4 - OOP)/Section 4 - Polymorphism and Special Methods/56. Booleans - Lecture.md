# Understanding Booleans in Python

## Truth Values of Built-in Objects

Every Python object has a **truth value**:

- **Numbers**: Non-zero (e.g., `1`, `-5`) is `True`; `0` is `False`.
- **Collections**: Empty (e.g., `[]`, `()`, `{}`, `len() == 0`) is `False`; non-empty is `True`.

```python
print(bool(42))    # True
print(bool(0))     # False
print(bool([1, 2])) # True
print(bool([]))    # False
```

- **Key Point**: **Truthy/falsy depends on content**.

---

## Customizing Truthiness with `__bool__`

Override an object’s truth value using `__bool__`:

```python
class Item:
    def __init__(self, quantity):
        self.quantity = quantity
    
    def __bool__(self):
        return self.quantity > 0

item1 = Item(5)
item2 = Item(0)
print(bool(item1))  # True
print(bool(item2))  # False
```

- **Requirement**: Must return `True` or `False`.
- **Use Case**: Define what “truthy” means for your class.
- **Takeaway**: ** `__bool__` gives direct control**.

---

## Fallback to `__len__`

If `__bool__` isn’t defined, Python checks `__len__`:

```python
class Inventory:
    def __init__(self, items):
        self.items = items
    
    def __len__(self):
        return len(self.items)

inv1 = Inventory([1, 2])
inv2 = Inventory([])
print(bool(inv1))  # True (len > 0)
print(bool(inv2))  # False (len == 0)
```

- **Logic**: `len() == 0` → `False`; otherwise `True`.
- **Similarity**: Matches built-in collections (lists, sets, etc.).
- **Key Point**: ** `__len__` acts as a backup**.

---

## Default Truthiness

No `__bool__` or `__len__`? It’s always `True`:

```python
class Empty:
    pass

obj = Empty()
print(bool(obj))  # True
```

- **Reason**: Default behavior assumes objects exist, so they’re truthy.
- **Override**: Use `__bool__` or `__len__` to change this.
- **Takeaway**: **Custom objects are truthy by default**.
