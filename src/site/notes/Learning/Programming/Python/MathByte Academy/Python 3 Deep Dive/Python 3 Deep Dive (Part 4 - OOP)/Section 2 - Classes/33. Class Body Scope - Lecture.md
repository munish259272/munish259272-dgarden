# Understanding Class Scope in Python

## The Class Body Scope

A class definition creates its own scope, distinct from its containing module. Consider this:

```python
# module1.py
class Python:
    kingdom = "Animalia"
    phylum = "Chordata"
    family = "Pythonidae"
    
    def __init__(self):
        pass
    
    def say_hello(self):
        return "Hello"

p = Python()
```

- **Module Scope**: Contains `Python` (the class) and `p` (an instance).
- **Class Body Scope**: Holds `kingdom`, `phylum`, `family`, `__init__`, and `say_hello`.
- **Key Point**: **The class body has its own namespace**, separate from the module.

---

## Function Scopes: Not Nested in Class

The functions (`__init__`, `say_hello`) are defined in the class, but their scopes aren’t nested within it:

- **Symbols**: `init` and `say_hello` live in the class body namespace.
- **Function Objects**: Their code is scoped to the **module level**, not the class body.

Think of it like this:

```python
# Alternative view
def callable1(self):
    pass

def callable2(self):
    return "Hello"

class Python:
    kingdom = "Animalia"
    __init__ = callable1
    say_hello = callable2
```

- **Takeaway**: **Function scopes align with the module**, not the class body.

## Practical Implications with an Account Class

Let’s see how this affects variable lookup:

```python
class Account:
    comp_frequency = 12  # Compounds monthly
    apr = 0.05          # 5% annual percentage rate
    apy = (1 + apr / comp_frequency) ** comp_frequency - 1  # Annual percentage yield
    
    def instance_method(self):
        return self.apy  # Finds apy via instance
    
    @classmethod
    def class_method(cls):
        return cls.apy  # Finds apy via class
    
    @staticmethod
    def static_method():
        return Account.apy  # Explicitly references class
    
    def broken_method(self):
        return apy  # NameError: 'apy' not found
```

### How It Works
- **Class Body**: `comp_frequency`, `apr`, and `apy` are in the class namespace, so `apy` can reference `apr` and `comp_frequency` directly.
- **Instance Method**: `self.apy` works—it climbs from instance to class.
- **Class Method**: `cls.apy` works—directly accesses the class.
- **Static Method**: `Account.apy` works—explicitly names the class.
- **Broken Method**: Bare `apy` fails—Python looks in the **module scope**, not the class body.

### ⚠️ Warning: Module-Level Conflicts
If `apy` exists in the module scope:

```python
apy = 0.1  # Module-level
class Account:
    apy = 0.05
    def check(self):
        return apy  # Uses module-level 0.1, not class 0.05
```

- **Result**: `check` sees the module’s `apy`, causing subtle bugs.

**Key Point**: **Functions don’t see class scope**—they look outward to the module.
