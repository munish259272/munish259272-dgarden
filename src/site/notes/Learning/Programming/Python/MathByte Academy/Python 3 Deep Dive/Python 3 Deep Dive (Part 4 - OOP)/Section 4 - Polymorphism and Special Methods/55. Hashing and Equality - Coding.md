# Reviewing Hashing and Equality in Python

## Default Behavior from `object`

Every class inherits from `object`, providing default implementations:

```python
class Person:
    pass

p1 = Person()
p2 = Person()
print(id(p1))       # Unique memory address
print(id(p2))       # Different address
print(p1 == p2)     # False (identity-based)
print(hash(p1))     # Hash based on id(p1)
```

- **`__eq__`**: Compares memory addresses (`is`).
- **`__hash__`**: Derived from `id(obj)` (slightly adjusted).
- **`__repr__`**: Basic string like `<__main__.Person object at 0x...>`.
- **Key Point**: **Defaults use identity**, not data.

## Customizing Equality

Override `__eq__` for value-based comparison:

```python
class Person:
    def __init__(self, name):
        self.name = name
    
    def __eq__(self, other):
        if isinstance(other, Person):
            return self.name == other.name
        return NotImplemented
    
    def __repr__(self):
        return f"Person(name={self.name})"

p1 = Person("John")
p2 = Person("John")
p3 = Person("Eric")
print(p1 == p2)  # True (same name)
print(p1 == p3)  # False (different name)
# print(hash(p1))  # TypeError: unhashable
```

- **Effect**: `__eq__` checks `name`, but `__hash__` becomes `None`.
- **Why?**: Python disables hashing when `__eq__` is overridden to avoid mismatches (equal objects must hash the same).
- **Takeaway**: ** `__eq__` alone kills hashability**.

---

## Adding Hashing

Pair `__hash__` with `__eq__`:

```python
class Person:
    def __init__(self, name):
        self.name = name
    
    def __eq__(self, other):
        if isinstance(other, Person):
            return self.name == other.name
        return NotImplemented
    
    def __hash__(self):
        return hash(self.name)
    
    def __repr__(self):
        return f"Person(name={self.name})"

p1 = Person("John")
p2 = Person("John")
d = {p1: "data"}
print(d[p2])  # "data" (same hash)
```

- **`__hash__`**: Hashes `name`, consistent with `__eq__`.
- **Result**: `p1` and `p2` are equal and hashable, usable as dict keys or set elements.
- **Problem**: `name` is mutable—trouble ahead.

## Ensuring Immutability

Make `name` immutable with a property:

```python
class Person:
    def __init__(self, name):
        self._name = name
    
    @property
    def name(self):
        return self._name
    
    def __eq__(self, other):
        if isinstance(other, Person):
            return self.name == other.name
        return NotImplemented
    
    def __hash__(self):
        return hash(self.name)
    
    def __repr__(self):
        return f"Person(name={self.name})"

p1 = Person("Eric")
p2 = Person("Eric")
s = {p1, p2}  # One element (equal hashes)
print(s)      # {Person(name=Eric)}
d = {p1: 42}
print(d[p2])  # 42
```

- **Immutability**: `_name` is pseudo-private; `name` is read-only.
- **Why?**: Mutable keys (e.g., changing `name`) break hash consistency in dictionaries/sets.
- ⚠️ **Warning**: **Hashable objects must be immutable** for fields used in `__eq__` and `__hash__`.
- **Key Point**: **Match `__eq__` and `__hash__` data**.
