# The `object` Class in Python

## Implicit Inheritance from `object`

When you define a class without a parent:

```python
class Person:
    pass
```

- **Behind the Scenes**: Python makes it inherit from `object` implicitly.
- **Explicit Option**: `class Person(object):` works too, but isn’t needed in Python 3.
- **Key Point**: **Every class is a subclass of `object`**.

---

## `object` Naming Convention

- **Lowercase**: `object` uses C-style naming (no camel case) because it’s implemented in C.
- **Contrast**: Python convention is camel case (e.g., `MyClass`), but built-ins like `namedtuple` follow C style too.
- **Type**: `type(object)` returns `<class 'type'>`—it’s a class, not an instance.

---

## Universal Subclassing

All classes inherit from `object`:

```python
class Person:
    pass

class Teacher(Person):
    pass

class Student(Person):
    pass

print(issubclass(Student, object))  # True
print(issubclass(Teacher, object))  # True
print(issubclass(Person, object))   # True
```

- **Chain**: `Student` → `Person` → `object`.
- **Takeaway**: **Even hierarchies trace back to `object`**.

---

## Instances and `object`

Instances inherit too:

```python
s1 = Student()
print(isinstance(s1, Student))  # True
print(isinstance(s1, Person))   # True
print(isinstance(s1, object))   # True
```

- **Why**: If a class is a subclass of `object`, its instances are instances of `object`.

---

## What `object` Provides

Inheriting from `object` gives **default behaviors**:

```python
p = Person()
print(p.__name__)      # Attribute from object
print(repr(p))         # < __main__.Person object at ... >
print(hash(p))         # Some integer (uses id())
print(p == p)          # True (compares id())
```

- **Methods**: `__new__`, `__init__` (default no-op), `__repr__`, `__hash__`, `__eq__`, and more.
- **Override**: Define your own (e.g., `__init__`) to replace defaults.
- **Key Point**: **Empty classes aren’t empty—they inherit from `object`**.
