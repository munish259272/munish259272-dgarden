# Raising Exceptions in Python

## Using `raise`

Trigger exceptions with **`raise`**:

```python
raise ValueError("Custom exception")
```

- **Requirement**: Must be a `BaseException` subclass.
- **Key Point**: **`raise` initiates exception flow**.

### Flexible Arguments
`BaseException` supports **multiple args**:

```python
e = ValueError("Error", "Details")
print(e.args)      # ('Error', 'Details')
print(str(e))      # ('Error', 'Details')
print(repr(e))     # ValueError('Error', 'Details')
```

- **Common Use**: First arg as error message.
- **Key Point**: **`args` stores all inputs**.

---

## Reraising Exceptions

Resume propagation with bare **`raise`**:

```python
try:
    raise ValueError("Problem")
except ValueError as e:
    print("Caught:", e)
    raise  # Reraises "Problem"
```

- **Use Case**: Log then propagate:
  ```python
  try:
      raise ValueError("Issue")
  except:
      print("Logging exception")
      raise  # Continues up the stack
  ```
- **Key Point**: **Bare `raise` resumes flow**.

---

## Nested Exceptions & Tracebacks

Exceptions can **nest**, affecting tracebacks:

```python
try:
    raise ValueError("Outer")
except ValueError:
    try:
        raise TypeError("Inner")
    except TypeError:
        raise KeyError("Final")
```

- **Output**: Shows `KeyError`, `TypeError`, `ValueError` in stack.
- **Key Point**: **Tracebacks reveal nesting**.

### Controlling Tracebacks
Use **`raise ... from`** to **filter**:

- **Hide All Prior**:
  ```python
  try:
      raise ValueError("Outer")
  except ValueError:
      try:
          raise TypeError("Inner")
      except TypeError:
          raise KeyError("Final") from None
  ```
  - Result: Only `KeyError` in traceback.

- **Link Specific**:
  ```python
  try:
      raise ValueError("Outer")  # x1
  except ValueError as x1:
      try:
          raise TypeError("Inner")
      except TypeError:
          raise KeyError("Final") from x1
  ```
  - Result: `KeyError` linked to `ValueError`, skips `TypeError`.

- **Why**: Hide implementation details (e.g., for libraries).
- **Key Point**: **`from` shapes traceback**.
