# Python Enumerations Basics

## Enum Introduction

Python’s `enum` module (from PEP 435, Python 3.4) offers the `Enum` class for **structured constants**:

- **Key Point**: **Enums go beyond raw variables with powerful features**.

---

## Defining an Enum

Create enumerations by subclassing `Enum`:

```python
from enum import Enum

class Color(Enum):
    RED = 1
    GREEN = 2
    BLUE = 3
```

- **Terminology**:
  - `Color`: The **enumeration** (or "enum").
  - `Color.RED`: An **enumeration member**—an object, not just `1`.
- **Key Point**: **Members are instances of the enum class**.

---

## Member Properties

Members have built-in attributes:

```python
print(type(Color.RED))      # <enum 'Color'>
print(isinstance(Color.RED, Color))  # True
print(str(Color.RED))       # Color.RED
print(repr(Color.RED))      # <Color.RED: 1>
print(Color.RED.name)       # 'RED'
print(Color.RED.value)      # 1
```

- **Oddity**: `Color.RED` is a `Color` instance, not an `int`.
- **Key Point**: **Metaprogramming transforms attributes into members**.

---

## Comparing Members

Use `is` for **identity-based equality**:

```python
print(Color.RED is Color.RED)    # True
print(Color.RED is Color.BLUE)   # False
print(Color.RED is 1)            # False
print(Color.RED in Color)        # True
```

- **Note**: `Color.RED != 1`—**members aren’t their values**.
- **Key Point**: **Use `is`, not $==$, for member checks** (faster, customizable).

---

## Hashable Members

Members are **hashable** by default:

```python
pixel = {Color.RED: 255, Color.GREEN: 128}
print(pixel[Color.RED])  # 255
colors_set = {Color.RED, Color.GREEN}
```

- **Use**: Keys in dicts, elements in sets.
- **Key Point**: **Hashability holds unless `__eq__` is overridden**.

---

## Accessing Members

Look up members dynamically:

```python
# By value
print(Color(2))         # Color.GREEN
# By name
print(Color["GREEN"])   # Color.GREEN
print(Color(2) is Color["GREEN"])  # True
```

- **How**: Call the enum (`Color(2)`) or use brackets (`Color["GREEN"]`).
- **Key Point**: **Programmatic access simplifies lookups**.

---

## Iterating Members

Enums are **iterable** with **preserved order**:

```python
print(list(Color))  # [<Color.RED: 1>, <Color.GREEN: 2>, <Color.BLUE: 3>]
class Color(Enum):
    GREEN = 2
    BLUE = 3
    RED = 1
print(list(Color))  # [<Color.GREEN: 2>, <Color.BLUE: 3>, <Color.RED: 1>]
```

- **Note**: Order follows definition, not values.
- **Key Point**: **Definition order drives iteration**.

---

## Members Property

Access all members via `.__members__`:

```python
print(Color.__members__)  # {'RED': <Color.RED: 1>, 'GREEN': ...}
```

- **Returns**: Immutable dict of name-to-member mappings.
- **Key Point**: **Useful for inspection, especially with aliases**.

---

## Immutability

Enums are **constant**:
- **Members**: Can’t add, remove, or replace after definition.
- **Values**: Can’t reassign (e.g., `Color.RED.value = 5` fails), but mutable values (like lists) can be altered internally.
- **Subclassing**: Blocked if members exist, ensuring **fixed membership**.
