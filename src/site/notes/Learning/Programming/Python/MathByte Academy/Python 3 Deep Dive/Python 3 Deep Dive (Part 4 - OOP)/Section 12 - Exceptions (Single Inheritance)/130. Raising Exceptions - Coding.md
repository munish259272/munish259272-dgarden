# Raising Exceptions Deep Dive

## The `raise` Basics

Initiate exceptions with **`raise`**:

```python
raise ValueError("Something went wrong")
```

- **Rule**: Must be a `BaseException` subclass:
  ```python
  class Person: 
	  pass
	  
  raise Person()  # TypeError: must derive from BaseException
  ```
- **Key Point**: **`raise` requires exception types**.

### Flexible Arguments
`BaseException` handles **multiple args**:

```python
e = BaseException("A", "B", "C")
print(e.args)      # ('A', 'B', 'C')
print(str(e))      # ('A', 'B', 'C')
print(repr(e))     # BaseException('A', 'B', 'C')
```

- **Inherited**: Works for `ValueError`, etc.
- **Common**: Single message arg.
- **Key Point**: **`args` captures all inputs**.

### Shorthand
Pass class directly:

```python
try:
    raise ValueError  # Implicitly creates instance
except ValueError as e:
    print(e.args)  # ()
```

- **Key Point**: **No args still works**.

## Reraising Exceptions

Continue propagation with bare **`raise`**:

```python
def div(a, b):
    try:
        return a / b
    except ZeroDivisionError as e:
        print(f"Logging: {repr(e)}")
        raise  # Reraises original exception

div(1, 0)  # Logs, then propagates
```

- **Use**: Intercept (e.g., log) without silencing.
- **Key Point**: **Bare `raise` resumes workflow**.

## Changing Exceptions

Transform exceptions with **`raise`**:

```python
class CustomError(Exception): "Custom exception"

def myfunc(a, b):
    try:
        return a / b
    except ZeroDivisionError as e:
        raise CustomError(*e.args)  # New type, same args

myfunc(1, 0)  # Raises CustomError
```

- **Traceback**: Shows both `CustomError` and `ZeroDivisionError`.
- **Key Point**: **New exception builds on old**.

## Nested Exceptions

Exceptions can **stack**:

```python
try:
    raise ValueError("Level 1")
except ValueError:
    try:
        raise TypeError("Level 2")
    except TypeError:
        raise KeyError("Level 3")
```

- **Output**: `KeyError` atop `TypeError` atop `ValueError`.
- **Key Point**: **Tracebacks grow with nesting**.

### Controlling Tracebacks
Use **`raise ... from`** to **filter**:

- **Squash All**:
  ```python
  try:
    raise ValueError("Level 1")
  except ValueError:
    try:
        raise TypeError("Level 2")
    except TypeError:
        raise KeyError("Level 3") from None
  ```
  - Result: Only `KeyError`.

- **Link Specific**:
  ```python
  try:
    raise ValueError("Level 1")  # e1
  except ValueError as e1:
    try:
      raise TypeError("Level 2")
    except TypeError:
      raise KeyError("Level 3") from e1
  ```
  - Result: `KeyError` from `ValueError`, skips `TypeError`.

- **Why**: Hide implementation details.
- **Key Point**: **`from` shapes visibility**.

## Practical Example

From prior conversion code:

```python
class ConversionError(Exception):  
    pass  
  
  
def convert_int(val):  
    if not isinstance(val, int):  
        raise TypeError  
    if val not in (0, 1):  
        raise ValueError("Integer values 0 or 1 only")  
    return bool(val)  
  
  
def convert_str(val):  
    if not isinstance(val, str):  
        raise TypeError  
  
    v = val.casefold()  
    if v in ("0", "f", "false"):  
        return False  
    if v in ("1", "t", "true"):  
        return True  
    raise ValueError("Valid strings: 0, 1, t, f, true, false")  
  
  
def make_bool(val):  
    try:  
        try:  
            b = convert_int(val)  
        except TypeError:  
            try:  
                b = convert_str(val)  
            except TypeError:  
                raise ConversionError(f"Type {type(val)} inadmissible")  
  
    except ValueError as e:  
        raise ConversionError(f"Value {val} cannot be converted: {e}")  from None
    else:  
        return b

print(make_bool("abc")) 
```

- **Before**: Messy stack with `TypeError`, `ValueError`.
- **After**: Clean `ConversionError` only.
- **Key Point**: **`from None` simplifies output**.
