# Mastering Rich Comparisons in Python

## What Are Rich Comparisons?

Rich comparisons are the operators like `<`, `>`, `>=`, `<=`, $==$, and `!=`. We can customize them in our classes using special methods:

- **`__lt__`**: Less than (`<`)
- **`__le__`**: Less than or equal to (`<=`)
- **`__eq__`**: Equal to ($==$)
- **`__ne__`**: Not equal to (`!=`)
- **`__gt__`**: Greater than (`>`)
- **`__ge__`**: Greater than or equal to (`>=`)

**Key Point**: **Rich comparisons let us define custom ordering**.

---

## How They Work

For `a < b`, Python calls `a.__lt__(b)`:

```python
class Number:
    def __init__(self, value):
        self.value = value
    
    def __lt__(self, other):
        if not isinstance(other, Number):
            return NotImplemented
        return self.value < other.value

a = Number(5)
b = Number(10)
print(a < b)  # True
```

- **Flow**: `a.__lt__(b)` compares `self.value` to `other.value`.
- **Support**: Must return a boolean (or `NotImplemented`).

---

## Reflection in Action

If `__lt__` returns `NotImplemented` and types differ, Python tries the **reflected operator**:

```python
class Number:
    def __init__(self, value):
        self.value = value
    
    def __lt__(self, other):
        return NotImplemented
    
    def __gt__(self, other):
        if not isinstance(other, Number):
            return NotImplemented
        return self.value > other.value

a = Number(5)
b = Number(10)
print(a < b)  # True (via b.__gt__(a))
```

- **Process**:
  1. `a < b` → `a.__lt__(b)` → `NotImplemented`.
  2. Types match, so no reflection here, but if they differed (e.g., `5 < Number(10)`), it’d try `b.__gt__(a)`.
- **Reflections**:
  - `<` → `>`
  - `<=` → `>=`
  - $==$ → $==$
  - `!=` → `!=`
  - `>` → `<`
  - `>=` → `<=`

**Takeaway**: **Reflection flips the comparison** when needed.
