# Exploring Single Inheritance in Python

## Inheritance in Python

Python supports **single and multiple inheritance**:

- **Single Inheritance**: A class inherits from one parent—most common.
- **Multiple Inheritance**: A class can inherit from multiple parents (unlike Java, which restricts to one class but allows multiple interfaces).
- **Complexity**: Multiple inheritance is powerful but trickier—we’ll tackle it later.

For now, we focus on **single inheritance**.

---

## Single Inheritance Basics

A class inherits from one parent, forming a simple chain:

```python
class Animal:
    def speak(self):
        return "Sound"

class Dog(Animal):
    def wag_tail(self):
        return "Tail wags"

d = Dog()
print(d.speak())      # Sound (from Animal)
print(d.wag_tail())   # Tail wags (from Dog)
```

- **Chain**: `Dog` → `Animal`.
- **Key Point**: **Single inheritance builds straightforward hierarchies**.

---

## Implicit `object` Inheritance

Every class implicitly inherits from `object`:

```python
class MyClass:
    pass

m = MyClass()
print(isinstance(m, object))  # True
```

- **Why**: `object` is Python’s root class, providing basics like `__str__`, `__eq__`.
- **Even Here**: `Dog` → `Animal` → `object`.
- **Takeaway**: **All classes tie back to `object`**.

---

## Why Start with Single?

- **Common**: Most inheritance use cases are single-parent.
- **Simpler**: Easier to understand before diving into multiple inheritance’s complexity.
