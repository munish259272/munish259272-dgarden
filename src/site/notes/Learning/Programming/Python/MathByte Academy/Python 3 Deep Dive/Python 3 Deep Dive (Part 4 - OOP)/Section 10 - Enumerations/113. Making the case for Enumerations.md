# Why Use Enumerations?

## Problems with Raw Constants

Defining constants like this has issues:

```python
RED = 1
GREEN = 2
BLUE = 3
COLORS = (RED, GREEN, BLUE)

pixel_color = {RED: 255, GREEN: 128, BLUE: 0}
print(pixel_color[RED])  # 255 (works with 1 too!)
```

- **Downsides**:
  - `RED in COLORS` is `True`, but `1 in COLORS` is also `True`—**confuses intent**.
  - `RED < GREEN` works (1 < 2), but **ordering is meaningless** here.
  - `pixel_color["RED"]` vs. `pixel_color[RED]`—**symbols and values mix messily**.
- **Key Point**: **Plain constants lack control**.

---

## Strings as Constants?

Using strings repeats the problem:

```python
RED = "red"
GREEN = "green"
BLUE = "blue"
COLORS = (RED, GREEN, BLUE)
```

- **Issues**:
  - **Repetition**: `"red"` appears twice—prone to typos.
  - **Mutability**: `RED = "violet"` breaks code elsewhere.
  - **Nonsense**: `RED * 2` gives `"redred"`—**unintended operations**.
- **Key Point**: **Strings don’t enforce uniqueness or intent**.

---

## What We Want

Enumerations provide:
- **Immutable collection** of **constant members**.
- **Unique names** (e.g., `red`, `green`) with **semantic meaning**.
- **Constant values** that can’t change.
- **Unique values** (no accidental duplicates).
- **Restricted operations** (no `red * 2`).
- **Lookup by name** (e.g., `"red" → RED`) or **value** (e.g., `1 → RED`).
- **Iteration** over members.

---

## Class Attempt

A class improves things but falls short:

```python
class Colors:
    RED = 1
    GREEN = 2
    BLUE = 3

print(Colors.RED)  # 1
print(hasattr(Colors, "RED"))  # True
```

- **Limits**:
  - **Value lookup**: How to get `GREEN` from `2`? **Not easy**.
  - **Iteration**: Listing `RED`, `GREEN`, `BLUE`? **Complex**.
  - **Uniqueness**: `GREEN = 1` still possible—**duplicates sneak in**.
- **Key Point**: **Classes lack built-in enum features**.

---

## Aliases in Enumerations

Consider polygons with aliases:

```python
# Want: Poly4 = Rectangle = Square = Rhombus (all 4 sides)
#       Poly3 = Triangle = Equilateral = Isosceles (all 3 sides)
```

- **Goal**: 
  - `Poly4` is the main member; others are **aliases** (same object).
  - Lookup `3` → `Poly3`; lookup `"Square"` → `Poly4`.
- **Benefit**: **Unique values, multiple names**.
- **Key Point**: **Aliases need special handling**—hard with raw classes.
