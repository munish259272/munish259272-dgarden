# Exploring Function Attributes and Binding in Python

## Defining a Function in a Class

Let’s create a class with a function attribute:
```python
class Person:
    def say_hello():
        print("hello")
```

- **`say_hello`**: A **class attribute** that’s a function.
- Check it:
```python
print(Person.say_hello)  # <function Person.say_hello at ...>
print(type(Person.say_hello))  # <class 'function'>
Person.say_hello()  # "hello"
```

**Key Point**: At the class level, it’s just a **function**.

---

## Instance Binding

Create an instance:
```python
p = Person()
print(hex(id(p)))  # e.g., "0x7f8b1c0"
```

Access `say_hello` via the instance:
```python
print(p.say_hello)  # <bound method Person.say_hello of <__main__.Person object at 0x7f8b1c0>>
print(type(p.say_hello))  # <class 'method'>
```

- **Bound Method**: It’s no longer a function—it’s tied to `p`.
- **Different**: `type(p.say_hello) != type(Person.say_hello)` (method vs. function).

Try calling it:
```python
p.say_hello()  # TypeError: say_hello() takes 0 positional arguments but 1 was given
```

- **Error**: Python passes `p` as an argument, but `say_hello` doesn’t expect it.

**Takeaway**: **Instance access transforms functions into methods**.

---

## Understanding the Binding

Redefine with arguments:
```python
class Person:
    def say_hello(*args):
        print(f"say_hello args: {args}")
```

Test it:
```python
Person.say_hello()  # "say_hello args: ()"
p = Person()
print(hex(id(p)))  # e.g., "0x7f8b1c0"
p.say_hello()  # "say_hello args: (<__main__.Person object at 0x7f8b1c0>,)"
```

- **Class Call**: No arguments passed.
- **Instance Call**: `p` is injected as the first argument.

**Key Point**: Python **binds the instance** to the method, passing it as `self`.

---

## Using the Bound Instance

Let’s manipulate the instance:
```python
class Person:
    def set_name(self, new_name): # or def set_name(instance_object, new_name):
        self.name = new_name  # or setattr(self, "name", new_name)
```

Test it:
```python
p = Person()
p.set_name("Alex")
print(p.__dict__)  # {'name': 'Alex'}
```

Manual equivalent:
```python
p = Person()
Person.set_name(p, "Alex")
print(p.__dict__)  # {'name': 'Alex'}
```

- **`self`**: The instance (`p`) that the method is bound to.
- **Convention**: We use `self` (not `instance_object`) for clarity and tradition.

**Takeaway**: Methods access and modify **instance state** via `self`.

## Method Object Properties

Define a method:
```python
class Person:
    def say_hello(self):
        return f"{self} says hello"
```

Explore it:
```python
p = Person()
q = Person()
print(Person.say_hello) # <function Person.say_hello at 0x7fae787d5b20>


try:
	print(p.__self__) 
except AttributeError as e:
	print(e) # 'Person' object has no attribute '__self__'
try: 
	print(p.__func__) 
except AttributeError as e: 
	print(e) # 'Person' object has no attribute '__func_'

m_hello = p.say_hello
print(m_hello)  # <bound method Person.say_hello of <__main__.Person object at 0x7fae787df860>>
print(q.say_hello)  # <bound method Person.say_hello of <__main__.Person object at 0x7fae787defc0>>
print(m_hello.__func__)  # <function Person.say_hello at 0x7fae787d5b20>
print(m_hello.__self__)  # <__main__.Person object at 0x7fae787df860>
print(hex(id(p)) == hex(id(m_hello.__self__)))  # True
```

- **`__func__`**: The original class function.
- **`__self__`**: The bound instance (`p`).

Call it:
```python
print(p.say_hello())  # "<__main__.Person object at ...> says hello"
```

**Key Point**: **Methods are objects** with function and instance references.

> [!question]- Can we say that when we do `p.say_hello`. It creates reference to itself i.e `p` now has a reference to itself using `p.say_hello.__self__` ?
> Yes, we can say that `p.say_hello` creates a reference to a **bound method** that, in turn, holds a reference to `p` itself via `p.say_hello.__self__`. However, let’s clarify this carefully to avoid confusion.
> 
> When you access `p.say_hello`, Python doesn’t just give you the raw function defined in the class—it constructs a **bound method object** on the fly. This bound method is tied to the instance `p`, and its `__self__` attribute explicitly points back to `p`. So, in a sense, `p.say_hello.__self__` is a reference to `p`, but it’s not that `p` inherently "has a reference to itself" in its own structure—it’s the **method object** that maintains this reference.
> 
> Here’s the breakdown:
> 
> ### Code Example
> ```python
> class Person:
>     def say_hello(self):
>         return "Hello"
> 
> p = Person()
> method = p.say_hello  # Creates a bound method
> print(method.__self__)  # <Person object at 0x7fae787df860> (this is p)
> print(method.__self__ is p)  # True
> print(hex(id(p)))  # 0x7fae787df860
> print(hex(id(method.__self__)))  # Same as above i.e 0x7fae787df860
> ```
> 
> ### Explanation
> - **`p.say_hello`**: Accessing this doesn’t modify `p` itself—it generates a **bound method object**.
> - **`__self__`**: This attribute of the bound method holds a reference to the instance (`p`) that it’s bound to.
> - **Reference**: The bound method’s `__self__` points to `p`, meaning the method "knows" which instance it belongs to. This is how Python ensures that `self` in `say_hello(self)` is `p` when called.
> 
> ### Does `p` Reference Itself?
> - Not directly. `p` doesn’t store `p.say_hello` or a reference to itself in its `__dict__`. The reference exists in the **method object** created when you access `p.say_hello`.
> - Each time you access `p.say_hello`, Python dynamically creates this bound method, linking it to `p` via `__self__`.
> 
> ### Verification
> ```python
> print(p.__dict__)  # {} (no reference to say_hello stored here)
> print(p.say_hello.__self__ is p)  # True (the method references p)
> ```
> 
> ### Conclusion
> So, yes, `p.say_hello.__self__` is a reference to `p`, but it’s more accurate to say that **the bound method `p.say_hello` references `p`** rather than `p` having a self-reference in its own state. This binding is what enables instance methods to operate on the specific instance they’re called from.
> 
> **Key Point**: **`p.say_hello` binds `p` to the method**, and `__self__` exposes that reference.

## Dynamic (Monkey Patching)<i class="muc"><span>**Monkey patching in Python** means **dynamically changing or extending a class or module at runtime**, usually to modify or replace methods, functions, or attributes without altering the original source code</span></i> Methods

Add a function at runtime:
```python
class Person:
    def say_hello(self):
        return f"Instance method called from {self}"

p = Person()
Person.do_work = lambda self: f"do_work called from {self}"
```

Check:
```python
print(Person.__dict__)  # Includes 'do_work': <function <lambda> at ...>
print(p.do_work)  # <bound method <lambda> of <__main__.Person object at ...>>
print(p.do_work())  # "do_work called from <__main__.Person object at ...>"
```

- **Bound**: `do_work` becomes a method when called via `p`.

> [!note] After doing `Person.do_work = lambda self: f"do_work called from {self}"` we did not redefine `p` (like `p = Person()`) for `p.do_work` to work. So at this point python looked for `do_work` in the instance and it did not found so it looked in the class and found it.

Add to instance instead:
```python
p.other_func = lambda *args: f"other_func called with {args}"
print(p.__dict__)  # {'other_func': <function <lambda> at ...>}
print(p.other_func)  # <function <lambda> at ...> (not a method)
print(p.other_func())  # "other_func called with ()"
```

- **Not Bound**: Instance-level functions stay plain functions.

**Takeaway**: **Class-level functions become methods**; instance-level ones don’t.

> [!answer]-
> The behavior you're observing with the `Person` class, the `say_hello` method, and the attempts to access `__self__` and `__func__` involves understanding how Python handles instance methods, bound methods, and the descriptor protocol. Let’s break this down step by step to explain when `say_hello` becomes a bound method and why the attribute errors occur.
> 
> ### Code Analysis
> ```python
> class Person:
>     def say_hello(self):
>         return f"Instance method called from {self}"
> 
> p = Person()
> p.say_hello()  # This works and returns "Instance method called from <__main__.Person object at ...>"
> print(p.__self__)  # AttributeError: 'Person' object has no attribute '__self__'
> print(p.__func__)  # AttributeError: 'Person' object has no attribute '__func__'
> ```
> 
> #### 1. **What Happens When `p.say_hello()` is Called?**
> - In Python, when you define a method like `say_hello` inside a class with `self` as the first parameter, it is an **instance method**.
> - When you access `p.say_hello` (i.e., call it as `p.say_hello()`), Python automatically transforms it into a **bound method**. This transformation happens due to Python’s descriptor protocol, specifically through the `__get__` method of the function object.
> - **Bound Method Creation**:
>   - A bound method is created when an instance (e.g., `p`) accesses the method. The instance (`p`) is automatically passed as the first argument (`self`) to the method.
>   - This binding occurs at the time of attribute access (`p.say_hello`), not when the method is defined or called. The bound method is an object that "remembers" the instance (`p`) and the function (`say_hello`).
>   - You can verify this by checking the type:
>     ```python
>     print(type(p.say_hello))  # <class 'method'>
>     ```
>   - The bound method is a wrapper that calls `say_hello` with `p` as `self` when invoked.
> 
> #### 2. **When Does `say_hello` Become a Bound Method?**
> - **Timing**: `say_hello` becomes a bound method **at the moment you access it through an instance** (e.g., `p.say_hello`). This is handled by the descriptor protocol:
>   - The `function` object (defined by `def say_hello(self)`) implements the descriptor protocol via its `__get__` method.
>   - When `p.say_hello` is accessed, `__get__(self, obj, cls)` is called (where `obj` is `p`, the instance, and `cls` is `Person`), returning a `method` object that binds `p` to `say_hello`.
> - **Before Access**: If you access `say_hello` through the class (e.g., `Person.say_hello`), it remains an unbound function, and you must manually pass an instance as `self`:
>   ```python
>   print(type(Person.say_hello))  # <class 'function'>
>   Person.say_hello(p)  # Works, passing p as self
>   ```
> 
> #### 3. **Why the Attribute Errors?**
> - **`print(p.__self__)`**:
>   - The error `AttributeError: 'Person' object has no attribute '__self__'` occurs because `__self__` is not an attribute of the instance (`p`). Instead, `__self__` is an attribute of the **bound method** object returned by `p.say_hello`.
>   - To access it, you need to get the bound method first:
>     ```python
>     bound_method = p.say_hello
>     print(bound_method.__self__)  # <__main__.Person object at ...>
>     ```
>   - `__self__` holds the instance (`p`) that the method is bound to.
> 
> - **`print(p.__func__)`**:
>   - The error `AttributeError: 'Person' object has no attribute '__func__'` occurs for the same reason: `__func__` is an attribute of the bound method, not the instance.
>   - To access it:
>     ```python
>     bound_method = p.say_hello
>     print(bound_method.__func__)  # <function Person.say_hello at ...>
>     ```
>   - `__func__` holds the original function object (`say_hello`) that the method wraps.
> 
> - **Direct Access on Instance**: Calling `print(p.__self__)` or `print(p.__func__)` tries to access these as instance attributes of `p`, which don’t exist, hence the `AttributeError`.
> 
> #### 4. **Descriptor Protocol in Action**
> - When Python sees `p.say_hello`, it looks up `say_hello` in the `Person` class.
> - Since `say_hello` is a function (a descriptor), its `__get__` method is invoked:
>   - `__get__(self, p, Person)` returns a `method` object.
>   - The `method` object stores:
>     - `__self__`: The instance (`p`).
>     - `__func__`: The original function (`say_hello`).
> - This happens transparently when you write `p.say_hello()`, and the method is called with `p` as `self`.
> 
> ### Example to Clarify
> ```python
> class Person:
>     def say_hello(self):
>         return f"Instance method called from {self}"
> 
> p = Person()
> 
> # Access the bound method
> bound_method = p.say_hello
> print(type(bound_method))         # <class 'method'>
> print(bound_method.__self__)      # <__main__.Person object at ...>
> print(bound_method.__func__)      # <function Person.say_hello at ...>
> print(bound_method())             # Instance method called from <__main__.Person object at ...>
> 
> # Direct call works because it's bound
> print(p.say_hello())              # Instance method called from <__main__.Person object at ...>
> 
> # Unbound function requires manual self
> print(Person.say_hello(p))        # Instance method called from <__main__.Person object at ...>
> ```
> 
> ### Key Points
> - **Binding Occurs at Access**: `say_hello` becomes a bound method when accessed via an instance (`p.say_hello`), not when defined or called.
> - **Attributes on Bound Method**: `__self__` and `__func__` are attributes of the `method` object, not the instance. You must first get the bound method to access them.
> - **No Direct Instance Attributes**: `p` itself doesn’t have `__self__` or `__func__` because these are part of the method binding, not the instance’s state.
> 
> ### Conclusion
> `say_hello` becomes a bound method when you access it through an instance (e.g., `p.say_hello`), thanks to Python’s descriptor protocol. The `AttributeError`s occur because `__self__` and `__func__` are attributes of the bound method object, not the instance `p`.
