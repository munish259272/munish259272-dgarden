# Automatic Values in Enums

## Introducing `auto()`

Since Python 3.6, the `enum` module offers `auto()` for **automatic value assignment**:

```python
from enum import Enum, auto

class Color(Enum):
    RED = auto()
    GREEN = auto()
    BLUE = auto()
```

- **Default**: Assigns **sequential integers** starting at 1 (`RED=1`, `GREEN=2`, `BLUE=3`).
- **Key Point**: **`auto()` simplifies value setup**.

---

## How `auto()` Works

Behind `auto()`, the `Enum` class uses `_generate_next_value_`:

```python
class Color(Enum):
    RED = auto()  # Calls _generate_next_value_
    GREEN = auto()
    BLUE = auto()

print(Color.RED.value)    # 1
print(Color.GREEN.value)  # 2
```

- **Method**: Static, no `self`.
- **Args**:
  - `name`: Member name (e.g., `"RED"`).
  - `count`: Number of prior members (including aliases).
  - `last_values`: List of previous values.
- **Returns**: The next value (defaults to `count + 1`).
- **Key Point**: **Controls how values increment**.

---

## Mixing `auto()` and Manual Values

Be cautious mixing `auto()` with explicit values:

```python
class Mixed(Enum):
    A = 5
    B = auto()  # What’s next? 6? 2?
    C = auto()
```

- **Issue**: Default `_generate_next_value_` ignores `5`, starts at 1 (`B=1`, `C=2`).
- **Problem**: Unpredictable—Python docs don’t guarantee behavior.
- ⚠️ **Key Point**: **Avoid mixing `auto()` and manual values**.

---

## Customizing `_generate_next_value_`

Override it for custom logic:

```python
class Custom(Enum):
    def _generate_next_value_(name, start, count, last_values):
        return count * 10  # 0, 10, 20, ...

    A = auto()
    B = auto()
    C = auto()

print(Custom.A.value)  # 0
print(Custom.B.value)  # 10
```

- **Args Used**: `count` starts at 0, increments per call.
- **Key Point**: **Tailor values to your needs**.

### With Aliases
```python
class Aliased(Enum):
    def _generate_next_value_(name, start, count, last_values):
        return count + 1

    A = auto()  # 1
    B = auto()  # 2
    C = 2       # Alias to B
    D = auto()  # 3

print(list(Aliased))  # [A, B, D]
```

- **Note**: `count` includes aliases, but only masters iterate.
- **Key Point**: **Custom logic handles aliases**.
