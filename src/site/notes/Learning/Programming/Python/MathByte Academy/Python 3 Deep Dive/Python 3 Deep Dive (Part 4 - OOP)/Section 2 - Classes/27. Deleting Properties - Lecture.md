# Deleting Properties in Python

## What Deleting Properties Means

When we "delete" a property, we’re **not removing it from the class**—we’re affecting the instance. Let’s clarify with an example:

```python
class Circle:
    pass

c = Circle()
c.color = "yellow"    # Adds to instance dict
print(c.__dict__)  # {'color': 'yellow'}
del c.color        # Removes 'color' from instance
print(c.__dict__)  # {}
#print(c.color)   # AttributeError
print(Circle.color)   # AttributeError
```

- **`del`**: Removes attributes from the instance’s `__dict__`, not the class.
- **Options**: Use `del c.color` or `delattr(c, "color")`.

**Key Point**: **Deletion targets instance data**, not class definitions.

---

## Adding a Property Deleter

Properties can include a `deleter` to handle cleanup when deleted:

```python
class Circle:
    def __init__(self, color):
        self._color = color
    
    @property
    def color(self):
        return self._color
    
    @color.setter
    def color(self, value):
        self._color = value
    
    @color.deleter
    def color(self):
        del self._color  # Cleanup

c = Circle("yellow")
print(c.__dict__)  # {'_color': 'yellow'}
print(c.color)     # yellow
del c.color        # Runs deleter, removes _color
print(c.__dict__)  # {}
# print(c.color)   # AttributeError
print(Circle.color) # <property object at 0x7f10310ee660>
```

- **`fdel`**: The `deleter` method (set via `@color.deleter` or `property(fdel=...)`) executes custom code.
- **Effect**: Here, it deletes `_color` from the instance.

**Takeaway**: **Deleters manage instance cleanup**, not class structure.

---

## Manual Property Setup

Alternatively, use the `property` constructor:

```python
class Circle:
    def __init__(self, color):
        self._color = color
    
    def get_color(self):
        return self._color
    def set_color(self, value):
        self._color = value
    def del_color(self):
        del self._color
    
    color = property(get_color, set_color, del_color)
```

- **Same Result**: `del c.color` calls `del_color`, clearing `_color`.

**Key Point**: **Decorators or `property()` achieve the same goal**.

---

## Property Persists in Class

Post-deletion, the property remains in the class:

```python
c = Circle("red")
del c.color        # Clears _color
print(c.__dict__)  # {}
c.color = "blue"   # Setter works, adds _color back
print(c.__dict__)  # {'_color': 'blue'}
print(c.color)     # blue
```

- **Class-Level**: `Circle.color` (the property) is intact, ready to manage new instance data.
- **Error Cause**: `c.color` fails only if `_color` is missing, not because the property is gone.

**Takeaway**: **Properties stay in the class**, only instance attributes are deleted.
