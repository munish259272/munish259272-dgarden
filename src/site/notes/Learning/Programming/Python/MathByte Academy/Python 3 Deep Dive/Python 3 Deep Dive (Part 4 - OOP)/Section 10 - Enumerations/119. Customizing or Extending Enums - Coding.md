# Customizing and Extending Enums in Depth

## Adding Custom Methods

Enums are **classes**, so add methods freely:

```python
from enum import Enum

class Color(Enum):
    RED = 1
    GREEN = 2
    BLUE = 3

    def pure_color(self, value):
        return {self: value}  # Self is hashable

print(Color.RED.pure_color(100))  # {<Color.RED: 1>: 100}
print(Color.BLUE.pure_color(255)) # {<Color.BLUE: 3>: 255}
```

- **How**: `Color.RED` is an instance, so `pure_color` binds to it.
- **Key Point**: **Methods enhance member functionality**.

## Overriding Dunder Methods

Customize representations or behavior:

```python
class Color(Enum):
    RED = 1
    GREEN = 2
    BLUE = 3

    def __repr__(self):
        return f"{self.name}({self.value})"

print(Color.RED)  # RED(1)
```

- **Default**: `<Color.RED: 1>`.
- **Custom**: `RED(1)`—more compact.
- **Key Point**: **Dunder methods shape output**.

## Adding Ordering

Enable comparisons with `__lt__`:

```python
class Number(Enum):
    ONE = 1
    TWO = 2
    THREE = 3

    def __lt__(self, other):
        if isinstance(other, Number):
            return self.value < other.value
        return NotImplemented

print(Number.ONE < Number.TWO)    # True
print(Number.TWO > Number.ONE)    # True (reflected)
```

- **Default**: No ordering (`<` raises `TypeError`).
- **Custom**: Compares values if both are `Number`.
- **Key Point**: **Ordering requires explicit implementation**.

### Total Ordering
Use `@total_ordering` for full comparisons:

```python
from functools import total_ordering

@total_ordering
class Number(Enum):
    ONE = 1
    TWO = 2
    THREE = 3

    def __lt__(self, other):
        if isinstance(other, Number):
            return self.value < other.value
        return NotImplemented

print(Number.ONE >= Number.ONE)  # True
```

- **Effect**: Adds `<=`, `>`, `>=` from `__lt__`.
- **Key Point**: **`@total_ordering` simplifies comparisons**.

## Custom Equality

Override `__eq__` for flexible equality:

```python
class Number(Enum):
    ONE = 1
    TWO = 2
    THREE = 3

    def __eq__(self, other):
        if isinstance(other, Number):
            return self is other
        if isinstance(other, int):
            return self.value == other
        return False

print(Number.ONE == Number.ONE)  # True
print(Number.ONE == 1)           # True
print(Number.ONE == 1.0)         # False
```

- **Caution**: Breaks hashability unless `__hash__` is added.
- **Key Point**: **Custom `__eq__` needs `__hash__` for dicts**.

## Truthiness Control

Adjust truthiness with `__bool__`:

```python
class State(Enum):
    READY = 1
    BUSY = 0

    def __bool__(self):
        return bool(self.value)

state = State.BUSY
if state:
    print("Ready")
else:
    print("Busy")  # Busy
```

- **Default**: All members are `True`.
- **Custom**: Ties to `value` (0 → `False`).
- **Key Point**: **`__bool__` aligns logic with values**.

## Complex Example: Phases

Combine methods for meaning:

```python
class Phase(Enum):
    READY = "ready"
    RUNNING = "running"
    FINISHED = "finished"

    def __str__(self):
        return self.value

    def __eq__(self, other):
        if isinstance(other, Phase):
            return self is other
        if isinstance(other, str):
            return self.value == other
        return False

    def __lt__(self, other):
        ordered = list(Phase)  # [READY, RUNNING, FINISHED]
        self_idx = ordered.index(self)
        if isinstance(other, Phase):
            return self_idx < ordered.index(other)
        if isinstance(other, str):
            try:
                other_member = Phase(other)
                return self_idx < ordered.index(other_member)
            except ValueError:
                return False
        return False

print(Phase.READY < Phase.RUNNING)  # True
print(Phase.READY == "ready")       # True
```

- **Order**: Defined by declaration, not value.
- **Key Point**: **Custom logic reflects intent**.

## Subclassing Limits

Enums **can’t extend members**, only empty bases:

```python
class ColorBase(Enum):
    def hello(self):
        return f"{self} says hello"

class Color(ColorBase):
    RED = "red"
    BLUE = "blue"

print(Color.RED.hello())  # "red says hello"
```

- **Rule**: No members in `ColorBase`.
- **Key Point**: **Subclass for shared methods**.

### Ordered Base
```python
from functools import total_ordering

@total_ordering
class OrderedEnum(Enum):
    """Create an ordering based on the member values. 
    So member values have to support rich comparisons."""
    
    def __lt__(self, other):
        if isinstance(other, OrderedEnum):
            return self.value < other.value
        return NotImplemented

class Dimension(OrderedEnum):
    D1 = (1,)
    D2 = (1, 1)
    D3 = (1, 1, 1)

print(Dimension.D1 < Dimension.D2)  # True
```

- **Assumption**: Values support `<`.
- **Key Point**: **Base classes add reusable logic**.

## Multi-Value Enums

Use `__new__` for complex values:

```python
class AppStatus(Enum):
    def __new__(cls, value, phrase):
        member = object().__new__(cls)
        member._value_ = value
        member.phrase = phrase
        return member

    OKAY = (0, "No problem")
    FAILED = (1, "Crap")

print(AppStatus.OKAY.value)   # 0
print(AppStatus.OKAY.name)    # OKAY
print(AppStatus.OKAY.phrase)  # "No problem"
print(AppStatus(0))           # AppStatus.OKAY
```

- **How**: `__new__` splits tuple into `value` and attributes.
- **Key Point**: **Custom `__new__` enables lookups**.

### Reusable Base
```python
class TwoValueEnum(Enum):
    def __new__(cls, value, phrase):
        member = object().__new__(cls)
        member._value_ = value
        member.phrase = phrase
        return member

class AppStatus(TwoValueEnum):
    OKAY = (0, "No problem")
    FAILED = (1, "Crap")
```

- **Benefit**: Reuse across enums.
- **Key Point**: **Base enums streamline multi-value setups**.
