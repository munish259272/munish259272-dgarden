# Understanding Slots in Python

## The Problem with Dictionaries

Normally, instance attributes live in a `__dict__`:

```python
class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y

p = Point(0, 0)
print(p.__dict__)  # {'x': 0, 'y': 0}
```

- **Issue**: Each instance gets its own dictionary, which has memory overhead.
- **Scale**: Thousands of instances (e.g., 10,000 `Point`s) mean thousands of dictionaries—costly!
- **Python 3.3+**: Key-sharing dictionaries reduce this, but there’s a better way.

## Introducing `__slots__`

Use `__slots__` to **replace dictionaries with a compact structure**:

```python
class Point:
    __slots__ = ['x', 'y']  # Define allowed attributes
    def __init__(self, x, y):
        self.x = x
        self.y = y

p = Point(0, 0)
print(p.x, p.y)  # 0 0
# print(p.__dict__) # 'Point' object has no attribute '__dict__'
print(dir(p)) # [..., 'x', 'y']

```

- **How**: List attribute names in `__slots__`—Python skips `__dict__`.
- **Result**: `p.__dict__`  or `var(p)` raises `AttributeError`—no dictionary exists.
- **Access**: `dir(p)` shows `x` and `y`; use them as usual (`p.x = 5`).
- **Key Point**: **Slots cut memory use**.

---

## Memory Savings

Compare with and without slots:

```python
class PointDict:
    def __init__(self, x, y):
        self.x = x
        self.y = y

class PointSlots:
    __slots__ = ['x', 'y']
    def __init__(self, x, y):
        self.x = x
        self.y = y

# 10,000 instances
points_dict = [PointDict(i, i) for i in range(10_000)]
points_slots = [PointSlots(i, i) for i in range(10_000)]
```

- **Memory**: ~1,700 KB (dict) vs. ~635 KB (slots)—about **1/3 the size**.
- **Use Case**: Great for tons of instances (e.g., database records).

---

## Speed Boost

Slots are **faster too**:

```python
class PersonDict:
    def __init__(self, name):
        self.name = name

class PersonSlots:
    __slots__ = ['name']
    def __init__(self, name):
        self.name = name

p1 = PersonDict("John")
p2 = PersonSlots("John")
# Time setting, getting, deleting 'name'
```

- **Result**: Slots ~30% faster on average for attribute operations.
- **Why**: No dictionary lookups—just direct memory access.

---

## Trade-Offs

Slots limit flexibility:

```python
p = PointSlots(0, 0)
p.z = 100  # AttributeError: 'PointSlots' object has no attribute 'z'
```

- **No [[Learning/Programming/Python/MathByte Academy/Python 3 Deep Dive/Python 3 Deep Dive (Part 4 - OOP)/Section 2 - Classes/16. Function Attributes - Coding#Dynamic (Monkey Patching) Methods\|Monkey Patching]]**: Can’t add new attributes post-creation.
- **Fixed Set**: Even in `__init__`, `self.z = 100` fails—only `x` and `y` are allowed.
- ⚠️ **Warning**: Issues with multiple inheritance or data descriptors (shown later).
- **Key Point**: **Slots lock down attributes**.

---

## When to Use Slots

- **Yes**: Use slots for **huge instance counts** with **known attributes** (memory + speed wins).
- **No**: Avoid if you need **dynamic attributes** or lack performance pressure.
- **Takeaway**: **Reserve slots for big gains**.
