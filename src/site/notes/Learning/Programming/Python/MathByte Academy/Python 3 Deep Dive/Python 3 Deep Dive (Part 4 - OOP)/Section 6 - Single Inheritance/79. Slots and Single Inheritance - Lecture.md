# Slots with Single Inheritance

## Slots in a Base Class

Let’s define a base class with slots:

```python
class Person:
    __slots__ = ('name',)  # Single-item tuple needs comma
    
class Student(Person):
    pass

s = Student()
s.name = "Alex"
print(s.name)  # Alex
s.age = 18
print(s.__dict__)  # {'age': 18}
```

- **Behavior**: `name` uses slots from `Person`; `age` goes into `__dict__`.
- **Key Point**: **Subclasses inherit slots but add a dictionary**.

---

## Adding Slots to Subclass

Specify only new slots in the subclass:

```python
class Person:
    __slots__ = ('name',)

class Student(Person):
    __slots__ = ('age',)  # Just the new one

s = Student()
s.name = "Alex"
s.age = 18
print(s.name, s.age)  # Alex 18
# s.__dict__  # AttributeError: no dictionary
```

- **Correct**: `Student` adds `age` to slots; `name` stays from `Person`.
- **Takeaway**: **Don’t repeat parent slots**.

---

## Redefining Slots (Avoid!)

Repeating slots causes issues:

```python
class Person:
    __slots__ = ('name',)
    @property
    def name(self):
        return self._name.upper()
    @name.setter
    def name(self, value):
        self._name = value

class Student(Person):
    __slots__ = ('name', 'age')  # Repeats 'name'
    def __init__(self, name, age):
        self.name = name
        self.age = age

p = Person()
p.name = "alex"
print(p.name)  # ALEX

s = Student("alex", 18)
print(s.name)  # alex (not ALEX)
```

- **Problem**: `Student`’s `name` slot hides `Person`’s property.
- **Memory**: Extra slot increases usage.
- ⚠️ **Warning**: Python may block this in the future—avoid it!
- **Key Point**: **Redefining slots hides parent behavior**.

---

## Slots in Subclass Only

Parent without slots, subclass with:

```python
class Person:
    pass

class Student(Person):
    __slots__ = ('name', 'age')

s = Student()
s.name = "Alex"
s.age = 18
s.grade = "A"
print(s.__dict__)  # {'grade': 'A'}
```

- **Result**: `name` and `age` use slots; `grade` uses `__dict__`.
- **Takeaway**: **Slots and dictionaries coexist**.

---

## Slots vs. Properties

Both avoid `__dict__` for storage:

```python
class Useless:
    @property
    def useless(self):
        return "nothing"
    @useless.setter
    def useless(self, value):
        pass

u = Useless()
u.useless = "something"
print(u.__dict__)  # {} (empty, no backing variable)
```

- **Similarity**: Slots and properties use **data descriptors** (covered later).
- **Slots**: Auto-create getters/setters/deleters for listed attributes.
- **Key Point**: **Slots are like pre-built properties**.

---

## Hybrid Approach

Mix slots and dictionaries in one class:

```python
class Person:
    __slots__ = ('name', '__dict__')
    
p = Person()
p.name = "Alex"  # Uses slot
p.age = 18       # Uses __dict__
print(p.name, p.age)  # Alex 18
print(p.__dict__)     # {'age': 18}
```

- **How**: Include `__dict__` in `__slots__`.
- **Benefit**: Slots for fixed attributes, `__dict__` for runtime additions.
- **Takeaway**: **Hybrid gives flexibility with efficiency**.
