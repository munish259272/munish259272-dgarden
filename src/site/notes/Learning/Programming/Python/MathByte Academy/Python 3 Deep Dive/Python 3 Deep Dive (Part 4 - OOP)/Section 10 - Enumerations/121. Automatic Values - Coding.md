# Using `auto()` for Enum Values

## Basic `auto()` Usage

Assign values automatically with `auto()`:

```python
from enum import Enum, auto

class State(Enum):
    WAITING = auto()
    STARTED = auto()
    FINISHED = auto()

for member in State:
    print(f"{member.name}: {member.value}")
# WAITING: 1
# STARTED: 2
# FINISHED: 3
```

- **Default**: Starts at 1, increments by 1.
- **Key Point**: **`auto()` auto-increments integers**.

---

## How It Works

`auto()` calls `_generate_next_value_` during class compilation:

- **Trigger**: Each `auto()` invokes it.
- **Default**: Returns `count + 1` (starts at 1).

---

## Mixing Manual and `auto()` Values

Mixing can lead to surprises:

```python
class State(Enum):
    WAITING = 100
    STARTED = auto()
    FINISHED = auto()

print(list(State))  # [<State.WAITING: 100>, <State.STARTED: 101>, <State.FINISHED: 102>]
```

- **Works**: Continues from 100.
- ⚠️ **Caution**: Undocumented behavior—don’t rely on it.

### Alias Risk
```python
class State(Enum):
    WAITING = auto()  # 1
    STARTED = 1       # Alias to WAITING
    FINISHED = auto() # 2

print(list(State))  # [<State.WAITING: 1>, <State.FINISHED: 2>]
print(State.__members__)  # {'WAITING': <State.WAITING: 1>, 'STARTED': <State.WAITING: 1>, ...}
```

- **Issue**: `STARTED` becomes an alias; `@unique` catches it but doesn’t fix intent.
- **Key Point**: **Avoid mixing to prevent alias confusion**.

## Customizing `_generate_next_value_`

Override for custom logic:

```python
class State(Enum):
    def _generate_next_value_(name, start, count, last_values):
        print(f"Name: {name}, Count: {count}, Last: {last_values}")
        return 100  # Constant

    A = auto()
    B = auto()
    C = auto()
```

- **Args**:
  - `name`: Member name (e.g., `"A"`).
  - `count`: Prior members (starts at 0).
  - `last_values`: List of previous values.
- **Output**: All get 100 (becomes aliases).

### Random Unique Values
```python
import random
random.seed(0)

class State(Enum):
    def _generate_next_value_(name, start, count, last_values):
        while True:
            new_val = random.randint(1, 100)
            if new_val not in last_values:
                return new_val

    A = auto()
    B = auto()
    C = auto()

for m in State:
    print(f"{m.name}: {m.value}")
# A: 50
# B: 98
# C: 54
```

- **Uses**: `last_values` ensures uniqueness.
- **Key Point**: **Customize for specific patterns**.

### Name-Based Values
```python
class State(Enum):
    def _generate_next_value_(name, start, count, last_values):
        return name.lower()

    WAITING = auto()
    STARTED = auto()
    FINISHED = auto()

print(State.WAITING.value)  # "waiting"
```

- **Uses**: `name` to derive values.
- **Key Point**: **Eliminates repetition**.

## Reusable Base Class

Make it reusable:

```python
class NameAsString(Enum):
    def _generate_next_value_(name, start, count, last_values):
        return name.lower()

class State(NameAsString):
    WAITING = auto()
    STARTED = auto()

print(State.WAITING.value)  # "waiting"
```

- **Benefit**: Share logic across enums.
- **Key Point**: **Inheritance streamlines `auto()`**.

## Meaningless Values

Hide values to enforce name usage:

```python
class State(Enum):
    WAITING = object()
    STARTED = object()
    FINISHED = object()

print(State.WAITING in State)  # True
# State(1)  # ValueError: no match
```

- **Why**: `object()` prevents value-based lookups.
- **Key Point**: **Obscure values to focus on names**.

## Aliases with `auto()`

Control aliases dynamically:

```python
class AliasEnum(Enum):
    def _generate_next_value_(name, start, count, last_values):
        return last_values[-1]  # Always alias to previous

class Color(AliasEnum):
    RED = object()
    CRIMSON = auto()
    CARMINE = auto()
    BLUE = object()
    AQUA = auto()

print(list(Color))  # [<Color.RED: ...>, <Color.BLUE: ...>]
print(Color.CARMINE is Color.RED)  # True
```

- **Logic**: Reuses last value for aliases.
- **Key Point**: **`auto()` can enforce aliasing**.
