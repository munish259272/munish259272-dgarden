```python
import  types  
def my_func():  
    pass  
  
# print(type(my_func) is function)  
i = 10  
print(type(i)) # <class 'int'>  
print(type(i) is int) # True  
  
s = 'abc'  
print(type(s)) # <class 'str'>  
print(type(s) is str) # True  
  
l = [1, 2, 3]  
print(type(l)) # <class 'list'>  
print(type(l) is list) # True  
  
t = (1, 2, 3)  
print(type(t)) # <class 'tuple'>  
print(type(t) is tuple) # True  
  
print(type(my_func)) # <class 'function'>  
# print(type(my_func) is function) # Error : NameError: name 'function' is not defined  
print(isinstance(my_func, types.FunctionType))  
print(type(my_func) is types.FunctionType)
```


> [!question]- Even though `print(type(my_func))` returns `<class 'function'>` why is the type of `my_func` not 'function' instead it is `types.FunctionType` ?
> 
> Great question! The key point here is how Python internally defines and categorizes types.
> 
> ### Understanding `type(my_func)`
> 
> 1. When you define a function in Python:
>     
>     ```python
>     def my_func():
>         pass
>     ```
>     
>     - The function is an **instance** of a class that represents functions.
>     - When you check `print(type(my_func))`, it prints `<class 'function'>`. This is the name assigned to this type in Python.
> 1. However, `function` is **not a built-in name** in Python. Unlike `int`, `str`, `list`, etc., Python does **not** provide a direct reference to `function` in the global namespace.
>     
> 3. Instead, functions belong to the type **`types.FunctionType`**, which is defined in the `types` module.
>     
> 
> ### Why `type(my_func) is not function`?
> 
> - Python **does not** have `function` as a globally available type name.
> - The correct way to check the type of a function is:
>     
>     ```python
>     import types
>     print(type(my_func) is types.FunctionType)  # True
>     ```
>     
> 
> ### Why does `print(type(my_func))` return `<class 'function'>`?
> 
> - `FunctionType` is defined in C, and its `__name__` attribute is set to `"function"`, making it display as `<class 'function'>` for readability.
> - Internally, it maps to `types.FunctionType`.
> 
> ### Key Takeaways:
> 
> 1. `int`, `str`, `list`, `tuple`, etc., are built-in types with globally available names.
> 2. Functions belong to `types.FunctionType`, but Python **displays** this as `<class 'function'>` for clarity.
> 3. `function` is **not a globally available identifier**, so `type(my_func) is function` raises a `NameError`.
> 4. The correct way to check if an object is a function:
>     
>     ```python
>     import types
>     isinstance(my_func, types.FunctionType)  # True
>     type(my_func) is types.FunctionType  # True
>     ```
