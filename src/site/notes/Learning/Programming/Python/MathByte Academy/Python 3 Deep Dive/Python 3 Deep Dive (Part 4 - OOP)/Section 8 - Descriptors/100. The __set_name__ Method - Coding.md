# Leveraging `__set_name__` for Descriptors

## Capturing Property Names

Use `__set_name__` (Python 3.6+) to grab the attribute name at class definition:

```python
class ValidString:
    def __set_name__(self, owner, name):
        print(f"Set name: owner={owner}, name={name}")
        self.property_name = name

class Person:
    name = ValidString()  # Runs here, not at instantiation
```

- **When**: Called at **class compile time**, not when instantiating (`p = Person()`).
- **Output**: `Set name: owner=<class '__main__.Person'>, name=name`.
- **Key Point**: **Knows its name before instances exist**.

## Enhancing Descriptors

Store the name and use it:

```python
class ValidString:
    def __set_name__(self, owner, name):
        self.property_name = name
    def __get__(self, instance, owner):
        if instance is None:
            return self
        print(f"Get called for {self.property_name} on {instance}")
        return instance.__dict__.get(self.property_name)

class Person:
    first_name = ValidString()
    last_name = ValidString()

p = Person()
print(p.first_name)  # Get called for first_name on <Person object>
print(p.last_name)   # Get called for last_name on <Person object>
```

- **Why**: `self.property_name` tracks `first_name` or `last_name` per descriptor instance.
- **Key Point**: **Simplifies messaging and usage**.

## Validation and Storage

Add validation and store in `__dict__`:

```python
class ValidString:
    def __init__(self, min_length=None):
        self.min_length = min_length
    def __set_name__(self, owner, name):
        self.property_name = name
    def __set__(self, instance, value):
        if not isinstance(value, str):
            raise ValueError(f"{self.property_name} must be a string")
        if self.min_length is not None and len(value) < self.min_length:
            raise ValueError(f"{self.property_name} must be at least {self.min_length} characters")
        instance.__dict__[self.property_name] = value
    def __get__(self, instance, owner):
        if instance is None:
            return self
        return instance.__dict__.get(self.property_name)

class Person:
    first_name = ValidString(1)
    last_name = ValidString(2)

p = Person()
try:
	p.first_name = "Alex"
	p.last_name = "M"  # ValueError: last_name must be at least 2 characters
except ValueError as e:
	print(e)           #outputlast_name must be at least 2 characters
```

- **How**: Use `self.property_name` for errors and storage key.
- **Fixes**: No need to pass names manually—cleaner than before.
- **Key Point**: **Validation with precise feedback**.

## Shadowing Concern

Storing in `__dict__` under the same name:

```python
p = Person()
p.first_name = "Alex"
print(p.__dict__)  # {'first_name': 'Alex'}
print(p.first_name)  # Alex (via __get__, not __dict__)
```

- **Question**: Doesn’t `__dict__['first_name']` shadow the descriptor?
- **Answer**: **No, data descriptors (with `__set__`) override `__dict__` lookup**.
- **Avoiding Recursion**: Use `instance.__dict__` directly, not `setattr` (infinite loop risk).
- **Key Point**: **Descriptor priority beats shadowing**.

> [!question]- "<font color="#000000" style="font-weight:normal;font-style:italic;">**Avoiding Recursion**: Use `instance.__dict__` directly, not `setattr` (infinite loop risk).</font>" - How is `setattr(instance, self.property_name, value)` going to cause infinite recursion ?
> 
> Using `setattr(instance, self.property_name, value)` in a descriptor’s `__set__` method causes **infinite recursion** because it triggers the descriptor’s `__set__` method again, creating an endless loop. Here’s how:
> 
> ### Why It Loops
> - When you write `p.first_name = "Alex"`, Python sees `first_name` is a descriptor with `__set__` and calls `ValidString.__set__(self, instance, value)`.
> - Inside `__set__`, if you use `setattr(instance, self.property_name, value)` (e.g., `setattr(instance, "first_name", "Alex")`), Python interprets this as another assignment to `instance.first_name`.
> - Since `first_name` is still tied to the descriptor, this assignment **re-calls `__set__`** with the same arguments, and the cycle repeats indefinitely.
> 
> ### Example
> ```python
> class ValidString:
>     def __set_name__(self, owner, name):
>         self.property_name = name
>     def __set__(self, instance, value):
>         print(f"Setting {self.property_name} to {value}")
>         setattr(instance, self.property_name, value)  # Triggers recursion
> 
> class Person:
>     name = ValidString()
> 
> p = Person()
> p.name = "Alex"  # Infinite recursion: RecursionError
> ```
> 
> - **Output**: Keeps printing `"Setting name to Alex"` until a `RecursionError: maximum recursion depth exceeded`.
> 
> ### Stack Trace
> 1. `p.name = "Alex"` → `ValidString.__set__(p, "Alex")`
> 2. `setattr(p, "name", "Alex")` → `p.name = "Alex"` → `ValidString.__set__(p, "Alex")`
> 3. Repeat step 2 forever.
> 
> ### Fix: Use `__dict__`
> Bypass the descriptor by directly setting the value in `instance.__dict__`:
> 
> ```python
> def __set__(self, instance, value):
>     instance.__dict__[self.property_name] = value  # No recursion
> ```
> 
> - **Why Safe**: This updates the dictionary directly, avoiding attribute assignment logic (and thus the descriptor’s `__set__`).
> - **Key Point**: **`setattr` re-triggers descriptors; `__dict__` doesn’t`**.

## Caveat

Still assumes `__dict__` exists:

- **Issue**: Fails with slots unless `__dict__` is in `__slots__`.
- **Risk**: Could overwrite user-defined keys (less likely with exact name).
