# Understanding Single Inheritance in Python

## What is Inheritance?

**Inheritance** is a core concept in object-oriented programming where classes form hierarchies:

- **Classes**: Define properties (attributes) and methods (behaviors).
- **Hierarchy**: A natural "is-a" relationship between classes.

---

## Building a Shape Hierarchy

Imagine a geometry example:

```python
class Shape:
    def translate_center(self, dx, dy):
        return f"Moved by ({dx}, {dy})"

class Polygon(Shape):
    def __init__(self, sides):
        self.sides = sides

class Triangle(Polygon):
    def __init__(self):
        super().__init__(3)  # Fixed 3 sides

class Equilateral(Triangle):
    pass

class Quadrilateral(Polygon):
    def __init__(self):
        super().__init__(4)  # Fixed 4 sides

class Rectangle(Quadrilateral):
    pass

class Square(Rectangle):
    pass
```

- **Chain**: `Square` → `Rectangle` → `Quadrilateral` → `Polygon` → `Shape`.
- **Key Point**: **Each subclass specializes its parent**.

---

## Inheriting and Extending

Subclasses **inherit and extend**:

- **Inheritance**: `Triangle` gets `translate_center()` from `Shape`.
- **Extension**: `Polygon` adds `sides`, not in `Shape`.
- **Override**: `Triangle` fixes `sides` to `3`, overriding `Polygon`’s flexibility.
- **Takeaway**: **Subclasses build on and tweak parents**.

---

## Terminology with Example

Consider a people hierarchy:

```python
class Person:
    def __init__(self, name):
        self.name = name
    
    def eat(self):
        return "Eating"

class Teacher(Person):
    def teach(self):
        return "Teaching"

class Student(Person):
    def study(self):
        return "Studying"
```

- **Parent**: `Person` (aka base class).
- **Children**: `Teacher`, `Student` (aka subclasses).
- **Synonyms**: `Student` extends/derives from/is a subclass of `Person`.
- **Key Point**: **Single inheritance = one parent per class**.

---

## Instances and `isinstance`

Instances reflect the hierarchy:

```python
p1 = Person("Alex")
s1 = Student("Bob")
t1 = Teacher("Clara")

print(isinstance(s1, Student))  # True (direct)
print(isinstance(s1, Person))   # True (inherited)
print(isinstance(s1, Teacher))  # False (sibling)
print(isinstance(p1, Student))  # False (parent not child)
```

- **`isinstance(obj, Class)`**: Checks if `obj` is an instance of `Class` or its ancestors.
- **Contrast `type`**:
  ```python
  print(type(s1) == Student)  # True (exact class)
  print(type(s1) == Person)   # False (ignores inheritance)
  ```
- **Key Point**: **`isinstance` follows the chain; `type` doesn’t**.

---

## Practical Use

Use `isinstance` for behavior checks:

```python
def can_eat(obj):
    if isinstance(obj, Person):  # Covers Person, Teacher, Student
        return obj.eat()
    return "No eating"

print(can_eat(s1))  # Eating
print(can_eat(t1))  # Eating
print(can_eat([]))  # No eating
```

- **Why**: Tests for `eat()` capability, not exact type.
- **Tip**: Use the **least restrictive parent** needed.

---

## Class Relationships with `issubclass`

Check class hierarchies:

```python
class CollegeStudent(Student):
    pass

print(issubclass(Student, Person))         # True (direct)
print(issubclass(CollegeStudent, Student)) # True (direct)
print(issubclass(CollegeStudent, Person))  # True (indirect)
print(issubclass(Student, Teacher))        # False (siblings)
print(issubclass(Person, Student))         # False (parent not subclass)
```

- **`issubclass(Sub, Parent)`**: True if `Sub` inherits from `Parent`, directly or indirectly.
- **Parent vs Ancestor**: `Person` is parent of `Student`, ancestor (not parent) of `CollegeStudent`.
- **Key Point**: **Subclasses trace up the chain**.

---

## Defining Subclasses

Simple syntax:

```python
class Person: pass
class Student(Person): pass
class CollegeStudent(Student): pass
```

- **How**: `class Sub(Parent):`.
- **Implicit**: `Person` inherits from `object`.
- **Chain**: `CollegeStudent` → `Student` → `Person` → `object`.
- **Takeaway**: **All classes link to `object`**.
