# Function Attributes as Methods in Python

## Defining a Function Attribute

Let’s create a class with a function attribute:
```python
class MyClass:
    def say_hello():
        return "Hello World"
```

- **`say_hello`**: A **class attribute** that’s a function.

**Key Point**: Functions in classes behave **differently** from data attributes.

---

## Class vs. Instance Access

### From the Class
```python
print(MyClass.say_hello) # <function MyClass.say_hello at 0x7ff505a819e0>
print(MyClass.say_hello())  # "Hello World"
```

- Works directly—`say_hello` is a plain function here.

### From an Instance
Create an instance:
```python
my_object = MyClass()
```

Access the function:
```python
print(my_object.say_hello)  # <bound method MyClass.say_hello of <__main__.MyClass object at ...>>
```

- **Bound Method**: Not just a function anymore—it’s tied to `my_object`.
- Try calling it:
```python
my_object.say_hello()  # TypeError: say_hello() takes 0 positional arguments but 1 was given
```

**Why the Error?** Python passes `my_object` as an argument, but `say_hello` doesn’t expect it.

**Takeaway**: **Instance calls bind the function** to the object.

---

## What’s a Bound Method?

- **Method**: A callable object, distinct from a raw function.
- **Bound**: Tied to an instance (`my_object`), which is passed as the first argument.

When we call:
```python
my_object.say_hello()
```

It’s equivalent to:
```python
MyClass.say_hello(my_object)
```

- Python **injects the instance** (`self`) automatically.

**Key Point**: **Methods receive the instance** as the first parameter.

---

## Fixing the Function

Adjust `say_hello` to accept the instance:
```python
class MyClass:
    def say_hello(self):
        return "Hello World"
```

Now:
```python
my_object = MyClass()
print(my_object.say_hello())  # "Hello World"
```

- **`self`**: The bound instance (`my_object`).

**Takeaway**: Define functions with **`self`** for instance methods.

---

## Method Attributes

Bound methods have properties:
```python
method = my_object.say_hello
print(method.__self__)  # <__main__.MyClass object at ...> (my_object)
print(method.__func__)  # <function MyClass.say_hello at ...> (original function)
```

- **`__self__`**: The instance it’s bound to.
- **`__func__`**: The underlying function from the class.

**Key Point**: **Methods combine instance and function**.

---

## Understanding Bound Methods in Depth

Calling `obj.method(args)` $\large{\xrightarrow[\text{translates under the hood to}]{}}$ `method.__func__(method.__self__, args)`

### Example Breakdown
```python
class Person:
    def hello(self):
        pass

p = Person()
print(p.hello.__self__)  # p (the instance itself)
print(p.hello.__func__)  # <function Person.hello at ...> (the original function)
```

- **`p.hello.__self__`**: Points to `p`—the instance the method is bound to.
- **`p.hello.__func__`**: The raw `hello` function from the class.

> [!question]- is **init** and other dunder functions also become bound to the object ?
> 
> **Yes — `__init__` and *all* other methods (including dunder methods) become **bound methods** when accessed via an instance.**
> 
> Let’s break this down **clearly and completely**.
> 
> ---
> 
> ## 1. What is a **Bound Method**?
> 
> When you access a method on an **instance**, Python **automatically binds** it:
> 
> ```python
> obj.method  →  method.__func__(method.__self__, ...)
> ```
> 
> | Attribute | Meaning |
> |---------|--------|
> | `method.__self__` | The **instance** (`obj`) |
> | `method.__func__` | The **original function** from the class |
> 
> ---
> 
> ## 2. `__init__` is Also a Bound Method
> 
> ```run-python
> class Person:
>     def __init__(self, name):
>         self.name = name
>         print("__init__ called")
> 
> p = Person("Alice")
> ```
> 
> Under the hood:
> 
> ```run-python
> # This:
> p = Person("Alice")
> 
> # Is roughly:
> p = Person.__new__(Person)           # create object
> Person.__init__(p, "Alice")          # ← __init__ is called as a bound method
> ```
> 
> But if you access it:
> 
> ```run-python
> print(p.__init__)           # <bound method Person.__init__ of <__main__.Person object>>
> print(p.__init__.__self__)  # <__main__.Person object at 0x...> → p
> print(p.__init__.__func__)  # <function Person.__init__ at 0x...>
> ```
> 
> **Yes — `__init__` is a bound method** when accessed via `p.__init__`.
> 
> ---
> 
> ## 3. All Dunder Methods Are Bound Methods
> 
> ```run-python
> class Person:
>     def __init__(self, name):
>         self.name = name
> 
>     def __str__(self):
>         return f"Person({self.name})"
> 
>     def __len__(self):
>         return len(self.name)
> 
> p = Person("Alice")
> 
> print(p.__str__)          # <bound method Person.__str__ of ...>
> print(p.__len__)          # <bound method Person.__len__ of ...>
> print(p.__add__)          # AttributeError! (unless defined)
> ```
> 
> > **Every instance method — dunder or not — becomes a bound method.**
> 
> ---
> 
> ## 4. How Python Does This (Internals)
> 
> ```run-python
> class Person:
>     def hello(self):
>         print("Hi")
> 
> p = Person()
> ```
> 
> 1. `Person.hello` → **function object** (unbound)
> 2. `p.hello` → Python creates a **bound method object** on the fly
>    - Wraps `Person.hello` (the function)
>    - Binds `p` as the first argument (`self`)
> 
> ```run-python
> # Equivalent:
> bound_hello = types.MethodType(Person.hello, p)
> ```
> 
> ---
> 
> ## 5. Proof: All Methods Are Bound
> 
> ```run-python
> class Demo:
>     def regular(self): pass
>     def __init__(self): pass
>     def __str__(self): return "hi"
>     def __call__(self): pass
> 
> d = Demo()
> 
> print(d.regular)   # <bound method Demo.regular ...>
> print(d.__init__)  # <bound method Demo.__init__ ...>
> print(d.__str__)   # <bound method Demo.__str__ ...>
> print(d.__call__)  # <bound method Demo.__call__ ...>
> ```
> 
> All are **bound methods**.
> 
> ---
> 
> ## 6. When Are They *Not* Bound?
> 
> Only when accessed via the **class**:
> 
> ```run-python
> print(Person.__init__)   # <function Person.__init__> → NOT bound
> print(Person.hello)      # <function Person.hello>     → NOT bound
> ```
> 
> > **Class → function**  
> > **Instance → bound method**
> 
> ---
> 
> ## 7. Practical Use: `__func__` and `__self__`
> 
> ```run-python
> class Logger:
>     def log(self, msg):
>         func_name = self.log.__func__.__name__
>         instance = self.log.__self__
>         print(f"[{instance}] {func_name}: {msg}")
> 
> l = Logger()
> l.log("test")
> ```
> 
> Output:
> ```
> [<__main__.Logger object at ...>] log: test
> ```
> 
> > Use `__func__` to get the original function (e.g., in decorators, logging, debugging).
> 
> ---
> 
> ## Summary Table
> 
> | Method Type | `Class.method` | `obj.method` | Has `__self__`? | Has `__func__`? |
> |------------|----------------|--------------|------------------|------------------|
> | Regular | Function | Bound Method | Yes | Yes |
> | `__init__` | Function | Bound Method | Yes | Yes |
> | `__str__`, `__len__`, etc. | Function | Bound Method | Yes | Yes |
> | `__call__` | Function | Bound Method | Yes | Yes |
> 
> ---
> 
> ## Final Answer
> 
> > **Yes — `__init__` and all dunder methods become bound methods when accessed via an instance.**  
> > They have:
> > - `.__self__` → the instance
> > - `.__func__` → the original function from the class
> 
> ---
> 
> ## Bonus: How to Call the Raw Function
> 
> ```run-python
> # Call __init__ manually (rarely needed)
> Person.__init__(p, "Bob")  # p must already exist
> ```
> 
> Or:
> 
> ```run-python
> p.__init__.__func__(p, "Bob")  # same thing
> ```
>

### Another Example
```python
class MyClass:
    def say_hello(obj):  # 'obj' as the first parameter
        return f"Hello World!"

my_obj = MyClass()
print(my_obj.say_hello)  # <bound method MyClass.say_hello of <__main__.MyClass object at ...>>
```

Calling it:
```python
print(my_obj.say_hello())      # "Hello World!"
print(MyClass.say_hello(my_obj))  # "Hello World!"
```

- **Bound Behavior**: `my_obj.say_hello()` auto-passes `my_obj` to `say_hello`.
- **Manual Call**: `MyClass.say_hello(my_obj)` does the same explicitly.

### Why This Matters
When defining functions in a class:
- **Include a first parameter** (e.g., `obj` or `self`) to accept the instance.
- Without it, instance calls fail because Python **injects the instance** automatically.

**Why `self` Instead of `obj`?**
- **Convention**: `self` is the standard name in Python for the instance parameter—it’s instantly recognizable to Python developers.
- **Clarity**: Using `obj` works (it’s just a name), but `self` signals “this is the instance” by tradition, avoiding confusion.
- **Consistency**: Python’s built-in methods and libraries use `self`, so sticking with it keeps code uniform.

**Key Point**: Functions become **instance methods** when called on an instance, binding to it with `self`.

## Adding Parameters

Enhance the method:
```python
class MyClass:
    language = "Python"
    def say_hello(self, name):
        return f"Hello {name}, I am {self.language}"
```

Test it:
```python
python = MyClass()
print(python.say_hello("John"))  # "Hello John, I am Python"
```

Override at instance level:
```python
java = MyClass()
java.language = "Java"
print(java.say_hello("John"))  # "Hello John, I am Java"
```

- **Lookup**: `self.language` checks `java.__dict__` first, then `MyClass.__dict__`.

**Takeaway**: Methods access **instance or class state** via `self`.
