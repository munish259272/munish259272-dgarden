# Customizing Truth Values in Python

## How Truthiness Works

Every object in Python has a **truth value**, accessed via `bool()`. For custom classes, Python follows this order:

1. **`__bool__`**: If defined, it determines the truth value directly.
2. **`__len__`**: If no `__bool__`, it checks length—`0` is `False`, non-zero is `True`.
3. **Default**: If neither exists, the object is `True`.

---

## Default Truthiness

Without overrides, objects are always truthy:

```python
class Person:
    pass

p = Person()
print(bool(p))  # True
```

- **Reason**: No `__bool__` or `__len__`, so default is `True`.
- **Key Point**: **Custom objects default to truthy**.

---

## Using `__len__` for Collections

Implement `__len__` to mimic collection behavior:

```python
class MyList:
    def __init__(self, length):
        self._length = length
    
    def __len__(self):
        print("Len called")
        return self._length

l1 = MyList(0)
l2 = MyList(10)
print(bool(l1))  # Len called, False
print(bool(l2))  # Len called, True
```

- **Logic**: `len() == 0` → `False`; otherwise `True`.
- **Consistency**: Matches lists, tuples, etc.
- **Takeaway**: ** `__len__` is enough for iterables**.

---

## Prioritizing `__bool__`

Add `__bool__`—it takes precedence:

```python
class MyList:
    def __init__(self, length):
        self._length = length
    
    def __bool__(self):
        print("Bool called")
        return self._length > 0
    
    def __len__(self):
        print("Len called")
        return self._length

l1 = MyList(0)
l2 = MyList(10)
print(bool(l1))  # Bool called, False
print(bool(l2))  # Bool called, True
```

- **Priority**: `__bool__` overrides `__len__`.
- **Key Point**: ** `__bool__` is checked first**.

---

## Custom Truthiness with `__bool__`

Define truthiness for a 2D point—`(0, 0)` is falsy:

```python
class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y
    
    def __bool__(self):
        return self.x != 0 or self.y != 0

p1 = Point(0, 0)
p2 = Point(1, 1)
print(bool(p1))  # False
print(bool(p2))  # True
```

- **Logic**: `(0, 0)` → `False`; any non-origin → `True`.
- **Contrast**: Without `__bool__`, both would be `True`.

---

## Pitfall: `__bool__` Must Return a Boolean

Watch out—`__bool__` can’t return non-booleans:

```python
class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y
    
    def __bool__(self):
        return self.x or self.y  # Returns int, not bool

p1 = Point(0, 0)
# print(bool(p1))  # TypeError: __bool__ should return bool, returned int
```

- **Fix**: Explicitly convert to boolean:
  ```python
  def __bool__(self):
      return bool(self.x or self.y)
  print(bool(p1))  # False
  ```
- ⚠️ **Warning**: ** `__bool__` must return `True` or `False`**, not raw values.
