# Modular Arithmetic Project Overview

In this project, we will explore **modular arithmetic** by implementing a `Mod` class to handle key concepts. Below is a clear, structured explanation of the requirements and concepts to guide you through the implementation.

## Understanding Modular Arithmetic
- **Modular arithmetic** deals with integers where numbers "wrap around" after reaching a certain value, called the **modulus** (`N`). Think of it like a clock: on a 12-hour clock, 13:00 is the same as 1:00 because $\large{13 \mod 12 = 1}$.
- **Applications**: Modular arithmetic is widely used in:
  - **Encryption** (e.g., RSA cryptography).
  - **Timekeeping** (e.g., 12-hour or 24-hour clocks).
  - Other fields like computer science and number theory.
- For learning more, check resources like **Khan Academy** or **Wikipedia** for introductory material on modular arithmetic.

## Key Definitions
- **Modulus (`N`)**: A positive integer that defines the "wrap-around" point. For example, in modulo 12, numbers are reduced to their remainder when divided by 12.
- **Residue**: The remainder of a number `a` when divided by the modulus `N`, calculated as $\large{a \mod N}$. For example:
  - $\large{8 \mod 3 = 2}$, so the residue of 8 modulo 3 is 2.
  - $\large{7 \mod 12 = 7}$, and $\large{19 \mod 12 = 7}$, so 7 and 19 are **congruent modulo 12**.
- **Congruence**: Two numbers `A` and `B` are congruent modulo `N` if they have the same residue, written as $\large{A \equiv B \pmod{N}}$. For example:
  - $\large{7 \equiv 19 \pmod{12}}$ because $\large{7 \mod 12 = 7}$ and $\large{19 \mod 12 = 7}$.

## Project Requirements
You need to create a Python class called `Mod` to implement modular arithmetic operations. Below are the detailed requirements:

### 1. Class Initialization
- **Class Name**: `Mod`
- **Constructor (`__init__`)**:
  - Takes two arguments:
    - `value`: The number to store (can be any integer, positive or negative).
    - `modulus`: The modulus `N` (must be a positive integer).
  - **Validation**:
    - Ensure both `value` and `modulus` are integers.
    - Ensure `modulus` is positive ($\large{N > 0}$).
  - **Storage**:
    - Store the **residue** of `value` modulo `modulus`. For example, if `value = 8` and `modulus = 3`, store `2` because $\large{8 \mod 3 = 2}$.
  - **Read-Only Properties**:
    - Make `value` and `modulus` read-only to prevent modification after initialization.

### 2. Congruence Comparison
- Implement the **equality operator** (`__eq__`) to check congruence:
  - Two `Mod` objects are congruent if they have the **same modulus** and their **residues** are equal.
  - Allow comparison between a `Mod` object and an integer. For example:
    - `Mod(8, 3) == 11` should return `True` because $\large{8 \mod 3 = 2}$ and $\large{11 \mod 3 = 2}$.
  - **Restriction**: Do not allow comparison between two `Mod` objects with different moduli.

### 3. Hashability
- Ensure `Mod` objects are **hashable** (implement `__hash__`) so they can be used as keys in dictionaries or sets.
- The hash should be based on the residue and modulus to ensure consistency.

### 4. Integer Conversion
- Implement the `__int__` method to return the **residue** when `int()` is called on a `Mod` object. For example:
  - If `Mod(8, 3)` is created, `int(Mod(8, 3))` should return `2`.

### 5. String Representation
- Implement `__repr__` to provide a clear string representation of the `Mod` object. For example:
  - `Mod(8, 3)` could return something like `"Mod(2, 3)"` to show the residue and modulus.

### 6. Arithmetic Operations
- Implement the following arithmetic operators:
  - **Addition** (`__add__`): Add two `Mod` objects or a `Mod` object and an integer.
  - **Subtraction** (`__sub__`): Subtract two `Mod` objects or a `Mod` object and an integer.
  - **Multiplication** (`__mul__`): Multiply two `Mod` objects or a `Mod` object and an integer.
  - **Power** (`__pow__`): Raise a `Mod` object to the power of another `Mod` object or an integer.
- **Rules**:
  - For operations between two `Mod` objects, ensure they have the **same modulus**.
  - Convert integer operands to a `Mod` object with the same modulus before performing the operation. For example:
    - `Mod(2, 3) + 16` is equivalent to `Mod(2, 3) + Mod(16, 3)`.
    - Since $\large{16 \mod 3 = 1}$, this becomes `Mod(2, 3) + Mod(1, 3) = Mod(2 + 1, 3) = Mod(0, 3)`.
  - Always return a new `Mod` object with the computed residue and the same modulus.
- **In-Place Operators**:
  - Implement in-place versions: `__iadd__` (`+=`), `__isub__` (`-=`), `__imul__` (`*=`), `__ipow__` (`**=`).

### 7. Ordering
- Implement ordering methods (`__lt__`, `__le__`, `__gt__`, `__ge__`) to compare `Mod` objects or a `Mod` object with an integer.
- Ordering is based on the **residues** (which range from 0 to $\large{N-1}$).
- Allow comparison with:
  - Another `Mod` object with the same modulus.
  - An integer (converted to a `Mod` object with the same modulus).

## Example Scenarios
- **Congruence**:
  - `Mod(7, 12) == Mod(19, 12)` returns `True` because $\large{7 \mod 12 = 7}$ and $\large{19 \mod 12 = 7}$.
  - `Mod(8, 3) == 11` returns `True` because $\large{8 \mod 3 = 2}$ and $\large{11 \mod 3 = 2}$.
- **Arithmetic**:
  - `Mod(2, 3) + 16` → `Mod(2 + 16, 3) = Mod(18, 3) = Mod(0, 3)`.
  - `Mod(2, 3) * Mod(4, 3)` → Since $\large{4 \mod 3 = 1}$, compute `Mod(2 * 1, 3) = Mod(2, 3)`.

## Important Notes
- 📌 **Keep it simple**: All operations should work with the residues, and the `__init__` method will handle reducing values modulo `N`.
- ⚠️ **Modulus consistency**: Always check that moduli match when operating on two `Mod` objects.
- ✅ **Residue storage**: Always store the residue ($\large{a \mod N}$) rather than the original value.
- 🎯 **Hashability**: Ensure the hash is consistent with equality to support dictionary/set usage.
