# Class and Static Methods in Python

## Instance Methods: The Default

Functions in a class typically become **instance methods**, binding to the instance:

```python
class MyClass:
    def hello(self):
        return f"hello"

print(MyClass.hello)  # <function MyClass.hello at 0x7f9cfdbce0c0>
m = MyClass()
print(m.hello)        # <bound method MyClass.hello of <__main__.MyClass object at 0x7f9cfdbd7b00>>
print(m.hello())           # hello
# m.hello(1)          # TypeError: too many arguments
```

- **Unbound**: `MyClass.hello` is a plain function.
- **Bound**: `m.hello` is a method, expecting `self` (the instance).

**Key Point**: **Instance methods bind to instances** by default.

---

## Class Methods: Binding to the Class

Want a method always bound to the class, not the instance? Use `@classmethod`:

```python
class MyClass:
    @classmethod
    def hello(cls):
        return f"hello from {cls.__name__}"

print(MyClass.hello)  # <bound method MyClass.hello of <class '__main__.MyClass'>>
print(MyClass.hello())  # hello from MyClass
m = MyClass()
print(m.hello)        # <bound method MyClass.hello of <class '__main__.MyClass'>>
# print(dir(m.hello))   # [..., '__func__', ..., '__self__',...']
print(m.hello())      # hello from MyClass
```

- **`cls`**: The class (e.g., `MyClass`) is passed, not the instance.
- **Consistency**: Called via class or instance, it’s bound to the class.

**Takeaway**: **Class methods unify class-level behavior**.

---

## Static Methods: No Binding

Need a function in a class that’s never bound? Use `@staticmethod`:

```python
class Circle:
    @staticmethod
    def help():
        return "help available"

print(Circle.help)  # <function Circle.help at 0x7f9cfdbce020>
print(Circle.help())  # help available
c = Circle()
print(c.help)       # <function Circle.help at 0x7f9cfdbce020>
print(c.help())     # help available
```

- **No `self` or `cls`**: Just a plain function, no automatic arguments.
- **Controversy**: Some argue these belong at the module level, not in classes.

**Key Point**: **Static methods stay unbound**, acting like regular functions.

---

## When to Use Each?

Consider a `Timer` class:

```python
from time import time, ctime

class Timer:
    _timezone = "EST"  # Class-level attribute
    
    def start(self):  # Instance method
        self._start = time()
        return "Timer started"
    
    def end(self):    # Instance method
        return time() - self._start
    
    @classmethod
    def current_time_tz(cls):  # Class method
        return f"{ctime()} {cls._timezone}"
    
    @staticmethod
    def current_time_utc():  # Static method
        return ctime()
```

- **Instance Methods (`start`, `end`)**: Operate on instance state (`_start`).
- **Class Method (`current_time_tz`)**: Uses class state (`_timezone`), shared across instances.
- **Static Method (`current_time_utc`)**: Independent of class or instance state.

**Takeaway**: **Choose based on binding needs**—instance, class, or none.
