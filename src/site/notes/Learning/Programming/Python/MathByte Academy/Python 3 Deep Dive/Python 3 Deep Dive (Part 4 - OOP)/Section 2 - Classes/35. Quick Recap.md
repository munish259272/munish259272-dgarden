# Recap of Class Features in Python

## Class Basics

Let’s start with a `Circle` class:

```python
class Circle:
    origin = (0, 0)  # Class attribute

c = Circle()
print(Circle.origin)  # (0, 0)
print(c.origin)      # (0, 0)
```

- **Class Attribute**: `origin` is shared across all instances.
- **Access**: Available via `Circle.origin` or `c.origin`.
- **Change Impact**: Modifying `Circle.origin` affects all instances.

**Key Point**: **Class attributes are shared**, tied to the class itself.

---

## Custom Initializer

Add an `__init__` method:

```python
class Circle:
    origin = (0, 0)
    
    def __init__(self, r):
        self._r = r  # Instance attribute

c = Circle(5)
print(c._r)  # 5
```

- **Instance Method**: `__init__` binds to the instance (`self`) after creation.
- **Private Convention**: `_r` uses a leading underscore—private by convention, not enforcement.
- **Access**: Still reachable via `c._r`, though discouraged.

**Takeaway**: **Instance attributes live in the instance namespace**.

---

## Static Methods

Static methods don’t bind to anything:

```python
class Circle:
    origin = (0, 0)
    
    def __init__(self, r):
        self._r = r
    
    @staticmethod
    def new_circle(r):
        return Circle(r)

c = Circle.new_circle(3)
print(c._r)  # 3
```

- **Unbound**: Works the same from `Circle.new_circle()` or `c.new_circle()`—no `self` or `cls`.
- **Use Case**: Handy for utility functions, like creating instances.

**Key Point**: **Static methods are plain functions** within a class.

---

## Class Methods

Class methods bind to the class:

```python
class Circle:
    origin = (0, 0)
    
    @classmethod
    def move_origin(cls, x, y):
        cls.origin = (x, y)

Circle.move_origin(1, 2)
print(Circle.origin)  # (1, 2)
c = Circle(5)
print(c.origin)       # (1, 2)
```

- **Bound to Class**: Uses `cls`, affects `Circle.origin` whether called via `Circle` or `c`.
- **Effect**: Updates class-level data for all instances.

**Takeaway**: **Class methods manage class state**.

---

## Instance Methods

Instance methods operate on instance data:

```python
class Circle:
    def __init__(self, r):
        self._r = r
    
    def double_radius(self):
        self._r *= 2

c = Circle(5)
c.double_radius()
print(c._r)  # 10
```

- **Bound**: `double_radius` uses `self` to modify `c._r`.
- **Instance-Specific**: Only affects `c`, not other instances.

**Key Point**: **Instance methods target instance attributes**.

---

## Properties

Properties manage access to instance data:

```python
class Circle:
    def __init__(self, r):
        self._r = r
    
    @property
    def radius(self):
        return self._r
    
    @radius.setter
    def radius(self, value):
        self._r = value

c = Circle(5)
print(c.radius)  # 5
c.radius = 7
print(c.radius)  # 7
```

- **Getter**: `@property` makes `radius` readable like an attribute.
- **Setter**: `@radius.setter` allows setting `radius`, updating `_r`.
- **Bound**: Both methods are instance-bound.

**Takeaway**: **Properties control instance data access**.
