# Class Scope and Nested Functions

## Class Attributes and Scope

Class attributes live in the class body’s scope, accessible within that namespace:

```python
class Language:
    major = 3
    minor = 7
    revision = 4
    full = "{}.{}.{}".format(major, minor, revision)

print(Language.full)  # 3.7.4
```

- **Class Scope**: `major`, `minor`, `revision`, and `full` are symbols in `Language`’s namespace.
- **Works**: `full` sees `major`, etc., because they’re in the same scope.

**Key Point**: **Class body scope resolves internal references**.

---

## Functions and External Scope

Functions defined in a class don’t inherit the class scope—they’re nested in the module scope:

```python
class Language:
    major = 3
    minor = 7
    revision = 4
    
    @property
    def version(self):
        return "{}.{}.{}".format(self.major, self.minor, self.revision)
    
    @classmethod
    def class_version(cls):
        return "{}.{}.{}".format(cls.major, cls.minor, cls.revision)
    
    @staticmethod
    def static_version():
        return "{}.{}.{}".format(Language.major, Language.minor, Language.revision)

l = Language()
print(l.version)           # 3.7.4
print(Language.class_version())  # 3.7.4
print(Language.static_version()) # 3.7.4
```

- **Instance Method**: Uses `self.major` to access class attributes.
- **Class Method**: Uses `cls.major` for the same.
- **Static Method**: Needs `Language.major` explicitly—bare `major` fails (no class scope visibility).

**Takeaway**: **Functions look to the module scope**, not the class body.

---

## External Functions as Attributes

You can assign a module-level function to a class attribute:

```python
def full_version():
    return "{}.{}.{}".format(Language.major, Language.minor, Language.revision)

class Language:
    major = 3
    minor = 7
    revision = 4
    version = full_version

print(Language.version())  # 3.7.4
print(Language.version is full_version)  # True
```

- **Scope**: `full_version` lives in the module, not the class, but works when called via `Language.version()`.

**Key Point**: **Class attributes can point to external functions**.

## Broken Example: No Scope Access

Without explicit reference, functions fail:

```python
class Language:
    major = 3
    def version(self):
        return "{}.{}.{}".format(major, minor, revision)  # NameError

l = Language()
# l.version()  # NameError: 'major' not defined
```

- **Why?**: `major` isn’t in the function’s module scope—it’s in the class scope, which isn’t searched.

## Nested Scopes and Closures

Nested functions close over their enclosing scope, not the class:

```python
major, minor, revision = 0, 0, 0

def gen_class():
    major, minor, revision = 0, 4, 2
    class Language:
        major, minor, revision = 3, 7, 4
        @classmethod
        def version(cls):
            return "{}.{}.{}".format(major, minor, revision)
    return Language

Lang = gen_class()
print(Lang.version())  # 0.4.2 (from gen_class scope)
```

- **Closure**: `version` uses `major` from `gen_class`, not `Language`.
- **Proof**:
  ```python
  import inspect
  print(inspect.getclosurevars(Lang.version).nonlocals)  # {'major': 0, 'minor': 4, 'revision': 2}
  ```

**Takeaway**: **Class methods close over outer scopes**, not class body.

## Comprehensions and Scope Pitfalls

Comprehensions act like functions, bypassing class scope:

```python
name = "Guido"
class MyClass:
    name = "Raymond"
    list_1 = [name] * 3
    list_2 = [name for i in range(3)]
    
    @classmethod
    def hello(cls):
        return f"{name} says hello"

print(MyClass.list_1)      # ['Raymond', 'Raymond', 'Raymond']
print(MyClass.list_2)      # ['Guido', 'Guido', 'Guido']
print(MyClass.hello())     # Guido says hello
```

- **`list_1`**: Uses `name` from class scope (`Raymond`).
- **`list_2`**: Comprehension uses module scope (`Guido`).
- **`hello`**: Class method uses module scope (`Guido`).

⚠️ **Warning**: **Comprehensions skip class scope**, risking unexpected values.
