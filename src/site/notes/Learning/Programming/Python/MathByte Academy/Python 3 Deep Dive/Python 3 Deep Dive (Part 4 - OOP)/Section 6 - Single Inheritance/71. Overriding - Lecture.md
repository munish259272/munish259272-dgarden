# Overriding Methods in Python

## What is Overriding?

When a subclass redefines a parent’s method, it’s **overriding**:

```python
class Person:
    def say_hello(self):
        return "Hello"
    def say_bye(self):
        return "Bye"

class Student(Person):
    def say_hello(self):  # Override
        return "Yo"

p = Person()
s = Student()

print(p.say_hello())  # Hello
print(p.say_bye())    # Bye
print(s.say_hello())  # Yo (overridden)
print(s.say_bye())    # Bye (inherited)
```

- **Behavior**: `s.say_hello()` uses `Student`’s version; `s.say_bye()` uses `Person`’s.
- **Key Point**: **Overrides replace parent methods for the instance**.

---

## Overriding Across the Hierarchy

You can override methods from any ancestor, including `object`:

```python
class Person:
    def __init__(self, name):
        self.name = name
    def __repr__(self):
        return f"Person(name={self.name})"

class Student(Person):
    def __repr__(self):  # Overrides Person’s __repr__
        return f"Student(name={self.name})"

p = Person("John")
s = Student("Eric")

print(str(p))  # Person(name=John)
print(str(s))  # Student(name=Eric)
```

- **Inheritance**: `Student` inherits `__init__` from `Person`.
- **Override**: `Student` redefines `__repr__`; `Person`’s is ignored for `s`.
- **From `object`**: If `Person` had no `__init__`, `object`’s (no-args) version would run.
- **Takeaway**: **Override targets the closest definition**.

## Dynamic Class Names

Avoid hardcoding class names in overrides:

```python
class Person:
    def __init__(self, name):
        self.name = name
    def __repr__(self):
        return f"{self.__class__.__name__}(name={self.name})"

class Student(Person):
    pass  # No need to redefine __repr__

p = Person("John")
s = Student("Eric")

print(repr(p))  # Person(name=John)
print(repr(s))  # Student(name=Eric)
```

- **`__class__.__name__`**: Gets the instance’s class name dynamically.
- **Tip**: Use `type(self).__name__` instead of `__class__` (more reliable, shown later).
- **Key Point**: **Make methods adaptable to subclasses**.

## Overriding and Method Calls

Overrides affect calls within inherited methods:

```python
class Person:
    def eat(self):
        return "Person eats"
    def work(self):
        return "Person works"
    def sleep(self):
        return "Person sleeps"
    def routine(self):
        return f"{self.eat()}, {self.work()}, {self.sleep()}"

class Student(Person):
    def work(self):  # Override
        return "Student studies"

p = Person()
s = Student()

print(p.routine())  # Person eats, Person works, Person sleeps
print(s.routine())  # Person eats, Student studies, Person sleeps
```

- **Binding**: `s.routine()` uses `Person`’s `routine`, but `self` is `s`.
- **Lookup**: `self.work()` finds `Student`’s override; `eat` and `sleep` stay from `Person`.
- **Key Point**: **Overrides apply based on the instance, not the method’s class**.
