# Property Lookup with Descriptors

## The Lookup Question

How does Python resolve `obj.x` or `obj.x = value` when `x` is both a descriptor and in `__dict__`?

- **Scenario**: Class has `x = Descriptor()`, and instance `__dict__` has `'x': value`.
- **Key Point**: **Resolution depends on descriptor type**.

---

## Data Descriptors

**Data descriptors** (with `__get__` and `__set__` or `__delete__`) override `__dict__`:

```python
class DataDescriptor:
    def __set_name__(self, owner, name):
        self.name = name
    def __set__(self, instance, value):
        instance.__dict__[self.name] = value
    def __get__(self, instance, owner):
        if instance is None:
            return self
        return instance.__dict__.get(self.name)

class MyClass:
    prop = DataDescriptor()

m = MyClass()
m.prop = "value1"          # Uses __set__
print(m.prop)              # value1 (via __get__)
m.__dict__["prop"] = "value2"  # Direct dict set
print(m.prop)              # Still value1 (descriptor wins) 🐞 I THINK THIS IS WRONG. THE CORRECT OUTPUT SHOULD BE value2
m.prop = "value3"          # Uses __set__
print(m.__dict__)          # {'prop': 'value3'} (updated by descriptor)
```

- **Behavior**: `__get__` and `__set__` control access, ignoring `__dict__` unless explicitly used.
- **Key Point**: **Data descriptors take priority**.

## Non-Data Descriptors

**Non-data descriptors** (only `__get__`) let `__dict__` win:

```python
class NonDataDescriptor:
    def __get__(self, instance, owner):
        if instance is None:
            return self
        return "default"

class MyClass:
    prop = NonDataDescriptor()

m = MyClass()
print(m.prop)         # default (via __get__, no dict entry)
m.prop = 100          # Sets __dict__, no __set__ to intercept
print(m.__dict__)     # {'prop': 100}
print(m.prop)         # 100 (dict shadows descriptor)
```

- **Behavior**: Checks `__dict__` first; falls back to `__get__` if absent.
- **Key Point**: **Instance dict shadows non-data descriptors**.

---

## Why It Matters

- **Data Descriptors**: Safe to store values in `__dict__` under the same name (e.g., `prop`)—descriptor still rules.
- **Non-Data**: Assignment bypasses descriptor, so `__dict__` takes over.
- **Key Point**: **Descriptor type dictates lookup order**.
