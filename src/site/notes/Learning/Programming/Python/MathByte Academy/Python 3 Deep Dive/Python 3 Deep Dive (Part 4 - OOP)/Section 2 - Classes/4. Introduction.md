# Classes in Python

## Understanding Objects

In Python and **object-oriented programming (OOP)**, **objects** are the core building blocks. An object is a bundle of **data** (attributes) and **behavior** (functions or methods) that represents something—like a car with doors (data) and a start function (behavior).

### What Are Classes?
A **class** is a blueprint for creating objects. It defines the structure and capabilities of objects, and we use it to **instantiate** (create) and **initialize** (set up) them.

- **Instantiation**: Creating the object (Python handles this).
- **Initialization**: Setting initial values (what we mostly focus on).

**Key Point**: **Classes define objects**; instantiation makes them real.

---

## Creating Objects

### Instantiation and Initialization
```python
class Car:
    def __init__(self, doors):  # Initialization
        self.doors = doors      # Set initial data

# Instantiation happens here, Python calls __init__ automatically
my_car = Car(4)
print(my_car.doors)  # 4
```

- `__init__`: The **initializer** method runs after instantiation to set up the object.
- **Instantiation**: `Car(4)` creates the object; Python manages this step.

**Takeaway**: We control **initialization**, not instantiation.

## Attributes

Attributes give objects **data** and **functionality**.

### Class vs. Instance Attributes
- **Class Attributes**: Shared across all instances, defined in the class.
- **Instance Attributes**: Unique to each object, set via `self`.

```python
class Car:
    wheels = 4  # Class attribute (shared)

    def __init__(self, doors):
        self.doors = doors  # Instance attribute (unique)

car1 = Car(2)
car2 = Car(4)
print(car1.wheels, car1.doors)  # 4 2
print(car2.wheels, car2.doors)  # 4 4
```

**Key Point**: **Class attributes are shared**, **instance attributes are per-object**.

## Methods

Methods are **functions** tied to objects or classes, with a **binding mechanism** determining what they operate on.

### 1. Instance Methods
Bound to an **instance** (object), using `self`:

```python
class Car:
    def start(self):  # Instance method
        return "Car started"

my_car = Car()
print(my_car.start())  # "Car started"
```

- `self`: Refers to the calling instance.

### 2. Class Methods
Bound to the **class**, using `@classmethod` and `cls`:

```python
class Car:
    wheels = 4

    @classmethod
    def get_wheels(cls):  # Class method
        return cls.wheels

print(Car.get_wheels())  # 4
```

### 3. Static Methods
Not bound to class or instance, using `@staticmethod`:

```python
class Car:
    @staticmethod
    def honk():  # Static method
        return "Beep!"

print(Car.honk())  # "Beep!"
```

**Takeaway**: **Instance methods use `self`**, **class methods use `cls`**, **static methods stand alone**.

## Properties

**Properties** blend **data** and **functions**, letting you access data via methods while keeping a simple syntax.

```python
class Car:
    def __init__(self, doors):
        self._doors = doors

    @property
    def doors(self):  # Getter
        return self._doors

    @doors.setter
    def doors(self, value):  # Setter
        if value > 0:
            self._doors = value

my_car = Car(4)
print(my_car.doors)  # 4 (via property)
my_car.doors = 2     # Sets via setter
print(my_car.doors)  # 2
```

- **Why?**: Control access (e.g., validation) while looking like a data attribute.

**Key Point**: **Properties hide methods** behind attribute-like access.
