# Hashing and Equality in Python

## Why Hashing and Equality Matter

To use objects as keys in dictionaries or elements in sets (hash maps), they must be **hashable**. This requires implementing special methods: `__hash__` for hashing and `__eq__` for equality. These are crucial for custom classes, especially when you want equality based on data, not just object identity.

---

## Default Behavior

Without custom methods:

```python
class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y

p1 = Point(1, 2)
p2 = Point(1, 2)
print(p1 == p2)  # False (identity-based)
print(hash(p1))  # Some integer based on id(p1)
```

- **Equality**: $==$ defaults to `is`—compares memory addresses.
- **Hashing**: `__hash__` uses `id(obj)` (adjusted slightly), unique per object.
- **Problem**: `p1` and `p2` have same data but aren’t equal.

**Key Point**: **Defaults rely on identity**, not content.

---

## Custom Equality and Hashing

Implement both for meaningful comparisons:

```python
class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y
    
    def __eq__(self, other):
        if not isinstance(other, Point):
            return NotImplemented
        return self.x == other.x and self.y == other.y
    
    def __hash__(self):
        return hash((self.x, self.y))

p1 = Point(1, 2)
p2 = Point(1, 2)
print(p1 == p2)     # True (data-based)
print(hash(p1))     # Same as hash(p2)
d = {p1: "A"}
print(d[p2])        # "A" (works as dict key)
```

- **`__eq__`**: Compares `x` and `y` values.
- **`__hash__`**: Hashes a tuple of `x` and `y`—consistent with `__eq__`.
- **Result**: `p1` and `p2` are equal and hash to the same value, usable in hash maps.

---

## Gotcha with `__eq__` Alone

If you only define `__eq__`:

```python
class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y
    
    def __eq__(self, other):
        if not isinstance(other, Point):
            return NotImplemented
        return self.x == other.x and self.y == other.y

p1 = Point(1, 2)
# print(hash(p1))  # TypeError: unhashable type
```

- **Effect**: Python sets `__hash__` to `None`, making the object unhashable.
- **Why?**: Equality without a matching hash breaks hash map rules (equal objects must hash the same).
- **Takeaway**: **Pair `__eq__` with `__hash__`** for hashable objects.

---

## Immutability Matters

Only use these for **immutable objects**:

```python
class MutablePoint:
    def __init__(self, x, y):
        self.x = x
        self.y = y
    
    def __eq__(self, other):
        if not isinstance(other, MutablePoint):
            return NotImplemented
        return self.x == other.x and self.y == other.y
    
    def __hash__(self):
        return hash((self.x, self.y))

p = MutablePoint(1, 2)
d = {p: "A"}
p.x = 3  # Mutate after hashing
# print(d[p])  # KeyError (hash changed)
```

- **Issue**: Changing `x` alters the hash, breaking dictionary lookup.
- **Fix**: Use immutable data (e.g., tuples) or prevent changes.
- ⚠️ **Warning**: **Mutable hashables cause chaos** in hash maps.
