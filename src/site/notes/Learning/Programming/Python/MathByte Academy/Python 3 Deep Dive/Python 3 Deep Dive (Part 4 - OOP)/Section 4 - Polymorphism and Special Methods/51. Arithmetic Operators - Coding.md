# Customizing Arithmetic Operators with a Vector Class

## Building a Flexible Vector Class

Let’s create a `Vector` class with arbitrary dimensions, restricted to real numbers:

```run-python fold:1:3
from numbers import Real

class Vector:
    def __init__(self, *components):
        if len(components) < 1:
            raise ValueError("Cannot create an empty vector")
        for comp in components:
            if not isinstance(comp, Real):
                raise ValueError(f"Component {comp} is invalid; must be a real number")
        self._components = tuple(components)  # Store as tuple for immutability
    
    def __len__(self):
        return len(self._components)
    
    @property
    def components(self):
        return self._components
    
    def __repr__(self):
        return f"Vector{self._components}"

v1 = Vector(1, 2)
v2 = Vector(10, 20, 30, 40)
print(len(v1))  # 2
print(len(v2))  # 4
print(v1)       # Vector(1, 2)
print(str(v1))  # Vector(1, 2) (no __str__, falls back to __repr__)
```

- **Arbitrary Dimensions**: `*components` allows any number of args.
- **Validation**: Ensures non-empty and real numbers.
- **Immutability**: `_components` as a tuple (pseudo-private).
- **Key Point**: **Validate inputs upfront**.

## Adding and Subtracting Vectors

Support `+` and `-` with dimension checks:

```python
class Vector:
    # ... (previous code)
    
    def _validate_type_and_dim(self, other):
        return isinstance(other, Vector) and len(self) == len(other)
    
    def __add__(self, other):
        if not self._validate_type_and_dim(other):
            return NotImplemented
        new_components = tuple(x + y for x, y in zip(self.components, other.components))
        return Vector(*new_components)
    
    def __sub__(self, other):
        if not self._validate_type_and_dim(other):
            return NotImplemented
        new_components = tuple(x - y for x, y in zip(self.components, other.components))
        return Vector(*new_components)

v1 = Vector(1, 2)
v2 = Vector(10, 10)
print(v1 + v2)  # Vector(11, 12)
print(v2 - v1)  # Vector(9, 8)
v1 + Vector(1, 2, 3)  # TypeError: unsupported operand types
```

- **Validation**: `_validate_type_and_dim` ensures same type and size.
- **`NotImplemented`**: Signals unsupported ops without custom errors (simpler for now).
- **Key Point**: **Match dimensions for vector ops**.

## Scalar Multiplication

Add `*` for scalar multiplication (vector × real):

```python
class Vector:
    # ... (previous code)
    
    def __mul__(self, other):
        if isinstance(other, Real):
            return Vector(*(other * x for x in self.components))
        if self._validate_type_and_dim(other):
            return sum(x * y for x, y in zip(self.components, other.components))  # Dot product
        return NotImplemented
    
    def __rmul__(self, other):
        return self * other  # Reuse __mul__

v1 = Vector(1, 2)
print(v1 * 10)  # Vector(10, 20)
print(10 * v1)  # Vector(10, 20) (via __rmul__)
print(v1 * Vector(3, 4))  # 11 (dot product)
```

- **Scalar**: If `other` is real, scale the vector.
- **Dot Product**: If `other` is a same-dimension vector, compute the scalar result.
- **`__rmul__`**: Handles `10 * v1` by flipping to `v1 * 10`.
- **Key Point**: ** `__mul__` can multitask**—scalar or dot product.

## In-Place Addition

Implement `+=` with mutation:

```python
class Vector:
    # ... (previous code)
    
    def __iadd__(self, other):
        if not self._validate_type_and_dim(other):
            return NotImplemented
        self._components = tuple(x + y for x, y in zip(self.components, other.components))
        return self

v1 = Vector(1, 2)
v2 = Vector(10, 10)
print(id(v1))  # e.g., 140712...
v1 += v2
print(v1)      # Vector(11, 12)
print(id(v1))  # Same as before—mutated
```

- **Mutation**: Updates `_components` in-place.
- **Contrast**: Without `__iadd__`, `v1 += v2` ==would create a new object via `__add__` (i.e when we do v1 = v1 + v2)== .
- **Key Point**: ** `__iadd__` enables mutation** for mutable types ( ==not for immutable like tuples== ).

## Unary Operators and Absolute Value

Add negation and `abs()`:

```python
from math import sqrt

class Vector:
    # ... (previous code)
    
    def __neg__(self):
        return Vector(*(-x for x in self.components))
    
    def __abs__(self):
        return sqrt(sum(x * x for x in self.components))

v1 = Vector(1, 2)
print(-v1)      # Vector(-1, -2)
print(abs(v1))  # 2.236... (sqrt(5))
```

- **`__neg__`**: Flips signs, returns a new vector.
- **`__abs__`**: Computes magnitude (sqrt of sum of squares).
- **Takeaway**: **Unary ops extend functionality**.

## Beyond Numbers: Creative Example

Operators aren’t just for numbers—try a `Family` class:

```python
class Person:
    def __init__(self, name):
        self.name = name
    def __repr__(self):
        return f"Person('{self.name}')"

class Family:
    def __init__(self, mother, father):
        self.mother = mother
        self.father = father
        self.children = []
    
    def __iadd__(self, child):
        self.children.append(child)
        return self

f = Family(Person("Mary"), Person("John"))
f += Person("Eric")
f += Person("Michael")
print(f.children)  # [Person('Eric'), Person('Michael')]
```

- **`+=`**: Adds children—non-numeric but intuitive.
- **Key Point**: **Be creative with operators**—they’re not number-only.
