# Customizing and Extending Enums

## Customizing Enums

Enums are **classes**, so we can add methods:

```python
from enum import Enum

class State(Enum):
    READY = 1
    BUSY = 0

    def __bool__(self):
        return bool(self.value)
```

- **Default**: All members are truthy (`bool(State.READY)` and `bool(State.BUSY)` are `True`).
- **Customization**: `__bool__` ties truthiness to the value.
- **Result**:
  ```python
  print(bool(State.READY))  # True (1 is truthy)
  print(bool(State.BUSY))   # False (0 is falsy)
  ```
- **Key Point**: **Add methods to control member behavior**.

> [!note] **Customizing**
> 
> - Enums are **classes**
> 	- class attributes become **instances** of that class -> **members**
> 
> - We can define **functions** in the enumeration class
> 
> 	- -> become **bound methods** when called from a member (instance of the class)
> 
> 	- -> custom methods
> 
> 	- -> implement dunder methods
> 
> 		- **str** **repr** **eq** **lt** etc...

## Dunder Methods

Implement special methods like `__str__` or `__eq__`:

```python
class State(Enum):
    READY = 1
    BUSY = 0

    def __str__(self):
        return f"{self.name} ({self.value})"
```

- **Effect**: `print(State.READY)` → "READY (1)".
- **Key Point**: **Dunder methods enhance functionality**.

## Extending Enums

Enums support **subclassing**, but with a catch:

- **Rule**: **Only empty enums can be extended** (no members).
- **Why**: Preserves **immutability** of member lists.

### Example
```python
class BaseEnum(Enum):
    def greet(self):
        return f"Hello from {self.name}"

class Status(BaseEnum):
    ON = 1
    OFF = 0

print(Status.ON.greet())  # "Hello from ON"
```

- **Base**: `BaseEnum` adds `greet` with no members.
- **Subclass**: `Status` inherits `greet` and adds members.
- **Key Point**: **Use inheritance for reusable methods**.

---

## Why It Works

- **Members**: Become instances of the enum class, callable as bound methods.
- **Limit**: Can’t subclass `State` to add more members—immutability blocks it.
- **Key Point**: **Extend functionality, not membership**.
