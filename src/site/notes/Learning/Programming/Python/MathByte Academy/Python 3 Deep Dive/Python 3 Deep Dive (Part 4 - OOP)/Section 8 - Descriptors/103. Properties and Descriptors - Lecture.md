# Properties as Data Descriptors

## Properties Unpacked

Properties are **data descriptors** with `__get__`, `__set__`, and `__delete__`:

```python
class Person:
    def get_age(self):
        return self._age
    def set_age(self, value):
        self._age = value
    age = property(fget=get_age, fset=set_age)

p = Person()
p.age = 10    # Calls __set__
print(p.age)  # Calls __get__, prints 10
```

- **How**: `property()` creates a descriptor delegating to `fget` and `fset`.
- **Key Point**: **Properties leverage descriptor mechanics**.

---

## How It Works

- **Get**: `p.age` → `__get__(p, Person)` → calls `get_age(p)`.
- **Set**: `p.age = 10` → `__set__(p, 10)` → calls `set_age(p, 10)`.

```python
p = Person()
p.age = 10     # __set__ delegates to set_age
print(p.age)   # __get__ delegates to get_age
```

- **Flow**: Descriptor methods pass the instance to your functions.
- **Key Point**: **Delegation drives property behavior**.

---

## Missing `fset`

If `fset` is omitted, `__set__` fails:

```python
class Person:
    def get_age(self):
        return self._age
    age = property(fget=get_age)  # No fset

p = Person()
p.age = 10  # AttributeError: can't set attribute
```

- **Why**: `__set__` exists but checks `fset`; if `None`, it raises an error.
- **Key Point**: **Read-only properties lack `fset`**.

## DIY Property Example

Mimic properties with a custom descriptor:

```python
class MyProperty:
    def __init__(self, fget, fset=None):
        self.fget = fget
        self.fset = fset
    def __get__(self, instance, owner):
        if instance is None:
            return self
        return self.fget(instance)
    def __set__(self, instance, value):
        if self.fset is None:
            raise AttributeError("can't set attribute")
        self.fset(instance, value)

class Person:
    def get_age(self):
        return self._age
    def set_age(self, value):
        self._age = value
    age = MyProperty(fget=get_age, fset=set_age)

p = Person()
p.age = 10
print(p.age)  # 10
```

- **Purpose**: Shows how descriptors underpin properties.
- **Key Point**: **Custom descriptors replicate property logic**.
