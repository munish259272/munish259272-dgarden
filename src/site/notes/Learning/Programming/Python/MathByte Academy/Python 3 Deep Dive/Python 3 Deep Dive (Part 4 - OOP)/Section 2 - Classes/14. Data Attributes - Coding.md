# Understanding Data Attributes in Classes

Let's start by focusing on **data attributes of classes**.

## Creating a Bank Account Class

We'll create a `BankAccount` class and set an **annual percentage rate (apr)** as a class attribute.

```python
class BankAccount:
    apr = 1.2  # Class attribute
```

A **class attribute** belongs to the class itself and is shared across all instances. We can access it using:

```python
print(BankAccount.apr)  # Output: 1.2
```

Alternatively, we can retrieve it using the `__dict__` property:

```python
print(BankAccount.__dict__)  # Shows all class attributes
```

## Creating Instances

Let's create two instances of `BankAccount`:

```python
account_1 = BankAccount()
account_2 = BankAccount()
```

Each instance is a separate object, but they both share the same **class attribute**.

```python
print(account_1.apr)  # Output: 1.2
print(account_2.apr)  # Output: 1.2
```

Even though the **instance dictionary** (`__dict__`) for both accounts is empty initially:

```python
print(account_1.__dict__)  # Output: {}
print(account_2.__dict__)  # Output: {}
```

⚠️ **Important:** The apr value is retrieved from the class because it is not found in the instance dictionary.

## Adding Class Attributes Dynamically

Python allows us to add **new attributes** to a class even after its definition.

```python
BankAccount.account_type = "Savings"
```

Now, all instances of `BankAccount` can access this new class attribute:

```python
print(account_1.account_type)  # Output: Savings
print(account_2.account_type)  # Output: Savings
```

## Overriding a Class Attribute in an Instance

We can **override** the class attribute for a specific instance by assigning a new value to it:

```python
account_1.apr = 0  # Now apr is an instance attribute for account_1
```

This creates an **instance attribute** in `account_1`, which shadows the class attribute.

```python
print(account_1.__dict__)  # Output: {'apr': 0}
print(account_2.__dict__)  # Output: {}
```

Now, accessing `apr` gives different results for the two instances:

```python
print(account_1.apr)  # Output: 0  (Instance attribute)
print(account_2.apr)  # Output: 1.2 (Class attribute)
```

✅ **Python Lookup Order:** When accessing an attribute, Python follows this sequence:

1. Check if the attribute exists in the **instance dictionary**.
2. If not found, look for it in the **class**.
3. If still not found, follow the **inheritance chain** (if applicable).

## Using `setattr()` and `getattr()`

We can manipulate attributes dynamically using `setattr()` and `getattr()`:

```python
setattr(account_2, "apr", 10)  # Adds apr as an instance attribute to account_2
```

Now, `apr` for `account_2` is no longer inherited from the class:

```python
print(account_2.__dict__)  # Output: {'apr': 10}
print(getattr(account_2, "apr"))  # Output: 10
```

However, creating a new instance will still inherit from the class:

```python
account_3 = BankAccount()
print(account_3.apr)  # Output: 1.2 (Class attribute)
```

## Adding New Attributes to Instances

We can dynamically add **new attributes** to an instance that **do not exist in the class**:

```python
account_1.bank = "Acme Savings & Loans"
```

Now, `account_1` has an additional attribute:

```python
print(account_1.__dict__)  # Output: {'apr': 0, 'bank': 'Acme Savings & Loans'}
```

🔹 **Note:** `account_2` does not have this attribute because it was only added to `account_1`.

```python
print(account_2.__dict__)  # Output: {'apr': 10}
```

## Class vs Instance Attributes

**Key Differences:**

- **Class Attributes**: Shared across all instances; lives in the class itself.
- **Instance Attributes**: Specific to each object; lives in the object's namespace.

## Exploring `__dict__`

Let's inspect `__dict__` for both a class and an instance:

```python
print(BankAccount.__dict__)  # Shows a mapping proxy (immutable)
print(account_1.__dict__)    # Shows a mutable dictionary
```

🔹 **Mapping Proxy vs Dictionary:**

- The `__dict__` of a **class** is a **mapping proxy** (read-only view of class attributes).
- The `__dict__` of an **instance** is a regular **dictionary** (mutable).

### Example

```python
class Program:
    language = "Python"

p = Program()
print(p.__dict__)  # Output: {}

# Adding an attribute dynamically
p.version = "3.7"
print(p.__dict__)  # Output: {'version': '3.7'}
```

We can access the new attribute using:

```python
print(p.version)  # Output: 3.7
print(getattr(p, "version"))  # Output: 3.7
```
