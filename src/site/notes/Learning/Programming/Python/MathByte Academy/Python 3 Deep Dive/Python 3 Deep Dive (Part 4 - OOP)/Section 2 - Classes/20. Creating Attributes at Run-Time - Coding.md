# Adding Attributes and Methods at Runtime

## Runtime Instance Attributes

We’ve seen that attributes can be added to instances at runtime, like in `__init__`:
```python
class Person:
    pass

p1 = Person()
p2 = Person()
p1.name = "Alex"
print(p1.__dict__)  # {'name': 'Alex'}
print(p2.__dict__)  # {}
```

- **Dynamic**: `name` is added to `p1`’s namespace after creation.
- **Instance-Specific**: `p2` remains unaffected.

**Key Point**: **`__init__` and manual assignments** both modify instances at runtime.

---

## Adding Functions to Instances

Add a function as an attribute:
```python
p1.say_hello = lambda: "Hello"
print(p1.say_hello())  # "Hello"
print(p1.say_hello)  # <function <lambda> at ...>
```

- **Not a Method**: It’s a plain function in `p1.__dict__`, not bound to `p1`.
- **Why?**: Only class-defined functions become bound methods when accessed via an instance.

**Takeaway**: **Instance-level functions** don’t auto-bind.

---

## Creating Bound Methods at Runtime

To make a function a bound method on a specific instance, use `types.MethodType`:
```python
from types import MethodType

class Person:
    def __init__(self, name):
        self.name = name

p1 = Person("Eric")
p2 = Person("Alex")
```

Define a function:
```python
def say_hello(self):
    return f"{self.name} says hello"
```

Bind it to `p1`:
```python
p1.say_hello = MethodType(say_hello, p1)
print(p1.__dict__)  # {'name': 'Eric', 'say_hello': <bound method say_hello of <__main__.Person object at 0x7f92579e6f90>>}
print(hex(id(p1))) # 0x7f92579e6f90
print(p1.say_hello())  # "Eric says hello"
print(p2.__dict__)  # {'name': 'Alex'} (no say_hello)
```

- **`MethodType`**: Takes a function and the instance to bind it to.
- **Bound**: `say_hello` is now a method tied to `p1`, with `self` as `p1`.

**Key Point**: **Create instance-specific methods** with `MethodType`.

---

## Flexible Binding

The bound object can differ from the instance holding the method:
```python
p1.say_hello = MethodType(lambda self: f"{self.name} says hi", p2)
print(p1.say_hello())  # "Alex says hi" (uses p2’s name)
```

- **Bound to `p2`**: `say_hello` uses `p2`’s state, even though it’s on `p1`.

**Takeaway**: **Methods are self-contained**—binding determines behavior.

---

## Practical Example: Custom Functionality

Let’s create a class with pluggable methods:
```python
from types import MethodType

class Person:
    def __init__(self, name):
        self.name = name
    
    def register_do_work(self, func):
        setattr(self, "_do_work", MethodType(func, self))
    
    def do_work(self):
        method = getattr(self, "_do_work", None)
        if method:
            return method()
        raise AttributeError("You must first register a do_work method")
```

Test it:
```python
math_teacher = Person("Eric")
english_teacher = Person("John")

def work_math(self):
    return f"{self.name} will teach differentials today"

def work_english(self):
    return f"{self.name} will analyze Hamlet today"

math_teacher.register_do_work(work_math)
english_teacher.register_do_work(work_english)
```

Run:
```python
print(math_teacher.do_work())  # "Eric will teach differentials today"
print(english_teacher.do_work())  # "John will analyze Hamlet today"
print(math_teacher.__dict__)  # {'name': 'Eric', '_do_work': <bound method ...>}
```

- **`register_do_work`**: Binds a function as a method to the instance.
- **`do_work`**: Calls the registered method or raises an error if missing.
- **Benefit**: Same method name, different implementations.

Loop example:
```python
persons = [math_teacher, english_teacher]
for p in persons:
    print(p.do_work())
# Output:
# Eric will teach differentials today
# John will analyze Hamlet today
```

**Key Point**: **Runtime methods enable plug-in-like behavior** without inheritance.
