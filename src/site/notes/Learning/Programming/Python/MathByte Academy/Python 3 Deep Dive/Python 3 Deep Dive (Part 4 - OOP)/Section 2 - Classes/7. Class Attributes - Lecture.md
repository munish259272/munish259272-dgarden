# Class Attributes in Python

## Defining Class Attributes

Let’s create a class with attributes:
```python
class MyClass:
    language = "Python"
    version = 3.6
```

- **Focus**: These are **class attributes**, not instance attributes.
- **MyClass**: An **object** and a **type**, with `type(MyClass)` returning `<class 'type'>`.

**Key Point**: **Class attributes** are defined directly in the class body.

---

## Built-In and Custom Attributes

Python provides **automatic attributes** (e.g., `__name__ = "MyClass"`), and we’ve added:
- `language = "Python"`
- `version = 3.6"`

These are **stored in the class object**, not instances.

```python
print(MyClass.__name__)  # "MyClass" (built-in)
print(MyClass.language)  # "Python" (custom)
```

**Takeaway**: Classes have **state** from both Python and our definitions.

---

## Accessing Attributes

### Using `getattr()`
Retrieve attributes with the built-in **`getattr()`** function:
```python
value = getattr(MyClass, "language")  # "Python"
```

- **Syntax**: `getattr(object, "attribute_name", [default])`.
- **Default**: Returns a fallback if the attribute doesn’t exist:
```python
value = getattr(MyClass, "x", "N/A")  # "N/A" (no "x" attribute)
```

- Without default, it raises an `AttributeError`:
```python
getattr(MyClass, "x")  # Raises AttributeError
```

### Using Dot Notation
Simpler shorthand:
```python
print(MyClass.language)  # "Python"
```

- **Limitation**: No default—`MyClass.x` raises `AttributeError`.

**Key Point**: Use **`getattr()` for defaults**, **dot notation for simplicity**.

---

## Setting Attributes

### Using `setattr()`
Modify or add attributes with **`setattr()`**:
```python
setattr(MyClass, "version", 3.7)  # Updates version
setattr(MyClass, "x", 100)        # Adds new attribute if it does not exist
```

### Using Dot Notation
Equivalent and cleaner:
```python
MyClass.version = 3.7
MyClass.x = 100
```

- **Dynamic Nature**: Python lets us **add attributes** at runtime, even if not defined initially.

```python
print(MyClass.version)  # 3.7
print(MyClass.x)        # 100
```

**Takeaway**: **Classes are mutable**—state can change or grow.

---

## Where Is State Stored?

Class state lives in a **dictionary**, accessed via `__dict__`:
```python
print(MyClass.__dict__)
# Output: mappingproxy({'__module__': '__main__', 'language': 'Python', 'version': 3.7, 'x': 100, ...})
```

- **`__dict__`**: A **mapping proxy** (read-only dictionary-like object).Since it is read only we cannot add directly key,values to it, we need `setattr()` for doing that.
- **Keys**: Always **strings** (e.g., `"language"`), enforced by Python.
- **Values**: The attribute values (e.g., `"Python"`, `3.7`).

**Key Point**: **State is in a namespace dictionary**, optimized with string interning.

---

## Modifying Attributes

Update or add attributes dynamically:
```python
MyClass.version = 3.8  # Mutates existing
MyClass.y = 200        # Adds new
print(MyClass.__dict__)  # Reflects changes
```

- **Interning**: String keys are interned for faster lookups (see earlier string interning discussion).

---

## Deleting Attributes

Remove attributes at runtime with **`delattr()`** or **`del`**:
```python
delattr(MyClass, "version")  # Deletes version
del MyClass.x                # Deletes x
```

- **Result**: Removed from `__dict__`:
```python
print(MyClass.__dict__)  # No "version" or "x"
```

**Takeaway**: **Attributes can be deleted**, shrinking the namespace.

---

## Accessing the Namespace

Read `__dict__` for debugging:
```python
print(MyClass.__dict__["language"])  # "Python"
```

- ⚠️ **Warning**: Avoid using `__dict__` to get attributes routinely—use `getattr()` or dot notation. Not all attributes live here (more later).
- **Use Case**: Handy for inspecting state.

**Key Point**: **`__dict__` is read-only** but reflects mutations via `setattr()` or `del`.

```python
print(getattr(MyClass, "language", "my_default_value"))
```
