# Reusable Descriptors for Type Validation

## Basic Type Descriptors

Create descriptors for specific types:

```python
class Int:
    def __set_name__(self, owner, name):
        self.prop_name = name
    def __set__(self, instance, value):
        if not isinstance(value, int):
            raise ValueError(f"{self.prop_name} must be an integer")
        instance.__dict__[self.prop_name] = value
    def __get__(self, instance, owner):
        if instance is None:
            return self
        return instance.__dict__.get(self.prop_name)

class Float:
    def __set_name__(self, owner, name):
        self.prop_name = name
    def __set__(self, instance, value):
        if not isinstance(value, float):
            raise ValueError(f"{self.prop_name} must be a float")
        instance.__dict__[self.prop_name] = value
    def __get__(self, instance, owner):
        if instance is None:
            return self
        return instance.__dict__.get(self.prop_name)

class Person:
    age = Int()
    height = Float()

p = Person()
p.age = 12.5    # ValueError: age must be an integer
p.height = "10" # ValueError: height must be a float
```

- **Purpose**: Enforce type-specific properties.
- **Key Point**: **Descriptors ensure type validation**.

## The Copy-Paste Problem

Duplicating code for `Int`, `Float`, `List`, etc., screams for reusability:

- **Issue**: Each descriptor repeats the same logic, just swapping types.
- **Key Point**: **Repetition signals a need for generics**.

## Generic `ValidType` Descriptor

Refactor into a single, reusable descriptor:

```python
class ValidType:
    def __init__(self, type_):
        self._type = type_
    def __set_name__(self, owner, name):
        self.prop_name = name
    def __set__(self, instance, value):
        if not isinstance(value, self._type):
            raise ValueError(f"{self.prop_name} must be of type {self._type.__name__}")
        instance.__dict__[self.prop_name] = value
    def __get__(self, instance, owner):
        if instance is None:
            return self
        return instance.__dict__.get(self.prop_name)

class Person:
    age = ValidType(int)
    height = ValidType(float)
    tags = ValidType(list)

p = Person()
p.age = 12.5    # ValueError: age must be of type int
p.height = 10.5 # Works
p.tags = "abc"  # ValueError: tags must be of type list
```

- **How**: Pass the desired type to `__init__`; use `__name__` for readable errors.
- **Key Point**: **One class handles all types**.

## Flexible Type Checking

Use abstract base classes for broader validation:

```python
from numbers import Real

class Person:
    age = ValidType(int)
    height = ValidType(Real)  # Accepts int or float
    tags = ValidType(list)

p = Person()
p.height = 10    # Works (int is a Real)
p.height = 10.5  # Works (float is a Real)
p.height = "10"  # ValueError: height must be of type Real
```

- **Why**: `numbers.Real` covers both `int` and `float`.
- **Key Point**: **Abstract types expand flexibility**.
