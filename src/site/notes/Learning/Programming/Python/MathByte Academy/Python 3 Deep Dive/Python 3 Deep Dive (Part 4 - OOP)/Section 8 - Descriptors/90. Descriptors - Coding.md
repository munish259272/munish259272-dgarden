Lets  rewrite the code from the previous class again.

```python
from datetime import datetime,timezone

class TimeUTC:
    def __get__(self, instance, owner):
        return datetime.now(timezone.utc).isoformat()

class Logger:
    current_time = TimeUTC()

l = Logger()
print(Logger.current_time) # 2025-03-31T04:24:49.563578+00:00
print(l.current_time)  # 2025-03-31T04:24:49.563597+00:00
```

We see that `__get__` is called when we use `Logger.current_time` or `l.current_time`

Lets look at another code to pull 10 cards from a deck randomly.
```python
from random import choice, seed

class Deck:
    @property
    def suit(self):
        return choice(('Spade', 'Heart', 'Diamond', 'Club'))

    @property
    def card(self):
        return choice(tuple('23456789JQKA') + ('10',))

d = Deck()

seed(0)

for _ in range(10):
    print(d.card, d.suit)
```

The problem is that suit and card property are essentially doing the same thing i.e they choose from a given iterable.

Let's try doing the same thing using descripters where we can send the parameters to choose from.

```python
from random import choice, seed

class Choice:
    def __init__(self, *choices):
        self.choices = choices

    def __get__(self, instance, owner_class):
        return choice(self.choices)

class Deck:
    suit = Choice('Spade', 'Heart', 'Diamond', 'Club')
    card = Choice(*'23456789JQKA', '10')

seed(0)
d = Deck()
for _ in range(10):
    print(d.card, d.suit)
```

> [!question]- Does `*'23456789JQKA'` causes the unpacking of these strings into tuples ? Can you show a simple example of this ?
> 
> Yes, using the `*` operator with a string like `*'23456789JQKA'` causes the unpacking of the string into individual characters, effectively creating a tuple of those characters. Here is a simple example to illustrate this:
> 
> ```python
> # Original string
> cards = '23456789JQKA'
> 
> # Unpacking the string into individual characters
> unpacked_cards = *cards,
> 
> # Displaying the result
> print(unpacked_cards)  # Output: ('2', '3', '4', '5', '6', '7', '8', '9', 'J', 'Q', 'K', 'A')
> ```
> 
> In this example, `unpacked_cards` becomes a tuple containing each character from the original string `cards`. The use of the comma at the end (`*cards,`) is necessary to ensure that Python recognizes it as a tuple rather than a sequence of arguments.

Unlike property, use of descripters make our choice more generic now. We can apply it to another class for choosing something else like choosing a throw of dice.

```python
class Dice:
    die_1 = Choice(1, 2, 3, 4, 5, 6)
    die_2 = Choice(1, 2, 3, 4, 5, 6)
    die_3 = Choice(1, 2, 3, 4, 5, 6)

seed(0)

dice = Dice()
for _ in range(10):
	print(dice.die_1, dice.die_2, dice.die_3)
```
