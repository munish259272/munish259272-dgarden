# Advanced Descriptors: Polygon Vertices

Suppose we have a Polygon class that has a vertices property that needs to be defined as a sequence of Point2D instances. So here, not only do we want the vertices attribute of our Polygon to be an iterable of some kind, we also want the elements to all be instances of the Point2D class. In turn we'll 1 also want to make sure that coordinates for Point2D are non-negative integer values (as might be expected in computer screen coordinates)

## Integer Descriptor with Bounds

Define an `Int` descriptor for constrained integers:

```python
class Int:
    def __init__(self, min_value=None, max_value=None):
        self.min_value = min_value
        self.max_value = max_value
    def __set_name__(self, owner, name):
        self.name = name
    def __set__(self, instance, value):
        if not isinstance(value, int):
            raise ValueError(f"{self.name} must be an int")
        if self.min_value is not None and value < self.min_value:
            raise ValueError(f"{self.name} must be at least {self.min_value}")
        if self.max_value is not None and value > self.max_value:
            raise ValueError(f"{self.name} cannot exceed {self.max_value}")
        instance.__dict__[self.name] = value
    def __get__(self, instance, owner):
        if instance is None:
            return self
        return instance.__dict__.get(self.name)
```

- **Purpose**: Validates integers within a range.
- **Key Point**: **Flexible bounds via `__init__`**.

## Point2D Class

Build a `Point2D` class with validated coordinates:

```python
class Point2D:
    x = Int(min_value=0, max_value=800)
    y = Int(min_value=0, max_value=600)
    def __init__(self, x, y):
        self.x = x
        self.y = y
    def __repr__(self):
        return f"Point2D(x={self.x}, y={self.y})"
    def __str__(self):
        return f"({self.x}, {self.y})"
    def __eq__(self, other):
        return isinstance(other, Point2D) and self.x == other.x and self.y == other.y
    def __hash__(self):
        return hash((self.x, self.y))

p = Point2D(0, 10)  # Works
print(p)            # (0, 10)
try:
    Point2D(0, 800) # ValueError: y cannot exceed 600
except ValueError as e:
    print(e)
```

- **Features**: Non-negative integer `x` and `y`, equality, and hashability.
- **Key Point**: **Descriptors enforce screen coordinate rules**.

## Sequence Descriptor for Points

Create a `Point2DSequence` descriptor for polygon vertices:

```python
from collections.abc import Sequence

class Point2DSequence:
    def __init__(self, min_length=None, max_length=None):
        self.min_length = min_length
        self.max_length = max_length
    def __set_name__(self, owner, name):
        self.name = name
    def __set__(self, instance, value):
        if not isinstance(value, Sequence):
            raise ValueError(f"{self.name} must be a sequence")
        if self.min_length is not None and len(value) < self.min_length:
            raise ValueError(f"{self.name} must contain at least {self.min_length} elements")
        if self.max_length is not None and len(value) > self.max_length:
            raise ValueError(f"{self.name} cannot contain more than {self.max_length} elements")
        for idx, item in enumerate(value):
            if not isinstance(item, Point2D):
                raise ValueError(f"Item at index {idx} is not a Point2D instance")
        instance.__dict__[self.name] = list(value)  # Store as mutable list
    def __get__(self, instance, owner):
        if instance is None:
            return self
        if self.name not in instance.__dict__:
            instance.__dict__[self.name] = []
        return instance.__dict__[self.name]
```

- **Validation**: Ensures a sequence of `Point2D` with length constraints.
- **Key Point**: **Returns mutable list for flexibility**.

> [!question]- What is a `Sequence` ?
> 
> A `Sequence` in Python is an abstract base class from the `collections.abc` module that represents a finite ordered collection of elements. Sequences are a type of iterable, meaning that they can be looped over, but they also provide additional functionality such as indexing and slicing.
> 
> The key characteristics of sequences include:
> 
> 1. **Ordered Collection**: Elements in a sequence have a specific order, and this order is maintained.
> 
> 2. **Indexable**: You can access elements by their index position using square bracket notation (e.g., `sequence[index]`).
> 
> 3. **Slicable**: You can create sub-sequences using slicing (e.g., `sequence[start:end]`).
> 
> 4. **Iterable**: You can iterate over the elements in the sequence using a loop or comprehension.
> 
> Common built-in types that implement the `Sequence` abstract base class include:
> - Lists (`list`)
> - Tuples (`tuple`)
> - Strings (`str`)
> - Ranges (`range`)
> 
> These types provide concrete implementations of all the methods and properties defined by the `Sequence` interface, such as `__getitem__`, `__len__`, and others required for complete sequence behavior.
> 
> In the provided code snippet for the `Point2DSequence` descriptor, using a `Sequence` ensures that any value assigned to it supports these operations, making it suitable for storing an ordered collection of points with constraints on length and element type.

## Polygon Base Class

Define a `Polygon` class with sequence behavior:

```python
class Polygon:
    vertices = Point2DSequence(min_length=3)
    def __init__(self, *vertices):
        self.vertices = vertices
    def append(self, point):
        if not isinstance(point, Point2D):
            raise ValueError("Can only append Point2D instances")
        max_length = type(self).vertices.max_length
        if max_length is not None and len(self.vertices) >= max_length:
            raise ValueError(f"Vertices length is at max {max_length}")
        self.vertices.append(point)
    def __len__(self):
        return len(self.vertices)
    def __getitem__(self, key):
        return self.vertices[key]
    def __iadd__(self, point):
        self.append(point)
        return self
    def __contains__(self, point):
        return point in self.vertices

p = Polygon(Point2D(0, 0), Point2D(1, 0), Point2D(1, 1))
print(p.vertices)  # [Point2D(x=0, y=0), ...]
p.append(Point2D(10, 10))
print(p.vertices)
print(len(p))      # 4
print(p[1])        # Point2D(x=1, y=0)
```

- **Features**: Mutable vertices, sequence methods (`len`, `getitem`, etc.).
- **Key Point**: **Polygon acts like a sequence**.

> [!question]- Why are we using `type(self)` in `max_length = type(self).vertices.max_length` ?
> The use of `type(self)` in `max_length = type(self).vertices.max_length` is crucial to **access the descriptor instance defined in the actual class of the object (`self`), rather than hardcoding the base class (`Polygon`)**. This ensures flexibility, especially with inheritance. HereŌĆÖs why:
> 
> ### Context
> In the `Polygon` classŌĆÖs `append` method, we need to check the `max_length` of the `vertices` descriptor to enforce size constraints. The `vertices` descriptor is defined in the class, but its `max_length` can vary depending on the specific class (e.g., `Polygon`, `Triangle`, `Rectangle`).
> 
> ### Why `type(self)`?
> - **`self` is the instance**: `self` refers to the current object (e.g., a `Triangle` instance), but its attributes (like `vertices`) are resolved via its class.
> - **`type(self)` is the class**: `type(self)` gives the actual class of `self` (e.g., `Triangle`, not just `Polygon`), allowing access to the class-level `vertices` descriptor.
> - **Inheritance matters**: Subclasses like `Triangle` override `vertices` with different `max_length` values. Using `type(self)` ensures we get the descriptor from the correct class.
> 
> ### Code Breakdown
> ```python
> def append(self, point):
>     if not isinstance(point, Point2D):
>         raise ValueError("Can only append Point2D instances")
>     max_length = type(self).vertices.max_length  # Get descriptorŌĆÖs max_length
>     if max_length is not None and len(self.vertices) >= max_length:
>         raise ValueError(f"Vertices length is at max {max_length}")
>     self.vertices.append(point)
> ```
> 
> - **`type(self).vertices`**: Accesses the `Point2DSequence` descriptor instance defined in the class of `self`.
> - **`.max_length`**: Retrieves the `max_length` set for that specific descriptor.
> 
> ### Example
> ```python
> class Polygon:
>     vertices = Point2DSequence(min_length=3)  # No max_length
> 
> class Triangle(Polygon):
>     vertices = Point2DSequence(min_length=3, max_length=3)  # max_length=3
> 
> t = Triangle(Point2D(0, 0), Point2D(1, 0), Point2D(1, 1))
> t.append(Point2D(10, 10))  # Checks TriangleŌĆÖs max_length (3)
> ```
> 
> - **If `Polygon.vertices` was used**: `max_length` would be `None` (base class default), allowing unlimited appends, ignoring `Triangle`ŌĆÖs limit.
> - **With `type(self).vertices`**: `self` is a `Triangle`, so `type(self)` is `Triangle`, and `max_length` is 3, enforcing the subclass constraint.
> 
> ### Why Not `Polygon.vertices`?
> Hardcoding `Polygon.vertices` would:
> - Always use the base classŌĆÖs descriptor (`max_length=None`).
> - Ignore overrides in subclasses, breaking inheritance (e.g., `Triangle`ŌĆÖs `max_length=3` would be bypassed).
> 
> ### Why Not `self.vertices`?
> - `self.vertices` calls the descriptorŌĆÖs `__get__`, returning the instanceŌĆÖs `vertices` list, not the descriptor itself. The list doesnŌĆÖt have a `max_length` attributeŌĆöonly the descriptor does.
> 
> ### Key Point
> - **`type(self)` ensures the correct class descriptor is used**, making the code work seamlessly with inheritance and subclass-specific constraints like `max_length`.

> [!question]- Why do we need `len(self.vertices) >= max_length` ?
> 
> The condition `len(self.vertices) >= max_length` in the `append` method of the `Polygon` class is necessary to **prevent adding more points when the polygon has already reached or exceeded its maximum allowed number of vertices**. Here's why:
> 
> ### Context
> In the `Point2DSequence` descriptor, `max_length` defines an upper limit for the number of `Point2D` objects in the `vertices` sequence. The `append` method in `Polygon` needs to respect this limit when adding new points.
> 
> ### Why `>=`?
> - **Equality (`=`)**: If `len(self.vertices) == max_length`, the polygon is already at its maximum capacity (e.g., a `Triangle` with `max_length=3` already has 3 points). Adding another point would exceed the limit, which should be disallowed.
> - **Greater Than (`>`)**: Including `>` handles edge cases where the sequence might somehow exceed `max_length` (e.g., if initialized with too many points bypassing the descriptor). While the descriptorŌĆÖs `__set__` prevents this during initialization, `>=` ensures robustness if the list is manipulated directly.
> 
> ### Code Breakdown
> ```python
> def append(self, point):
>     if not isinstance(point, Point2D):
>         raise ValueError("Can only append Point2D instances")
>     max_length = type(self).vertices.max_length
>     if max_length is not None and len(self.vertices) >= max_length:
>         raise ValueError(f"Vertices length is at max {max_length}")
>     self.vertices.append(point)
> ```
> 
> - **Check**: `len(self.vertices) >= max_length` tests if the current number of vertices meets or exceeds the cap.
> - **Action**: If true, it raises an error instead of appending, preserving the constraint.
> 
> ### Example
> ```python
> class Triangle(Polygon):
>     vertices = Point2DSequence(min_length=3, max_length=3)
> 
> t = Triangle(Point2D(0, 0), Point2D(1, 0), Point2D(1, 1))  # 3 vertices
> print(len(t.vertices))  # 3
> t.append(Point2D(10, 10))  # ValueError: Vertices length is at max 3
> ```
> 
> - **Without `>=`**: If we only checked `>`, appending when `len(vertices) == 3` would succeed, breaking the `max_length=3` rule.
> - **With `>=`**: Stops at 3, enforcing the triangleŌĆÖs limit.
> 
> ### Why Not Just `>`?
> Using `len(self.vertices) > max_length` would allow appending when the length equals `max_length`, pushing the total beyond the intended limit. For a `Triangle`, this would mean 4 vertices, which contradicts the definition.
> 
> ### Key Point
> - **Purpose**: **`>=` ensures the polygon stays within its max vertex count**, making `append` consistent with the descriptorŌĆÖs `max_length` constraint.

## Specialized Polygons via Inheritance

Use inheritance to specialize:

```python
class Triangle(Polygon):
    vertices = Point2DSequence(min_length=3, max_length=3)

class Rectangle(Polygon):
    vertices = Point2DSequence(min_length=4, max_length=4)

t = Triangle(Point2D(0, 0), Point2D(1, 0), Point2D(1, 1))  # Works
try:
    t.append(Point2D(10, 10))  # ValueError: Vertices length is at max 3
except ValueError as e:
    print(e)
```

- **How**: Override `vertices` with specific constraints.
- **Key Point**: **Inheritance avoids code duplication**.
