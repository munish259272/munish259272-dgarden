# Building Custom Exceptions

## Basic Custom Exceptions

Define custom exceptions by **inheriting**:

```python
class TimeoutError(Exception):
    "Timeout exception"
```

- **Why**: Specific meaning (e.g., `TimeoutError`) beats vague `ValueError("timeout")`.
- **Docstring**: Always add one—no `pass` needed.
- **Key Point**: **Custom names improve clarity**.

### Handling
```python
try:
    raise TimeoutError("Timeout occurred")
except TimeoutError as e:
    print(repr(e))  # TimeoutError('Timeout occurred')
```

- **Broader Catch**: Works with `except Exception` too (not ideal).
- **Key Point**: **Specificity aids users**.

## Flexible Inheritance

Inherit from **any exception subclass**:

```python
class ReadOnlyError(AttributeError):
    "Indicates an attribute is read-only"

try:
    raise ReadOnlyError("Account number is read-only", 1001)
except ReadOnlyError as e:
    print(repr(e))  # ReadOnlyError('Account number is read-only', 1001)
except AttributeError:
    print("Caught as AttributeError")  # Also works
```

- **Options**: Catch as `ReadOnlyError`, `AttributeError`, or `Exception`.
- **Key Point**: **Inheritance sets catch scope**.

## Exception Hierarchies

Structure exceptions for **complex apps**:

```python
class WebScraperException(Exception):
    "Base exception for WebScraper library"

class HTTPException(WebScraperException):
    "General HTTP exception"

class InvalidURLException(HTTPException):
    "URL is invalid (e.g., DNS failure)"

class TimeoutException(HTTPException):
    "General HTTP timeout"

class PingTimeoutException(TimeoutException):
    "Ping timeout"

class LoadTimeoutException(TimeoutException):
    "Page load timeout"

class ParserException(WebScraperException):
    "General page parsing exception"
```

- **Usage**:
  ```python
  try:
      raise PingTimeoutException("Ping to www.example.com timed out")
  except HTTPException:
      print("HTTP issue")  # Catches all HTTP-related
  except WebScraperException:
      print("Scraper issue")  # Catches everything
  ```
- **Key Point**: **Hierarchy mirrors Python’s**.

## Enhancing Exceptions

Add **functionality** to exceptions:

```python
from datetime import datetime, timezone
from http import HTTPStatus
import json

class APIException(Exception):
    http_status = HTTPStatus.INTERNAL_SERVER_ERROR
    internal_error_message = "API exception occurred"
    user_error_message = "We’re sorry, an unexpected error occurred on our end"

    def __init__(self, *args, user_error_message=None):
        if args:
            self.internal_error_message = args[0]
            super().__init__(*args)
        else:
            super().__init__(self.internal_error_message)
        if user_error_message:
            self.user_error_message = user_error_message

    def to_json(self):
        return json.dumps({
            "status": self.http_status.value,
            "message": self.user_error_message
        })

    def log_exception(self):
        exc = {
            "type": type(self).__name__,
            "http_status": self.http_status.value,
            "message": self.args[0] if self.args else self.internal_error_message,
            "args": self.args[1:] if self.args else ()
        }
        print(f"{datetime.now(timezone.utc).isoformat()} - {exc}")
```

- **Test**:
  ```python
  try:
      raise APIException("Custom error")
  except APIException as e:
      e.log_exception()  # Logs with timestamp
      print(e.to_json()) # {"status": 500, "message": "We’re sorry..."}
  ```
- **Key Point**: **Custom logic ensures consistency**.

### Refined Hierarchy
```python
class ApplicationException(APIException):
    http_status = HTTPStatus.INTERNAL_SERVER_ERROR
    internal_error_message = "Generic server-side exception"
    user_error_message = "We’re sorry, an unexpected error occurred"

class DBException(ApplicationException):
    internal_error_message = "Database error"

class DBConnectionError(DBException):
    internal_error_message = "Database connection exception"

class ClientException(APIException):
    http_status = HTTPStatus.BAD_REQUEST
    internal_error_message = "Client submitted bad request"
    user_error_message = "Bad request received"

class NotFoundError(ClientException):
    http_status = HTTPStatus.NOT_FOUND
    internal_error_message = "Resource not found"
    user_error_message = "Requested resource was not found"

class NotAuthorizedError(ClientException):
    http_status = HTTPStatus.UNAUTHORIZED
    internal_error_message = "Client not authorized"
    user_error_message = "You’re not authorized to perform this request"
```

- **Key Point**: **Overrides tailor responses**.

## Practical Example: API Endpoint

Mock lookup with **enhanced exceptions**:

```python
class Account:
    def __init__(self, account_id, account_type):
        self.account_id = account_id
        self.account_type = account_type

def lookup_account_by_id(account_id):
    if not isinstance(account_id, int) or account_id <= 0:
        raise ClientException(f"Account id={account_id} is invalid",
                            account_id, "TypeError: not an integer")
    elif account_id < 100:
        raise DBConnectionError("Permanent failure connecting to database",
                              "DB-01")
    elif account_id < 200:
        raise NotAuthorizedError("User lacks permission",
                               f"account_id={account_id}")
    elif account_id < 300:
        raise NotFoundError("Account not found",
                          f"account_id={account_id}")
    else:
        return {"account_id": account_id, "type": "savings"}

def get_account(account_id):
    try:
        account = lookup_account_by_id(account_id)
    except APIException as e:
        e.log_exception()
        return e.to_json()
    else:
        return (HTTPStatus.OK.value,
                json.dumps({"id": account["account_id"],
                          "type": account["type"]}))

print(get_account("abc"))  # Logs, returns JSON with 400
print(get_account(50))     # Logs, returns JSON with 500
print(get_account(150))    # Logs, returns JSON with 401
print(get_account(250))    # Logs, returns JSON with 404
print(get_account(350))    # (200, '{"id": 350, "type": "savings"}')
```

- **Benefit**: Consistent logging and JSON output.
- **Key Point**: **Exceptions handle the heavy lifting**.

## Multiple Inheritance

Exceptions can inherit from **multiple parents**:

```python
class AppException(Exception):
    "Generic app exception"

class NegativeIntegerError(AppException, ValueError):
    "Negative integer error"

e = NegativeIntegerError("Age cannot be negative")
print(isinstance(e, AppException))  # True
print(isinstance(e, ValueError))    # True
```

- **Usage**:
  ```python
  def set_age(age):
      if age < 0:
          raise NegativeIntegerError("Age cannot be negative")

  try:
      set_age(-10)
  except ValueError:
      print("Caught as ValueError")  # Works
  except AppException:
      print("Caught as AppException")  # Also works
  ```
- **Key Point**: **Multiple inheritance adds flexibility**.
