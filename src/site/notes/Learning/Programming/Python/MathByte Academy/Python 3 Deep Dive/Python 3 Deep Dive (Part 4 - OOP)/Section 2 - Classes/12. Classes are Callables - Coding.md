# Classes and Instances in Python

## Classes Are Callable

When we define a class, Python makes it **callable**:
```python
class Program:
    language = "Python"
    def say_hello():
        print(f"Hello from {Program.language}")
```

- **Calling it**: `Program()` creates an **instance**.
- **Purpose**: Produces a new object of type `Program`.

**Key Point**: **Classes create instances** when called.

---

## Creating an Instance

Instantiate the class:
```python
p = Program()
print(type(p))          # <class '__main__.Program'>
print(isinstance(p, Program))  # True
```

- **`type(p)`**: Shows `p` is a `Program` object.
- **`isinstance()`**: Confirms `p` is an instance of `Program`.

**Takeaway**: **Instances are distinct objects** from the class.

---

## Namespace Separation

Classes and instances have **separate namespaces** and each instance gets its own **namespace**:
```python
print(Program.__dict__)  # mappingproxy({'language': 'Python', 'say_hello': <function ...>, ...})
print(p.__dict__)        # {}
```

- **Class**: Holds `language` and `say_hello`.
- **Instance**: Empty by default—its own space.

**Key Point**: **Class attributes** live in the class, not instances.

---

## Instance Attributes from Python

Instances get built-in attributes:
```python
print(p.__class__)  # <class '__main__.Program'>
```

- **`__class__`**: Links to `Program`, like `type(p)`.

**Takeaway**: **Instances have their own state**, even if empty.

---

## Checking Types Safely

### Using `type()` and `__class__`
```python
print(type(p) == p.__class__)  # True
```

- Both work, but **`__class__` can be tampered with**.

### Tampering Example
```python
class MyClass:
    __class__ = str  # Overrides __class__

m = MyClass()
print(m.__class__)    # <class 'str'> (misleading)
print(type(m))        # <class '__main__.MyClass'> (correct)
```

- **`isinstance()`**: Handles this better:
```python
print(isinstance(m, MyClass))  # True
print(isinstance(m, str))      # True (due to tampering)
```

Without tampering:
```python
class MyClass:
    pass

m = MyClass()
print(isinstance(m, str))  # False
```

**Key Point**: Use **`type()`** or **`isinstance()`**, not `__class__`, for reliability.

---

## Why `isinstance()`?

- **Safe**: Unaffected by attribute overrides.
- **Inheritance**: More useful later (we’ll see why).

**Takeaway**: **`isinstance()` is preferred** for type checking.
