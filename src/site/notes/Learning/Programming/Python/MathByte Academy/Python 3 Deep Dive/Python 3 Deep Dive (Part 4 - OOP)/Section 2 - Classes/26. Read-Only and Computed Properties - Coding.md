# Exploring Computed Properties in Python

## Read-Only vs. Write-Only Recap

**Read-only properties** (getter, no setter) are common, while write-only (setter, no getter) are rare but possible. Today, we’ll focus on **computed properties**.

---

## Basic Computed Property: Circle Area

A circle’s area feels like a property, not a method. Here’s how:

```python
from math import pi

class Circle:
    def __init__(self, radius):
        self.r = radius
    
    @property
    def area(self):
        print("Calculating area")
        return pi * self.r ** 2

c = Circle(1)
print(c.r)      # 1
print(c.area)   # Calculating area, 3.14159...
print(c.area)   # Calculating area, 3.14159...
```

- **Bare Attribute**: `r` is directly accessible.
- **Property**: `area` computes on demand, no `()` needed.
- **Preference**: `c.area` reads cleaner than `c.area()`.

**Key Point**: **Properties suit computed values** naturally.

---

## Issue: Recalculation Overhead

Notice `area` recalculates every time—inefficient if `r` doesn’t change. Let’s cache it:

```python
from math import pi

class Circle:
    def __init__(self, radius):
        self._r = radius
        self._area = None  # Cache
    
    @property
    def radius(self):
        return self._r
    
    @radius.setter
    def radius(self, value):
        self._r = value
        self._area = None  # Invalidate cache
    
    @property
    def area(self):
        if self._area is None:
            print("Calculating area")
            self._area = pi * self.radius ** 2  # Use property
        return self._area

c = Circle(1)
print(c.area)      # Calculating area, 3.14159...
print(c.area)      # 3.14159... (cached)
c.radius = 2       # Resets cache
print(c.area)      # Calculating area, 12.566...
```

- **`_area`**: Caches the result; `None` means “recompute.”
- **Setter**: Tracks `radius` changes, clears cache.
- **Why `self.radius`?**: Uses the getter, safer than `_r` if logic changes later.

**Takeaway**: **Setters enable caching** with change detection.

## Advanced Example: Web Page Metrics

Let’s build a `WebPage` class with lazy-loaded, computed properties:

```python
from urllib.request import urlopen
from time import perf_counter

class WebPage:
    def __init__(self, url):
        self._url = url
        self._page = None
        self._page_size = None
        self._load_time = None
    
    @property
    def url(self):
        return self._url
    
    @url.setter
    def url(self, value):
        self._url = value
        self._page = None  # Invalidate cache
    
    @property
    def page(self):
        if self._page is None:
            self.download_page()
        return self._page
    
    @property
    def page_size(self):
        if self._page is None:
            self.download_page()
        return self._page_size
    
    @property
    def load_time(self):
        if self._page is None:
            self.download_page()
        return self._load_time
    
    def download_page(self):
        self._page_size = None
        self._load_time = None
        start = perf_counter()
        with urlopen(self.url) as f:
            self._page = f.read()
        end = perf_counter()
        self._page_size = len(self._page)
        self._load_time = end - start

urls = ["https://google.com", "https://python.org", "https://yahoo.com"]
for url in urls:
    page = WebPage(url)
    print(f"{page.url}\tSize: {format(page.page_size, '_d')}\tTime: {page.load_time:.2f} sec")
```

- **Lazy Loading**: Downloads only when `page`, `page_size`, or `load_time` is accessed.
- **Cache Invalidation**: `url` setter resets `_page`, triggering recalculation.
- **Output Example**:
  ```
  https://google.com    Size: 12_345    Time: 0.18 sec
  https://python.org    Size: 49_876    Time: 0.19 sec
  https://yahoo.com     Size: 528_000   Time: 0.77 sec
  ```

**Note**: Use the `requests` library over `urllib` for real projects—simpler and more robust.

**Key Point**: **Properties handle complex, lazy computations** efficiently.

---

## When to Use Bare Attributes?

Start with bare attributes (`self.url = url`)—simple and fast. Convert to properties later if you need control (e.g., caching, validation) without breaking code.

**Takeaway**: **Properties scale from simple to sophisticated** use cases.
