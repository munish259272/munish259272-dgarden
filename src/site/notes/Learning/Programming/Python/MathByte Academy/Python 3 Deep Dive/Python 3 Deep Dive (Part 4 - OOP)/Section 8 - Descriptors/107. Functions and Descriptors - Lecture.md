# Functions as Descriptors

## Functions and Methods

Explore how instance methods transform from functions:

```python
class Person:
    def __init__(self, name):
        self.name = name
    def say_hello(self):
        return f"Hello, {self.name}"

p = Person("Alex")
print(p.say_hello())  # Hello, Alex
print(Person.say_hello)  # <function Person.say_hello ...>
```

- **Observation**: `p.say_hello()` binds to `p`, but `Person.say_hello` is just a function.
- **Key Point**: **Functions become methods when called on instances**.

---

## The Descriptor Trick

Functions are **non-data descriptors** with `__get__`:

- **How**: When accessed, a function’s `__get__` decides what to return:
  - **Class access**: `Person.say_hello` → raw function.
  - **Instance access**: `p.say_hello` → bound method (ties `self` to `p`).
- **Proof**:

```python
print(hasattr(Person.say_hello, "__get__"))  # True
print(p.say_hello)  # <bound method Person.say_hello of <__main__.Person ...>>
```

- **Key Point**: **No magic—descriptors handle the binding**.

---

## Why It Matters

- **No C-level hacks**: Python doesn’t use special if-statements; it’s pure descriptor logic.
- **Consistency**: Mirrors custom descriptors—functions adapt based on context.
- **Key Point**: **Understanding descriptors unlocks Python’s method mechanics**.
