# Callable Classes in Python

## Classes Are Callable

When we define a class with the **`class` keyword**, Python makes it **callable** automatically:
```python
class MyClass:
    pass
```

- **Callable**: Means we can use `MyClass()` to create something.
- **Result**: An **instance**—a new object of type `MyClass`.

**Key Point**: **Classes are callable** by default, producing instances.

---

## Instantiation Explained

Call the class to **instantiate** it:
```python
obj = MyClass()
print(type(obj))          # <class '__main__.MyClass'>
print(isinstance(obj, MyClass))  # True
```

- **`type(obj)`**: Returns the class (`MyClass`).
- **`isinstance()`**: Confirms `obj` is an instance of `MyClass`.
- **Terminology**: “Object” often means instance; “class” is the blueprint.

**Takeaway**: **Calling a class creates an instance**.

---

## How It Works

Python adds **behaviors** to classes:
- **Automatic**: The **`__call__`** method (more on this later) makes classes callable.
- **Instance**: The returned object has type `MyClass`.

---

## Instance Namespace

Each instance gets its own **namespace**:
```python
print(obj.__dict__)  # {}
print(MyClass.__dict__)  # mappingproxy({'__module__': '__main__', ...})
```

- **`__dict__`**: A dictionary holding the instance’s state (empty here).
- **Separate**: Distinct from the class’s `__dict__`.

**Key Point**: **Instances have unique namespaces**, unlike the class.

---

## Built-In Instance Attributes

Python provides instance attributes:
```python
print(obj.__class__)  # <class '__main__.MyClass'>
```

- **`__class__`**: Links back to `MyClass` (like `type(obj)`).
- **Usage**: Prefer `type()` or `isinstance()` over `__class__` for type checks.

**Takeaway**: **Instances inherit behaviors** from Python too.
