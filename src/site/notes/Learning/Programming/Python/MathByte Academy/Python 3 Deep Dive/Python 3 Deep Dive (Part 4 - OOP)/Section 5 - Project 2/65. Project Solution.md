> [!question]- What is the different between += and + i.e iadd and add ?

Below is a clear explanation of the differences, tailored to the context of your modular arithmetic project, with proper Markdown formatting and key points highlighted.
> 
> ## Difference Between `+` (`__add__`) and `+=` (`__iadd__`)
> 
> ### 1. **Purpose and Behavior**
> - **+ Operator (`__add__`)**:
>   - Used for **binary addition**, creating a new object as the result.
>   - When you write `a + b`, Python calls `a.__add__(b)`.
>   - It **does not modify** the original object (`a` or `b`); instead, it returns a new object with the computed result.
>   - In the context of the `Mod` class, `Mod(2, 3) + Mod(4, 3)` would return a new `Mod` object, e.g., `Mod(0, 3)` (since \( \large{2 + 4 = 6 \mod 3 = 0} \)).
>   
> - **+= Operator (`__iadd__`)**:
>   - Used for **in-place addition**, modifying the left-hand object directly.
>   - When you write `a += b`, Python calls `a.__iadd__(b)` if `__iadd__` is defined; otherwise, it falls back to `a = a.__add__(b)`.
>   - It is **intended to modify** the object `a` in place, but you can still return a new object if in-place modification isn't feasible.
>   - In the `Mod` class, `Mod(2, 3) += Mod(4, 3)` would typically modify the left-hand `Mod` object to store the residue \( \large{0} \) (if implemented to modify in place).
> 
> ### 2. **Return Value**
> - **`__add__`**:
>   - Must return a new object representing the result of the addition.
>   - Example:
>     ```python
>     def __add__(self, other):
>         if isinstance(other, Mod):
>             if self._modulus != other._modulus:
>                 return NotImplemented
>             return Mod(self._value + other._value, self._modulus)
>         elif isinstance(other, int):
>             return Mod(self._value + other, self._modulus)
>         return NotImplemented
>     ```
>     - `a = Mod(2, 3); b = Mod(4, 3); c = a + b` results in `c` being `Mod(0, 3)`, while `a` and `b` remain unchanged.
> 
> - **`__iadd__`**:
>   - Should return the modified object (or a new object if in-place modification isn't possible).
>   - If you modify the object in place, return `self`. If you create a new object, return that instead.
>   - Example:
>     ```python
>     def __iadd__(self, other):
>         if isinstance(other, Mod):
>             if self._modulus != other._modulus:
>                 return NotImplemented
>             self._value = (self._value + other._value) % self._modulus
>             return self  # Modify in place
>         elif isinstance(other, int):
>             self._value = (self._value + other) % self._modulus
>             return self
>         return NotImplemented
>     ```
>     - `a = Mod(2, 3); a += Mod(4, 3)` modifies `a` to become `Mod(0, 3)`.
> 
> ### 3. **In-Place Modification vs. New Object**
> - **`__add__`**: Always creates a new object, leaving the original objects unchanged. This is useful for maintaining immutability or when you want to preserve the original values.
> - **`__iadd__`**: Designed for in-place modification, which can be more efficient because it avoids creating a new object. However, for immutable objects (like `Mod` if you choose to make it immutable), `__iadd__` may return a new object instead, mimicking the behavior of `__add__`.
> 
> ### 4. **Fallback Behavior**
> - If `__iadd__` is not defined, Python falls back to using `__add__` for `+=`. For example:
>   - `a += b` becomes `a = a.__add__(b)` if `__iadd__` is not implemented.
>   - This means `+=` will still work, but it won't modify `a` in place; it will reassign `a` to a new object.
> 
> ### 5. **Use in the `Mod` Class**
> In your modular arithmetic project, you need to implement both `__add__` and `__iadd__` for the `Mod` class:
> - **For `__add__`**:
>   - Compute the sum of residues and return a new `Mod` object.
>   - Example: `Mod(2, 3) + Mod(4, 3)` → `Mod((2 + 4) % 3, 3) = Mod(0, 3)`.
>   - Ensure the moduli match for two `Mod` objects, and handle integer operands by converting them to the same modulus.
> - **For `__iadd__`**:
>   - Modify the residue of the left-hand `Mod` object (or return a new `Mod` object if you prefer immutability).
>   - Example: `a = Mod(2, 3); a += Mod(4, 3)` → `a` becomes `Mod(0, 3)` if modified in place.
>   - Check for modulus compatibility and handle integer operands similarly.
> 
> ### 6. **Immutability Consideration**
> - If you want `Mod` to be **immutable** (like Python's `int` or `str`), you can implement `__iadd__` to return a new `Mod` object instead of modifying `self`. For example:
>   ```python
>   def __iadd__(self, other):
>       if isinstance(other, Mod):
>           if self._modulus != other._modulus:
>               return NotImplemented
>           return Mod(self._value + other._value, self._modulus)
>       elif isinstance(other, int):
>           return Mod(self._value + other, self._modulus)

>   - Here, `a += b` creates a new `Mod` object, just like `a + b`, and reassigns it to `a`.
