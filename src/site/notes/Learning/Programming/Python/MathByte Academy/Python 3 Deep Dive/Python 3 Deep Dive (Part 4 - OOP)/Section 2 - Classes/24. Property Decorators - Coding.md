# Mastering Property Decorators in Python

## Properties Outside Classes

We’ve seen **properties** as class attributes, but they’re just objects of the `property` class. You can create them standalone:

```python
p = property(fget=lambda self: print("Getting property"))
print(p.fget)  # <function <lambda> at ...>
```

- **`p`**: A `property` object with a getter, no class required.
- **Attributes**: `p.fget`, `p.fset`, `p.fdel` hold the getter, setter, and deleter (if defined).

**Key Point**: **Properties are independent objects**, configurable with `fget`, `fset`, etc.

---

## Quick Decorator Recap

Decorators wrap functions, replacing the original name with a modified version. Example:

```python
def my_decorator(fn):
    print("Decorating function")
    def inner(*args, **kwargs):
        print("Running decorated function")
        return fn(*args, **kwargs)
    return inner

@my_decorator
def my_func(a, b):
    print("Running original function")
    return a + b

my_func(1, 2)
# Output:
# Decorating function
# Running decorated function
# Running original function
# 3
```

- **Equivalent**: `my_func = my_decorator(my_func)`.
- **Closure**: `inner` remembers `fn` (the original function).

**Takeaway**: **Decorators redefine names** with enhanced behavior.

---

## Property as a Decorator

Since `property` takes a function and returns a `property` object, it’s a natural decorator:

```python
class Person:
    def __init__(self, name):
        self._name = name
    
    @property
    def name(self):
        print("Getter called")
        return self._name

p = Person("John")
print(type(Person.name)) # <class 'property'>
print(type(p.name)) # <class 'str'>
print(Person.name.fget) # <function Person.name at 0x7faea3dde7a0>
print(Person.name.fset) # None
print(p.name)  # Getter called, John
```

- **`@property`**: Turns `name` into a `property` with `name` as the getter.
- **Manual Version**:
  ```python
  def get_name(self):
      print("Getter called")
      return self._name
  name = property(get_name)
  ```

**Key Point**: **`@property` simplifies getter setup**.

---

## Property Methods in Action

The `property` object has `getter`, `setter`, and `deleter` methods to modify it:

```python
def get_prop(self):
    return "Getter called"
def set_prop(self, value):
    print("Setter called")
def del_prop(self):
    print("Deleter called")

p = property(get_prop)  # Start with getter
print(p.fget)  # <function get_prop at ...>
print(p.fset)  # None

p = p.setter(set_prop)  # Add setter
p = p.deleter(del_prop)  # Add deleter
print(p.fset)  # <function set_prop at ...>
print(p.fdel)  # <function del_prop at ...>
```

- **New Object**: Each call (`p.setter`, etc.) returns a **new `property` instance**, copying existing attributes (like `fget`).
- **Class Use**:
  ```python
  class Person:
      name = p
  p = Person()
  print(p.name)  # Getter called
  p.name = "Eric"  # Setter called
  del p.name  # Deleter called
  ```

**Takeaway**: **Property methods build behavior incrementally**.

## Decorators with Setter/Deleter

Combine decorators for a full property:

```python
class Person:
    def __init__(self, name):
        self._name = name
    
    @property
    def name(self):
        return self._name
    
    @name.setter
    def name(self, value):
        self._name = value

p = Person("Alex")
p.name = "Eric"  # Sets _name
print(p.name)  # Eric
```

- **`@name.setter`**: Updates the `name` property with a setter.
- **Name Consistency**: Both functions must be `name` to target the same property.

**Mistake**:
```python
@name.setter
def name_set(self, value):  # Wrong name
    self._name = value
```
- **Result**: Creates a separate `name_set` attribute, not a setter for `name`.

**Key Point**: **Same name ensures one cohesive property**.

---

## Manual vs. Decorator Steps

Manually:
```python
name = property(lambda self: "Getter")
name = name.setter(lambda self, value: print("Setter"))
```

With decorators:
- `@property def name`: Sets the getter.
- `@name.setter def name`: Adds the setter.

**Proof**:
- `name` starts as a function, gets redefined as a `property`, then updated with a setter—all under the same name.

**Takeaway**: **Decorators automate symbol reassignment**.

---

## Docstrings with Decorators

Add a docstring to the getter:
```python
class Person:
    def __init__(self, name):
        self._name = name
    
    @property
    def name(self):
        """The person's name."""
        return self._name
    
    @name.setter
    def name(self, value):
        self._name = value

help(Person.name)  # "The person's name"
```

- **Setter Docstring**: Ignored—only the getter’s docstring counts.

**Key Point**: **Docstrings go in the getter** for properties.

---

## Write-Only Properties

For a setter-only property:
```python
class Person:
    prop = property(fset=lambda self, value: print("Setter called"))
    # OR
    prop = property().setter(lambda self, value: print("Setter called"))

p = Person()
p.prop = "Hello"  # Setter called
# p.prop  # AttributeError: unreadable attribute
```

- **Docstring**:
  ```python
  prop = property(fset=lambda self, value: print("Setter called"), doc="Write-only prop")
  ```

**Takeaway**: **Set-only properties skip `@property`**—use `fset` directly.
