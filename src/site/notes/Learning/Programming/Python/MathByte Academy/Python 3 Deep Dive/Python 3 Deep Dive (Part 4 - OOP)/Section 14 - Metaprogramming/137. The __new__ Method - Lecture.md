# Tutorial: Understanding `__new__` and `__init__`

This video explains the two-step process of creating an instance: **object creation** (`__new__`) and **object initialization** (`__init__`).

## The Two-Step Instance Creation Process

When you "call" a class like `Person("Guido")`, Python executes a two-phase process:

1.  **Creation:** `__new__(cls, ...)` creates and returns a **new object**.
2.  **Initialization:** `__init__(self, ...)` initializes that new object.

### 🎯 Key Analogy
Think of building a house:
*   `__new__` is like **constructing the empty house frame** (creating the object in memory).
*   `__init__` is like **furnishing and decorating the house** (setting up initial attributes).

## How Python Calls `__new__` and `__init__`

```python
class Person:
    def __init__(self, name):
        print(f"__init__ called with name={name}")
        self.name = name

# When you do this:
p = Person("Guido")

# Python internally does something like this:
# 1. Call __new__ to CREATE the object
p = Person.__new__(Person, "Guido")  # Returns a new "blank" Person instance

# 2. Then call __init__ to INITIALIZE it
if isinstance(p, Person):  # Only if __new__ returned a Person instance
    Person.__init__(p, "Guido")  # Initialize the new object
```

## The Default `__new__` Method

Every class inherits `__new__` from the `object` base class:

```python
# You can call it directly (though you shouldn't normally)
p = object.__new__(Person)  # Creates a Person instance WITHOUT calling __init__
print(type(p))  # <class '__main__.Person'>
print(hasattr(p, 'name'))  # False - __init__ wasn't called!

# To fully initialize it, you'd need to call __init__ manually
Person.__init__(p, "Guido")
print(p.name)  # "Guido"
```

## Overriding `__new__` in Custom Classes

You can override `__new__` to customize object creation:

```python
class Person:
    def __new__(cls, *args, **kwargs):
        print(f"__new__ called for class {cls}")
        print(f"Args: {args}, Kwargs: {kwargs}")
        
        # Delegate actual creation to the parent class
        # Use super(), not object.__new__(), for proper inheritance
        instance = super().__new__(cls)
        
        # You can modify the instance before returning it
        print(f"Created instance: {instance}")
        
        return instance
    
    def __init__(self, name, age):
        print(f"__init__ called with name={name}, age={age}")
        self.name = name
        self.age = age

# Testing it
p = Person("Guido", 67)
# Output:
# __new__ called for class <class '__main__.Person'>
# Args: ('Guido', 67), Kwargs: {}
# Created instance: <__main__.Person object at 0x...>
# __init__ called with name=Guido, age=67
```

## 📌 Critical Rules for `__new__`

### 1. **`__new__` is a Static Method**
It receives the class (`cls`) as its first argument, not an instance (`self`).

### 2. **Arguments Must Match**
The arguments in `__new__` must be compatible with `__init__` since Python passes the same arguments to both.

### 3. **`__new__` Must Return an Object**
It can return **any object**, not necessarily an instance of the class.

### 4. **`__init__` is Only Called If...**
Python only calls `__init__` if `__new__` returns **an instance of the same class** (or a subclass).

```python
class Person:
    def __new__(cls, name):
        print("__new__ called")
        # Returning something that's NOT a Person instance
        return "I'm not a Person object!"
    
    def __init__(self, name):
        print("This will NEVER be called!")
        self.name = name

p = Person("Guido")  # __init__ is NOT called!
print(p)             # "I'm not a Person object!"
print(type(p))       # <class 'str'> - Not a Person!
```

## Practical Use Case: Singleton Pattern

`__new__` is often used to control instance creation, like implementing the Singleton pattern:

```python
class Singleton:
    _instance = None
    
    def __new__(cls):
        if cls._instance is None:
            print("Creating new singleton instance")
            cls._instance = super().__new__(cls)
        else:
            print("Returning existing singleton instance")
        return cls._instance
    
    def __init__(self):
        print("Singleton initialized")

# Testing
s1 = Singleton()  # "Creating new singleton instance", "Singleton initialized"
s2 = Singleton()  # "Returning existing singleton instance", "Singleton initialized"
print(s1 is s2)   # True - Same object!
```

## ⚠️ Important Warning

*   **Don't override `__new__` unless you have a specific need.** Most customization should be done in `__init__`.
*   **Always use `super().__new__(cls)`** instead of `object.__new__(cls)` to maintain proper inheritance.
*   **Remember:** `__new__` creates the object, `__init__` initializes it.
