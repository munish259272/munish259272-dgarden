# Objects and Classes in Python

## What Are Objects?

In Python and **object-oriented programming (OOP)**, an **object** is like a **container**. It holds:
- **Data** (state or attributes): Information like a car’s brand or year.
- **Behavior** (functions or methods): Actions like accelerating or braking.

For example:
- A **list** object has data (its elements) and behavior (methods like `sort()` or `remove()`).
- A **car** object might have `brand = "Ferrari"` (state) and `accelerate()` (behavior).

**Key Point**: **Objects combine state and behavior** into one unit.

---

## Accessing Object Contents

We use **dot notation** to access attributes:
```python
my_car = ...  # Some car object
print(my_car.brand)         # "Ferrari" (data attribute)
my_car.purchase_price = 1.6e6  # Set state
print(my_car.accelerate())  # Call a method (behavior)
```

- **Attributes**: Data (e.g., `brand`) or methods (e.g., `accelerate`).
- **Methods**: Functions tied to the object.

**Takeaway**: **Dot notation links us to object contents**.

---

## What Are Classes?

A **class** is a **blueprint** or **template** for creating objects. It defines the structure (state) and capabilities (behavior) of objects. In Python:
- **Class** = **Type**: Terms used interchangeably (e.g., `list` is both a class and a type).
- **Instances**: Objects created from a class.

```python
class Car:
    pass

my_car = Car()  # Instance of Car
print(type(my_car))  # <class '__main__.Car'>
```

**Key Point**: **Classes define types**; instances are the objects.

---

## State and Behavior: Class vs. Instance

### Class State and Behavior
- **Classes are objects too**: They have their own attributes and methods.
- Example: The class name is stored as state, and the ability to create instances is behavior.

```python
class Car:
    pass

print(Car.__name__)  # "Car" (state)
my_car = Car()       # Callable behavior
```

### Instance State and Behavior
- Defined within the class, unique to each object.

```python
class Car:
    def __init__(self, brand):
        self.brand = brand  # Instance state
    def accelerate(self):
        return "Vroom!"

my_car = Car("Ferrari")
print(my_car.brand)        # "Ferrari"
print(my_car.accelerate()) # "Vroom!"
```

**Takeaway**: **Classes have shared traits**, **instances have unique ones**.

---

## Creating Classes

Use the **`class` keyword**:
```python
class MyClass:
    pass  # Empty class
```

- When Python encounters this:
  - It **compiles** the class into an object (like a function with `def`).
  - It’s stored as `MyClass`, a **callable object** that creates instances.

### Built-In Features
Even an "empty" class has:
- **State**: `MyClass.__name__` = `"MyClass"`.
- **Behavior**: Calling `MyClass()` creates an instance.

```python
print(MyClass.__name__)  # "MyClass"
obj = MyClass()
```

**Key Point**: **Classes are objects** with automatic attributes from Python.

---

## Types and Instances

- **Class Type**: The type of `MyClass` is `type` (a built-in class).
- **Instance Type**: The type of `obj` is `MyClass`.

```python
print(type(MyClass))      # <class 'type'>
print(type(obj))          # <class '__main__.MyClass'>
print(isinstance(obj, MyClass))  # True
print(isinstance(MyClass, type)) # True
```

- **Terminology**: In OOP, “objects” often mean instances, but in Python, **everything is an object** (classes included).

**Takeaway**: **Classes are types**, and their instances reflect that type.

---

## How Are Classes Created?

Classes come from **metaclasses** (e.g., `type`), but that’s advanced territory for later. For now:
- **Classes are callable**: `MyClass()` invokes behavior to make instances.
- Python handles this behind the scenes.
