# Diving into Rich Comparisons with Vectors

## Setting Up a 2D Vector Class

Let’s create a `Vector` class for 2D points with a custom representation:

```python
class Vector:
    def __init__(self, x, y):
        self.x = x
        self.y = y
    
    def __repr__(self):
        return f"Vector(x={self.x}, y={self.y})"

v1 = Vector(0, 0)
v2 = Vector(0, 0)
print(id(v1))  # Different memory address
print(id(v2))  # Different memory address
print(v1 == v2)  # False (default is identity)
```

- **Default Equality**: Without `__eq__`, $==$ checks object identity (`is`), not values.
- **Key Point**: **Default $==$ compares memory, not content**.

## Implementing `__eq__`

Add equality to compare components:

```python
class Vector:
    def __init__(self, x, y):
        self.x = x
        self.y = y
    
    def __repr__(self):
        return f"Vector(x={self.x}, y={self.y})"
    
    def __eq__(self, other):
        if isinstance(other, Vector):
            return self.x == other.x and self.y == other.y
        return NotImplemented

v1 = Vector(1, 1)
v2 = Vector(1, 1)
v3 = Vector(10, 10)
print(v1 == v2)  # True (same values)
print(v1 is v2)  # False (different objects)
print(v1 == v3)  # False (different values)
```

- **Logic**: Checks if `other` is a `Vector`, then compares `x` and `y`.
- **Fallback**: `NotImplemented` for non-`Vector` types.
- **Takeaway**: ** `__eq__` defines value-based equality**.

## Extending Equality to Tuples

Support comparing with tuples:

```python
class Vector:
    def __init__(self, x, y):
        self.x = x
        self.y = y
    
    def __repr__(self):
        return f"Vector(x={self.x}, y={self.y})"
    
    def __eq__(self, other):
        print("EQ called")
        if isinstance(other, tuple):
            # print(*other)  # 10 11
            other = Vector(*other)  # Convert tuple to Vector
        if isinstance(other, Vector):
            return self.x == other.x and self.y == other.y
        return NotImplemented

v1 = Vector(10, 11)
#print(v1 == (10, 11))  # EQ called, True
print((11, 11) == v1)  # EQ called, True (via reflection)
```

- **Conversion**: Tuples become `Vector`s; errors if not 2D.
- **Reflection**: `(10, 11) == v1` fails tuple’s `__eq__`, flips to `v1.__eq__((10, 11))`.
- **Key Point**: **Reflection flips operands** when needed.

## Adding `__lt__` with Euclidean Norm

Compare vectors by their magnitude (distance from origin):

```python
from math import sqrt

class Vector:
    def __init__(self, x, y):
        self.x = x
        self.y = y
    
    def __repr__(self):
        return f"Vector(x={self.x}, y={self.y})"
    
    def __eq__(self, other):
        if isinstance(other, tuple):
            other = Vector(*other)
        if isinstance(other, Vector):
            return self.x == other.x and self.y == other.y
        return NotImplemented
    
    def __abs__(self):
        return sqrt(self.x**2 + self.y**2)
    
    def __lt__(self, other):
        print("LT called")
        if isinstance(other, tuple):
            other = Vector(*other)
        if isinstance(other, Vector):
            return abs(self) < abs(other)
        return NotImplemented

v1 = Vector(0, 0)
v2 = Vector(1, 1)
print(v1 < v2)        # LT called, True
print(v2 > v1)        # LT called, True (via reflection)
print(v1 < (1, 1))    # LT called, True
print((1, 1) > v1)    # LT called, True (via reflection)
```

- **`__abs__`**: Magnitude via $\large{\sqrt{x^2 + y^2}}$.
- **`__lt__`**: Compares magnitudes.
- **Reflection**: `v2 > v1` → `v1 < v2` since `__gt__` isn’t defined.
- **Takeaway**: **One method can enable its opposite** via reflection.

## Supporting `__le__`

Add less-than-or-equal-to:

```python
class Vector:
    # ... (previous code)
    
    def __le__(self, other):
        print("LE called")
        if isinstance(other, tuple):
            other = Vector(*other)
        if isinstance(other, Vector):
            return self == other or self < other
        return NotImplemented

v1 = Vector(0, 0)
v2 = Vector(0, 0)
v3 = Vector(1, 1)
print(v1 <= v2)  # LE called, EQ called, True (short-circuits)
print(v1 <= v3)  # LE called, EQ called, LT called, True
print(v3 >= v1)  # LE called, EQ called, LT called, True (via reflection)
print(v1 != v2)  # False (derived as not (v1 == v2))
```

- **Logic**: `self == other or self < other`, short-circuits if equal.
- **Derived**: `>=` works via reflection; `!=` via `not __eq__`.
- **Key Point**: **Combine existing methods for efficiency**.

## Total Ordering with `functools.total_ordering`

Simplify with a decorator:

```python
from functools import total_ordering

@total_ordering
class Number:
    def __init__(self, x):
        self.x = x
    
    def __eq__(self, other):
        print("EQ called")
        if isinstance(other, Number):
            return self.x == other.x
        return NotImplemented
    
    def __lt__(self, other):
        print("LT called")
        if isinstance(other, Number):
            return self.x < other.x
        return NotImplemented

a = Number(1)
b = Number(2)
c = Number(1)
print(a < b)    # LT called, True
print(a <= b)   # LT called, True
print(a >= c)   # EQ called, True
print(a != b)   # EQ called, True
```

- **Decorator**: Fills in `>`, `>=`, `<=`, `!=` from `__eq__` and `__lt__`.
- ⚠️ **Warning**: Without `__eq__`, defaults to identity, causing odd results (e.g., `Number(1) != Number(1)`).
- **Key Point**: **Total ordering needs `__eq__` for consistency**.
