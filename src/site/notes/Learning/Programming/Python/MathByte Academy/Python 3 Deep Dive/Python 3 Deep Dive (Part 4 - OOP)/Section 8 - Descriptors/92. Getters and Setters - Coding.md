# Descriptor Getters and Setters in Depth

## Unpacking `__get__`

Revisit the `TimeUTC` descriptor:

```python
from datetime import datetime, timezone

class TimeUTC:
    def __get__(self, instance, owner):
        print(f"__get__ called: self={self}, instance={instance}, owner={owner}")
        if instance is None:
            return self
        return datetime.now(timezone.utc).isoformat()

class Logger1:
    current_time = TimeUTC()

class Logger2:
    current_time = TimeUTC()

# Class access
print(Logger1.current_time)  # __get__ called: self=<TimeUTC obj>, instance=None, owner=Logger1
# Instance access
l1 = Logger1()
print(l1.current_time)      # __get__ called: self=<TimeUTC obj>, instance=<Logger1 obj>, owner=Logger1
l2 = Logger2()
print(l2.current_time)      # __get__ called: self=<TimeUTC obj>, instance=<Logger2 obj>, owner=Logger2
```

- **Args**:
  - `self`: The `TimeUTC` instance (class attribute).
  - `instance`: `None` (class call) or the instance (`l1`, `l2`).
  - `owner`: The class (`Logger1`, `Logger2`).
- **Pattern**: Return `self` for class access, value for instance access.
- **Key Point**: **`__get__` reveals caller context**.


> [!note] It is same as the property decorator as well.
> 
> ```python
> from datetime import datetime, timezone
> 
> class Logger:
>     @property
>     def curent_time(self):
>         return datetime.now(timezone.utc).isoformat()
> 
> print(Logger.curent_time) # <property object at 0x7f05d1db2d40>
> 
> l = Logger()
> 
> print(l.curent_time) # 2025-03-31T17:17:30.414793+00:00
> ```
> If called using class `Logger.curent_time`, then it returns the callers instance/object.
> If called using instance `l.curent_time`, then it returns the value of the attribute.

## Shared Descriptor Behavior

Descriptors are class attributes:

```python
l1 = Logger1()
l2 = Logger1()
print(id(l1.current_time) == id(l2.current_time))  # False (different times)
print(id(Logger1.current_time))  # Same TimeUTC instance
```

- **Shared**: `current_time` is one `TimeUTC` object per class.
- **No Issue Here**: No state, just current time—safe for sharing.

## State in Descriptors (Pitfall)

A countdown example shows the problem:

```python
class Countdown:
    def __init__(self, start):
        self.start = start + 1
    def __get__(self, instance, owner):
        if instance is None:
            return self
        self.start -= 1
        return self.start

class Rocket:
    countdown = Countdown(10)

r1 = Rocket()
r2 = Rocket()
print(r1.countdown)  # 10
print(r2.countdown)  # 9 (shared state!)
print(r1.countdown)  # 8
```

- ⚠️ **Warning**: **Shared state in descriptors affects all instances**.
- **Fix Later**: Store per-instance data.

## Exploring `__set__`

Now a setter for integers:

```python
class IntegerValue:
    def __set__(self, instance, value):
        print(f"__set__ called: instance={instance}, value={value}")
    def __get__(self, instance, owner):
        if instance is None:
            print("Get called from class")
            return self
        print(f"Get called: instance={instance}")

class Point2D:
    x = IntegerValue()
    y = IntegerValue()

p = Point2D()
p.x = 1.1  # __set__ called: instance=<Point2D obj>, value=1.1
p.x        # Get called: instance=<Point2D obj>
Point2D.x  # Get called from class
```

- **Signature**: `__set__(self, instance, value)`—no `owner` (instance-only).
- **Purpose**: Sets instance-specific values.

## Naive Storage Mistake

Storing in the descriptor fails:

```python
class IntegerValue:
    def __set__(self, instance, value):
        self._value = value
    def __get__(self, instance, owner):
        if instance is None:
            return self
        return self._value

class Point2D:
    x = IntegerValue()

p1 = Point2D()
p2 = Point2D()
p1.x = 1.1
print(p1.x, p2.x)  # 1.1 1.1 (shared!)
p2.x = 10.0
print(p1.x, p2.x)  # 10.0 10.0 (shared again!)
```

- **Issue**: `_value` is in the descriptor—shared across `p1` and `p2`.
- **Key Point**: **Descriptors need instance-specific storage**.

> [!question]- Why is `_value` becoming a shared even though `self._value` is used ? Is it becoming a class attribute somewhere ?
> 
> The issue with `_value` becoming shared across instances, even though `self._value` is used, is due to the nature of descriptors and how they are implemented.
> 
> When you define a descriptor like `IntegerValue` and use it as a class attribute (e.g., `x = IntegerValue()`), the descriptor instance (`IntegerValue`) itself is shared across all instances of the class (`Point2D`). This means that when you set or get a value using this descriptor, you're interacting with the same descriptor instance, not separate ones for each instance of `Point2D`.
> 
> Here’s why this results in shared state:
> 
> - **Shared Descriptor Instance**: The descriptor object itself (e.g., an instance of `IntegerValue`) is created once per class and shared among all instances of that class. Therefore, any attributes on the descriptor (like `_value`) are also shared.
>   
> - **Class Attribute**: The descriptor is a class attribute, so any data stored within it as an instance variable (like `self._value`) becomes effectively a "class-level" attribute of sorts for the purposes of storage. 
> 
> To store data specific to each instance of the owner class (`Point2D`), you need to change where you store values. Instead of storing them on the descriptor object itself, you should store them on the owner class's instances. A common pattern is to use an attribute on `instance`, typically by using a naming convention or dictionary-based storage keyed by unique identifiers.
> 
> Here's how you can fix it:
> 
> ```python
> class IntegerValue:
>     def __set__(self, instance, value):
>         # Store data in the instance's dictionary using a unique key
>         if not isinstance(value, int):
>             raise ValueError("Must be an integer")
>         instance.__dict__['_x'] = value
> 
>     def __get__(self, instance, owner):
>         if instance is None:
>             return self
>         return instance.__dict__.get('_x', None)
> 
> class Point2D:
>     x = IntegerValue()
> 
> p1 = Point2D()
> p2 = Point2D()
> p1.x = 1
> print(p1.x)  # 1
> print(p2.x)  # None (or whatever default value handling logic you'd want)
> p2.x = 10
> print(p1.x)  # 1
> print(p2.x)  # 10
> ```
> 
> In this solution:
> 
> - We use `instance.__dict__` to store `_x`. This ensures each `Point2D` object manages its own state independently.
> - We add type checking in `__set__` to ensure only integers are assigned.
> 
> This approach leverages Python's dynamic attributes system effectively while maintaining encapsulated control over how values are stored and retrieved.
