# Callable Class Attributes in Action

## Defining a Class with Functions

Let’s create a class with both data and callable attributes:
```python
class Program:
    language = "Python"
    def say_hello():
        print(f"Hello from {Program.language}")
```

- **`language`**: A **data attribute** set to `"Python"`.
- **`say_hello`**: A **callable attribute**—a function defined in the class.

**Key Point**: **Attributes can be functions**, not just data.

---

## Inspecting Attributes

Check the class’s namespace:
```python
print(Program.__dict__)
# Output: mappingproxy({'language': 'Python', 'say_hello': <function Program.say_hello ...>, ...})
```

- **`language`**: The string `"Python"`.
- **`say_hello`**: A function object, stored as an attribute.

**Takeaway**: **Functions are class attributes** too.

---

## Accessing and Calling the Function

### Dot Notation
Access and call it directly:
```python
Program.say_hello()  # Output: "Hello from Python"
```

- **`Program.language`**: Referenced inside the function using the class name.

### `getattr()`
Retrieve and call:
```python
func = getattr(Program, "say_hello")
func()  # Output: "Hello from Python"
```

### `__dict__`
Access via the namespace (not recommended for production):
```python
Program.__dict__["say_hello"]()  # Output: "Hello from Python"
```

- ⚠️ **Warning**: Use `__dict__` for **debugging only**, not regular code.

**Key Point**: **Dot notation** is the simplest way to call class functions.

---

## Why This Works

- **Class Scope**: The function is an attribute of the **class object**, not instances.
- **Dynamic Access**: Python’s flexibility lets us treat functions like any attribute.

**Takeaway**: Define a function in a class, and it becomes a **callable class attribute**.
