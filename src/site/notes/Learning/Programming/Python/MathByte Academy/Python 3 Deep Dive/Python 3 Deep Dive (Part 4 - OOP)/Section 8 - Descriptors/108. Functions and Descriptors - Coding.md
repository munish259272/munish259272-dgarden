# Functions as Non-Data Descriptors

## Functions Have `__get__`

Functions are **non-data descriptors** with a `__get__` method:

```python
def add(a, b):
    return a + b

print(hasattr(add, "__get__"))  # True
print(hasattr(add, "__set__"))  # False
```

- **Evidence**: `__get__` exists; `__set__` doesn’t—classic non-data descriptor.
- **Key Point**: **Functions use descriptors to adapt behavior**.

---

## `__get__` in Action

See how `__get__` behaves:

```python
import sys
main_module = sys.modules["__main__"]
f = add.__get__(None, main_module)  # Mimic module-level call
print(f) # <function add at 0x7f8e95345300>
print(f is add)  # True
```

- **Module-Level**: `add.__get__(None, main_module)` returns the function itself.
- **Key Point**: **No instance = raw function**.

> [!note] Add is defined in our module and we can think of our module as class that contains add so if we call it from the module the instance is going to be `None` as per descriptor protocol and the owner class if going to be the main module(as per descriptor protocol). `__main__` is the main module that loads up when our python application starts.

## Class Methods vs. Instance Methods

In a class, `__get__` switches behavior:

```python
class Person:
    def __init__(self, name):
        self.name = name
    def say_hello(self):
        return f"{self.name} says hello"

p = Person("Alex")
print(Person.say_hello)  # <function Person.say_hello ...>
print(p.say_hello)       # <bound method Person.say_hello ...>
print(p.say_hello.__get__(p, Person)) # <bound method Person.say_hello ...>
```

- **Class Access**: `Person.say_hello` → function (instance is `None`).
- **Instance Access**: `p.say_hello` → bound method (instance is `p`).
- **Manual Call**: `Person.say_hello.__get__(p, Person)` → bound method.
- **Key Point**: **`__get__` binds `self` dynamically**.

---

## Bound Method Details

Bound methods are **fresh objects each time**:

```python
f1 = p.say_hello
f2 = p.say_hello
print(f1 is f2)  # False
print(type(f1)) # <class 'method'>
print(type(f1.__get__(f1, Person))) # <class 'method'>
print(f1.__func__)  # <function Person.say_hello ...>
print(f1())  # Alex says hello
```

- **Why**: `__get__` creates a new method object per access.
- **Link**: `__func__` points to the original function.
- **Key Point**: **Methods track their function via `__func__`**.

## Mimicking with a Custom Descriptor

Build a descriptor to replicate this:

```python
from types import MethodType

class MyFunc:
    def __init__(self, func):
        self._func = func
    def __get__(self, instance, owner):
        if instance is None:
            return self._func
        print("Get called from instance")
        return MethodType(self._func, instance)

def hello(self):
    return f"{self.name} says hello"

class Person:
    def __init__(self, name):
        self.name = name
    say_hello = MyFunc(hello)

p = Person("Alex")
print(Person.say_hello)  # <function hello ...>
print(p.say_hello)      # Get called from instance, <bound method ...>
print(p.say_hello())    # Alex says hello
```

- **Logic**: Returns function if no instance; binds via `MethodType` if instance exists.
- **Key Point**: **Custom descriptor mimics function-to-method magic**.
