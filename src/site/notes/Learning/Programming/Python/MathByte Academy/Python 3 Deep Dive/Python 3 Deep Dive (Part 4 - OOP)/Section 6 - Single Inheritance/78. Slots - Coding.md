# Using Slots in a Location Class

## Defining a Class with Slots

Let’s create a `Location` class with `__slots__`:

```python
class Location:
    __slots__ = ('name', '_longitude', '_latitude')  # Tuple of attribute names
    
    def __init__(self, name, *, longitude, latitude):  # Keyword-only args
        self.name = name
        self._longitude = longitude
        self._latitude = latitude
    
    @property
    def longitude(self):
        return self._longitude
    
    @property
    def latitude(self):
        return self._latitude

l = Location("Mumbai", longitude=72.8777, latitude=19.0760)
print(l.name, l.longitude, l.latitude)  # Mumbai 72.8777 19.0760
```

- **`__slots__`**: Lists allowed instance attributes—here, `name`, `_longitude`, `_latitude`.
- **Keyword Args**: Forces `longitude=` and `latitude=` to avoid mix-ups.
- **Properties**: Read-only access to `_longitude` and `_latitude`.
- **Key Point**: **Slots define a fixed set of attributes**.

> [!question]- I know about the syntax `(*args, **kargs)` but what is a solo `*` in `def __init__(self, name, *, longitude, latitude):` doing and how does it enforces Keyword-only args ?
> 
> In Python, the use of a solo `*` in a function signature is a way to enforce that all subsequent parameters must be specified as keyword arguments. This means you cannot pass these arguments positionally, and you must explicitly name them when calling the function.
> 
> Here's how it works in your `Location` class example:
> 
> ```python
> def __init__(self, name, *, longitude, latitude):
> ```
> 
> - `name` is a positional argument. You can pass it without specifying the keyword.
> - The `*` indicates that all parameters following it must be specified as keyword arguments.
> - Thus, `longitude` and `latitude` must be passed with their names, like so: `longitude=72.8777`, `latitude=19.0760`.
> 
> This design helps prevent errors that can arise from mixing up positional arguments, especially when multiple parameters are of the same type or similar meaning. It makes the function call more readable and explicit about which value corresponds to which parameter.
> 
> For example:
> 
> ```python
> l = Location("Mumbai", longitude=72.8777, latitude=19.0760)
> ```
> 
> Here, "Mumbai" is passed positionally for `name`, but both `longitude` and `latitude` are specified as keyword arguments due to the presence of the solo `*`. If you tried to pass them positionally (e.g., `Location("Mumbai", 72.8777, 19.0760)`), Python would raise an error because those arguments must be passed with their respective keywords.

## Class vs. Instance Attributes

Slots affect instances, not the class:

```python
print(Location.__dict__)  # Mappingproxy with class attributes
Location.map_service = "Google Maps"  # Add class attribute
print(Location.map_service)  # Google Maps
```

- **Class**: `__dict__` exists (a `mappingproxy`), modifiable via dot notation.
- **Instance**: `l.__dict__` raises `AttributeError`—no instance dictionary.
- **Takeaway**: **Slots lock instances, not classes**.

## Instance Limitations

Slots restrict instance flexibility:

```python
# Can’t add new attributes
l.map_link = "https://..."  # AttributeError: 'Location' object has no attribute 'map_link'

# Can delete slotted attributes
del l.name
print(l.name)  # AttributeError: 'name' not found
l.name = "Delhi"  # Reset works
print(l.name)  # Delhi

# Can’t delete properties (no deleter)
del l.longitude  # AttributeError: can't delete attribute
# But can delete backing attribute
del l._longitude  # Works
```

- **No Monkey Patching**: Only `name`, `_longitude`, `_latitude` allowed.
- **Delete**: Slotted attributes (`l._longitude`) can be deleted; properties need a `@longitude.deleter`.
- **Key Point**: **Slots enforce attribute rigidity**.
