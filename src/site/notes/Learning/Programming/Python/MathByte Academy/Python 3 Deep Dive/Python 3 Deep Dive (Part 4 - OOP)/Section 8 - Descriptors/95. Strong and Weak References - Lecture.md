# Strong and Weak References in Descriptors

## Strong References Explained

Default references are strong:

```python
class Person:
    pass

p1 = Person()  # Strong reference
p2 = p1        # Another strong reference
del p1         # One reference gone, object still alive via p2
del p2         # No strong refs, object garbage collected
```

- **Behavior**: Strong refs keep objects alive until all are gone.
- **Problem**: Descriptors with strong refs (e.g., `self.data[instance]`) cause memory leaks.
- **Key Point**: **Strong refs increment ref count**.

---

## Weak References Introduced

Weak refs don’t affect garbage collection:

```python
import weakref

p1 = Person()         # Strong ref (count: 1)
p2 = weakref.ref(p1)  # Weak ref (count still 1)
print(p2())           # <Person object> (access via callable)
del p1                # No strong refs, object collected
print(p2())           # None (dead weak ref)
```

- **How**: Use `weakref.ref(object)`—callable returns object or `None`.
- **Takeaway**: **Weak refs let objects die when strong refs vanish**.

> [!note] weak refs also create reference but without increasing the ref count for the memory manager. When all the strong references are removed the weak refs are dead.

---

## Pitfall: Creating Strong Refs

Be cautious with weak ref results:

```python
p1 = Person()
p2 = weakref.ref(p1)
p3 = p2()    # Strong ref from p2()!
del p1       # p3 keeps object alive
print(p3)    # <Person object>
del p3       # Now collected
```

- ⚠️ **Warning**: Assigning `p2()` creates a strong ref.
- **Key Point**: **Handle weak refs carefully**.

> [!note] `p2` is another object that contains a weak reference to the object and it is a callable that returns the object if it exists or else it returns `None` when called.

---

## Fixing Descriptors with WeakKeyDictionary

Use `WeakKeyDictionary` for cleanup:

```python
from weakref import WeakKeyDictionary

class IntegerValue:
    def __init__(self):
        self.values = WeakKeyDictionary()
    def __set__(self, instance, value):
        self.values[instance] = int(value)
    def __get__(self, instance, owner):
        if instance is None:
            return self
        return self.values.get(instance)

class Point2D:
    x = IntegerValue()

p1 = Point2D()
p1.x = 10
print(p1.x)          # 10
print(p1 in Point2D.x.values)  # True
del p1               # Weak ref dies, auto-removed
print(len(Point2D.x.values))  # 0
```

- **Why**: Keys are weak refs—entries vanish when instances are deleted.
- **Bonus**: No manual cleanup needed.
- **Key Point**: **WeakKeyDictionary solves memory leaks**.

---

## Iteration Caveat

Watch out when iterating:

```python
# Hypothetical risky code
d = WeakKeyDictionary()
p = Person()
d[p] = 1
for k in d:  # Could raise RuntimeError if p is deleted elsewhere
    pass
```

- ⚠️ **Warning**: Dictionary size may change during iteration.
- **Solution**: Avoid mid-iteration deletions in our use case.
