# Single Inheritance Hierarchy in Python

## Creating a Shape Hierarchy

Let’s build a class structure for geometric shapes:

```python
class Shape:
    pass

class Ellipse(Shape):
    pass

class Circle(Ellipse):
    pass

class Polygon(Shape):
    pass

class Rectangle(Polygon):
    pass

class Square(Rectangle):
    pass

class Triangle(Polygon):
    pass
```

- **Hierarchy**:
  - `Shape` (root)
  - `Ellipse` → `Circle`
  - `Polygon` → `Rectangle` → `Square`, and `Polygon` → `Triangle`
- **Relationships**: `Ellipse` and `Polygon` are siblings; `Rectangle` and `Triangle` are siblings.
- **Key Point**: **Single inheritance forms a clear chain**.

---

## Checking Class Relationships with `issubclass`

Use `issubclass()` to test inheritance:

```python
print(issubclass(Ellipse, Shape))    # True (direct)
print(issubclass(Circle, Shape))     # True (indirect via Ellipse)
print(issubclass(Polygon, Ellipse))  # False (siblings)
print(issubclass(Square, Shape))     # True (indirect via Rectangle, Polygon)
print(issubclass(Square, Square))    # True (self is a subclass)
```

- **Logic**: True if the first class inherits from the second, directly or indirectly.
- **Note**: A class is a subclass of itself—useful for functionality checks.
- **Takeaway**: **`issubclass` traces the hierarchy**.

---

## Instances with `isinstance`

Test instances against the hierarchy:

```python
s = Shape()
c = Circle()
sq = Square()

print(isinstance(s, Shape))     # True (direct)
print(isinstance(sq, Square))   # True (direct)
print(isinstance(sq, Rectangle))# True (parent)
print(isinstance(sq, Shape))    # True (ancestor)
print(isinstance(sq, Ellipse))  # False (unrelated branch)
```

- **Contrast**: `issubclass(s, Shape)` fails—use classes, not instances.
- **Key Point**: **`isinstance` checks instance compatibility up the chain**.

---

## `type` vs `isinstance`

`type()` pinpoints the exact class:

```python
print(type(s) == Shape)    # True
print(type(sq) == Square)  # True
print(type(sq) == Shape)   # False (ignores inheritance)
```

- **`type`**: Exact creation class, no hierarchy lookup.
- **`isinstance`**: Broader, checks ancestry.
- **Use Case**: Prefer `isinstance` for behavior (e.g., “Can it act as a `Shape`?”).

---

## Mixing `type` and `isinstance`

Relate two instances’ classes:

```python
p = Polygon()
sq = Square()

# Check if sq’s class is in p’s hierarchy
print(isinstance(sq, type(p)))      # True (Square is a Polygon)
print(issubclass(type(sq), type(p))) # True (Square subclasses Polygon)
```

- **Why**: `type(p)` gets `Polygon`, then test `sq` against it.
- **Key Point**: **Combine tools to probe relationships**.

---

## Implicit `object` Inheritance

All classes inherit from `object`:

```python
class Person:
    pass

print(issubclass(Person, object))  # True
print(issubclass(Square, object))  # True
```

- **Default**: Even `Shape` inherits from `object`.
- **Chain**: `Square` → `Rectangle` → `Polygon` → `Shape` → `object`.
- **Takeaway**: **`object` is the root of all classes**.
