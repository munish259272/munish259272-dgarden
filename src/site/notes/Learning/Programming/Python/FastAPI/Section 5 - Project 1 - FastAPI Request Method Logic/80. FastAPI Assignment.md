Why it is not giving the correct output , i have mentioned `books` as the path parameter ? #flashcard 
> Ah, that clarifies things! You wanted it to be a **path parameter**, but there is a specific way FastAPI recognizes them.
> 
> Right now, FastAPI is treating `books` as a **query parameter** instead of a path parameter. Here is why that's happening and how to fix it:
> 
> ### The Problem: Missing `{}` in the Path
> 
> FastAPI decides whether a function argument is a path parameter or a query parameter based on the URL string in your decorator (`@app.get(...)`):
> 
> - **Query Parameter (Current Behavior):** If a variable is in your function arguments (`books: str`) but **not** in the URL path, FastAPI automatically assumes you want it to be a query parameter (e.g., `/books?books=something`).
>     
> - **Path Parameter (What you want):** To tell FastAPI that a variable belongs in the URL path, you _must_ enclose it in curly braces `{}` inside the decorator string.
>     
> 
> ### The Fix
> 
> If you want to pass a specific book parameter (like a dynamic title or category) directly inside the URL path, change your code to look like this:
> 
> Python
> 
> ```
> from fastapi import FastAPI
> 
> app = FastAPI()
> 
> books = [
>     {"title": "Title 1", "author": "Author 1", "category": "Science"},
>     {"title": "Title 2", "author": "Author 2", "category": "History"},
>     {"title": "Title 3", "author": "Author 3", "category": "Math"},
>     {"title": "Title 4", "author": "Author 4", "category": "Science"},
>     {"title": "Title 5", "author": "Author 5", "category": "History"}
> ]
> 
> # 1. Added '{book_title}' to the URL path
> # 2. Renamed the parameter to avoid clashing with your global 'books' list
> @app.get("/books/{book_title}")
> def get_book_by_title(book_title: str):
>     # Search for the book in your global list
>     for book in books:
>         if book["title"].lower() == book_title.lower():
>             return book
>             
>     return {"error": "Book not found"}
> ```
> 
> ### What Changed?
> 
> 1. **`@app.get("/books/{book_title}")`**: The `{book_title}` syntax explicitly tells FastAPI: _"Expect a dynamic value at the end of this URL path."_
>     
> 2. **Variable Renaming**: It is highly recommended to change the parameter name from `books` to something else (like `book_title`). Keeping it named `books` overrides your global list inside that function, making it impossible to search through your actual data!
>     
> 
> Now, if you navigate to `/books/Title 1`, it will correctly match the path parameter and return:
> 
> JSON
> 
> ```
> {
>   "title": "Title 1",
>   "author": "Author 1",
>   "category": "Science"
> }
> ```
