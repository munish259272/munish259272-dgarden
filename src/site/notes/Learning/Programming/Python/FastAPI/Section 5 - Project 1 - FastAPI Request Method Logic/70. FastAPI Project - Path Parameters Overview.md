# Path Parameters in FastAPI

## 🎯 What are Path Parameters?

**Path Parameters** are values embedded directly inside the URL path that allow an API endpoint to handle **dynamic requests**.

Think of them like folders in a file system:

```text
/users/mukesh/documents/python/fastapi/section1
```

Each part of the path helps identify a specific resource.

Similarly, in an API:

```text
http://localhost:8000/books
```

The path is:

```text
/books
```

and FastAPI matches it with:

```python
@app.get("/books")
async def read_all_books():
    return books_list
```

Since `/books` is hardcoded, this is called a **static path**.

---

## 📌 Static Path Example

```python
@app.get("/books")
async def read_all_books():
    return books_list
```

Request:

```text
http://localhost:8000/books
```

Response:

```json
[
    {...},
    {...},
    {...}
]
```

Here, the URL must always be exactly:

```text
/books
```

You cannot change it to:

```text
/books/book1
/books/book2
```

because FastAPI is only listening for:

```text
/books
```

---

# 🎯 Why Do We Need Path Parameters?

Imagine you have 10,000 books.

❌ Bad approach:

```python
@app.get("/books/book1")
@app.get("/books/book2")
@app.get("/books/book3")
...
```

Creating an endpoint for every book would be impossible.

Instead, FastAPI allows us to use **dynamic values**.

---

## ✅ Dynamic Path Parameter

```python
@app.get("/books/{dynamic_param}")
async def read_book(dynamic_param):
    return dynamic_param
```

Request:

```text
http://localhost:8000/books/book1
```

FastAPI extracts:

```text
book1
```

and passes it into:

```python
dynamic_param
```

Equivalent to:

```python
read_book("book1")
```

Response:

```json
"book1"
```

---

## 🔍 What Happens Internally?

Endpoint:

```python
@app.get("/books/{dynamic_param}")
```

Request:

```text
/books/book1
```

FastAPI sees:

```text
/books/{something}
```

and matches:

```text
dynamic_param = "book1"
```

Then it calls:

```python
async def read_book(dynamic_param):
```

with:

```python
dynamic_param = "book1"
```

and returns:

```python
"book1"
```

---

## ⚠️ Parameter Names Must Match

The name inside `{}` must match the function parameter.

✅ Correct:

```python
@app.get("/books/{book_id}")
async def read_book(book_id):
    return book_id
```

---

❌ Incorrect:

```python
@app.get("/books/{book_id}")
async def read_book(id):
    return id
```

FastAPI will raise an error because:

```text
book_id ≠ id
```

---

# 🎯 Order Matters in Path Parameters

FastAPI checks routes **from top to bottom**.

Consider:

```python
@app.get("/books/{book_name}")
async def read_book(book_name):
    return book_name
```

and below it:

```python
@app.get("/books/mybook")
async def read_my_book():
    return "My Favorite Book"
```

Request:

```text
/books/mybook
```

What happens?

FastAPI checks the first route:

```python
/books/{book_name}
```

It matches because:

```text
book_name = "mybook"
```

So FastAPI immediately executes:

```python
read_book("mybook")
```

The second endpoint is never reached.

---

## ✅ Correct Order

Place specific routes first.

```python
@app.get("/books/mybook")
async def read_my_book():
    return "My Favorite Book"

@app.get("/books/{book_name}")
async def read_book(book_name):
    return book_name
```

Now:

Request:

```text
/books/mybook
```

matches:

```python
/books/mybook
```

---

Request:

```text
/books/python
```

does not match the static route, so FastAPI falls back to:

```python
/books/{book_name}
```

---

## 📌 Rule to Remember

> [!tip]  
> **Always place more specific (static) routes before more general (dynamic) routes.**
> 
> FastAPI evaluates routes **top-to-bottom**.

# 🎯 Real Example: Finding a Book by Title

Suppose we have:

```python
books = [
    {"title": "Title 1", "author": "Author 1"},
    {"title": "Title 2", "author": "Author 2"},
    {"title": "Title 4", "author": "Author 4"}
]
```

We want:

```text
/books/Title 4
```

to return:

```json
{
    "title": "Title 4",
    "author": "Author 4"
}
```

---

## URL Encoding for Spaces

URLs cannot contain spaces.

Instead:

```text
Title 4
```

becomes:

```text
Title%204
```

because:

```text
%20 = space
```

Request:

```text
http://localhost:8000/books/Title%204
```

FastAPI automatically decodes it back into:

```text
Title 4
```

---

## Implementation

```python
@app.get("/books/{book_title}")
async def read_book(book_title: str):
    for book in books:
        if book["title"].casefold() == book_title.casefold():
            return book
```

---

## 🔍 Why Use `casefold()`?

User may type:

```text
title 4
TITLE 4
Title 4
tItLe 4
```

Without normalization:

```python
"Title 4" == "title 4"
```

returns:

```python
False
```

---

Using:

```python
casefold()
```

```python
"Title 4".casefold()
```

becomes:

```python
"title 4"
```

and

```python
"TITLE 4".casefold()
```

also becomes:

```python
"title 4"
```

Now comparison succeeds.

---

> [!info]  
> `casefold()` is a more aggressive and Unicode-aware version of `lower()`.
> 
> It is generally preferred when performing case-insensitive string comparisons.

# 📌 Key Takeaways

✅ **Path parameters allow dynamic values inside URLs**

```python
/books/{book_id}
```

---

✅ **Values are extracted from the URL and passed into function parameters**

```python
@app.get("/books/{book_id}")
async def read_book(book_id):
```

---

✅ **Parameter names inside `{}` must match function arguments**

```python
/books/{book_id}
```

```python
def read_book(book_id):
```

---

✅ **Static routes must come before dynamic routes**

```python
/books/mybook
```

before

```python
/books/{book_name}
```

---

✅ **Spaces in URLs are encoded as `%20`**

```text
Title 4
```

becomes

```text
Title%204
```

---

✅ **Use `casefold()` for reliable case-insensitive string matching**

```python
book["title"].casefold() == book_title.casefold()
```

This allows users to search for resources dynamically using URLs such as:

```text
/books/Title%204
/books/Python%20Basics
/books/FastAPI%20Guide
```

without creating a separate endpoint for every book.
