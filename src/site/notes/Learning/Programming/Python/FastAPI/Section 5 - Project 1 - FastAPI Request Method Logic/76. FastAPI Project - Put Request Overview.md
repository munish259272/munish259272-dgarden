# PUT Request Method in FastAPI: Updating Existing Data

So far, we have learned two important HTTP request methods:

|Method|Purpose|
|---|---|
|**GET**|Retrieve data|
|**POST**|Create new data|

Now we'll learn another commonly used HTTP method:

# 🎯 PUT Request

A **PUT request** is used to **update existing data**.

Think of it this way:

- **POST** → Create a new book.
- **PUT** → Modify an existing book.

---

## 📌 Real-World Analogy

Imagine a library.

### POST Request

You bring a brand-new book to the librarian:

```text
"Please add this new book to the library."
```

✅ New record created.

---

### PUT Request

You discover that an existing book has incorrect information:

```text
"Please update the author/category of this existing book."
```

✅ Existing record modified.

---

# POST vs PUT

|HTTP Method|Purpose|
|---|---|
|POST|Create new data|
|PUT|Update existing data|

---

## 📦 Can PUT Have a Request Body?

Yes.

Just like a POST request, a PUT request can contain a **Request Body**.

Example:

```json
{
    "title": "Title One",
    "author": "Author Two",
    "category": "Math"
}
```

This body contains the updated information for the book.

---

> [!tip]  
> 📌 **Important**
> 
> GET requests cannot have a body.
> 
> Both **POST** and **PUT** requests can have a body.

---

# How Will We Update a Book?

Suppose our current books list contains:

```python
books = [
    {
        "title": "Title One",
        "author": "Author One",
        "category": "Science"
    }
]
```

Now suppose we want to change:

```text
Category: Science → Math
```

or

```text
Author: Author One → Author Two
```

Instead of creating a new book, we want to **replace the existing book information**.

---

# Endpoint URL

Our endpoint might look like:

```text
http://localhost:8000/books/update_book
```

Notice:

✅ No query parameters

✅ No path parameters

✅ Updated book data will be sent in the request body

---

# Creating a PUT Endpoint

```python
@app.put("/books/update_book")
async def update_book(updated_book=Body()):
    ...
```

---

## Understanding the Function

### Route Definition

```python
@app.put("/books/update_book")
```

This tells FastAPI:

> "When a PUT request arrives at `/books/update_book`, execute this function."

---

### Reading the Request Body

```python
updated_book = Body()
```

This tells FastAPI:

> "Take whatever data is present in the Request Body and store it inside `updated_book`."

---

# Finding the Existing Book

We need to search through our books list.

```python
for i in range(len(books)):
```

This allows us to inspect every book.

---

### Compare Titles

```python
if books[i]["title"].casefold() == updated_book["title"].casefold():
```

We compare titles because the title identifies which book should be updated.

Using:

```python
.casefold()
```

makes the comparison case-insensitive.

Example:

```python
"title one"
```

and

```python
"TITLE ONE"
```

are treated as equal.

---

# Replacing the Old Book

Once we find a matching title:

```python
books[i] = updated_book
```

The old book is replaced by the updated version.

---

## Complete Example

```python
from fastapi import FastAPI, Body

app = FastAPI()

books = [
    {
        "title": "Title One",
        "author": "Author One",
        "category": "Science"
    }
]

@app.put("/books/update_book")
async def update_book(updated_book=Body()):
    for i in range(len(books)):
        if books[i]["title"].casefold() == updated_book["title"].casefold():
            books[i] = updated_book
```

# Comparing POST and PUT Internally

### POST

```python
books.append(new_book)
```

Result:

```text
Existing Books
      +
New Book
      ↓
Book Added
```

Creates a new record.

---

### PUT

```python
books[i] = updated_book
```

Result:

```text
Existing Book
      ↓
Replaced By
      ↓
Updated Book
```

Updates an existing record.

---

## ⚠️ Potential Problem

Suppose the request body is missing fields:

```json
{
    "title": "Title One"
}
```

but your code expects:

```json
{
    "title": "Title One",
    "author": "Author Two",
    "category": "Math"
}
```

The application may fail because required data is missing.

Example:

```python
updated_book["author"]
```

would cause an error if `"author"` does not exist.

---

> [!warning]  
> In production applications, we use **Pydantic Models** to validate incoming request bodies and prevent these kinds of errors.

---

> [!tip]  
> 🎯 **Key Takeaways**
> 
> - **PUT** is used to update existing data.
>     
> - PUT requests can contain a **Request Body**.
>     
> - The body usually contains the updated version of an existing object.
>     
> - FastAPI reads the body using `Body()`.
>     
> - A common update pattern is:
>     
> 
> ```python
> books[i] = updated_book
> ```
> 
> - **POST creates**, while **PUT updates**.
>     
> - Both POST and PUT can carry JSON data inside the request body.
>     
> - GET requests cannot have a request body.
