## POST Request Method in FastAPI: Creating Data

So far, we have learned how to send data to an API using:

- **Path Parameters** → Data embedded inside the URL path.
- **Query Parameters** → Data appended to the URL after `?`.

Now we'll learn a new HTTP request method:

# 🎯 POST Request

A **POST request** is primarily used to **create new data** on the server.

Unlike GET requests, which are mainly used for retrieving data, POST requests allow us to send a much larger amount of information through a special section of the HTTP request called the **Request Body**.

---

## 📌 Why Do We Need a Request Body?

Suppose a book in our application requires:

```python
{
    "title": "Title One",
    "author": "Author One",
    "category": "Science"
}
```

A book contains multiple pieces of information:

- Title
- Author
- Category

Passing all of this through URL parameters would become cumbersome.

Instead, we can send the entire book as a dictionary (JSON object) inside the **Request Body**.

---

## 📦 Understanding the HTTP Request Structure

An HTTP request can contain several parts:

```text
HTTP Request
│
├── URL / Path
├── Headers
├── Query Parameters
└── Body (Optional)
```

### GET Request

```http
GET /books?author=Author%20One
```

Contains:

- URL
- Query Parameters

❌ No Request Body

---

### POST Request

```http
POST /books/create_book

{
    "title": "Title Seven",
    "author": "Author Two",
    "category": "Math"
}
```

Contains:

- URL
- Headers
- Request Body

✅ Can contain a Request Body

---

## ⚠️ Important: GET Requests Cannot Have a Body

FastAPI allows us to import and use `Body`, but that does **not** mean GET requests can accept a body.

This is an HTTP protocol rule:

|Request Type|Supports Body?|
|---|---|
|GET|❌ No|
|POST|✅ Yes|
|PUT|✅ Yes|
|PATCH|✅ Yes|

> [!warning]  
> Even if FastAPI compiles successfully, a GET request with a body is invalid according to the HTTP specification and will result in an error.

---

## Importing `Body`

To receive data from a request body, import `Body`:

```python
from fastapi import FastAPI, Body
```

---

## Creating a POST Endpoint

```python
@app.post("/books/create_book")
async def create_book(new_book=Body()):
    books.append(new_book)
```

### What's Happening Here?

#### Route

```python
@app.post("/books/create_book")
```

This tells FastAPI:

> "When a POST request arrives at `/books/create_book`, execute this function."

---

#### Function Parameter

```python
new_book = Body()
```

This tells FastAPI:

> "Read the Request Body and store its contents inside the variable `new_book`."

---

#### Appending the Book

```python
books.append(new_book)
```

The newly received book is added to our existing book list.

---

## 🔄 Request Flow

```text
Client
   │
   │ POST /books/create_book
   │
   ├── Body
   │
   │ {
   │   "title": "Title Seven",
   │   "author": "Author Two",
   │   "category": "Math"
   │ }
   │
   ▼
FastAPI
   │
   ▼
new_book
   │
   ▼
books.append(new_book)
   │
   ▼
Book Added Successfully
```

## ⚠️ JSON Strings Must Use Double Quotes

Correct:

```json
{
    "title": "Title Seven"
}
```

Incorrect:

```json
{
    'title': 'Title Seven'
}
```

> [!warning]  
> JSON requires **double quotes (`"`)** around string values and property names.
> 
> Python allows both single and double quotes, but JSON does not.

## Executing the Request

After clicking **Execute**, Swagger generates a request similar to:

```bash
curl -X POST \
"http://127.0.0.1:8000/books/create_book" \
-H "Content-Type: application/json" \
-d '{
  "title":"Title Seven",
  "author":"Author Two",
  "category":"Math"
}'
```

The request contains:

- URL
- Headers
- Body

The body is automatically sent to FastAPI.

---

## Understanding the Response

You may receive:

```json

```

because our function does not return anything:

```python

```

However, the important part is:

```text
Status Code: 200
```

which indicates success.

## What Happens If We Try Using a Body in a GET Request?

Suppose we write:

```python
@app.get("/books/book_title")
async def read_book(book_title: str,
                    new_book=Body()):
    ...
```

The application may start without compilation errors.

Swagger may even display a Request Body section.

However, when the request is executed, FastAPI will reject it because:

```text
GET requests cannot have a request body.
```

> [!danger]  
> The fact that code compiles does **not** mean it is valid according to the HTTP protocol.
> 
> GET requests are intended for retrieving data and do not support request bodies.

---

## 📌 Why Does FastAPI Use `Body()`?

FastAPI uses:

```python
new_book = Body()
```

to tell the framework:

> "Take the data from the Request Body and assign it to the variable `new_book`."

Conceptually:

```text
HTTP Request
│
├── URL
├── Headers
└── Body
     │
     ▼
{
    "title": "Title Seven",
    "author": "Author Two",
    "category": "Math"
}
     │
     ▼
new_book
```

---

## ⚠️ Data Structure Must Match Expectations

Suppose your code expects:

```python
{
    "title": "...",
    "author": "...",
    "category": "..."
}
```

but the client sends:

```python
{
    "name": "Random"
}
```

the application may fail because the expected structure is missing.

Error handling and validation will be discussed later using **Pydantic Models**, which provide a safer and more robust solution.

>
