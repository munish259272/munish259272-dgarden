# 🎯 FastAPI Path Parameters

In the previous tutorial, we created a `/books` API endpoint that returned all books from our `books` list.

Now we'll learn about **Path Parameters**, one of the most important concepts in FastAPI.

---

## 📌 What is a Path Parameter?

A **Path Parameter** is a dynamic part of a URL that allows us to pass values directly through the URL path.

For example:

```text
/books/science
/books/history
/books/python
```

Here:

- `/books` is the fixed part of the URL.
- `science`, `history`, and `python` are dynamic values.

FastAPI can automatically capture these values and pass them into a Python function.

---

## 🧠 Simple Example

```python
@app.get("/books/{dynamic_param}")
async def read_all_books(dynamic_param):
    return {"dynamic_param": dynamic_param}
```

### How it works

When a request comes in:

```text
/books/science
```

FastAPI automatically does something similar to:

```python
dynamic_param = "science"
```

and calls:

```python
read_all_books("science")
```

which returns:

```json
{
    "dynamic_param": "science"
}
```

---

## 🔄 Flow of Execution

```text
User Request
      │
      ▼
/books/science
      │
      ▼
dynamic_param = "science"
      │
      ▼
read_all_books(dynamic_param)
      │
      ▼
{"dynamic_param": "science"}
```

---

## 📌 Why Does This Work?

Notice that the name inside `{}` matches the function parameter name:

```python
@app.get("/books/{dynamic_param}")
async def read_all_books(dynamic_param):
```

FastAPI automatically maps:

```text
{dynamic_param}
```

to

```python
dynamic_param
```

behind the scenes.

✅ No extra code is required.

# 📌 Adding Type Hints

We can specify the expected data type:

```python
@app.get("/books/{dynamic_param}")
async def read_all_books(dynamic_param: str):
    return {"dynamic_param": dynamic_param}
```

Here:

```python
dynamic_param: str
```

means:

> "I expect this value to be a string."

---

## 🧠 Interesting Observation

Suppose we enter:

```text
/books/1
```

FastAPI still returns:

```json
{
    "dynamic_param": "1"
}
```

because we explicitly declared:

```python
dynamic_param: str
```

So FastAPI converts the value to a string.

---

## ⚠️ Important: Order Matters

Consider these two endpoints:

### Dynamic Route

```python
@app.get("/books/{book_title}")
async def read_book(book_title: str):
    return {"book_title": book_title}
```

### Static Route

```python
@app.get("/books/mybook")
async def my_book():
    return {"book_title": "My Favorite Book"}
```

---

## ❌ Problem

If the dynamic route appears first:

```python
@app.get("/books/{book_title}")
```

and then:

```python
@app.get("/books/mybook")
```

A request to:

```text
/books/mybook
```

will match:

```python
/books/{book_title}
```

instead of:

```python
/books/mybook
```

FastAPI thinks:

```python
book_title = "mybook"
```

---

## ✅ Correct Order

Always place **more specific routes first**:

```python
@app.get("/books/mybook")
async def my_book():
    return {"book_title": "My Favorite Book"}

@app.get("/books/{book_title}")
async def read_book(book_title: str):
    return {"book_title": book_title}
```

Now:

```text
/books/mybook
```

matches the first endpoint.

Everything else:

```text
/books/science
/books/history
/books/python
```

matches the dynamic endpoint.

# 🎯 Real Use Case: Finding a Specific Book

Returning the parameter itself is not very useful.

Instead, we can use the path parameter to search for a book.

```python
@app.get("/books/{book_title}")
async def read_book(book_title: str):
    for book in books:
        if book["title"].casefold() == book_title.casefold():
            return book
```

---

## 📌 What is `casefold()`?

`casefold()` is a stronger version of `lower()`.

Example:

```python
"SCIENCE".casefold()
```

becomes:

```python
"science"
```

This allows comparisons to ignore letter casing.

---

## 🔄 Execution Flow

Suppose we request:

```text
/books/Title One
```

FastAPI performs:

```python
book_title = "Title One"
```

Then:

```python
for book in books:
```

loops through all books.

For each book:

```python
if book["title"].casefold() == book_title.casefold():
```

If a match is found:

```python
return book
```

books = [

# 📌 URL Encoding

When a URL contains spaces, browsers automatically encode them.

Example:

```text
/books/Title One
```

becomes:

```text
/books/Title%20One
```

### Common Encodings

|Character|Encoded Form|
|---|---|
|Space|`%20`|
|`#`|`%23`|
|`&`|`%26`|
|`?`|`%3F`|

So:

```text
/books/Title One
```

and

```text
/books/Title%20One
```

represent the same URL.

---

## Swagger UI vs Browser

Swagger UI displays responses in a cleaner format:

```json
{
    "title": "Title One",
    "author": "Author One",
    "category": "Science"
}
```

Internally, the API still returns normal JSON.

The browser simply displays the raw JSON response.

> 
> Think of a path parameter like a blank space in an address:
> 
> ```text
> /books/{book_title}
> ```
> 
> is similar to:
> 
> ```text
> "Go to shelf ____"
> ```
>

> 
> ```text
> /books/science
> ```
> 
> then FastAPI understands:
> 
> ```text
> Go to shelf "science"
> ```
> 
> The value changes dynamically, but the route pattern remains the same.
