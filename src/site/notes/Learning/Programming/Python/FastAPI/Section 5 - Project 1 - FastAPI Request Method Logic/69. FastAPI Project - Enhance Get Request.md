# Enhance Your FastAPI Endpoint – Return a List of Books

Let’s **upgrade** your API to return a **real list of books** and explore **Swagger UI** for interactive testing.

---

## Step 1: Define the Books List (In-Memory Data)

Add this **above** your endpoint in `books.py`:

```python
BOOKS = [
    {"title": "Title 1", "author": "Author 1", "category": "Science"},
    {"title": "Title 2", "author": "Author 2", "category": "Science"},
    {"title": "Title 3", "author": "Author 3", "category": "History"},
    {"title": "Title 4", "author": "Author 4", "category": "Math"},
    {"title": "Title 5", "author": "Author 5", "category": "Math"},
    {"title": "Title 6", "author": "Author 2", "category": "Math"}
]
```

> **6 books**  
> **Author 2 wrote 2 books** → Realistic data variation

---

## Step 2: Update the GET Endpoint

Replace your old endpoint:

```python
@app.get("/books")
async def read_all_books():
    return BOOKS
```

### Changes Made:
| Before | After |
|-------|-------|
| `@app.get("/api-endpoint")` | `@app.get("/books")` |
| `first_api()` | `read_all_books()` |
| `{"message": "Hello, Eric"}` | `BOOKS` |

> **Descriptive paths & function names** = Better API design

---

## Step 3: Run the Server

```bash
uvicorn books:app --reload
```

> `--reload` → **Auto-restart** on save

---

## Step 4: Test in Browser

**Direct URL**:  
**http://127.0.0.1:8000/books**

Returns raw JSON (works, but hard to read).

## Final `books.py` (Complete)

```python
from fastapi import FastAPI

app = FastAPI()

BOOKS = [
    {"title": "Title 1", "author": "Author 1", "category": "Science"},
    {"title": "Title 2", "author": "Author 2", "category": "Science"},
    {"title": "Title 3", "author": "Author 3", "category": "History"},
    {"title": "Title 4", "author": "Author 4", "category": "Math"},
    {"title": "Title 5", "author": "Author 5", "category": "Math"},
    {"title": "Title 6", "author": "Author 2", "category": "Math"}
]

@app.get("/books")
async def read_all_books():
    return BOOKS
```

> [!question]- So, when i run `'http://localhost:8000/books'` in browser the function `read_all_books` runs becuase it is decorated by `@app.get("/books")` . Am i correct ? How is BOOKs displayed in the browser. Is it stored somewhere ?
> 
> **Yes — You're 100% Correct!**  
> 
> Let’s break it down clearly:
> 
> ---
> 
> ## 1. How `read_all_books()` Runs on `/books`
> 
> ```python
> @app.get("/books")        # ← This is the **trigger**
> async def read_all_books():
>     return BOOKS          # ← This runs when URL is visited
> ```
> 
> | Action | What Happens |
> |-------|--------------|
> | You open **http://localhost:8000/books** in browser | → Sends **HTTP GET** request |
> | FastAPI sees `@app.get("/books")` | → Matches the path |
> | **Calls `read_all_books()`** | → Function executes |
> | `return BOOKS` | → FastAPI converts Python list → **JSON** |
> | Browser receives JSON | → Displays raw text |
> 
> > **Decorator = URL → Function Mapping**  
> > **No HTML, no templates — just JSON**
> 
> ---
> 
> ## 2. How Is `BOOKS` Displayed in Browser?
> 
> ### **It's NOT stored anywhere permanently**  
> - `BOOKS` is a **Python variable in memory**  
> - Lives only while the **Uvicorn server is running**  
> - **Lost when you stop the server**
> 
> ```python
> BOOKS = [ ... ]  # ← In RAM, not saved to disk
> ```
> 
> ### **Browser sees this (raw JSON):**
> 
> ```json
> [
>   {"title": "Title 1", "author": "Author 1", "category": "Science"},
>   {"title": "Title 2", "author": "Author 2", "category": "Science"},
>   ...
> ]
> ```
> 
> > **FastAPI auto-converts** `list[dict]` → **valid JSON**  
> > Sets correct header: `Content-Type: application/json`
> 
> ---
> 
> ## 3. Visual in Browser vs Swagger UI
> 
> | View | What You See |
> |------|-------------|
> | **Browser (`/books`)** | Raw, unformatted JSON text |
> | **Swagger UI (`/docs`)** | Pretty, clickable, structured response |
> 
> ---
> 
> ## 4. Where Is Data *Actually* Stored?
> 
> | Storage | Current | Future |
> |--------|--------|--------|
> | **In-memory (RAM)** | `BOOKS = [...]` | Only during runtime |
> | **File (JSON, CSV)** | Not yet | Can save/load |
> | **Database (SQLite, PostgreSQL)** | Not yet | Real persistence |
> 
> > **Right now: Data lives in code → temporary**  
> > **Later: Connect to real DB → permanent**
> 
> ---
> 
> ## Summary (Key Points)
> 
> | Question | Answer |
> |--------|--------|
> | Does `@app.get("/books")` run the function? | **Yes — exactly when URL is hit** |
> | How is `BOOKS` shown? | **Auto-converted to JSON** and sent to browser |
> | Is it stored? | **Only in memory (RAM)** — gone when server stops |
> | Can I see it nicely? | Yes — use **http://localhost:8000/docs** |
