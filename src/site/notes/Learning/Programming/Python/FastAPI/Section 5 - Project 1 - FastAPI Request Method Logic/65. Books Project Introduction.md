# Books Project Overview with FastAPI

This is the **first hands-on project** using **FastAPI** — perfect for mastering **core HTTP methods** and **real-world API development**.

---

## Project Goal 🎯
Build a **Books API** to manage a list of books using **CRUD operations**:
- **Create** new books  
- **Read** all or specific books  
- **Update** existing books  
- **Delete** books from the list

---

## Book Data Structure 📚
Each book is a **key-value object** with:
```python
{
  "title": "Title 1",
  "author": "Author 1",
  "category": "Science"  # e.g., Science, History, Math
}
```

We'll start with **5 sample books**:
- Titles: `Title 1` → `Title 5`  
- Authors: `Author 1` → `Author 5`  
- Categories: `Science`, `History`, `Math`

---

## CRUD ↔ HTTP Methods Mapping ✅

| CRUD       | HTTP Method | FastAPI Decorator |
|------------|-------------|-------------------|
| **Create** | `POST`      | `@app.post()`     |
| **Read**   | `GET`       | `@app.get()`      |
| **Update** | `PUT`       | `@app.put()`      |
| **Delete** | `DELETE`    | `@app.delete()`   |

> **🎯 Key Insight**: **CRUD operations = HTTP verbs** — FastAPI makes this mapping **clean and intuitive**.

---

## Request & Response Flow 🔄

```
[Web Page / Client]
       ↓ (HTTP Request)
[FastAPI Server] → Processes → Validates → Responds
       ↑ (HTTP Response: JSON data)
[Web Page / Client]
```

**Example**:  
Client: *"Give me Book 2 and Book 4"*  
FastAPI: Returns → `{ "book2": {...}, "book4": {...} }`

---

## Built-in Swagger UI (Interactive Docs) 🚀

FastAPI **auto-generates** API documentation at:  
```
http://localhost:8000/docs
```
