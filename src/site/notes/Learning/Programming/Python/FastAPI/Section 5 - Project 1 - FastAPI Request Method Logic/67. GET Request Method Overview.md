# FastAPI: First GET Endpoint Tutorial

Let’s build your **first FastAPI application** and create a **GET HTTP request** to return a list of books.

---

## Step 1: Create `books.py` 📌

```python
from fastapi import FastAPI

app = FastAPI()
```

> **Key Point**:  
> - `from fastapi import FastAPI` → Imports the framework  
> - `app = FastAPI()` → Creates your **API instance** (the "app" object)

---

## Step 2: Define Sample Books Data

```python
books = [
    {"title": "Title 1", "author": "Author 1", "category": "Science"},
    {"title": "Title 2", "author": "Author 2", "category": "History"},
    {"title": "Title 3", "author": "Author 3", "category": "Math"},
    {"title": "Title 4", "author": "Author 4", "category": "Science"},
    {"title": "Title 5", "author": "Author 5", "category": "History"}
]
```

> **5 books** with **title**, **author**, and **category** — stored in-memory.

---

## Step 3: Create Your First GET Endpoint

```python
@app.get("/books")
async def read_all_books():
    return books
```

### Breakdown:
| Part | Meaning |
|------|--------|
| `@app.get("/books")` | **Decorator** → Maps `GET /books` URL to this function |
| `async def` | **Asynchronous function** (FastAPI handles async automatically) |
| `return books` | Returns **JSON** response automatically |

> **Why `async`?**  
> Not required — but **explicitly recommended** in this course.  
> FastAPI will run it asynchronously **even if you use `def`**.

---

## Step 4: Run the FastAPI App via Uvicorn

Open terminal and run:

```bash
uvicorn books:app --reload
```

### Command Explained:
| Part | Purpose |
|------|--------|
| `books` | Your Python file (`books.py`) |
| `app` | The `FastAPI()` instance inside the file |
| `--reload` | **Auto-restart** on code changes (great for development!) |

> **Uvicorn** = Built-in **ASGI server** for FastAPI (installed automatically)

---

## Step 5: Test Your Endpoint

Visit in browser:  
**http://127.0.0.1:8000/books**

**Expected JSON Response**:
```json
[
  {"title": "Title 1", "author": "Author 1", "category": "Science"},
  ...
]
```

Also check auto-docs:  
**http://127.0.0.1:8000/docs** → Interactive Swagger UI!

---

## Best Practice: Meaningful Paths 🎯

Avoid vague paths like `/api-endpoint`  
Use **descriptive URLs**:

```python
# Good
@app.get("/books")

# Avoid
@app.get("/api-endpoint")
```

> **Clients should know what data they’ll get from the URL!**

---

## Final Code (`books.py`)

```run-python
from fastapi import FastAPI

app = FastAPI()

books = [
    {"title": "Title 1", "author": "Author 1", "category": "Science"},
    {"title": "Title 2", "author": "Author 2", "category": "History"},
    {"title": "Title 3", "author": "Author 3", "category": "Math"},
    {"title": "Title 4", "author": "Author 4", "category": "Science"},
    {"title": "Title 5", "author": "Author 5", "category": "History"}
]

@app.get("/books")
async def read_all_books():
    return books
```

> ## Core Purpose of FastAPI
> 
> **FastAPI is for building APIs (Application Programming Interfaces)** — the **bridge between frontend and backend**.
> 
> It lets your **web/mobile app** talk to your **server logic & database** securely and efficiently.
> 
> ---
> 
> ## Your Questions — Answered Clearly
> 
> | Question | Answer |
> |--------|--------|
> | **Is FastAPI to grab data from SQL databases and update websites?** | **Yes — but more than that!**<br>FastAPI **connects** your **database** (SQL, NoSQL, etc.) to your **frontend**.<br>It **fetches**, **creates**, **updates**, or **deletes** data and sends it as **JSON**. |
> | **Does user submit a form → FastAPI handles request/response?** | **Exactly!**<br>User fills form → Frontend sends data → FastAPI receives, validates, saves to DB → returns success/data. |
> 
> ---
> 
> ## Real-World Flow Example
> 
> ```mermaid
> graph LR
>     A[User fills form: Add Book] --> B(Frontend: HTML/React)
>     B -->|POST /books| C{FastAPI Server}
>     C --> D[Validate Data]
>     D --> E[Save to SQL DB]
>     E --> F[Return: Book added!]
>     F --> B
>     B --> G[Show success message]
> ```
> 
> ---
> 
> ## What FastAPI Does (Step-by-Step)
> 
> | Step | FastAPI’s Role |
> |------|----------------|
> | 1. Receive Request | `@app.post("/books")` gets form data |
> | 2. Validate Input | Uses **Pydantic** → auto-checks types, required fields |
> | 3. Process Logic | Query **SQLAlchemy**, **SQLite**, **PostgreSQL**, etc. |
> | 4. Send Response | Returns **JSON** → `{"id": 6, "title": "New Book"}` |
>
