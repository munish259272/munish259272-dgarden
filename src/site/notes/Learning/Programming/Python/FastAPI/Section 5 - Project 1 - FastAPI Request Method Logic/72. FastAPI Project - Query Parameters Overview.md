# 🎯 FastAPI Query Parameters

In the previous tutorial, we learned about **Path Parameters**, where values are passed as part of the URL path:

```text
/books/Title One
/books/science
/books/Author Four
```

Now we'll learn about **Query Parameters**, which allow us to **filter or customize a request without changing the path itself**.

---

# 📌 What are Query Parameters?

A **Query Parameter** is a parameter attached to a URL **after a question mark (`?`)**.

Query parameters follow a **name=value** format.

### General Syntax

```text
/path?parameter=value
```

Example:

```text
/books?category=math
```

Here:

|Part|Meaning|
|---|---|
|`/books`|Path|
|`?`|Start of query parameters|
|`category`|Parameter name|
|`math`|Parameter value|

---

# 🧠 Simple Example

Suppose we have the URL:

```text
http://localhost:8000/books?category=science
```

Here:

```text
category = science
```

FastAPI automatically extracts the value `"science"` and passes it to the corresponding function parameter.

---

# 📌 FastAPI Implementation

```python
@app.get("/books")
async def read_category_by_query(category: str):
    books_to_return = []

    for book in books:
        if book["category"].casefold() == category.casefold():
            books_to_return.append(book)

    return books_to_return
```

---

## 🔍 How FastAPI Knows What to Pass?

Notice the URL:

```text
/books?category=science
```

and the function parameter:

```python
category: str
```

The names match:

```text
category=science
         │
         ▼
category: str
```

FastAPI automatically performs:

```python
category = "science"
```

and passes it into the function.

✅ No extra code is required.

# 📌 Query Parameters vs Path Parameters

|Feature|Path Parameter|Query Parameter|
|---|---|---|
|Location|Inside URL path|After `?`|
|Purpose|Identify a specific resource|Filter or modify results|
|Example|`/books/Title One`|`/books?category=science`|

### Path Parameter Example

```text
/books/Title One
```

Means:

> Give me **this specific book**.

---

### Query Parameter Example

```text
/books?category=science
```

Means:

> Give me **all books matching this filter**.

---

# 🎯 Using Path Parameters and Query Parameters Together

FastAPI allows both to be used in the same endpoint.

Consider the URL:

```text
/books/Author%20Four?category=science
```

Remember:

```text
%20 = Space
```

so this actually means:

```text
/books/Author Four?category=science
```

---

## 📌 What Information Does This URL Contain?

### Path Parameter

```text
Author Four
```

represents:

```python
book_author
```

---

### Query Parameter

```text
category=science
```

represents:

```python
category
```

# FastAPI Implementation

```python
@app.get("/books/{book_author}")
async def read_author_category_by_query(
    book_author: str,
    category: str
):
    pass
```

---

## 🔄 What Happens Internally?

Request:

```text
/books/Author Four?category=science
```

FastAPI performs:

```python
book_author = "Author Four"
category = "science"
```

and then calls:

```python
read_author_category_by_query(
    "Author Four",
    "science"
)
```

# 🎯 Multiple Query Parameters

A URL can contain more than one query parameter.

Syntax:

```text
/books?category=science&published=2025
```

Here:

```text
category = science
published = 2025
```

The `&` symbol separates query parameters.

---

# 📌 Key Takeaways

> [!important]- Query Parameters Summary
> 
> ✅ Query parameters appear **after a question mark (`?`)**
> 
> ✅ They follow a **name=value** format:
> 
> ```text
> /books?category=science
> ```
> 
> ✅ FastAPI automatically matches query parameter names with function parameters.
> 
> ✅ Query parameters are commonly used for:
> 
> - Filtering
>     
> - Searching
>     
> - Sorting
>     
> - Pagination
>     
> 
> ✅ Query parameters can be combined with path parameters.
> 
> ```text
> /books/Author Four?category=science
> ```
> 
> ✅ Multiple query parameters are separated using:
> 
> ```text
> &
> ```
> 
> Example:
> 
> ```text
> /books?category=science&year=2025
> ```

> [!tip]- Quick Rule to Remember
> 
> **Path Parameters identify _what resource_ you want.**
> 
> ```text
> /books/Title One
> ```
> 
> **Query Parameters describe _how you want the results filtered or customized_.**
> 
> ```text
> /books?category=science
> ```
> 
> Think:
> 
> ```text
> Path Parameter  → "Which item?"
> Query Parameter → "Which filter?"
> ```
