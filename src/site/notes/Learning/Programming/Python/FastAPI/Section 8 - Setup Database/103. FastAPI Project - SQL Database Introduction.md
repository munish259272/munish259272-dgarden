# Introduction to Databases and SQL

As we move into Project 3, one of the most important concepts we need to understand is the **database**. Databases are the foundation of most modern applications because they provide a way to **store, organize, retrieve, and manage data efficiently**.

---

# What is a Database?

A **database** is an organized collection of structured information (data) stored inside a computer system.

A database allows us to:

✅ Store data

✅ Retrieve data

✅ Update data

✅ Delete data

✅ Organize data efficiently

Without a database, applications would have no reliable way to remember information after they restart.

---

# What is Data?

Before understanding databases, we first need to understand **data**.

Data is simply information about an object.

For example, consider a user in an application:

|Attribute|Example Value|
|---|---|
|Name|Mukesh|
|Age|45|
|Email|[mukesh@example.com](mailto:mukesh@example.com)|
|Password|password123|

Each piece of information is considered **data**.

Together, all these values describe a single user.

---

# Real-World Example

Imagine a library system.

Each book might contain:

|Field|Example|
|---|---|
|Title|FastAPI Guide|
|Author|John Doe|
|Rating|5|
|Publish Date|2025|

These individual values are data.

A collection of thousands of such records becomes a database.

---

# Why Do We Need a Database?

Suppose we store users inside a Python list:

```python
users = [
    {
        "name": "Mukesh",
        "age": 45
    }
]
```

This works temporarily.

However:

⚠️ If the application stops, the data disappears.

⚠️ Data cannot easily be shared across multiple users.

⚠️ Searching large datasets becomes inefficient.

A database solves these problems by storing information permanently and organizing it efficiently.

---

# Database vs Data

|Concept|Meaning|
|---|---|
|Data|Individual pieces of information|
|Database|Organized collection of data|

### Example

```text
Data:
------
Mukesh
45
mukesh@example.com

Database:
---------
Collection of thousands of user records
```

📌 A database is essentially a **system for managing data**.

---

# Database Management System (DBMS)

A database is usually controlled by a special piece of software called a:

## DBMS

```text
DBMS = Database Management System
```

A DBMS is responsible for:

- Storing data
    
- Retrieving data
    
- Updating data
    
- Deleting data
    
- Managing security
    
- Managing relationships between data
    

Think of a DBMS as the manager responsible for organizing everything inside a database.

```

```

# What is a Relational Database?

A relational database stores data in tables.

Think of it like an Excel spreadsheet.

Example:

|ID|Name|Email|
|---|---|---|
|1|Mukesh|[mukesh@example.com](mailto:mukesh@example.com)|
|2|John|[john@example.com](mailto:john@example.com)|

Each column represents an attribute.

Each row represents a record.

---

# Important Terminology

## Table

A collection of related data.

Example:

```text
Users Table
```

---

## Column

Defines a type of information.

Example:

```text
Name
Age
Email
Password
```

---

## Row (Record)

A single entry in the table.

Example:

|Name|Age|
|---|---|
|Mukesh|45|

This entire row is called a **record**.

---

# SQL and CRUD Operations

SQL allows us to perform four fundamental operations.

These operations should look familiar because they closely match our API endpoints.

|CRUD|SQL Action|
|---|---|
|Create|Insert data|
|Read|Retrieve data|
|Update|Modify data|
|Delete|Remove data|

---

# Create

Add new records.

Example:

```text
Add a new user
```

SQL:

```sql
INSERT INTO users (name, age)
VALUES ('Mukesh', 45);
```

---

# Read

Retrieve existing records.

Example:

```text
Find all users
```

SQL:

```sql
SELECT * FROM users;
```

---

# Update

Modify existing records.

Example:

```text
Change a user's age
```

SQL:

```sql
UPDATE users
SET age = 46
WHERE id = 1;
```

---

# Delete

Remove records.

Example:

```text
Delete a user
```

SQL:

```sql
DELETE FROM users
WHERE id = 1;
```

---
