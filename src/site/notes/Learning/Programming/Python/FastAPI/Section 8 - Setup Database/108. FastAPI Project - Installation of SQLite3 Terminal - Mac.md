# Installing SQLite3 on macOS

In this lesson, we will install **SQLite3** on macOS so that we can interact directly with databases from the terminal.

Once installed, you'll be able to:

✅ Open SQLite databases

✅ Create tables

✅ Run SQL queries

✅ View records

✅ Insert, update, and delete data

✅ Inspect databases created by SQLAlchemy

---

# Why Install SQLite3?

In previous lessons, SQLAlchemy automatically created:

```text
todos.db
```

for us.

Example:

```python
models.Base.metadata.create_all(bind=engine)
```

However, SQLAlchemy hides most of the database operations behind Python code.

Installing SQLite3 allows us to interact with the database directly and see what is happening behind the scenes.

---

# Step 1: Install Homebrew

The easiest way to install software on macOS is using **Homebrew**.

Think of Homebrew as the macOS equivalent of:

|Operating System|Package Manager|
|---|---|
|Ubuntu/Debian|`apt`|
|Fedora/RHEL|`dnf`|
|Arch Linux|`pacman`|
|macOS|`brew`|

---

## Download Homebrew

Visit:

[Homebrew Official Website](https://brew.sh/?utm_source=chatgpt.com)

You will see an installation command similar to:

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

---

## Install Homebrew

Open Terminal and run:

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

The installer will:

- Download Homebrew
    
- Configure your environment
    
- Install required components
    

You may be prompted to enter your macOS password:

```text
Password:
```

Enter your password and continue.

---

# Step 2: Restart Terminal

After installation completes:

```text
Close Terminal
        ↓
Reopen Terminal
```

This ensures the new Homebrew configuration is loaded.

---

# Step 3: Verify Homebrew Installation

Run:

```bash
brew --version
```

Example output:

```text
Homebrew 4.x.x
```

If you see a version number, Homebrew is installed successfully.

---

# Step 4: Check Whether SQLite Is Already Installed

Many macOS systems already include SQLite.

Check by running:

```bash
brew list
```

or

```bash
sqlite3 --version
```

Example:

```text
3.49.1
```

If a version is displayed, SQLite is already available.

---

# Step 5: Install SQLite (If Needed)

If SQLite is missing:

```bash
brew install sqlite
```

Homebrew will:

- Download SQLite
    
- Install dependencies
    
- Configure SQLite binaries
    

---

# Verify SQLite Installation

Run:

```bash
sqlite3 --version
```

Example output:

```text
3.49.1 2025-02-18 ...
```

🎯 Seeing a version number confirms SQLite is installed.

---

# Starting the SQLite Shell

Launch SQLite:

```bash
sqlite3
```

Example:

```text
SQLite version 3.49.1
Enter ".help" for usage hints.
sqlite>
```

When you see:

```text
sqlite>
```

you are inside the SQLite interactive shell.

---

# What Can You Do Inside SQLite?

For example:

## Check Version

```sql
SELECT sqlite_version();
```

---

## List Commands

```sql
.help
```

---

## Exit SQLite

```sql
.quit
```

or

```sql
.exit
```

---

# SQLite and Our FastAPI Project

Remember our project structure:

```text
todoapp/
│
├── database.py
├── models.py
├── main.py
└── todos.db
```

SQLAlchemy creates:

```text
todos.db
```

SQLite allows us to open that same database directly:

```bash
sqlite3 todos.db
```

Now we can inspect exactly what SQLAlchemy created.

---

# SQLAlchemy vs SQLite CLI

## SQLAlchemy

Works through Python:

```python
todos = db.query(Todos).all()
```

---

## SQLite CLI

Works through SQL:

```sql
SELECT * FROM todos;
```

Both interact with the same database.

---

# Architecture Overview

```text
FastAPI
   │
   ▼
SQLAlchemy
   │
   ▼
SQLite Database
   ▲
   │
SQLite CLI
```

📌 SQLAlchemy and the SQLite command-line tool are simply two different ways of interacting with the same database file.
