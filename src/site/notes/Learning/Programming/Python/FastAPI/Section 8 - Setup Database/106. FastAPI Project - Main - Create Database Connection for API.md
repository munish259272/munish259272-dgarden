# Creating the SQLite Database and Tables with SQLAlchemy

At this point, we have already created:

```text
database.py
models.py
```

However, even though we have defined:

- Where the database should be stored (`database.py`)
    
- What tables should exist (`models.py`)
    

we still do **not actually have a database file or any tables created yet**.

The good news is that **SQLAlchemy can create both automatically for us** with just a few lines of code.

---

# Current Situation

So far, we have:

```text
todoapp/
│
├── database.py
└── models.py
```

### `database.py`

Defines:

```python
SQLALCHEMY_DATABASE_URL = "sqlite:///./todos.db"
```

This tells SQLAlchemy:

> "Store the SQLite database in a file called `todos.db`."

---

### `models.py`

Defines:

```python
class Todos(Base):
```

which describes the structure of the `todos` table.

But nothing has been created yet.

---

# Create `main.py`

Create a new file:

```text
todoapp/
│
├── database.py
├── models.py
└── main.py
```

This file will become the **entry point** of our FastAPI application.

📌 Most of the application's logic will eventually live here.

```text

        │
```

# Import Models and Engine

Import the models:

```python
import models
```

Import the database engine:

```python
from database import engine
```

---

# Why Import `models`?

This is important.

When Python imports:

```python
import models
```

it executes:

```python
class Todos(Base):
```

and registers the table definition with SQLAlchemy.

Without importing `models`, SQLAlchemy won't know that the `Todos` table exists.

> [!warning]  
> Many beginners accidentally remove:
> 
> ```python
> import models
> ```
> 
> and then wonder why their tables are not being created.

---

# Create the Database and Tables

Add:

```python
models.Base.metadata.create_all(bind=engine)
```

Complete code:

```python
from fastapi import FastAPI

import models
from database import engine

app = FastAPI()

models.Base.metadata.create_all(bind=engine)
```

---

# Understanding `create_all()`

This is where the magic happens.

```python
models.Base.metadata.create_all(bind=engine)
```

SQLAlchemy:

1. Looks at all classes that inherit from `Base`
    
2. Finds all table definitions
    
3. Connects to the database using the engine
    
4. Creates any missing tables
    

---

## Visual Flow

```text
models.py
    │
    ▼
Todos Table Definition
    │
    ▼
Base.metadata
    │
    ▼
create_all()
    │
    ▼
SQLite Database
```

---

# Understanding `bind=engine`

```python
bind=engine
```

means:

> "Use this database connection to create the tables."

Visualized:

```text
create_all()
      │
      ▼
    Engine
      │
      ▼
 SQLite
```

Without an engine, SQLAlchemy wouldn't know where to create the tables.

---

# What Happens Behind the Scenes?

Suppose your model is:

```python
class Todos(Base):
    __tablename__ = "todos"

    id = Column(Integer, primary_key=True)
    title = Column(String)
```

When:

```python
models.Base.metadata.create_all(bind=engine)
```

runs, SQLAlchemy secretly generates SQL similar to:

```sql
CREATE TABLE todos (
    id INTEGER PRIMARY KEY,
    title VARCHAR
);
```

📌 SQLAlchemy writes the SQL for you.

You don't have to manually create tables.

---

# How Does SQLAlchemy Know What to Create?

SQLAlchemy combines information from two files.

---

## From `database.py`

It learns:

```python
SQLALCHEMY_DATABASE_URL
```

and

```python
engine
```

which tell it:

```text
Where to create the database
```

---

## From `models.py`

It learns:

```python
class Todos(Base):
```

which tells it:

```text
What tables to create
```

# Why Is This Useful?

Without SQLAlchemy:

You would need to write SQL manually:

```sql
CREATE TABLE todos (
    id INTEGER PRIMARY KEY,
    title TEXT,
    description TEXT,
    priority INTEGER,
    complete BOOLEAN
);
```

With SQLAlchemy:

```python
models.Base.metadata.create_all(bind=engine)
```

That's it.

✅ Database created

✅ Tables created

✅ Columns created

✅ No SQL required
