# Creating the `models.py` File

In the previous lesson, we created `database.py`, which was responsible for configuring our database connection.

In this lesson, we will create **`models.py`**, which defines the actual database tables that will exist inside our database.

---

# What is a Database Model?

A **database model** describes the structure of a database table.

Think of it as a blueprint.

For example, if we want a table called `todos`, we need to define:

- What columns exist
    
- What data types they contain
    
- Which column is the primary key
    
- Any default values
    

---

# Real-World Analogy

Imagine creating a spreadsheet.

Before entering data, you first create the columns:

|ID|Title|Description|Priority|Complete|
|---|---|---|---|---|

This structure is the **model**.

Later, each row becomes a **record**.

Example:

|ID|Title|Description|Priority|Complete|
|---|---|---|---|---|
|1|Learn FastAPI|Study SQLAlchemy|5|False|

The table structure comes from the model.

The row is a record.

# Create `models.py`

Inside the `todoapp` package:

```text
todoapp/
│
├── database.py
└── models.py
```

---

# Import `Base` From `database.py`

In `database.py`, we previously created:

```python
Base = declarative_base()
```

Now import it:

```python
from database import Base
```

---

# Why Import `Base`?

Every database table must inherit from `Base`.

Think of `Base` as the parent class for all database tables.

```text
Base
 │
 ▼
Todo Table
```

Without `Base`, SQLAlchemy does not know that the class should become a database table.

---

# Creating the Todo Model

Create a class:

```python
class Todos(Base):
```

This tells SQLAlchemy:

> "This class represents a database table."

---

# Naming the Database Table

Inside the class:

```python
__tablename__ = "todos"
```

---

## Why is `__tablename__` Needed?

SQLAlchemy needs to know the table name that should be created inside the database.

Example:

```python
__tablename__ = "todos"
```

Results in:

```text
Database
   │
   ▼
todos Table
```

---

# Database Table Structure

We want the table to contain:

|Column|Purpose|
|---|---|
|id|Unique identifier|
|title|Todo title|
|description|Todo description|
|priority|Importance level|
|complete|Completion status|

---

# Import SQLAlchemy Column Types

Before creating columns, import the required types:

```python
from sqlalchemy import Column
from sqlalchemy import Integer
from sqlalchemy import String
from sqlalchemy import Boolean
```

or:

```python
from sqlalchemy import Column, Integer, String, Boolean
```

---

# Creating the `id` Column

```python
id = Column(
    Integer,
    primary_key=True,
    index=True
)
```

---

## Understanding Each Parameter

### `Integer`

```python
Integer
```

The ID stores numeric values.

Example:

```text
1
2
3
4
```

---

### `primary_key=True`

```python
primary_key=True
```

The primary key uniquely identifies every record.

Example:

|ID|Title|
|---|---|
|1|Learn FastAPI|
|2|Learn SQLAlchemy|

No two rows can share the same primary key.

---

### `index=True`

```python
index=True
```

Creates a database index.

Indexes improve search performance.

Example:

Without index:

```text
Database searches row-by-row
```

With index:

```text
Database can find records much faster
```

🎯 Frequently searched columns often use indexes.

---

# Creating the `complete` Column

```python
complete = Column(
    Boolean,
    default=False
)
```

---

## Why Use a Boolean?

A todo can only be:

```text
Complete
```

or

```text
Incomplete
```

Boolean values represent this perfectly.

|Value|Meaning|
|---|---|
|True|Completed|
|False|Not completed|

---

## Default Value

```python
default=False
```

Every new todo automatically starts as:

```text
Incomplete
```

unless explicitly changed.

Example:

|Title|Complete|
|---|---|
|Learn FastAPI|False|

---

# Complete `models.py`

```python
from database import Base
from sqlalchemy import Column
from sqlalchemy import Integer
from sqlalchemy import String
from sqlalchemy import Boolean


class Todos(Base):
    __tablename__ = "todos"

    id = Column(
        Integer,
        primary_key=True,
        index=True
    )

    title = Column(String)
    description = Column(String)
    priority = Column(Integer)

    complete = Column(
        Boolean,
        default=False
    )
```

---

# How SQLAlchemy Interprets This

When SQLAlchemy sees:

```python
class Todos(Base):
```

it converts the class into a database table.

Conceptually:

```text
Python Class
      │
      ▼
SQLAlchemy
      │
      ▼
Database Table
```

Resulting table:

|id|title|description|priority|complete|
|---|---|---|---|---|

---

# Model vs Record

## Model

```python
class Todos(Base):
```

Defines the table structure.

---

## Record

Example row:

|id|title|description|priority|complete|
|---|---|---|---|---|
|1|Learn FastAPI|Study SQLAlchemy|5|False|

Represents actual data stored inside the table.
