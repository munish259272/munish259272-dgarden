Unlike previous projects where data was stored in Python lists, this project introduces a real database using **SQLite** and **SQLAlchemy**.

# What is SQLAlchemy?

**SQLAlchemy** is an ORM.

ORM stands for:

```text
Object Relational Mapper
```

An ORM acts as a bridge between:

```text
Python Objects
       ↔
 Database Tables
```

Instead of writing SQL manually:

```sql
SELECT * FROM todos;
```

you can work with Python objects:

```python
todos = db.query(Todo).all()
```

---

# Why Use an ORM?

Without ORM:

```text
Python
   │
   ▼
Raw SQL Queries
   │
   ▼
Database
```

With ORM:

```text
Python Objects
      │
      ▼
SQLAlchemy
      │
      ▼
Database
```

✅ Easier to write

✅ Easier to maintain

✅ More Pythonic

# Step 6: Create a Database Engine

First import:

```python
from sqlalchemy import create_engine
```

---

## Create Engine

```python
engine = create_engine(
    SQLALCHEMY_DATABASE_URL,
    connect_args={"check_same_thread": False}
)
```

---

# What is a Database Engine?

The engine is responsible for:

✅ Opening connections

✅ Managing connections

✅ Sending queries

✅ Communicating with SQLite

Architecture:

```text
FastAPI
    │
    ▼
Engine
    │
    ▼
SQLite Database
```

Think of the engine as a translator between FastAPI and the database.

---

# Understanding `check_same_thread=False`

SQLite normally expects:

```text
One Thread
     │
     ▼
One Database Connection
```

However, FastAPI may process multiple requests simultaneously.

Example:

```text
Request 1
Request 2
Request 3
```

all accessing the database at the same time.

Therefore we use:

```python
connect_args={"check_same_thread": False}
```

to allow multiple FastAPI threads to interact with SQLite safely.

> [!warning]  
> SQLite is designed primarily for lightweight applications. Larger production systems typically use PostgreSQL or MySQL instead.

# Step 7: Create `SessionLocal`

Import:

```python
from sqlalchemy.orm import sessionmaker
```

Create:

```python
SessionLocal = sessionmaker(
    autocommit=False,
    autoflush=False,
    bind=engine
)
```

---

# What is a Session?

A session represents a conversation with the database.

Example:

```text
Open Session
      │
      ▼
Read Data
      │
      ▼
Update Data
      │
      ▼
Save Changes
      │
      ▼
Close Session
```

---

# Understanding the Parameters

## `autocommit=False`

```python
autocommit=False
```

Changes are not automatically saved.

Instead:

```python
db.commit()
```

must be called manually.

---

## `autoflush=False`

```python
autoflush=False
```

SQLAlchemy will not automatically send pending changes to the database.

You remain in complete control.

---

## `bind=engine`

```python
bind=engine
```

Connects the session to the database engine.

```text
Session
    │
    ▼
Engine
    │
    ▼
Database
```

---

# Step 8: Create a Base Class

Import:

```python
from sqlalchemy.ext.declarative import declarative_base
```

Create:

```python
Base = declarative_base()
```

---

# What is `Base`?

`Base` is the parent class for all future database models.

Later we will create tables like:

```python
class Todo(Base):
```

The inheritance relationship looks like:

```text
Base
 │
 ├── Todo
 │
 ├── User
 │
 └── Other Tables
```

---

# Why Do We Need `Base`?

Without `Base`:

```python
class Todo:
```

SQLAlchemy does not know this should become a database table.

With:

```python
class Todo(Base):
```

SQLAlchemy understands:

```text
Python Class
      │
      ▼
Database Table
```

---

# Final `database.py`

```python
from sqlalchemy import create_engine
from sqlalchemy.orm import sessionmaker
from sqlalchemy.ext.declarative import declarative_base

SQLALCHEMY_DATABASE_URL = "sqlite:///./todos.db"

engine = create_engine(
    SQLALCHEMY_DATABASE_URL,
    connect_args={"check_same_thread": False}
)

SessionLocal = sessionmaker(
    autocommit=False,
    autoflush=False,
    bind=engine
)

Base = declarative_base()
```
