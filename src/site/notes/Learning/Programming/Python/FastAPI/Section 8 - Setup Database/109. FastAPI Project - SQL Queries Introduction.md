# Basic SQL Queries (CRUD Operations)

Before we start using SQLAlchemy extensively, it is important to understand the underlying SQL operations it performs for us.

SQL revolves around four core operations, commonly known as **CRUD**:

|CRUD Operation|SQL Command|
|---|---|
|Create|`INSERT`|
|Read|`SELECT`|
|Update|`UPDATE`|
|Delete|`DELETE`|

🎯 Understanding these commands helps you understand what SQLAlchemy is doing behind the scenes.

---

# Sample `todos` Table

Throughout these examples, assume we have a table structured like this:

|id|title|description|priority|complete|
|---|---|---|---|---|
|1|Go to Store|Pick up eggs|4|False|
|2|Haircut|Need 1mm haircut|3|False|
|3|Feed Dog|Use new food brand|5|False|
|4|Water Plants|Indoor and outdoor plants|4|False|
|5|Learn Something New|Learn Python|5|False|
|6|Shower|You haven't showered in days|5|False|

---

# Table Columns Explained

|Column|Purpose|
|---|---|
|`id`|Primary key (unique identifier)|
|`title`|Short todo title|
|`description`|Detailed information|
|`priority`|Importance level|
|`complete`|Whether task is finished|

Future versions may also include:

```text
owner
```

which acts as a **foreign key** linking todos to specific users.

---

# CREATE — Inserting Records

To add data into a table, use:

```sql
INSERT INTO todos (
    title,
    description,
    priority,
    complete
)
VALUES (
    'Go to Store',
    'Pick up eggs',
    4,
    FALSE
);
```

---

## Why Isn't `id` Included?

Notice:

```sql
id
```

is missing.

This is intentional.

The database automatically generates IDs:

```text
1
2
3
4
...
```

This process is called **auto-incrementing**.

✅ Prevents duplicate IDs

✅ Ensures uniqueness

✅ Removes manual work

---

## Result

After execution:

|id|title|description|priority|complete|
|---|---|---|---|---|
|1|Go to Store|Pick up eggs|4|False|

---

# Adding Another Record

```sql
INSERT INTO todos (
```

Result:


# READ — Retrieving Data

To retrieve data:

```sql
SELECT * FROM todos;
```

---

## What Does `*` Mean?

The asterisk means:

```text
ALL COLUMNS
```

So:

```sql
SELECT * FROM todos;
```

means:

> Return every column and every row from the table.

---

## Result

```text
All records
All columns
```

---

# Retrieving Specific Columns

Suppose we only want titles.

```sql
SELECT title
FROM todos;
```

---

## Result

```text
Go to Store
Haircut
Feed Dog
Water Plants

Shower
```

Only the `title` column is returned.

# WHERE Clause

The `WHERE` clause filters records.

General form:

```sql
SELECT *
FROM table_name
WHERE condition;
```

Think of it as:

> "Only show rows matching this condition."

---

# Example: Priority Equals 5

```sql
SELECT *
FROM todos
WHERE priority = 5;
```

---

## Result

Only:

|id|title|priority|
|---|---|---|
|3|Feed Dog|5|
|5|Learn Something New|5|
|6|Shower|5|

are returned.

---

# Example: Search by Title

```sql
SELECT *
FROM todos
WHERE title = 'Feed Dog';
```

---

## Result

|id|title|
|---|---|
|3|Feed Dog|

Only one matching record is returned.

---

# Most Common Use Case: Search by ID

```sql
SELECT *
FROM todos
WHERE id = 2;
```

Plain English:

> Return the record whose ID equals 2.

Result:

|id|title|
|---|---|
|2|Haircut|

---

> [!tip]  
> 🎯 **Primary keys are the safest way to identify records because they are guaranteed to be unique.**

---

# UPDATE — Modifying Existing Records

The `UPDATE` statement changes existing data.

General form:

```sql
UPDATE table_name
SET column = value
WHERE condition;
```

---

# Mark Todo as Complete

```sql
UPDATE todos
SET complete = TRUE
WHERE id = 5;
```

---

## What Does This Mean?

```text
Find record with id = 5
        ↓
Change complete to TRUE
```

Result:

|id|title|complete|
|---|---|---|
|5|Learn Something New|True|

---

# Updating Using Title

```sql
UPDATE todos
SET complete = TRUE
WHERE title = 'Learn Something New';
```

This works, but it is risky.

---

> [!warning]  
> Titles are usually **not unique**.
> 
> If multiple records have:
> 
> ```text
> Learn Something New
> ```
> 
> then all matching records will be updated.

---

## Safer Approach

Always prefer:

```sql
WHERE id = ...
```

because IDs are unique.

---

# DELETE — Removing Records

General form:

```sql
DELETE FROM table_name
WHERE condition;
```

---

# Delete by ID

```sql
DELETE FROM todos
WHERE id = 5;
```

---

## What Happens?

Before:

|id|
|---|
|1|
|2|
|3|
|4|
|5|
|6|

After:

|id|
|---|
|1|
|2|
|3|
|4|
|6|

Record 5 is permanently removed.

---

# Dangerous Delete Example

```sql
DELETE FROM todos
WHERE complete = FALSE;
```

---

## What Could Happen?

Suppose all todos are incomplete:

```text
complete = FALSE
```

Then:

```sql
DELETE FROM todos
WHERE complete = FALSE;
```

would delete:

```text
EVERY RECORD
```

⚠️ This is why delete operations should be used very carefully.

---

# Soft Delete vs Hard Delete

Many production applications avoid permanently deleting records.

Instead they use a field such as:

```sql
is_deleted
```

or

```sql
deleted_at
```

Example:

|id|title|is_deleted|
|---|---|---|
|5|Learn Python|True|

The record still exists but is hidden from normal users.

This is called a **soft delete**.

# CRUD Summary

|Operation|SQL Command|
|---|---|
|Create|`INSERT INTO`|
|Read|`SELECT`|
|Update|`UPDATE`|
|Delete|`DELETE`|

---

# SQL vs SQLAlchemy

The good news is that in FastAPI we usually don't write these SQL statements manually.

For example:

### SQL

```sql
SELECT *
FROM todos
WHERE id = 2;
```

---

### SQLAlchemy

```python
todo = db.query(Todos).filter(
    Todos.id == 2
).first()
```

SQLAlchemy converts Python code into SQL automatically.
