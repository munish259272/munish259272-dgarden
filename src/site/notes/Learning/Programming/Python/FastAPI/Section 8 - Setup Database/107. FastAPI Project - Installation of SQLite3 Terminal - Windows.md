# Installing SQLite3 on Windows

In this lesson, we will install **SQLite3** on Windows so that we can interact directly with SQLite databases from the command line.

After installation, you'll be able to:

✅ Open SQLite databases

✅ Execute SQL queries

✅ View tables

✅ Insert records

✅ Update records

✅ Delete records

✅ Explore database contents directly from the terminal

---

# Why Install SQLite3?

So far, SQLAlchemy has been creating databases and tables automatically for us.

Example:

```python
models.Base.metadata.create_all(bind=engine)
```

creates:

```text
todos.db
```

However, we currently have no easy way to inspect what's inside the database.

Installing SQLite3 allows us to manually examine and manipulate the database.

---

# Step 1: Download SQLite3

Open your browser and visit:

```text
https://www.sqlite.org
```

Or directly:

```text
https://www.sqlite.org/download.html
```

---

# Step 2: Open the Downloads Page

From the SQLite homepage:

```text
Home
 └── Download
```

Click:

```text
Download
```

---

# Step 3: Locate Windows Precompiled Binaries

Scroll down until you find:

```text
Precompiled Binaries for Windows
```

You will see several download options.

Choose:

```text
sqlite-tools-win32-x86-<version>.zip
```

📌 This package contains the SQLite command-line tools required to manage SQLite databases.

---

# Why Download the Tools Package?

The tools package includes utilities such as:

```text
sqlite3.exe
sqldiff.exe
sqlite3_analyzer.exe
```

The most important file for us is:

```text
sqlite3.exe
```

which allows us to interact with databases directly from the command prompt.

---

# Step 4: Extract the ZIP File

After downloading:

```text
sqlite-tools-win32-x86-<version>.zip
```

Extract it.

Example:

```text
Downloads/
│
└── sqlite-tools-win32-x86-<version>/
```

Inside you'll find several executable files.

---

# Step 5: Move SQLite to a Permanent Location

Copy or move the extracted folder to:

```text
C:\
```

Example:

```text
C:\
│
└── SQLite3\
```

Final structure:

```text
C:\SQLite3\
│
├── sqlite3.exe
├── sqldiff.exe
└── sqlite3_analyzer.exe
```

---

# Why Use `C:\SQLite3`?

Keeping SQLite in a fixed location makes it easy to:

- Add to the system PATH
    
- Access from any terminal
    
- Avoid accidental deletion from Downloads
    

---

# Step 6: Copy the Folder Path

Open:

```text
C:\SQLite3
```

Copy the full path:

```text
C:\SQLite3
```

You'll use this in the next step.

---

# Step 7: Open Environment Variables

Press:

```text
Windows Key
```

Search for:

```text
Environment Variables
```

Open:

```text
Edit the system environment variables
```

---

# Step 8: Open Environment Variables Window

Click:

```text
Environment Variables
```

You will see:

```text
User Variables
System Variables
```

---

# Step 9: Edit PATH

Under:

```text
System Variables
```

select:

```text
Path
```

Then click:

```text
Edit
```

---

# Step 10: Add SQLite Path

Click:

```text
New
```

Paste:

```text
C:\SQLite3
```

Click:

```text
OK
```

Repeat:

```text
OK
```

until all dialogs are closed.

---

# What Does PATH Do?

The PATH environment variable tells Windows:

> "These are directories where executable programs can be found."

Without PATH:

```cmd
sqlite3
```

would only work if you were inside:

```text
C:\SQLite3
```

---

With PATH configured:

```cmd
sqlite3
```

works from anywhere.

Example:

```cmd
C:\Users\Mukesh>
sqlite3
```

or

```cmd
D:\Projects>
sqlite3
```

---

# Visual Representation

Before PATH:

```text
Command Prompt
      │
      ▼
sqlite3
      │
      ▼
❌ Command not found
```

---

After PATH:

```text
Command Prompt
      │
      ▼
sqlite3
      │
      ▼
Windows searches PATH
      │
      ▼
C:\SQLite3\sqlite3.exe
      │
      ▼
SQLite Starts
```

---

# Step 11: Verify Installation

Open a new Command Prompt.

Run:

```cmd
sqlite3
```

If installation succeeded, you should see something similar to:

```text
SQLite version 3.x.x
Enter ".help" for usage hints.
sqlite>
```

---

# Understanding the SQLite Prompt

When you see:

```text
sqlite>
```

you are now inside the SQLite command-line shell.

Example:

```sql
SELECT sqlite_version();
```

Output:

```text
3.x.x
```

---

# Exit SQLite

To leave the SQLite shell:

```sql
.quit
```

or

```sql
.exit
```

---

# Why We Need SQLite CLI

In upcoming lessons, we'll use SQLite directly to:

- View tables
    
- Inspect records
    
- Execute SQL queries
    
- Verify SQLAlchemy operations
    

Example:

```sql
SELECT * FROM todos;
```

This lets us see exactly what data exists in the database.

---

# SQLAlchemy vs SQLite CLI

## SQLAlchemy

```python
todos = db.query(Todos).all()
```

Python code interacts with the database.

---

## SQLite CLI

```sql
SELECT * FROM todos;
```

Direct SQL interaction with the database.

---

```text

   ▲
   │

```

Both SQLAlchemy and the SQLite command-line tool interact with the same database file.
