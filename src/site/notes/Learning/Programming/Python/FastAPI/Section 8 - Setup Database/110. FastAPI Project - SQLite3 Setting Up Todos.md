# Manipulating SQLite Databases from the Terminal

Now that SQLite3 is installed, we can interact directly with our database using the terminal.

This allows us to:

✅ View tables

✅ Insert records

✅ Query data

✅ Delete records

✅ Experiment with SQL commands

All without writing any Python code.

---

# Step 1: Navigate to the Project Directory

Move into the directory that contains your database:

```bash
cd todoapp
```

Example:

```text
todoapp/
│
├── database.py
├── models.py
├── main.py
└── todos.db
```

---

# Step 2: Open SQLite

Launch SQLite and connect to the database:

```bash
sqlite3 todos.db
```

Example output:

```text
SQLite version 3.x.x
Enter ".help" for usage hints.
sqlite>
```

The prompt:

```text
sqlite>
```

indicates you are now inside the SQLite shell.

---

# Step 3: View Database Schema

To see all tables and their structure:

```sql
.schema
```

Example output:

```sql
CREATE TABLE todos (
    id INTEGER NOT NULL,
    title VARCHAR,
    description VARCHAR,
    priority INTEGER,
    complete BOOLEAN,
    PRIMARY KEY (id)
);
```

---

## What Does `.schema` Show?

It displays:

✅ Tables

✅ Columns

✅ Data types

✅ Primary keys

Essentially, it shows how the database is structured.

---

# Understanding the `todos` Table

|Column|Purpose|
|---|---|
|`id`|Primary key|
|`title`|Todo title|
|`description`|Detailed description|
|`priority`|Importance level|
|`complete`|Completion status|

---

# Step 4: Insert Data

Add a todo:

```sql
INSERT INTO todos (
    title,
    description,
    priority,
    complete
)
VALUES (
    'Go To Store',
    'Pick Up Eggs',
    5,
    FALSE
);
```

---

> [!warning]  
> ⚠️ Every SQL statement must end with:
> 
> ```sql
> ;
> ```
> 
> Forgetting the semicolon causes SQLite to wait for the remainder of the command.

Example prompt:

```text
...>
```

means SQLite is still waiting for the command to finish.

---

# Step 5: View All Records

Retrieve everything:

```sql
SELECT * FROM todos;
```

Result:

```text
1|Go To Store|Pick Up Eggs|5|0
```

---

## Understanding Boolean Values

SQLite stores booleans as integers.

|Value|Meaning|
|---|---|
|`0`|False|
|`1`|True|

Example:

```text
complete = 0
```

means:

```text
False
```

---

# Understanding Auto-Increment IDs

Notice:

```text
1
```

was automatically assigned as the ID.

We never specified:

```sql
id
```

during insertion.

The database handled it automatically.

---

## Why Auto-Increment?

Without auto-increment:

```text
User A → id = 1
User B → id = 1
```

Duplicate primary keys would occur.

Auto-increment ensures uniqueness.

# Making Output Easier to Read

By default SQLite output can be difficult to read.

SQLite supports multiple display modes.

---

# Column Mode

```sql
.mode column
```

Now:

```sql
SELECT * FROM todos;
```

appears as:

```text
id  title         description          priority  complete
--  ------------  -------------------  --------  --------
1   Go To Store   Pick Up Eggs         5         0
2   Cut The Lawn  Grass Is Getting Long 3        0
3   Feed The Dog  He Is Getting Hungry 5         0
```

---

# Markdown Mode

```sql
.mode markdown
```

Produces Markdown-style output.

Useful when copying results into documentation.

---

# Box Mode

```sql
.mode box
```

Example:

```text
┌────┬──────────────┬─────────────────────┐
│ id │ title        │ description         │
├────┼──────────────┼─────────────────────┤
│ 1  │ Go To Store  │ Pick Up Eggs        │
└────┴──────────────┴─────────────────────┘
```

---

# Table Mode (Recommended)

```sql
.mode table
```

Produces:

```text
+----+--------------+----------------------+
| id | title        | description          |
+----+--------------+----------------------+
| 1  | Go To Store  | Pick Up Eggs         |
+----+--------------+----------------------+
```

🎯 Many developers prefer:

```sql
.mode table
```

for readability.

# Viewing Command History

Inside SQLite, you can use:

```text

```

(Up Arrow Key)

to cycle through previously executed commands.

Example:

```text

```

can be recalled without retyping.

# Deleting a Record

Delete by primary key:

```sql
DELETE FROM todos
WHERE id = 4;
```

---

## Why Use the Primary Key?

The primary key:

```sql
id
```

is unique.

Example:

```text
Only one record can have id = 4
```

This guarantees we delete exactly one record.

---

> [!tip]  
> 🎯 When updating or deleting records, always prefer:
> 
> ```sql
> WHERE id = ...
> ```
> 
> because primary keys are unique.

---

# Verify Deletion

```sql
SELECT * FROM todos;
```

Result:

|id|
|---|
|1|
|2|
|3|

Record 4 is gone.

# Interesting SQLite Behavior

Create another record:

```sql
INSERT INTO todos (
    title,
    description,
    priority,
    complete
)
VALUES (
    'New Test Element',
    'Testing Again',
    1,
    FALSE
);
```

---

## What Happens?

Depending on table configuration, SQLite may reuse deleted IDs.

Example:

```text
Deleted:
id = 4
```

New record:

```text
id = 4
```

instead of:

```text
id = 5
```

---

> [!info]  
> SQLite's behavior depends on how the primary key was defined.
> 
> A simple:
> 
> ```sql
> INTEGER PRIMARY KEY
> ```
> 
> behaves differently from:
> 
> ```sql
> INTEGER PRIMARY KEY AUTOINCREMENT
> ```
> 
> The latter guarantees IDs are never reused.

# Exiting SQLite

Leave the SQLite shell:

```sql
.quit
```

```sql

```

# Common SQLite Commands

|Command|Purpose|
|---|---|
|`.schema`|Show table definitions|
|`.tables`|Show all tables|
|`.mode table`|Table view|
|`.mode column`|Column view|
|`.mode box`|Box view|
|`.help`|Show available commands|
|`.quit`|Exit SQLite|
