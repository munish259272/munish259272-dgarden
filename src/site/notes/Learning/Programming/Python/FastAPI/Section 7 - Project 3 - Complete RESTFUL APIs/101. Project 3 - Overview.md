# Project 3 Overview: Building a Production-Style FastAPI Application

Now that we have completed **Project 1** and **Project 2**, it's time to move on to **Project 3**, where we will start building a much more realistic FastAPI application.

Instead of working with **Books**, this project will focus on a **Todo Application** and introduce several important concepts used in real-world software development.

---

# What Changes in Project 3?

In the previous projects:

```text
Client
   │
   ▼
FastAPI
```

Data was stored in memory using Python lists.

Example:

```python
books = []
```

This works for learning purposes, but it has several limitations:

❌ Data disappears when the application restarts.

❌ Data cannot be shared across multiple users.

❌ Not suitable for production environments.

---

# Introducing Databases

In Project 3, we will store data inside a database.

New architecture:

```text
Client/Web Browser
         │
         ▼
      FastAPI
         │
         ▼
      Database
```

Now data will persist even if the application stops and restarts.

✅ Data is stored permanently.

✅ Multiple users can access the same data.

✅ Scales much better than in-memory storage.

# Authentication with JWT

One of the biggest additions in Project 3 is **Authentication**.

Authentication answers the question:

> "Who are you?"

---

## Without Authentication

Current system:

```text
Anyone can call the API.
```

Example:

```http
GET /todos
```

No login required.

---

## With Authentication

Users must first log in.

Example:

```text
Username: mukesh
Password: ********
```

After successful login:

```text
JWT Token Generated
```

The token is then sent with future requests.

Architecture:

```text
User
  │
  ▼
Login
  │
  ▼
JWT Token
  │
  ▼
Protected API Endpoints
```

---

# What is JWT?

JWT stands for:

```text
JSON Web Token
```

A JWT is a digitally signed token used to verify a user's identity.

Instead of sending:

```text
Username + Password
```

with every request, the user sends:

```text
JWT Token
```

which proves they have already authenticated.

---

## JWT Workflow

```text
Login Request
      │
      ▼
Username + Password
      │
      ▼
FastAPI
      │
      ▼
JWT Token Created
      │
      ▼
Client Stores Token
      │
      ▼
Future Requests Include Token
```

🎯 This is how modern APIs typically authenticate users.

# Password Hashing

One of the most important security concepts introduced in this project is **password hashing**.

---

## Bad Practice

Never store passwords like this:

|Username|Password|
|---|---|
|mukesh|password123|

If the database is compromised:

```text
Attacker can immediately see all passwords.
```

⚠️ Extremely dangerous.

---

## Good Practice

Store hashed passwords instead:

|Username|Password Hash|
|---|---|
|mukesh|$2b$12$xA...|

Now the original password cannot be easily recovered.

---

## Password Storage Flow

```text
User Password
      │
      ▼
Hash Function
      │
      ▼
Hash Value
      │
      ▼
Database
```

When the user logs in:

```text
Entered Password
       │
       ▼
Hash Again
       │
       ▼
Compare Hashes
```

If hashes match:

```text
Login Successful
```
