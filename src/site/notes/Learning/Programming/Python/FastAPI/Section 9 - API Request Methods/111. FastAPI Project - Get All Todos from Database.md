# Creating a Database-Powered API Endpoint

Now that our database and table have been created and populated with records, let's create our first FastAPI endpoint that reads data directly from the database. This endpoint will return all todos stored in `todos.db`.

---

# Understanding `create_all()`

Earlier, we created the database using:

```python
models.Base.metadata.create_all(bind=engine)
```

This command:

- Checks whether the database and tables already exist.
    
- Creates them if they do not exist.
    

However:

⚠️ It **does not automatically modify existing tables**.

For example, if you later change your model:

```python
class Todos(Base):
    __tablename__ = "todos"

    id = Column(Integer, primary_key=True)
    title = Column(String)

    # Newly added column
    due_date = Column(String)
```

`create_all()` will **not update** the existing table structure.

---

> [!warning]  
> `create_all()` is useful during development, but it is **not a database migration tool**.
> 
> If your table structure changes:
> 
> - Delete and recreate the database (development only)
>     
> - Or use migration tools such as **Alembic** (production environments)
>     

---

# Creating a Database Dependency

Before an API endpoint can query the database, it needs a database connection.

Create a dependency:

```python
from database import SessionLocal

def get_db():
    db = SessionLocal()

    try:
        yield db
    finally:
        db.close()
```

---

# Understanding `yield`

This is one of the most important FastAPI patterns.

```python
def get_db():
    db = SessionLocal()

    try:
        yield db
    finally:
        db.close()
```

Execution flow:

```text
Create DB Session
        │
        ▼
yield db
        │
        ▼
API Endpoint Runs
        │
        ▼
Response Sent
        │
        ▼
db.close()
```

---

## Why Use `yield`?

FastAPI pauses execution at:

```python
yield db
```

and gives the database session to the endpoint.

After the endpoint finishes and the response is sent:

```python
db.close()
```

runs automatically.

---

> [!tip]  
> Think of `yield` as:
> 
> ```text
> "Here is the database connection.
> Use it.
> When you're done,
> I'll automatically clean everything up."
> ```

---

# Why Close Database Connections?

Database connections are limited resources.

Bad practice:

```python
db = SessionLocal()

# Never closed
```

If thousands of requests arrive:

```text
Request 1 → Open Connection
Request 2 → Open Connection
Request 3 → Open Connection
...
```

Eventually:

```text
Too Many Connections
```

---

> [!success]  
> Using:
> 
> ```python
> try:
>     yield db
> finally:
>     db.close()
> ```
> 
> ensures every connection is cleaned up properly.

# Dependency Injection

FastAPI provides a feature called **Dependency Injection**.

Dependency Injection means:

> "Run some code first and provide the result to my endpoint."

Example:

```python
Depends(get_db)
```

means:

```text
Run get_db()
       ↓
Create DB Session
       ↓
Pass Session To Endpoint
```

---

# Creating the Endpoint

```python
@app.get("/")
async def read_all(
    db: Annotated[
        Session,
        Depends(get_db)
    ]
):
    return db.query(Todos).all()
```

---

## Step 1: Open Database Session


calls:

```python
get_db()
```

```python

```

creates a database connection.

## Step 2: Inject Session Into Endpoint

FastAPI automatically provides:

```python
db
```

to the endpoint.

```python

```

means:

> Query the `Todos` table.

## Step 4: Return All Records

```python
.all()
```

means:

Equivalent SQL:


# Avoiding Repeated Dependency Code

This annotation is long:

```python
db: Annotated[
    Session,
    Depends(get_db)
]
```

Since nearly every endpoint needs database access, we can store it in a variable.

---

# Create a Reusable Dependency

```python
DBDependency = Annotated[
    Session,
    Depends(get_db)
]
```

> [!question]- What is `Annotated[Session, Depends(get_db)]` doing? Is it a type annotation for `db`?
> 
> Yes, **it is a type annotation for `db`**, but it does more than a normal type annotation.
> 
> Think of it as combining:
> 
> 1. **The type of the variable**
>     
> 2. **Extra metadata/instructions for FastAPI**
>     
> 
> into a single declaration.
> 
> 
> ---
> 
> # Normal Type Annotation
> 
> Suppose you write:
> 
> ```python
> name: str
> ```
> 
> This tells Python and your IDE:
> 
> ```text
> name is a string
> ```
> 
> Similarly:
> 
> ```python
> db: Session
> ```
> 
> means:
> 
> ```text
> db is a SQLAlchemy Session object
> ```
> 
> ---
> 
> # Problem
> 
> FastAPI needs more information than just:
> 
> ```python
> db: Session
> ```
> 
> It also needs to know:
> 
> ```text
> Where should I get this Session from?
> ```
> 
> That's where:
> 
> ```python
> Depends(get_db)
> ```
> 
> comes in.
> 
> ---
> 
> # Before `Annotated`
> 
> Older FastAPI code looked like:
> 
> ```python
> async def read_all(
>     db: Session = Depends(get_db)
> ):
>     pass
> ```
> 
> Here:
> 
> ```python
> Session
> ```
> 
> provides the type
> 
> and
> 
> ```python
> Depends(get_db)
> ```
> 
> provides dependency information.
> 
> ---
> 
> # Modern Style with `Annotated`
> 
> Instead:
> 
> ```python
> from typing import Annotated
> 
> db: Annotated[
>     Session,
>     Depends(get_db)
> ]
> ```
> 
> FastAPI reads this as:
> 
> ```text
> Type = Session
> 
> Dependency Provider = get_db
> ```
> 
> ---
> 
> # Breaking It Down
> 
> ```python
> Annotated[
>     Session,
>     Depends(get_db)
> ]
> ```
> 
> contains two pieces:
> 
> ---
> 
> ## Part 1
> 
> ```python
> Session
> ```
> 
> means:
> 
> ```text
> The value passed into db
> will be a SQLAlchemy Session
> ```
> 
> ---
> 
> ## Part 2
> 
> ```python
> Depends(get_db)
> ```
> 
> means:
> 
> ```text
> To get that Session,
> call get_db() first
> ```
> 
> ---
> 
> # What FastAPI Sees
> 
> When FastAPI sees:
> 
> ```python
> async def read_all(
>     db: Annotated[
>         Session,
>         Depends(get_db)
>     ]
> ):
> ```
> 
> it internally thinks:
> 
> ```text
> Before calling read_all():
> 
> 1. Run get_db()
> 2. Get the yielded Session
> 3. Store it in db
> 4. Execute read_all()
> 5. Cleanup get_db()
> ```
> 
> ---
> 
> # Visual Flow
> 
> ```text
> HTTP Request
>       │
>       ▼
> Depends(get_db)
>       │
>       ▼
> get_db()
>       │
>       ▼
> yield db
>       │
>       ▼
> db becomes available
>       │
>       ▼
> read_all(db)
>       │
>       ▼
> Response Sent
>       │
>       ▼
> db.close()
> ```
> 
> ---
> 
> # What Does `db` Actually Contain?
> 
> After dependency injection:
> 
> ```python
> db
> ```
> 
> contains something like:
> 
> ```python
> <sqlalchemy.orm.session.Session object>
> ```
> 
> So you can do:
> 
> ```python
> db.query(Todos).all()
> ```
> 
> because:
> 
> ```text
> db is a real SQLAlchemy Session
> ```
> 
> ---
> 
> # Equivalent Manual Code
> 
> Without FastAPI dependency injection:
> 
> ```python
> async def read_all():
>     db = SessionLocal()
> 
>     try:
>         return db.query(Todos).all()
>     finally:
>         db.close()
> ```
> 
> You would have to write this in every endpoint.
> 
> ---
> 
> With:
> 
> ```python
> db: Annotated[
>     Session,
>     Depends(get_db)
> ]
> ```
> 
> FastAPI automatically does:
> 
> ```python
> db = get_db()
> ```
> 
> behind the scenes.
> 
> ---
> 
> # Why Use `Annotated`?
> 
> `Annotated` allows:
> 
> ```python
> Type Information
> +
> Extra Metadata
> ```
> 
> in one place.
> 
> General syntax:
> 
> ```python
> Annotated[
>     ActualType,
>     ExtraInformation
> ]
> ```
> 
> Example:
> 
> ```python
> age: Annotated[
>     int,
>     "Must be positive"
> ]
> ```
> 
> means:
> 
> ```text
> Type = int
> 
> Extra metadata =
> "Must be positive"
> ```
> 
> ---
> 
> # FastAPI Uses the Metadata
> 
> FastAPI ignores:
> 
> ```python
> Session
> ```
> 
> for dependency resolution.
> 
> It looks at:
> 
> ```python
> Depends(get_db)
> ```
> 
> and understands:
> 
> ```text
> This parameter is a dependency.
> ```
> 
> ---
> 
> # Why Create `DBDependency`?
> 
> Instead of repeating:
> 
> ```python
> db: Annotated[
>     Session,
>     Depends(get_db)
> ]
> ```
> 
> everywhere:
> 
> ```python
> DBDependency = Annotated[
>     Session,
>     Depends(get_db)
> ]
> ```
> 
> Now:
> 
> ```python
> async def read_all(
>     db: DBDependency
> ):
> ```
> 
> means exactly the same thing.
> 
> ---
> 
> > [!success]
> > 
> > ### Mental Model
> > 
> > ```python
> > db: Annotated[
> >     Session,
> >     Depends(get_db)
> > ]
> > ```
> > 
> > Read it as:
> > 
> > ```text
> > db will be a Session object.
> > 
> > To obtain that Session,
> > FastAPI must first run get_db().
> > ```
> > 
> > - `Session` → Type of `db`
> >     
> > - `Depends(get_db)` → How FastAPI creates/fetches `db`
> >     
> > - `Annotated` → Combines both into one declaration
> >     
> 
> 🎯 `Annotated[Session, Depends(get_db)]` is essentially a **type annotation plus FastAPI dependency metadata**. It tells FastAPI: "Inject the result of `get_db()` here, and treat the resulting object as a SQLAlchemy `Session`."

# SQL Equivalent

This SQLAlchemy code:

```python

```

is roughly equivalent to:

```sql
SELECT *
FROM todos;
```

SQLAlchemy translates Python code into SQL behind the scenes.
