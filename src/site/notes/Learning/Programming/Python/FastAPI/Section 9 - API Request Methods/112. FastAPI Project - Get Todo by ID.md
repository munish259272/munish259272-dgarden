# Reading a Single Todo by ID

In the previous lesson, we created a **Read All Todos** endpoint that returned every record from the database using dependency injection.

In this lesson, we will create a new endpoint that retrieves **one specific todo** based on its ID. Along the way, we'll introduce:

✅ Database filtering

✅ HTTP exceptions

✅ Explicit status codes

✅ Path parameter validation

---

# Creating the Endpoint

Under the `read_all()` function, create a new endpoint:

```python
@app.get("/todo/{todo_id}")
async def read_todo(
    db: DBDependency,
    todo_id: int
):
```

---

## Understanding the Route

```python
@app.get("/todo/{todo_id}")
```

This creates a dynamic path parameter.

Examples:

```text
/todo/1
/todo/2
/todo/3
```

Here:

```python
todo_id
```

receives the value from the URL.

Example:

```text
GET /todo/2
```

results in:

```python
todo_id = 2
```

---

# Querying the Database

Create a variable:

```python
todo_model = db.query(Todos)
```

This starts a query against the:

```python
Todos
```

table.

Think of it as:

```text
Database
    │
    ▼
Todos Table
    │
    ▼
Start Query
```

---

# Filtering by ID

Add a filter:

```python
todo_model = db.query(Todos).filter(
    Todos.id == todo_id
)
```

---

## SQL Equivalent

This SQLAlchemy query:

```python
db.query(Todos).filter(
    Todos.id == todo_id
)
```

is roughly equivalent to:

```sql
SELECT *
FROM todos
WHERE id = todo_id;
```

Example:

```python
todo_id = 2
```

becomes:

```sql
SELECT *
FROM todos
WHERE id = 2;
```

---

# Why Use a Filter?

Without a filter:

```python
db.query(Todos).all()
```

returns:

```text
Todo 1
Todo 2
Todo 3
Todo 4
...
```

---

With a filter:

```python
db.query(Todos).filter(
    Todos.id == 2
)
```

returns only:

```text
Todo 2
```

---

# Using `.first()`

Add:

```python
todo_model = db.query(Todos).filter(
    Todos.id == todo_id
).first()
```

---

## Why Use `.first()`?

The database doesn't know that:

```python
id
```

is unique.

Without `.first()`:

```python
db.query(Todos).filter(...)
```

returns a query object.

---

With:

```python
.first()
```

SQLAlchemy:

1. Executes the query
    
2. Returns the first matching record
    
3. Stops searching immediately
    

---

### Performance Benefit

Without `.first()`:

```text
Search Record 1
Search Record 2
Search Record 3
Search Record 4
...
```

---

With `.first()`:

```text
Found Match
     ↓
Stop Searching
```

🎯 Since `id` is a primary key, only one match can exist anyway.

---


# Checking for Valid Data

```python
if todo_model is not None:
    return todo_model
```

If a matching record exists:

```text
Return Todo
```

---

If no matching record exists:

```python
None
```

we need to inform the client.

# Raising an HTTP Exception

Import:

```python
from fastapi import HTTPException
```

Then:

```python
raise HTTPException(
    status_code=404,
    detail="Todo not found"
)
```

---

## Why Raise an Exception?

Instead of returning:

```json
null
```

we return a meaningful API error.

Response:

```json
{
  "detail": "Todo not found"
}
```

Status Code:

```text
404 Not Found
```

---

## Why 404?

A 404 means:

> The requested resource does not exist.

Example:

```text
/todo/9999
```

when no such todo exists.

---

# Complete Endpoint

```python
@app.get("/todo/{todo_id}")
async def read_todo(
    db: DBDependency,
    todo_id: int
):
    todo_model = db.query(Todos).filter(
        Todos.id == todo_id
    ).first()

    if todo_model is not None:
        return todo_model

    raise HTTPException(
        status_code=404,
        detail="Todo not found"
    )
```

---

# Being Explicit About Success Responses

Import:

```python
from fastapi import status
```

Then:

```python
@app.get(
    "/todo/{todo_id}",
    status_code=status.HTTP_200_OK
)
```

---

## Why Specify Status Codes Explicitly?

FastAPI already returns:

```text
200 OK
```

by default.

However:

```python
status_code=status.HTTP_200_OK
```

makes the API contract clearer.

Future developers immediately know:

```text
Successful Request
      ↓
Returns HTTP 200
```

---

# Adding Path Parameter Validation

Import:

```python
from fastapi import Path
```

---

Modify:

```python
todo_id: int
```

to:

```python
todo_id: int = Path(gt=0)
```

---

## What Does This Mean?

```python
Path(gt=0)
```

means:

```text
Todo ID must be > 0
```

---

### Valid

```text
1
2
3
100
```

---

### Invalid

```text
0
-1
-10
```

---

# Why Use Validation?

Without validation:

```text
GET /todo/-100
```

would reach your database query.

---

With validation:

```python
Path(gt=0)
```

FastAPI rejects the request before the endpoint executes.

---

# Example Error

Request:

```text
/todo/-1
```

Response:

```json
{
  "detail": [
    {
      "msg": "Input should be greater than 0"
    }
  ]
}
```

Status:

```text
422 Unprocessable Entity
```

---

## Why Is This Better?

Instead of writing:

```python
if todo_id <= 0:
    raise ...
```

for every endpoint,

FastAPI validates automatically.

🎯 Less code and fewer bugs.

---

# Final Version

```python
from fastapi import (
    FastAPI,
    HTTPException,
    Path,
    status
)

@app.get(
    "/todo/{todo_id}",
    status_code=status.HTTP_200_OK
)
async def read_todo(
    db: DBDependency,
    todo_id: int = Path(gt=0)
):
    todo_model = db.query(Todos).filter(
        Todos.id == todo_id
    ).first()

    if todo_model is not None:
        return todo_model

    raise HTTPException(
        status_code=404,
        detail="Todo not found"
    )
```

---
