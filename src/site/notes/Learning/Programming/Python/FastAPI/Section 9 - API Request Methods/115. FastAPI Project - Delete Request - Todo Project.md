# Deleting a Todo Using a DELETE Request

In the previous lessons, we implemented the **Create (POST)**, **Read (GET)**, and **Update (PUT)** operations for our Todo application.

In this lesson, we'll implement the **Delete** operation, completing all four **CRUD** operations.

CRUD stands for:

|Operation|HTTP Method|
|---|---|
|**Create**|`POST`|
|**Read**|`GET`|
|**Update**|`PUT`|
|**Delete**|`DELETE`|

---

# Creating the DELETE Endpoint

Add the following endpoint below your existing API endpoints:

```python
@app.delete(
    "/todo/{todo_id}",
    status_code=status.HTTP_204_NO_CONTENT
)
async def delete_todo():
    pass
```

---

# Why Return HTTP 204?

A successful delete operation usually returns:

```text
204 No Content
```

This indicates:

- ✅ The resource was successfully deleted.
    
- ✅ There is no response body to return.
    

---

# Complete Function Signature

```python
@app.delete(
    "/todo/{todo_id}",
    status_code=status.HTTP_204_NO_CONTENT
)
async def delete_todo(
    db: DBDependency,
    todo_id: int = Path(gt=0)
):
```

This endpoint accepts two parameters:

|Parameter|Purpose|
|---|---|
|`db`|Database session (Dependency Injection)|
|`todo_id`|ID of the todo to delete|

---

# Validate the Path Parameter

```python
todo_id: int = Path(gt=0)
```

This ensures:

- ✅ `todo_id` must be greater than `0`
    
- ❌ Negative numbers or `0` are rejected **before** the endpoint logic executes.
    

For example:

```text
DELETE /todo/-1
```

returns:

```text
422 Unprocessable Entity
```

without executing any of your code.

---

# Step 1: Check Whether the Todo Exists

Before deleting a record, first verify that it exists.

```python
todo_model = (
    db.query(Todos)
      .filter(Todos.id == todo_id)
      .first()
)
```

This query searches for the todo whose ID matches the path parameter.

Internally, SQLAlchemy generates SQL similar to:

```sql
SELECT *
FROM todos
WHERE id = ?
LIMIT 1;
```

---

# Step 2: Handle Missing Records

If no matching todo exists:

```python
if todo_model is None:
    raise HTTPException(
        status_code=404,
        detail="Todo not found"
    )
```

The client receives:

```text
404 Not Found
```

instead of silently pretending the delete succeeded.

---

> [!tip]  
> **Always verify that the resource exists before deleting it.**
> 
> This allows your API to return a meaningful error (`404 Not Found`) rather than reporting success for a record that doesn't exist.

---

# Step 3: Delete the Record

Now perform the delete operation.

```python
db.query(Todos) \
  .filter(Todos.id == todo_id) \
  .delete()
```

This tells SQLAlchemy:

1. Query the `todos` table.
    
2. Find the row whose `id` matches `todo_id`.
    
3. Delete that row.
    

---

## SQL Equivalent

The above SQLAlchemy code is roughly equivalent to:

```sql
DELETE
FROM todos
WHERE id = todo_id;
```

---

# Why Query Again?

You might notice that we already queried the record earlier:

```python
todo_model = (
    db.query(Todos)
      .filter(Todos.id == todo_id)
      .first()
)
```

and then immediately perform another query:

```python
db.query(Todos) \
  .filter(Todos.id == todo_id) \
  .delete()
```

This is the approach used in the course.

An alternative approach is to delete the object directly:

```python
db.delete(todo_model)
```

Both approaches work.

The tutorial simply demonstrates deletion using the **query API**.

---

# Step 4: Commit the Transaction

Deleting the row isn't enough.

You must commit the transaction.

```python
db.commit()
```

Without calling:

```python
db.commit()
```

the deletion only exists within the current database transaction and is **not permanently saved**.

---

# Complete DELETE Endpoint

```python
@app.delete(
    "/todo/{todo_id}",
    status_code=status.HTTP_204_NO_CONTENT
)
async def delete_todo(
    db: DBDependency,
    todo_id: int = Path(gt=0)
):
    todo_model = (
        db.query(Todos)
          .filter(Todos.id == todo_id)
          .first()
    )

    if todo_model is None:
        raise HTTPException(
            status_code=404,
            detail="Todo not found"
        )

    db.query(Todos) \
      .filter(Todos.id == todo_id) \
      .delete()

    db.commit()
```

---

# Request Flow

```text
Client Sends DELETE Request
            │
            ▼
Validate Path Parameter
(todo_id > 0)
            │
            ▼
Query Database
            │
            ▼
Todo Found?
      ┌─────┴─────┐
      │           │
     Yes          No
      │           │
      ▼           ▼
Delete Row    HTTP 404
      │
      ▼
db.commit()
      │
      ▼
Database Updated
      │
      ▼
HTTP 204 No Content
```

# Why Do We Call `commit()`?

Think of SQLAlchemy's **Session** as a notebook where changes are first recorded.

```text
Delete Request
       │
       ▼
Session Records Deletion
       │
       ▼
db.commit()
       │
       ▼
SQLite Permanently Deletes Row
```

Without:

```python
db.commit()
```

the deletion is never permanently written to the database.
