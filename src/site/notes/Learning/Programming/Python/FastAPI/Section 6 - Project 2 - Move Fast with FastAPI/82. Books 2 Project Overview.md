# 📖 Representing Books as Real Python Objects

Unlike Project 1, where books may have been represented using simple dictionaries or lists, Project 2 introduces a dedicated **Book class**.

This means every book will become an actual Python object.

---

## 🏗️ Book Class Structure

The `Book` class will contain the following attributes:

|Attribute|Data Type|Description|
|---|---|---|
|`id`|`int`|Unique identifier for the book|
|`title`|`str`|Book title|
|`author`|`str`|Author name|
|`description`|`str`|Book description|
|`rating`|`int`|Book rating|

Example structure:

```python
class Book:
    id: int
    title: str
    author: str
    description: str
    rating: int
```

---

## ⚙️ Constructor

The class will also contain a constructor (`__init__`) that initializes all book properties.

```python
class Book:
    def __init__(
        self,
        id,
        title,
        author,
        description,
        rating
    ):
        self.id = id
        self.title = title
        self.author = author
        self.description = description
        self.rating = rating
```

Now, creating a book becomes very easy:

```python
book = Book(
    1,
    "Python Basics",
    "John Doe",
    "Introduction to Python",
    5
)
```

After passing the values, you immediately get a fully initialized **Book object**.

---

# 🏢 Real-World Analogy

Think of the `Book` class as a **blueprint** for books.

🏗️ **Blueprint (Class)**

```text
Book
├── id
├── title
├── author
├── description
└── rating
```

📚 **Actual Book Object**

```text
Book(
    id=1,
    title="Python Basics",
    author="John Doe",
    description="Introduction to Python",
    rating=5
)
```

The class defines what a book should look like, while each object is an actual book instance created from that blueprint.

# 🔄 CRUD Operations

One of the primary goals of this project is to reinforce **CRUD operations**, which are fundamental to almost every API.

> [!important]  
> **CRUD is the foundation of most web applications and APIs.**
> 
> Whether you're building an e-commerce site, banking application, social media platform, or inventory system, CRUD operations are everywhere.

---

## CRUD Breakdown

|CRUD Operation|Meaning|HTTP Method|
|---|---|---|
|**C**|Create|POST|
|**R**|Read|GET|
|**U**|Update|PUT|
|**D**|Delete|DELETE|

---

### 🟢 Create

Creates a new resource.

```http
POST /books
```

Example:

```json
{
    "title": "Python Basics",
    "author": "John Doe"
}
```

---

### 🔵 Read

Retrieves existing data.

```http
GET /books
```

Example:

```text
Return all books
```

---

### 🟡 Update

Modifies an existing resource.

```http
PUT /books/1
```

Example:

```json
{
    "title": "Advanced Python"
}
```

---

### 🔴 Delete

Removes an existing resource.

```http
DELETE /books/1
```

Example:

```text
Delete book with ID = 1
```
