## Adding a `publish_date` Field and Filtering Books by Publish Date

In this enhancement, we will:

1. Add a new field called **`publish_date`** to both the `Book` and `BookRequest` classes.
    
2. Update our existing data model to store publication years.
    
3. Add validation for publication dates.
    
4. Create a new **GET endpoint** that allows users to filter books by publication year.
    
5. Verify that all existing CRUD operations continue to work correctly.
    

---

# Step 1: Add `publish_date` to the Book Class

We first extend our `Book` model by adding a new attribute:

```python
class Book:
    id: int
    title: str
    author: str
    description: str
    rating: int
    publish_date: int
```

### Update the Constructor

Since every book should now store its publication year, we must also update the constructor:

```python
def __init__(
    self,
    id,
    title,
    author,
    description,
    rating,
    publish_date
):
    self.id = id
    self.title = title
    self.author = author
    self.description = description
    self.rating = rating
    self.publish_date = publish_date
```

📌 A value like:

```python
publish_date = 2012
```

means the book was published in the year **2012**.

---

# Step 2: Add `publish_date` to the Request Model

The API should also accept publication years when creating or updating books.

```python
publish_date: int
```

### Add Validation

We can use Pydantic validation to ensure only valid years are accepted:

```python
publish_date: int = Field(
    gt=1999,
    lt=2031
)
```

### What Does This Mean?

|Validation|Meaning|
|---|---|
|`gt=1999`|Year must be greater than 1999|
|`lt=2031`|Year must be less than 2031|

Therefore:

✅ Allowed:

```python
2025
2030
```

❌ Not Allowed:

```python
1999
2031
2037
```

> [!tip]  
> Field validation helps catch bad data **before it reaches your application logic**.

# Step 6: Create a New Endpoint to Filter by Publish Date

Now let's create a new GET endpoint.

```python
@app.get("/books/publish/{publish_date}")
async def read_book_by_publish_date(
    publish_date: int
):
    books_to_return = []

    for book in books:
        if book.publish_date == publish_date:
            books_to_return.append(book)

    return books_to_return
```

---

## Understanding the Logic

### Create an Empty List

```python
books_to_return = []
```

This list will store matching books.

---

### Loop Through Every Book

```python
for book in books:
```

---

### Check the Publish Date

```python
if book.publish_date == publish_date:
```

If the book's publication year matches the requested year, add it to the result.

---

### Add Matching Books

```python
books_to_return.append(book)
```

---

### Return Results

```python
return books_to_return
```

The API returns all books published in that year.

# Testing Validation

Let's intentionally provide an invalid year:

```json
{
    "publish_date": 2037
}
```

FastAPI returns a validation error because:

```python
Field(
    gt=1999,
    lt=2031
)
```

requires:

$$\large{1999 < publish_date < 2031}$$

Therefore:

```python
2037
```

fails validation.

⚠️ **The request never reaches your endpoint because Pydantic blocks invalid data first.**
