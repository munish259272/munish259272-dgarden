# Creating a GET Endpoint to Retrieve a Book by ID

So far, our FastAPI application contains:

- A `Book` class representing book objects.
    
- A `BookRequest` Pydantic model used for request validation.
    
- Schema configuration and field validation.
    
- A `POST` endpoint that converts a `BookRequest` into a `Book` object and adds it to the list.
    

Now let's create another API endpoint that allows us to **retrieve a specific book using its ID**.

---

# Creating the Endpoint

Add a new GET endpoint:

```python
@app.get("/books/{book_id}")
async def read_book(book_id: int):
    for book in books:
        if book.id == book_id:
            return book
```

---

# Understanding the Route

```python
"/books/{book_id}"
```

The curly braces indicate a **path parameter**.

For example:

```text
/books/1
/books/2
/books/3
```

FastAPI automatically extracts the value from the URL and stores it in:

```python
book_id
```

---

# Path Parameter Type Validation

```python
book_id: int
```

FastAPI automatically converts the value to an integer.

Example:

```text
/books/4
```

becomes:

```python
book_id = 4
```

---

> [!tip]  
> FastAPI performs automatic validation on path parameters.
> 
> If a user sends:
> 
> ```text
> /books/abc
> ```
> 
> FastAPI returns a validation error because `"abc"` cannot be converted to an integer.

---

# How the Function Works

The endpoint loops through all books:

```python
for book in books:
```

Then checks:

```python
if book.id == book_id:
```

If a match is found:

```python
return book
```

The matching book is immediately returned to the client.

# How FastAPI Sees This Request

When a user visits:

```text
/books/4
```

FastAPI internally does something similar to:

```python
read_book(book_id=4)
```

The value `4` is extracted from the URL and passed to the function automatically.

# ✅ Key Takeaways

- `@app.get("/books/{book_id}")` creates an endpoint with a **path parameter**.
    
- `{book_id}` is extracted from the URL automatically.
    
- `book_id: int` ensures FastAPI validates the parameter as an integer.
    
- The endpoint searches through the `books` list.
    
- If a matching ID is found, the corresponding book is returned.
    
- This allows clients to retrieve **one specific book instead of the entire list**.
    
- Path parameters are one of the most common ways to identify individual resources in REST APIs.
