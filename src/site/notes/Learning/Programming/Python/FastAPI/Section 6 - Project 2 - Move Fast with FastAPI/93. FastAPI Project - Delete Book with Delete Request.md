# Implementing a DELETE Request in FastAPI

In the previous section, we created a **PUT request** to update books. Now we'll implement a **DELETE request** that removes a book from our in-memory list.

## Creating the DELETE Endpoint

We define a DELETE route and pass the book ID as a **path parameter**:

```python
@app.delete("/books/{book_id}")
async def delete_book(book_id: int):
    for i in range(len(books)):
        if books[i].id == book_id:
            books.pop(i)
            break
```

### How It Works

📌 **Step-by-step process:**

1. The client sends a DELETE request to:
    
    ```text
    /books/3
    ```
    
2. FastAPI extracts the value `3` and stores it in the parameter:
    
    ```python
    book_id
    ```
    
3. The function loops through all books:
    
    ```python
    for i in range(len(books)):
    ```
    
4. It checks whether the current book's ID matches the requested ID:
    
    ```python
    if books[i].id == book_id:
    ```
    
5. When a match is found, the book is removed using:
    
    ```python
    books.pop(i)
    ```
    
6. The loop stops immediately:
    
    ```python
    break
    ```

## Visual Example

Assume our list contains:

|ID|Title|
|---|---|
|1|Book A|
|2|Book B|
|3|Book C|
|4|Book D|

A request is sent:

```http
DELETE /books/3
```

The loop finds:

```python
books[2].id == 3
```

Then:

```python
books.pop(2)
```

The list becomes:

|ID|Title|
|---|---|
|1|Book A|
|2|Book B|
|4|Book D|

✅ **Book C has been deleted.**

At this point, the application supports the major CRUD operations:

✅ We now have working examples of **GET, POST, PUT, and DELETE** endpoints in FastAPI.

> [!tip]  
> Think of the `books` list as a small temporary database stored in memory. The DELETE request searches for a matching record and removes it, just like deleting a row from a database table. Later, when working with real databases, the logic will be similar, but instead of using `books.pop(i)`, you'll execute a database delete query.
