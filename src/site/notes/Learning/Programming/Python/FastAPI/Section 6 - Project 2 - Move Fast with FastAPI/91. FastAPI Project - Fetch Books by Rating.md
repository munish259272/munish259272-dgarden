# Filtering Books by Rating Using Query Parameters in FastAPI

In this example, we will create a **new API endpoint that filters books based on their rating**.

📌 **Goal:** Allow users to request only the books that have a specific rating.

For example:

```http
/books/?book_rating=5
```

should return all books whose rating is **5**.

---

## Creating the Endpoint

```python
@app.get("/books/")
async def read_book_by_rating(book_rating: int):
    books_to_return = []

    for book in books:
        if book.rating == book_rating:
            books_to_return.append(book)

    return books_to_return
```

---

## Understanding the Code

### 1. Query Parameter

```python
book_rating: int
```

FastAPI automatically treats this as a **query parameter** because it is not part of the URL path.

Example request:

```http
/books/?book_rating=5
```

Here:

```text
book_rating = 5
```

---

### 2. Create an Empty List

```python
books_to_return = []
```

This list will store all books whose rating matches the requested rating.

---

### 3. Loop Through All Books

```python
for book in books:
```

Iterate through every book in the collection.

---

### 4. Check the Rating

```python
if book.rating == book_rating:
```

Compare:

- Book's actual rating
    
- Rating supplied by the user
    

If they match, add the book to the result list.

---

### 5. Add Matching Books

```python
books_to_return.append(book)
```

Store matching books in the return list.

---

### 6. Return the Results

```python
return books_to_return
```

FastAPI automatically converts the list into JSON.

# Query Parameter vs Path Parameter

This example highlights an important FastAPI concept.

## Query Parameter

```http
/books/?book_rating=5
```

- Value comes **after `?`**
    
- Used mainly for **filtering, searching, sorting, pagination**
    
- Does **not** affect route matching
    

---

## Path Parameter

```http
/books/5
```

- Value is part of the URL path
    
- Usually used to identify a **specific resource**
    
- Route matching depends on it
    

Example:

```python
@app.get("/books/{book_id}")
async def read_book(book_id: int):
    ...
```

Request:

```http
/books/5
```

means:

```text
book_id = 5
```

---

## Why There Is No Conflict Here

Consider these two requests:

### Path Parameter

```http
/books/5
```

FastAPI interprets:

```text
book_id = 5
```

---

### Query Parameter

```http
/books/?book_rating=5
```

FastAPI interprets:

```text
book_rating = 5
```

Even though both contain the value **5**, they are completely different routes because one uses a **path parameter** and the other uses a **query parameter**.

✅ Therefore, they do not interfere with each other.

---

## When Could a Conflict Occur?

Suppose we define:

```python
@app.get("/books/{book_id}")
```

and

```python
@app.get("/books/{book_rating}")
```

Now FastAPI cannot determine whether:

```http
/books/5
```

means:

```text
book_id = 5
```

or

```text
book_rating = 5
```

⚠️ This creates route ambiguity.

That is why **path parameters with identical URL patterns can conflict with each other**.

---

## Important Rule About Route Order

🎯 **FastAPI evaluates routes from top to bottom.**

If two path routes could potentially match the same URL, the route declared first gets matched first.

Example:

```python
@app.get("/books/mybook")
```

should be placed before:

```python
@app.get("/books/{book_id}")
```

Otherwise:

```http
/books/mybook
```

may be incorrectly interpreted as:

```text
book_id = "mybook"
```

---

> [!tip]  
> **Use Path Parameters when identifying a specific resource.**
> 
> Example:
> 
> ```http
> /books/5
> ```
> 
> Use Query Parameters when filtering or searching.
> 
> Example:
> 
> ```http
> /books/?book_rating=5
> ```

## Analogy

Imagine a library:

- **Path Parameter** → "Give me Book #5."
    
- **Query Parameter** → "Show me all books with a rating of 5."
    

The first identifies **one specific book**, while the second filters **many books** based on a condition.

✅ **Path Parameters identify resources.**  
✅ **Query Parameters filter resources.**  
✅ **Query Parameters do not conflict with Path Parameters because they are processed differently by FastAPI.**
