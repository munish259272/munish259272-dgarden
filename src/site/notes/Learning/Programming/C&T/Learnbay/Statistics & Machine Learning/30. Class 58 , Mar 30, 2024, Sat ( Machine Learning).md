[[.Unit 5 - Exploring bivariate numerical data#c) <span style="color FF0000">IMPORTANT</span> [R-squared or coefficient of determination (video) Khan Academy](https //www.khanacademy.org/math/statistics-probability/describing-relationships-quantitative-data/assessing-the-fit-in-least-squares-regression/v/r-squared-or-coefficient-of-determination)\|.Unit 5 - Exploring bivariate numerical data#c) <span style="color FF0000">IMPORTANT</span> [R-squared or coefficient of determination (video) Khan Academy](https //www.khanacademy.org/math/statistics-probability/describing-relationships-quantitative-data/assessing-the-fit-in-least-squares-regression/v/r-squared-or-coefficient-of-determination)]]
a) $R^2$ also called regression coefficient or coefficient of regression
 - $\Large{0 \le R^{2} \le 1}$
 - higher the $R^2$ the better fit is the regression line
 $$\large{R^{2} = \frac{\text{Explained variance}} {\text{Total variance}} = \frac{\text{Total variance} - \text{Unxplained variance}} {\text{Total variance}}}$$

 Total error --> Unexplained variance
 Total difference from the mean --> Total variance
This implies,
 $$
 \large\bbox[white, 3pt, border: 1px solid black]{
 \begin{align*} \\
 R^2 &= \frac{\text{Explained variance}}{\text{Total variance}} \\ 
     &= \frac{\text{Total variance} - \text{Unexplained variance}}{\text{Total variance}} \\
     &= 1 - \frac {\sum(y - {\hat{y}})^{2}} {\sum(y - {\bar{y}})^{2}}\\
     &= 1 - \frac {\text{SSR}} {\text{SST}}
\end{align*}
 }
$$
SSR --> sum of square residuals
SST --> sum of square of totals

### At --start=1:12:53 --end=1:25:19<span style="color:#FF0000">There is a problem with $R^2$</span> --> If you increase the number of predictors , my $R^2$ will increase

So, instead of $R^2$ we use  $adjusted\ R^2$. It penalizes the model with every addition of the variable
$$
\large\bbox[white, 3pt, border: 1px solid black]{
\begin{align*}
adj\ R^{2} &= 1 - \frac {\frac{\text{Unexplained variance}}{df_e}} {\frac{\text{total variance}}{df_t}}
\end{align*}
}
$$
if the penalty > explained variance ==> $adjusted\ R^2$ will decrease
if the penalty < explained variance ==> $adjusted\ R^2$ will increase
so, we need to surpass the penalty imposed so that the difference, $adjusted\ R^{2} - \text{penalty}$ is +ve.
<span style="color:#FF0000">Note</span> : $\Large{adjusted\ R^{2} \le R^{2}}$ 

### At --start=1:25:19 --end=02:07:25 Assumptions of linear regression

1) **<span style="font-weight:bold; color:#000000">Linearity</span>** : The relationship between independent and dependent variable is linear
2) **<span style="font-weight:bold; color:#000000"> Independence of error</span>** : The residuals between the observed value and the predicted value should be independent of each other. ( i.e there is no pattern in error ( Ex. : My model is able to predict the prices of smaller houses but not the larger ones , this should not be the case))
3) **<span style="font-weight:bold; color:#000000"> Variance of the error should be constant (homoscedasticity)</span>** : Variance of errors is constant across all levels of independent variables. ( spread of residuals/errors should be consistent through the range of independent variables)
4) **<span style="font-weight:bold; color:#000000">Normality of errors</span>** : Errors are normally distributed
5) **<span style="font-weight:bold; color:#000000">No multicolinearity</span>** : There should not be any perfect relation between the independent variables. ( we can detect it using corelation matrix)
	- **How to detect multicolinearity**
		1) corelation matrix (Independent vs Independent)
		2) Variance inflation factor (VIF) - If there is multicolinearity, variance increases. VIF tells us how the variance of an estimated regression coeff. increases if your predictors are correlated 
			$$\Large{VIF = \frac {1} {1 - R^{2}}} \in [1, \infty)$$
			 Thumb rule is if $VIF \gt 5$ then there is multicolinearity
	- **Treatment**
		 1) drop one of the variables
		 2) feature engineer

> [!question]- what is get_dummies in pandas ?
> 
> `get_dummies()` in pandas is a **feature encoding utility** used to convert **categorical variables → numerical format** using **one-hot encoding**.
> 
> ---
> 
> # 🎯 Why `get_dummies()` exists
> 
> Most ML models (e.g., linear regression, neural networks) **cannot work with strings/categories directly**.
> 
> Example:
> 
> ```
> Color = ["Red", "Blue", "Green"]
> ```
> 
> Models need:
> 
> ```
> Red   Blue   Green
> 1     0      0
> 0     1      0
> 0     0      1
> ```
> 
> 👉 That transformation is exactly what `get_dummies()` does.
> 
> ---
> 
> # 🧠 Core Idea: One-Hot Encoding
> 
> Each **unique category → new column**
> 
> Value becomes:
> 
> - `1` → if present
>     
> - `0` → if not
>     
> 
> ---
> 
> # 🔧 Basic Usage
> 
> ```python
> import pandas as pd
> 
> df = pd.DataFrame({
>     "Color": ["Red", "Blue", "Green", "Blue"]
> })
> 
> encoded = pd.get_dummies(df)
> print(encoded)
> ```
> 
> ### Output:
> 
> ```
>    Color_Blue  Color_Green  Color_Red
> 0           0            0          1
> 1           1            0          0
> 2           0            1          0
> 3           1            0          0
> ```
> 
> ---
>

>

> 
> Specify which columns to encode
> 
> ```python
> pd.get_dummies(df, columns=["Color"])

> [!question]- What is `drop_first=True` in pandas ?
> 
> When you are preparing data for machine learning, specifically using **One-Hot Encoding** via `pd.get_dummies()`, the `drop_first=True` parameter is used to prevent a statistical trap called **Multicollinearity**.
> 
> Essentially, it removes the first category of each encoded feature to ensure the remaining columns are linearly independent.
> 
> ---
> 
> ## Why Use It? (The Dummy Variable Trap)
> 
> If you have a categorical variable like "Season" with four options (Spring, Summer, Fall, Winter), One-Hot Encoding typically creates four columns. However, if you know the values of three columns, the fourth is mathematically predictable.
> 
> For example, if it is **not** Spring, **not** Summer, and **not** Fall, it _must_ be Winter. Including all four columns creates a perfect relationship between them, which can break certain models (like Linear or Logistic Regression) because the input variables are no longer independent.
> 
> ### A Simple Comparison
> 
> Imagine a column called **Gender** with values `Male` and `Female`.
> 
> **1. With `drop_first=False` (Default)**
> 
> You get two columns. This is redundant.
> 
> | Male | Female |
> | :--- | :--- |
> | 1 | 0 |
> | 0 | 1 |
> 
> **2. With `drop_first=True`**
> 
> The "Female" column (the first alphabetically) is dropped. We only need one column to represent two states.
> 
> | Male |
> | :--- |
> | 1 | (This means Male) |
> | 0 | (This means Female) |
> 
> ---
> 
> ## Key Benefits
> 
> - **Reduces Dimensionality:** It keeps your dataset smaller by removing unnecessary columns.
>     
> - **Avoids the "Dummy Variable Trap":** This is crucial for models that rely on matrix inversion or assume no multicollinearity.
>     
> - **Cleaner Statistics:** It allows for a "baseline" or "reference" category. In the example above, "Female" becomes the baseline, and the coefficient for "Male" tells you the effect of being male _relative_ to being female.
>     
> 
> ## When to Skip It
> 
> While it's standard for **Linear Models**, you usually **don't** need `drop_first=True` for:
> 
> - **Tree-based models** (Random Forest, XGBoost), which handle redundancy just fine.
>     
> - **Neural Networks**, where extra information doesn't typically cause the same mathematical instability.
>     
>

> [!question]- What is the purpose of using `random_state` in `X_train, X_test, y_train, y_test = train_test_split(X, y, test_size = 0.2, train_size=0.8, random_state = 10)` ?
> 
> ### ✅ Purpose of `random_state` in `train_test_split`
> 
> ```python
> X_train, X_test, y_train, y_test = train_test_split(
>     X, y, test_size=0.2, train_size=0.8, random_state=10
> )
> ```
> 
> ---
> 
> ## 🎯 Core Idea
> 
> > `random_state` controls the **randomness of the data split**.
> 
> ---
> 
> ## 🧠 Why Do We Need It?
> 
> When you call:
> 
> ```python
> train_test_split(X, y)
> ```
> 
> 👉 The data is **shuffled randomly** before splitting.
> 
> That means:
> 
> - Every run → **different train/test split**
>     
> - Different split → **different model performance**
>     
> 
> ---
> 
> ## ⚠️ Problem Without `random_state`
> 
> ```python
> # Run 1
> R² = 0.82
> 
> # Run 2
> R² = 0.76
> 
> # Run 3
> R² = 0.80
> ```
> 
> 👉 You lose:
> 
> - Reproducibility
>     
> - Debugging consistency
>     
> - Fair comparison between models
>     
> 
> ---
> 
> ## ✅ What `random_state` Does
> 
> ```python
> random_state=10
> ```
> 
> 👉 Fixes the **seed of the random number generator**
> 
> So:
> 
> - Same shuffle every time
>     
> - Same split every time
>     
> - Same results every time
>     
> 
> ---
> 
> ## 🔬 Technical Insight
> 
> Internally:
> 
> - Uses a **pseudo-random number generator (PRNG)**
>     
> - `random_state` = seed for that generator
>     
> 
> > Same seed → same sequence → same split
> 
> ---
> 
> ## 🧪 Analogy
> 
> Think of shuffling a deck of cards:
> 
> - Without seed → shuffle differently every time 🎲
>     
> - With seed → **exact same shuffle every time** 🔁
>     
> 
> ---
> 
> ## 📌 Important Behavior
> 
> ### ✔ Same `random_state`
> 
> ```python
> train_test_split(..., random_state=42)
> ```
> 
> 👉 Always produces identical split
> 
> ---
> 
> ### ❌ Different `random_state`
> 
> ```python
> random_state=1 vs random_state=100
> ```
> 
> 👉 Different splits → different results
> 
> ---
> 
> ## ⚠️ Subtle but Important
> 
> > `random_state` does NOT improve your model
> 
> It only ensures:
> 
> - **Reproducibility**
>     
> - **Experiment consistency**
>     
> 
> ---
> 
> ## 🧠 When Should You Use It?
> 
> |Scenario|Use `random_state`?|
> |---|---|
> |Learning / debugging|✅ YES|
> |Comparing models|✅ YES|
> |Production pipeline|⚠️ Optional (depends on workflow)|
> 
> ---
> 
> ## 🔥 Pro Insight (Important for ML practice)
> 
> If you don’t fix `random_state`, you might:
> 
> - Think model improved ❌
>     
> - But actually data split just got easier ✔
>     
> 
> 👉 This leads to **false conclusions**
> 
> ---
> 
> ## 🎯 Final Answer
> 
> > `random_state` ensures that the **train-test split is reproducible** by fixing the randomness used in shuffling the data.
>

30.class - Class 58 , Mar 30, 2024 : What is $R^2$ and $\text{adjusted }R^2$ ? #flashcard 
a) $R^2$ also called regression coefficient or coefficient of regression
 - $\Large{0 \le R^{2} \le 1}$
 - higher the $R^2$ the better fit is the regression line
 $$\Large{R^{2} = \frac{\text{Explained variance}} {\text{Total variance}} = \frac{\text{Total variance} - \text{Unxplained variance}} {\text{Total variance}}}$$
 Total error --> Unexplained variance
 Total difference from the mean --> Total variance
This implies,
 $$
 \Large{
 \begin{align*} \\
 R^2 &= \frac{\text{Explained variance}}{\text{Total variance}} \\ 
     &= \frac{\text{Total variance} - \text{Unexplained variance}}{\text{Total variance}} \\
     &= 1 - \frac {\sum(y - {\hat{y}})^{2}} {\sum(y - {\bar{y}})^{2}}\\
     &= 1 - \frac {\text{SSR}} {\text{SST}}
\end{align*}
 }
$$
SSR --> sum of square residuals
SST --> sum of square of totals
<span style="color:#FF0000">There is a problem with $R^2$</span> --> If you increase the number of predictors , my $R^2$ will increase
So, instead of $R^2$ we use  $adjusted\ R^2$. It penalizes the model with every addition of the variable
$$
\large{
\begin{align*}
adj\ R^{2} &= 1 - \frac {\frac{\text{Unexplained variance}}{df_e}} {\frac{\text{total variance}}{df_t}}
\end{align*}
}
$$
if the penalty > explained variance ==> $adjusted\ R^2$ will decrease
if the penalty < explained variance ==> $adjusted\ R^2$ will increase
so, we need to surpass the penalty imposed so that the difference, $adjusted\ R^{2} - \text{penalty}$ is +ve.
<span style="color:#FF0000">Note</span> : $\Large{adjusted\ R^{2} \le R^{2}}$

30.class - Class 58 , Mar 30, 2024 : What are the assumptions of linear regression and also How to detect multicolinearity ? #flashcard 
Assumptions of linear regression
1) **<span style="font-weight:bold; color:#000000">Linearity</span>** : The relationship between independent and dependent variable is linear
2) **<span style="font-weight:bold; color:#000000"> Independence of error</span>** : The residuals between the observed value and the predicted value should be independent of each other. ( i.e there is no pattern in error ( Ex. : My model is able to predict the prices of smaller houses but not the larger ones , this should not be the case))
3) **<span style="font-weight:bold; color:#000000"> Variance of the error should be constant (homoscedasticity)</span>** : Variance of errors is constant across all levels of independent variables. ( spread of residuals/errors should be consistent through the range of independent variables)
4) **<span style="font-weight:bold; color:#000000">Normality of errors</span>** : Errors are normally distributed
5) **<span style="font-weight:bold; color:#000000">No multicolinearity</span>** : There should not be any perfect relation between the independent variables. ( we can detect it using corelation matrix)
	- **How to detect multicolinearity**
		1) corelation matrix (Independent vs Independent)
		2) Variance inflation factor (VIF) - If there is multicolinearity, VIF increases. VIF tells us how the variance of an estimated regression coeff. increases if your predictors are correlated 
			$$\Large{VIF = \frac {1} {1 - R^{2}}} \in [1, \infty)$$
			 Thumb rule is if $VIF \gt 5$ then there is multicolinearity
	- **Treatment**
		 1) drop one of the variables
		 2) feature engineer
