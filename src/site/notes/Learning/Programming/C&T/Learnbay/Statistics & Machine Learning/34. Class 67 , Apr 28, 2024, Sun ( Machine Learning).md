We need to transform the data into higher dimension so that the data becomes separable but We use kernel trick because actually transforming data from one dimension to another might be computationally costly.

> > [!question]- Does doing <span style="color:#000000">$f(x)=x^2$</span> changes the data(x) into 2d ?
> > The function <span style="color:#000000">$f(x)=x^2$</span> doesn't directly change the data <span style="color:#000000">$x$</span> into 2D. Instead, it transforms the input $x$ into a new value by squaring it. So if you have a set of data points represented by <span style="color:#000000">$x$</span>, applying the function <span style="color:#000000">$f(x)=x^2$</span> would transform each data point x into its square 2x2.
> > 
> > For example, if your original data points were <span style="color:#000000">$x=[1,2,3,4]$</span>, applying the function would result in <span style="color:#000000">$f(x)=[1,4,9,16]$</span>.
> > 
> > However, this doesn't necessarily mean the data becomes 2D. It simply means the data has been transformed according to the function. If you were plotting this data, you would still plot it on a 1D line, with the x-values representing the original data and the y-values representing the output of the function <span style="color:#000000">$f(x)$</span>.
>
> > [!question]- So, how does this type of conversion works in SVM(support vector machines) which helps to separate data?
> > Support Vector Machines (SVMs) are a type of supervised learning model used for classification and regression tasks. In the context of classification, SVMs work by finding the hyperplane that best separates the data points of different classes in a high-dimensional space.
> > 
> > When you apply a transformation like <span style="color:#000000">$f(x)=x^2$</span> to your data before feeding it into an SVM, you're essentially mapping your original data points into a higher-dimensional space. This process is called "kernel trick" or "kernelization".
> > 
> > In SVM, the choice of kernel function determines how the data is transformed. A kernel function computes the dot product between two data points in the transformed space without explicitly calculating the transformation itself. Common kernel functions include linear kernel, polynomial kernel, radial basis function (RBF) kernel, etc.
> > 
> > For instance, if you have a 1D dataset that is not linearly separable, applying a transformation like <span style="color:#000000">$f(x)=x^2$</span> can map it to a 2D space where it might become linearly separable. Then, by finding the optimal hyperplane in this higher-dimensional space, the SVM can effectively separate the data points into their respective classes.
> > 
> > So, the transformation of data through functions like <span style="color:#000000">$f(x)=x^2$</span> in SVM allows the algorithm to find complex decision boundaries in higher-dimensional spaces, making it possible to separate non-linearly separable data.

> > 
> > Here's a breakdown:
> > 
> > 1. **Original Data**: Suppose you have a one-dimensional dataset, meaning each data point is represented by a single feature. For example, your data might look like this: <span style="color:#000000">$x=[1,2,3,4]$</span>.
> >     
> > 2. **Transformation Function**: You apply a transformation function <span style="color:#000000">$f(x)=x^2$</span> to each data point in your dataset. After this transformation, your data becomes <span style="color:#000000">$f(x)=[1,4,9,16]$</span>.
> >     
> > 3. **Feature Space**: While your original data was one-dimensional (each data point had only one feature), the transformed data now has two dimensions. <span style="color:#000000">Each data point in the transformed space is represented by two features: the original x value and its square 2x2. So, in a sense, you've created a new feature space where each data point is represented by two dimensions</span>.
> >     
> > 4. **Visualization**: When we say the data becomes "2D," we mean that you can now visualize it in a two-dimensional space, with one axis representing the original x values and the other axis representing the squared values 2x2. This doesn't mean the original data itself has changed its dimensionality; it's just a way of visualizing the transformed data in a 2D plane.
> >     
> > 5. **SVM and Separation**: When you use an SVM on this transformed data, it operates in this new feature space. The SVM algorithm finds the hyperplane that best separates the data points in this higher-dimensional space. This separation might be easier to achieve compared to the original one-dimensional space, especially if the data becomes linearly separable after the transformation.

> > 
> > So, to summarize, the original data doesn't become 2D itself, but it's transformed into a higher-dimensional feature space where it can be effectively separated by an SVM. 1) **<span style="font-weight:bold; color:#000000">Linear kernel</span>** : simplest kernel which does dot product in the input space, it works well whenever your data can be separated by a line.
$$\Large{k(x,y) = x.y}$$ . A line will be a plane/hyperplane in higher dimension.
2) **<span style="font-weight:bold; color:#000000">Polynomial kernel</span>** : It allows for non-linear boundaries by considering the power of dot product.
$$\Large{k(x,y) = {(x.y + c)}^d}$$
Where, $d$ is degree of polynomial and $c$ is a constant
3) **<span style="font-weight:bold; color:#000000">Radial basis function(RBF) kernel</span>** : It is used for complex non linear separation.It calculates the similarity(nearest) between points in a way that spreads out in all directions from the support vectors.
$$\Large{k(x,y) = {e^{-\frac{1}{2}\frac{{(x_{1} -x_{2})}^{2}} {\sigma}}} = {e^{-\gamma{{(x - y)}^{2}} }}}$$
Where, $\gamma$ denotes the width of the kernel. It is like creating a bubble or group or nearest or similar points with a width
<span style="color:#FF0000">Note</span> : <span style="color:#000000">Highest power</span> of a polynomial is called <span style="color:#000000">degree of a polynomial</span>

SVM in regression is called SVR(support vector regressor)
SVR creates a line called hyperplane for the higher dimensions that best fits the data with a goal to keep maximum amount of data within defined margin.
$\large{\epsilon}$ is used to denote defined margin
If my data point lies within $\large{\epsilon}$ then no penalty is applied to that applied to that data point or else <span style="color:#000000">penalty is applied based on how far outside it is</span>.
**<span style="font-weight:bold; color:#000000">Default value of $\large{\epsilon}$</span>** : 0.1
