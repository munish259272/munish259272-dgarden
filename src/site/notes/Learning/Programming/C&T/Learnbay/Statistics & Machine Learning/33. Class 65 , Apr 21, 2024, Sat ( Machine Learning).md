> [!explanation]-
> 
> In K-Nearest Neighbors (KNN), a new data point is classified based on the majority class of its nearest neighbors.
> 
> Mathematically, all points are considered neighbors, but we do not use all points. Instead, we select a specific number of points, typically denoted as \(k\). For example, we might choose \(k = 3\) or \(k = 5\).
> 
> ## Key Concepts
> 
> 1. **Number of Points to Choose:**
>    Choosing the right value of \(k\) is crucial. The concept involves selecting a number that balances the classification accuracy.
> 
> 2. **Distance Calculation:**
>    To classify a new data point, we need to calculate distances from this point to all other points in the dataset. This requires a reference point.
> 
> ## Steps Involved
> ![33. Class 65 , Apr 21, 2024, Sat ( Machine Learning)-20240524090851259.png\|right\|600](/img/user/attachments/33.%20Class%2065%20,%20Apr%2021,%202024,%20Sat%20(%20Machine%20Learning)-20240524090851259.png)
> 1. **Assume Data:**
>    Let's assume our dataset looks like this:
> 
> | S-No. | Distance (d) | Class |
> | ----- | -------- | ----- |
> 
> 
> 2. **Finding Distances:**
>    If given a new data point with serial number 10, distances from this point to all others are calculated:
>    ```
>    d1, d2, d3, ..., d9
>    ```
> 
> 3. **Sorting Distances:**
>    Sort these distances in ascending order to find the nearest neighbors.
> 
> 4. **Assigning Classes:**
>    Based on the majority class among the nearest neighbors (\(k\)), assign a class to the new data point.
> 
> ### Example
> 
> Suppose we have 9 data points and need to classify the 10 th:
> ```
> | Serial Number | Distance (d) | Class |
> |---------------|---------------|-------|
> | 1             | d1            | 0     |
> | 2             | d2            | 0     |
> | ...           | ...           | ...   |
> | 9             | d9            | 1     |
> ```
> 
> After sorting by distance:
> ```
> d4 < d5 < d6 < ...
> ```
> If \(k = 3\), check classes of closest three points and assign class based on majority vote.
> 
> ### Handling Ties
> If there's a tie when choosing an even number of neighbors (e.g., \(k = 4\)), increase \(k\) until there’s no tie.
> 
> ## Lazy Learner Concept
> 
> KNN is referred to as a "lazy learner" because it performs minimal processing during training and does most computations during prediction:
> 
> - **Training:** 
>     - Memorizes all data points.
>     - $\large{ \text{model.fit} }$
> - **Prediction:** 
>     - Calculates distances.
>     - Sorts distances.
>     - Finds nearest neighbors.
>     - Determines majority class.
>     - $\large{ \text{model.predict} }$
> 
> In summary:
> ```markdown
> At training time (`fit`), KNN memorizes data points.
> At prediction time (`predict`), it calculates distances and assigns classes based on nearest neighbors.
> ```
> 
> This leads to minimal training time but potentially expensive prediction time due to distance calculations and sorting operations.
> 
> ## Summary
> Understanding KNN involves knowing how many neighbors (\(k\)) to select and how distance impacts classification. The lazy nature of KNN makes it efficient for training but computationally intensive during predictions.
> 
> ### Important Questions
> - Why is KNN called a lazy learner?
> - How do you decide the value of \(k\)?
> 
> Remember that while both KNN and k-means clustering involve choosing \(k\), they serve different purposes—classification for supervised learning (KNN) vs. Clustering for unsupervised learning (k-means).

> When using K-Nearest Neighbors (KNN) in your projects, you will inevitably face two crucial questions:
> 1. How to decide the value of \(k\)?
> 2. What are the methods to determine the optimal \(k\)?
> 
> ### Suggestions for Selecting \(k\)
> 
> 1. **Choose an Odd Number for \(k\)**:
>     - Choosing \(k\) as an odd number helps to avoid ties in most cases. While ties can still occur with odd values, they are less likely compared to even values.
> 
> 2. **Range of Values for \(k\)**:
>     - Take a range of values for \(k\) and evaluate them based on metrics such as precision and recall.
>     - For instance, if the goal is to maximize precision, plot precision against different values of \(k\). <font color="#000000">Choose the value where precision is both high and stable</font>.
> 
> 3. **Error Analysis**:
>     - Similarly, analyze error rates for different values of \(k\). Plot error on the y-axis and \(k\) on the x-axis.
>     - Select the value of \(k\) where error is at its stable minimum.

> ### Distance Metrics
> 
> 1. **Euclidean Distance**:
>    - Euclidean distance measures straight-line distance using Pythagoras' theorem.
>    - Formula: $\large{d = \sqrt{x^2 + y^2}}$
> 
> 2. **Manhattan Distance**:
>    - Also known as city block distance, it accounts for obstacles by summing absolute differences across axes.
>    - Formula: $\large{\sum |a_i - b_i|}$
> 
> 3. **Cosine Similarity**:
>    - Used primarily in NLP (Natural Language Processing), it calculates angular distance between vectors.
>    - Formula: $\large{\cos(\theta)}$
> $$
> \text{If } \cos(0^\circ) = 1 \text{ and } \cos(90^\circ) = 0,
> $$
> higher angular distances indicate greater dissimilarity between vectors.
> 
> ### Important Note
> 
> In K-Means clustering, a different method called the "elbow method" is used to find optimal clusters (\(k\)). This should not be confused with methods used in KNN.

- **Training:**

    - Sorts distances.
    - Finds nearest neighbors.
    - Determines majority class.
    - $\large{ \text{model.predict} }$
