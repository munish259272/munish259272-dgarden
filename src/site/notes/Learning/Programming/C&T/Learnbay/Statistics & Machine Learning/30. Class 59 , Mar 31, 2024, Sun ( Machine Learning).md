### 📘 Logistic Regression — Part 1: Intuition & Core Idea

---

### 🎯 1. What is Logistic Regression?

Despite the name, **Logistic Regression is NOT a regression model** in the usual sense.

> ⚠️ It is actually a **classification model**

---

#### 🔍 Regression vs Classification

|Concept|Linear Regression|Logistic Regression|
|---|---|---|
|Target variable|Continuous (e.g., price, charges)|Categorical (e.g., 0/1, spam/not spam)|
|Output|Any real value (−∞ → +∞)|Class label (0 or 1)|
|Goal|Predict a value|Predict a category|

---

#### 📌 Key Definition

- **Logistic Regression = Binary Classification Model**
    
- Output has **only 2 classes**:
    
    - 0 / 1
        
    - Yes / No
        
    - Win / Lose
        
    - Spam / Not Spam
        

> This is called **Binary Classification** (also known as _Dichotomous Classification_)

---

### 🧠 2. Core Problem — Why Not Use Linear Regression?

Let’s think carefully:

#### Linear Regression Output:

$$  
(-\infty, +\infty)  
$$

👉 Any real value is possible.

---

#### But Classification Needs:

- Only **2 outputs** → {0, 1}
    

---

#### ❗ Problem:

How do we convert:  
$$  
(-\infty, +\infty) \rightarrow {0,1}  
$$

---

### 🔄 3. Two-Step Transformation Strategy

Logistic Regression solves this using **2 steps**:

---

#### ✅ Step 1: Scale Output

Convert:  
$$  
(-\infty, +\infty) \rightarrow (0,1)  
$$

👉 This is a **continuous probability range**

---

#### ✅ Step 2: Thresholding

Convert:  
$$  
(0,1) \rightarrow {0,1}  
$$

Using a **decision boundary (threshold)**:

- If ≥ 0.5 → **1**
    
- If < 0.5 → **0**
    

---

### 🎯 Important Insight

> Logistic Regression =  
> **Linear Regression + Transformation + Thresholding**

---

### 📊 4. Why (0,1)? — Connection to Probability

The range **0 to 1** represents:

> 🎯 **Probability**

Examples:

- 0.8 → 80% chance of "Yes"
    
- 0.2 → 20% chance of "Yes"
    

---

### ⚙️ 5. The Key Tool — Sigmoid Function

To perform **Step 1 (scaling)**, we use:

#### 📌 Sigmoid Function:

$$  
\large\bbox[white, 3pt, border: 1px solid black]{
\sigma(y) = \frac{1}{1 + e^{-y}}  
}
$$

Where:

- $y = mx + c$ (output of linear regression)
    

---

#### 🧠 What it does:

|Input (y)|Output (σ(y))|
|---|---|
|−∞|→ 0|
|0|= 0.5|
|+∞|→ 1|

---

#### 🎯 Conclusion:

Sigmoid function **compresses infinite values into (0,1)**

---

### 🔁 6. Full Pipeline View

```
Linear Model → Sigmoid → Probability → Threshold → Class
```

Or mathematically:

$$  
y = mx + c  
\rightarrow  
\sigma(y)  
\rightarrow  
\text{Probability}  
\rightarrow  
\text{Class (0 or 1)}  
$$

---

### 📌 7. Why is it called “Regression”?

Good question — commonly asked in interviews.

👉 Because:

- It **starts with a regression equation**:  
    $$  
    y = mx + c  
    $$
    
- Then transforms it into classification using sigmoid.
    

---

#### 🔑 Final Understanding

> Logistic Regression is called “regression”  
> because it **uses a regression equation internally**,  
> but the **final goal is classification**

---

### ✅ Summary (Part 1)

- Logistic Regression is a **binary classification model**
- Converts continuous output → categorical labels
- Works in **2 steps**:
    1. Scale using **Sigmoid** → (0,1)
    2. Threshold → {0,1}
- Output represents **probability**
    

---

### 📘 Logistic Regression — Part 2: Math, Log-Odds & Decision Boundary

---

### 🔁 1. Recap (Where we are)

From Part 1:

- Linear model:  
    $$  
    y = mx + c  
    $$
    
- Apply sigmoid:  
    $$  
    p = \frac{1}{1 + e^{-y}}  
    $$
    
- Convert to class using threshold (0.5)
    

---

### 🎯 2. What is this “p”?

> ( p ) = **Probability of belonging to class 1**

So:

- ( p = 0.8 ) → 80% chance of class 1
    
- ( p = 0.2 ) → 20% chance of class 1
    

---

### ⚠️ Problem with Linear Regression

Linear regression outputs:  
$$  
(-\infty, +\infty)  
$$

But probabilities must satisfy:  
$$  
0 \le p \le 1  
$$

👉 Sigmoid fixes this constraint.

---

### 🔄 3. Transforming Sigmoid → Log-Odds Form

We start from:

$$  
p = \frac{1}{1 + e^{-y}}  
$$

---

#### Step 1: Rearrange

$$  
p(1 + e^{-y}) = 1  
$$

$$  
p + p e^{-y} = 1  
$$

$$  
p e^{-y} = 1 - p  
$$

---

#### Step 2: Isolate exponential

$$  
e^{-y} = \frac{1 - p}{p}  
$$

Take reciprocal:

$$  
e^y = \frac{p}{1 - p}  
$$

---

### 🎯 4. Log-Odds (Logit Function)

Take log on both sides:

$$  
\large\bbox[white, 3pt, border: 1px solid black]{
y = \log \left(\frac{p}{1 - p}\right)  
}
$$

---

#### 📌 Interpretation

- Left side:  
    $$  
    y = mx + c  
    $$
    
- Right side:  
    $$  
    \log\left(\frac{p}{1-p}\right)  
    $$
    

---

#### 🔥 This is the **core equation of Logistic Regression**

> Linear combination = **log of odds**

---

### 🧠 5. What are “Odds”?

#### Definition:

$$  
\large\bbox[white, 3pt, border: 1px solid black]{
\text{Odds} = \frac{p}{1 - p}  
}
$$

---

#### Example:

If:

- ( p = 0.8 )
    

Then:  
$$  
\text{Odds} = \frac{0.8}{0.2} = 4  
$$

👉 Interpretation:

> Event is **4 times more likely to happen than not**

---

#### Another Example:

If:

- ( p = 0.5 )
    

$$  
\text{Odds} = 1  
$$

👉 Equal chance (neutral point)

---

### 📌 6. Why Log of Odds?

Because:

- Odds range = ( $0, \infty$ )
    
- Log(odds) range = ( $-\infty, +\infty$ )
    

---

👉 This perfectly matches:  
$$  
mx + c \in (-\infty, +\infty)  
$$

---

#### 🔑 Key Insight

> Logistic Regression works because:  
> it converts probability → log-odds → linear model

---

### 🎯 7. Decision Boundary (Threshold)

We said:

- If $p \ge 0.5$ → Class = 1
    
- If $p < 0.5$ → Class = 0
    

---

#### Let’s connect this mathematically

At boundary:  
$$  
p = 0.5  
$$

Then:  
$$  
\frac{p}{1-p} = 1  
$$

$$  
\log(1) = 0  
$$

---

#### So:

$$  
y = mx + c = 0  
$$

---

### 🔥 Final Insight

> **Decision boundary happens where the linear equation = 0**

---

#### Example:

If:  
$$  
2x - 6 = 0  
$$

Then:  
$$  
x = 3  
$$

👉 That is your classification boundary.

---

### ⚖️ 8. Why Threshold = 0.5?

Because:

|Threshold|Effect|
|---|---|
|0.5|Balanced decision|
|0.1|Biased toward class 1|
|0.9|Biased toward class 0|

---

#### 🎯 Interpretation:

- 0.5 → “more likely than not”
- Changing threshold = changing **decision policy**

---

#### ⚠️ Important

- You **can change threshold**
- But it introduces **bias**

---

#### When do we change it?

- Medical diagnosis (favor detection → lower threshold)
- Spam detection (avoid false positives → higher threshold)

---

### 🔁 9. Full Concept Pipeline (Mathematical View)

$$  
mx + c  
\rightarrow  
\log\left(\frac{p}{1-p}\right)  
\rightarrow  
p = \frac{1}{1 + e^{-y}}  
\rightarrow  
\text{Threshold}  
\rightarrow  
{0,1}  
$$

---

### 📌 10. Summary (Part 2)

- Logistic regression models:  
    $$  
    \log\left(\frac{p}{1-p}\right) = mx + c  
    $$
    
- Odds:  
    $$  
    \frac{p}{1-p}  
    $$
- Log-odds maps probability → linear space
- Decision boundary:  
    $$  
    mx + c = 0  
    $$

* Leads to:

  * Multiple local minima
  * Difficult optimization

# Is likelihood same as probability ?

</div>


> [!question]- Is likelihood same as probability ?
> 
> Short answer: **No, likelihood and probability are not the same**, even though they use the same mathematical expression.
> 
> ---
> 
> # 1. Probability vs Likelihood (Core Difference)
> 
> The distinction is about **what is fixed and what is variable**.
> 
> ---
> 
> ## Probability
> 
> A **Probability** answers:
> 
> > “Given a model (or parameter), how likely is the data?”
> 
> $$
> P(\text{data} \mid \theta)
> $$
> 
> * $\theta$ = parameter (fixed)
> * data = random
> 
> ---
> 
> ### Example (your coin context)
> 
> $$
> P(4H \mid p=0.5)
> $$
> 
> Interpretation:
> 
> > “If the coin is fair, what is the probability of getting 4 heads?”
> 
> ---
> 
> ## Likelihood
> 
> A **Likelihood function** answers:
> 
> > “Given the observed data, how plausible is a parameter?”
> 
> $$
> L(\theta) = P(\text{data} \mid \theta)
> $$
> 
> Same formula—but now:
> 
> * data = fixed (observed)
> * $\theta$ = variable
> 
> ---
> 
> ### Example
> 
> You observed:
> 
> ```id="obs_data"
> 4 heads out of 6
> ```
> 
> Now consider different values of (p):
> 
> $$
> L(p=0.5), \quad L(p=0.8), \quad L(p=0.3)
> $$
> 
> Interpretation:
> 
> > “Which value of $p$ best explains the observed data?”
> 
> ---
> 
> # 2. Same Formula, Different Meaning
> 
> This is the key subtlety:
> 
> $$
> P(\text{data} \mid \theta)
> $$
> 
> * As **probability** → vary data
> * As **likelihood** → vary $\theta$
> 
> ---
> 
> # 3. Concrete Comparison
> 
> | Concept     | What varies? | What is fixed? | Question                       |
> | ----------- | ------------ | -------------- | ------------------------------ |
> | Probability | Data         | Parameter      | “What will happen?”            |
> | Likelihood  | Parameter    | Data           | “What explains what happened?” |
> 
> ---
> 
> # 4. Back to Your Expected Value Statement
> 
> You said:
> 
> > “Weighted sum accounts for each outcome’s likelihood.”
> 
> More precisely:
> 
> * In expectation:
>   $$
>   E(X) = \sum x \cdot P(x)
>   $$
>   these are **probabilities**, not likelihoods.
> 
> So a more accurate phrasing is:
> 
> > “Weighted sum accounts for each outcome’s **probability**.”
> 
> ---
> 
> # 5. Why People Confuse Them
> 
> Because:
> 
> * Numerically → same expression
> * Conceptually → different roles
> 
> This distinction becomes critical in:
> 
> * **Maximum Likelihood Estimation (MLE)**
> * **Bayesian inference**
> 
> ---
> 
> # 6. Intuition (Very Important)
> 
> Think of it like this:
> 
> * **Probability** → forward direction
>   (model → data)
> 
> * **Likelihood** → backward direction
>   (data → model)
> 
> ---
> 
> # Final Takeaway
> 
> * **Probability**: chance of observing data given a model
> * **Likelihood**: how well a model explains observed data
> 
> 👉 In your expected value calculation, you are using **probabilities**, not likelihoods.
> 
> ---
>

> > Great—this is exactly where the **probability vs likelihood** distinction becomes operational.
> > 
> > ---
> > 
> > # 1. What is MLE?
> > 
> > **Maximum Likelihood Estimation $MLE$** is a method to estimate unknown parameters by:
> > 
> > > **Choosing the parameter that makes the observed data most likely**
> > 
> > ---
> > 
> > # 2. General Idea
> > 
> > Suppose:
> > 
> > * Data: $x_1, x_2, ..., x_n$ $observed$
> > * Model parameter: $\theta$ $unknown$
> > 
> > We define the **likelihood function**:
> > 
> > $$
> > L(\theta) = P(\text{data} \mid \theta)
> > $$
> > 
> > Then MLE chooses:
> > 
> > $$
> > \hat{\theta} = \arg\max_{\theta} L(\theta)
> > $$
> > 
> > ---
> > 
> > # 3. Why Likelihood $not Probability$?
> > 
> > * In **probability**, $\theta$ is fixed and data varies
> > * In **likelihood**, data is fixed and $\theta$ varies
> > 
> > 👉 MLE treats the observed data as fixed and asks:
> > 
> > > “Which parameter best explains this data?”
> > 
> > ---
> > 
> > # 4. Example 1: Coin Toss $Bernoulli Model$
> > 
> > Suppose:
> > 
> > * You flip a coin 6 times
> > * Observe: 4 heads, 2 tails
> > 
> > Let:
> > 
> > $$
> > p = P(\text{heads})
> > $$
> > 
> > ---
> > 
> > ## Step 1: Likelihood Function
> > 
> > $$
> > L(p) = P(4H, 2T \mid p)
> > $$
> > 
> > Using binomial:
> > 
> > $$
> > L(p) = \binom{6}{4} p^4 (1-p)> > $$
> > 
> > $L(p) = \binom{6}{4} p^4 (1-p)^2$
> > 
> > ---
> > 
> > ## Step 2: Maximize Likelihood
> > 
> > We want:
> > 
> > $$
> > \hat{p} = \arg\max L(p)
> > $$
> > 
> > Instead of maximizing $L(p)$, we maximize **log-likelihood**:
> > 
> > $$
> > \log L(p) = \log \binom{6}{4} + 4\log p + 2\log(1-p)
> > $$
> > 
> > Take derivative and solve → result:
> > 
> > $$
> > \hat{p} = \frac{4}{6} = 0.667
> > $$
> > 
> > ---
> > 
> > ## Interpretation
> > 
> > > The value of $p$ that best explains the data is **proportion of heads observed**
> > 
> > ---
> > 
> > # 5. Example 2: Normal Distribution
> > 
> > Suppose:
> > 
> > $$
> > X_1, X_2, ..., X_n \sim N(\mu, \sigma^2)
> > $$
> > 
> > Unknown parameter: $\mu$
> > 
> > ---
> > 
> > ## Likelihood
> > 
> > $$
> > L(\mu) = \prod_{i=1}^n \frac{1}{\sigma\sqrt{2\pi}} \exp\left(-\frac{(x_i-\mu)^2}{2\sigma^2}\right)
> > $$
> > 
> > ---
> > 
> > > [!question]- $$L(\mu) = \prod_{i=1}^n \frac{1}{\sigma\sqrt{2\pi}} \exp\left(-\frac{(x_i-\mu)^2}{2\sigma^2}\right)$$ Why are we multiplying here using $\prod$ ? Does each value in this PDF at $x_i$ represents the probability of the random variable $X_i$ ?
> > > Excellent question—this goes right to the heart of how likelihood is constructed.
> > > 
> > > ---
> > > 
> > > # 1. Why are we multiplying (the (\prod))?
> > > 
> > > The key assumption is **independence**.
> > > 
> > > If
> > > $$
> > > X_1, X_2, ..., X_n
> > > $$
> > > are **independent**, then the joint probability is:
> > > 
> > > $$
> > > P(X_1 = x_1, X_2 = x_2, ..., X_n = x_n)
> > > = \prod_{i=1}^n P(X_i = x_i)
> > > $$
> > > 
> > > This is a fundamental rule:
> > > 
> > > > **Joint probability of independent events = product of individual probabilities**
> > > 
> > > ---
> > > 
> > > ## Continuous Case (your equation)
> > > 
> > > For continuous variables, we use **densities** instead of probabilities.
> > > 
> > > So:
> > > 
> > > $$
> > > L(\mu) = \prod_{i=1}^n f(x_i \mid \mu)
> > > $$
> > > 
> > > Each term:
> > > 
> > > $$
> > > f(x_i \mid \mu)
> > > $$
> > > 
> > > comes from the **Normal distribution**.
> > > 
> > > ---
> > > 
> > > # 2. Important Clarification
> > > 
> > > You asked:
> > > 
> > > > “Does each value in this PDF at $x_i$ represent the probability of $X_i$?”
> > > 
> > > 👉 **No — this is a very important distinction.**
> > > 
> > > Each term is a **probability density**, not a probability.
> > > 
> > > ---
> > > 
> > > ## Probability vs Density
> > > 
> > > For a continuous random variable:
> > > 
> > > $$
> > > P(X = x_i) = 0
> > > $$
> > > 
> > > Always zero.
> > > 
> > > Instead:
> > > 
> > > $$
> > > f(x_i) = \text{density at } x_i
> > > $$
> > > 
> > > Interpretation:
> > > 
> > > > It tells you how “likely” values **near $x_i$** are.
> > > 
> > > Actual probability comes from an interval:
> > > 
> > > $$
> > > P(a \le X \le b) = \int_a^b f(x),dx
> > > $$
> > > 
> > > ---
> > > 
> > > # 3. So What Are We Multiplying?
> > > 
> > > We are multiplying:
> > > 
> > > $$
> > > f(x_1 \mid \mu) \cdot f(x_2 \mid \mu) \cdot ... \cdot f(x_n \mid \mu)
> > > $$
> > > 
> > > This gives the **joint density** of observing all data points together.
> > > 
> > > ---
> > > 
> > > ## Intuition
> > > 
> > > Think of it as:
> > > 
> > > > “How plausible is it to observe this entire dataset under parameter $\mu$?”
> > > 
> > > Each data point contributes a “score”, and we combine them via multiplication.
> > > 
> > > ---
> > > 
> > > # 4. Why Multiplication Makes Sense
> > > 
> > > If any data point is very unlikely under $\mu$:
> > > 
> > > * its density is small
> > > * product becomes small
> > > 
> > > So:
> > > 
> > > 👉 Bad parameter → low likelihood
> > > 👉 Good parameter → high likelihood
> > > 
> > > ---
> > > 
> > > # 5. Discrete vs Continuous (Important Contrast)
> > > 
> > > ### Discrete case
> > > 
> > > $$
> > > P(X_i = x_i)
> > > $$
> > > 
> > > → actual probability
> > > → multiplication gives joint probability
> > > 
> > > ---
> > > 
> > > ### Continuous case
> > > 
> > > $$
> > > f(x_i)
> > > $$
> > > 
> > > → density (not probability)
> > > → multiplication gives joint density
> > > 
> > > ---
> > > 
> > > # 6. Final Intuition
> > > 
> > > * Each $f(x_i \mid \mu)$ = “how well $\mu$ explains this data point”
> > > * Product = “how well $\mu$ explains all data points together”
> > > 
> > > ---
> > > 
> > > # 7. Why We Take Log (Preview)
> > > 
> > > Multiplying many small numbers → numerical underflow
> > > 
> > > So we use:
> > > 
> > > $$
> > > \log L(\mu) = \sum \log f(x_i \mid \mu)
> > > $$
> > > 
> > > This turns:
> > > 
> > > * product → sum
> > > * easier optimization
> > > 
> > > ---
> > > 
> > > # Final Takeaway
> > > 
> > > * We multiply because of **independence**
> > > * Each term is a **density**, not a probability
> > > * The product represents the **joint likelihood of the dataset**
> > > 
> > > ---
> > > 
> > > If you want, I can go one level deeper and explain **why maximizing likelihood ends up minimizing squared error** from this exact expression (step-by-step derivation).
> > 
> > ## Log-Likelihood

> > > # Big Picture
> > > 
> > > **Maximum Likelihood Estimation (MLE)** says:
> > > 
> > > $$
> > > \hat{\theta} = \arg\max_{\theta} P(\text{data} \mid \theta)
> > > $$
> > > 
> > > In ML, we usually **minimize a loss**, not maximize likelihood.
> > > 
> > > So we transform:
> > > 
> > > $$
> > > \text{maximize likelihood} \Longleftrightarrow \text{minimize negative log-likelihood}
> > > $$
> > > 
> > > $$
> > > \hat{\theta} = \arg\min_{\theta} \big(-\log L(\theta)\big)
> > > $$
> > > 
> > > 👉 This **negative log-likelihood (NLL)** becomes the **loss function**.

> > > # 1. Linear Regression → Squared Error Loss
> > > 
> > > ## Assumption (Very Important)
> > > 
> > > We assume:
> > > 
> > > $$
> > > y_i = w^T x_i + \epsilon_i
> > > $$
> > > 
> > > where noise
> > > 
> > > $$
> > > \epsilon_i \sim N(0, \sigma^2)
> > > $$
> > > 
> > > So:
> > > 
> > > $$
> > > y_i \mid x_i \sim N(w^T x_i, \sigma^2)
> > > $$
> > > 
> > > ---
> > > 
> > > ## Likelihood
> > > 
> > > $$
> > > L(w) = \prod_{i=1}^n \frac{1}{\sigma\sqrt{2\pi}} \exp\left(-\frac{(y_i - w^T x_i)^2}{2\sigma^2}\right)
> > > $$
> > > 
> > > ---
> > > 
> > > ## Log-Likelihood
> > > 
> > > $$
> > > \log L(w) = \text{constant} - \frac{1}{2\sigma^2} \sum (y_i - w^T x_i)> > > $$
> > > 
> > > ---
> > > 
> > > ## Convert to Loss
> > > 
> > > Maximizing likelihood ⇔ minimizing:
> > > 
> > > $$
> > > \sum (y_i - w^T x_i)> > > $$
> > > 
> > > $$
> > > \sum_{i=1}^n (y_i - w^T x_i)> > > $$
> > > ---
> > > 
> > > ## Conclusion
> > > 
> > > 👉 **Linear regression = MLE under Gaussian noise**

> > > 👉 Loss function = **Mean Squared Error (MSE)**

> > >

> > > > 
> > > > ## 1. Why $w^T$ instead of $m$?
> > > > 
> > > > In machine learning, we typically use **vector notation**:
> > > > 
> > > > $$
> > > > y_i = w^T x_i + \epsilon_i
> > > > $$
> > > > 
> > > > This is actually the **general form** that works for:
> > > > 
> > > > ### Simple linear regression (1 feature):
> > > > - $x_i$ is just a number
> > > > - $w$ is just a number (like $m$ in $y = mx + b$)
> > > > - But wait — where's the intercept $b$?
> > > > 
> > > > We usually **augment** the input:
> > > > $$
> > > > x_i = [1, \text{(feature)}] \quad \text{and} \quad w = [b, m]
> > > > $$
> > > > 
> > > > Then:
> > > > $$
> > > > w^T x_i = b \cdot 1 + m \cdot (\text{feature})
> > > > $$
> > > > 
> > > > ### Multiple linear regression (many features):
> > > > - $x_i$ is a vector of features
> > > > - $w$ is a vector of coefficients
> > > > - $w^T x_i$ = dot product = weighted sum of features
> > > > 
> > > > So $w^T$ is just the **general notation** that works for any number of features.
> > > > 
> > > > ---
> > > > 
> > > > ## 2. Why is the mean of $\epsilon_i$ zero?
> > > > 
> > > > Great question! Let me explain:
> > > > 
> > > > ### Intuition
> > > > If the noise had a non-zero mean, say $\mu$, then:
> > > > 
> > > > $$
> > > > y_i = w^T x_i + \epsilon_i, \quad \epsilon_i \sim N(\mu, \sigma^2)
> > > > $$
> > > > 
> > > > We could rewrite:
> > > > $$
> > > > y_i = \underbrace{(w^T x_i + \mu)}_{\text{absorb into model}} + \underbrace{(\epsilon_i - \mu)}_{\text{now mean 0}}
> > > > $$
> > > > 
> > > > The $\mu$ just gets **absorbed** into the intercept term. So assuming zero mean is **without loss of generality**.
> > > > 
> > > > ### Visual intuition
> > > > Imagine data points scattered around a line. The "errors" are the vertical distances. If the average error wasn't zero, you could just shift the line up/down to make it zero.
> > > > 
> > > > ---
> > > > 
> > > > ## 3. What does $y_i \mid x_i \sim N(w^T x_i, \sigma^2)$ mean?
> > > > 
> > > > This is the **generative assumption** of linear regression:
> > > > 
> > > > ### Breaking it down
> > > > - $y_i \mid x_i$ = "the output given the input"
> > > > - $\sim N(\mu, \sigma^2)$ = "follows a normal distribution with mean $\mu$, variance $\sigma^2$"
> > > > - $\mu = w^T x_i$ = the mean depends linearly on the input
> > > > - $\sigma^2$ = constant variance (homoscedasticity)
> > > > 
> > > > ### In plain English
> > > > > "For any given input $x$, the possible outputs $y$ are normally distributed around the line $w^T x$, with constant spread $\sigma$."
> > > > 
> > > > ### Visual representation
> > > > ```
> > > >     y ↑
> > > >       |        .   
> > > >       |    .   |   .
> > > >       |   |    |  σ
> > > >       |  .|....|...
> > > >       |   |    |
> > > >       |   .    .
> > > >       |________________ x
> > > >          w^T x
> > > > ```
> > > > 
> > > > For each $x$, there's a **distribution** of possible $y$ values, centered at $w^T x$.
> > > > 
> > > > ### Why assume this?
> > > > Because it's:
> > > > 1. **Mathematically convenient** (leads to squared error loss)
> > > > 2. **Often reasonable** (many real processes have Gaussian noise)
> > > > 3. **Maximum entropy** distribution given mean and variance (least assumptions)
> > > > 
> > > > ---
> > > > 
> > > > ## 4. Why do we need to make the log negative?
> > > > 
> > > > This is a crucial point! Let me show both directions:
> > > > 
> > > > ### Maximizing likelihood (statistics view)
> > > > We want:
> > > > $$
> > > > \hat{w} = \arg\max_w L(w)
> > > > $$
> > > > 
> > > > ### Minimizing loss (ML view)
> > > > We want:
> > > > $$
> > > > \hat{w} = \arg\min_w \text{Loss}(w)
> > > > $$
> > > > 
> > > > ### The connection
> > > > $$
> > > > \max_w L(w) \iff \min_w [-L(w)]
> > > > $$
> > > > 
> > > > But wait — we use **negative log**, not just negative:
> > > > 
> > > > $$
> > > > \min_w [-\log L(w)]
> > > > $$
> > > > 
> > > > ### Why $\log$ + negative?
> > > > 4. **Log** converts product to sum (easier math)
> > > > 5. **Negative** flips maximization to minimization
> > > > 
> > > > ### Example with numbers
> > > > Suppose we have two models:
> > > > - Model A: $L(w) = 0.8$
> > > > - Model B: $L(w) = 0.6$
> > > > 
> > > > If we maximize likelihood:
> > > > - Pick A (0.8 > 0.6)
> > > > 
> > > > If we minimize negative log:
> > > > - Model A: $-\log(0.8) \approx 0.22$
> > > > - Model B: $-\log(0.6) \approx 0.51$
> > > > - Pick A (0.22 < 0.51)
> > > > 
> > > > **Same result** — just flipped the optimization direction!
> > > > 
> > > > ### Why minimization is preferred in ML
> > > > 1. **Optimization algorithms** are usually written for minimization
> > > > 2. **Gradient descent** naturally decreases loss
> > > > 3. **Loss** intuitively means "penalty" — lower is better
> > > >

### 📊 4. Likelihood Function

For each data point:

* If ( y = 1 ) → probability = ( p )
* If ( y = 0 ) → probability = ( 1 - p )

---

#### Combine both cases:

$$
\large\bbox[white, 3pt, border: 1px solid black]{
P(y) = p^y (1 - p)^{1-y}
}
$$

---

#### For entire dataset:

$$
\large\bbox[white, 3pt, border: 1px solid black]{
L = \prod p_i^{y_i} (1 - p_i)^{1-y_i}
}
$$

---

👉 This is called **Likelihood**

---

### 🔄 5. Log-Likelihood (Why log?)

We take log to simplify multiplication:

$$
\log L = \sum \left[ y_i \log p_i + (1 - y_i)\log(1 - p_i) \right]
$$

---

### 🎯 6. Loss Function (Cross-Entropy / Log Loss)

We convert maximization → minimization:

$$
\large\bbox[white, 3pt, border: 1px solid black]{
\text{Loss} = - \sum \left[ y_i \log p_i + (1 - y_i)\log(1 - p_i) \right]}
$$



---

### 🔥 Final Loss Function

$$
\boxed{
\text{Log Loss} = - \left[ y \log p + (1 - y)\log(1 - p) \right]
}
$$

---

### 🧠 7. Why this Loss Works (VERY IMPORTANT)

---

#### Case 1: Correct Prediction

If:

* ( y = 1 ), ( p = 0.9 )

Then:

* Loss is **small**

---

#### Case 2: Wrong Prediction

If:

* ( y = 1 ), ( p = 0.1 )

Then:

* Loss is **very large**

---

#### 🎯 Interpretation

> Model is heavily penalized for **confident wrong predictions**

---

### 📉 8. Shape of the Loss Function

* Convex function
* Single global minimum
* Easy optimization

---

👉 This is why we prefer **log loss over MSE**

---

### ⚙️ 9. Training via Gradient Descent

We update parameters:

$$
\large\bbox[white, 3pt, border: 1px solid black]{
m := m - \alpha \frac{\partial \text{Loss}}{\partial m}
}
$$

$$
\large\bbox[white, 3pt, border: 1px solid black]{
c := c - \alpha \frac{\partial \text{Loss}}{\partial c}
}
$$

---

#### 🔁 Process:

1. Initialize ( m, c )
2. Compute predictions ( p )
3. Compute loss
4. Compute gradients
5. Update parameters
6. Repeat until convergence

```id="lr_training"
X → Linear (mx+c) → Sigmoid → p → Log Loss → Gradient Descent → Update
```

### 🔹 L2 Regularization (Ridge)

Adds penalty:

$$
\large\bbox[white, 3pt, border: 1px solid black]{
\lambda \sum w^2
}
$$

---

#### Effect:

* Shrinks weights smoothly
* Keeps all features

---

### 🔹 L1 Regularization (Lasso)

Adds penalty:

$$
\large\bbox[white, 3pt, border: 1px solid black]{
\lambda \sum |w|
}
$$

---

#### Effect:

* Forces some weights → 0
* Performs **feature selection**

---

Where,
**<span style="font-weight:bold; color:#000000">n</span>** is the number of observations or data points
**<span style="font-style:italic; color:#000000">k</span>** is the number of predictors (independent variables) in the model
```run-python

# n number of observations or data points
# k: number of predictors (independent variables) in the model
ols_adjusted_r2_train = 1 - (1 - ols_r2_training) * (len(y_train) - 1 ) / (len(y_train) - X_train_scaled.shape[1] - 1)
```
